## Câu hỏi 71

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A web application is hosted on an EC2 instance that processes sensitive financial information. The EC2 instance is launched in a private subnet, and all data is stored in an Amazon S3 bucket. Users access financial information over the Internet through pre-signed URLs generated by the web application. The company's security team is concerned that **internet connectivity to Amazon S3 poses a security risk**.
>
> In this scenario, what will you do to resolve this security vulnerability in the **most cost-effective** manner?

---

### 🔤 DỊCH CHI TIẾT

Ứng dụng web chạy trên EC2 trong **private subnet** (mạng con riêng tư), xử lý dữ liệu tài chính nhạy cảm lưu trong S3. Users truy cập qua **pre-signed URLs** (URL có chữ ký tạm thời). Security team lo ngại việc EC2 kết nối S3 qua **internet** là rủi ro bảo mật.

**Yêu cầu:** Loại bỏ kết nối internet đến S3, theo cách **cost-effective nhất** (tiết kiệm chi phí nhất).

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Kết nối EC2 → S3 không qua internet** → dùng **VPC Endpoint**
2. **Cost-effective nhất** → **Gateway VPC Endpoint** (miễn phí) thay vì Interface VPC Endpoint (tính phí theo giờ + data)
3. S3 và DynamoDB là 2 service **duy nhất** hỗ trợ Gateway Endpoint → không cần PrivateLink

**Từ khóa bẫy:** "most cost-effective" → loại Interface VPC Endpoint dù kỹ thuật cũng đúng

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án D:** Change to access S3 through a **Gateway VPC Endpoint**.

- Traffic EC2 → S3 đi hoàn toàn trong **AWS network**, không qua internet ✅
- Cấu hình qua **route table** trong VPC → đơn giản ✅
- **Hoàn toàn miễn phí** — không có hourly charge hay data processing fee ✅
- S3 là một trong hai service hỗ trợ Gateway Endpoint (cùng DynamoDB) ✅

---

❌ **Đáp án A:** Create a custom **VPC endpoint service**.

- **VPC endpoint service** = bạn là **nhà cung cấp** dịch vụ, tạo endpoint cho người khác kết nối đến app của bạn ❌
- Hoàn toàn khác với **VPC endpoint** (bạn là người dùng kết nối đến AWS service) ❌
- Không giải quyết được yêu cầu kết nối EC2 → S3 ❌

---

❌ **Đáp án B:** Access S3 through a **VPN connection**.

- VPN vẫn đi qua **public internet** (chỉ được mã hóa) → không loại bỏ được rủi ro internet ❌
- Không phải giải pháp để kết nối VPC → S3 nội bộ AWS ❌

---

❌ **Đáp án C:** Access S3 through an **Interface VPC Endpoint** (powered by AWS PrivateLink).

- Về mặt kỹ thuật hoạt động được — traffic không qua internet ✅
- **Nhưng tốn tiền:** tính phí theo giờ (hourly) + data processing charges ❌
- **Không cost-effective bằng Gateway Endpoint** vốn miễn phí ❌
- Bài hỏi "most cost-effective" → Gateway luôn thắng Interface ở S3/DynamoDB ❌

---

### Note / Pipeline đúng

**Yêu cầu:** EC2 (private subnet) → S3 không qua internet, rẻ nhất.

```
TRƯỚC (có vấn đề):
EC2 (private subnet) → Internet Gateway → Public Internet → S3
⚠️ Traffic đi qua internet → rủi ro bảo mật

SAU (giải pháp đúng):
EC2 (private subnet) → Gateway VPC Endpoint → S3
✅ Traffic hoàn toàn trong AWS network
✅ Không qua internet
✅ Miễn phí
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Loại VPC Endpoint | Hoạt động như thế nào | Hỗ trợ service | Chi phí |
|---|---|---|---|
| **Gateway Endpoint** | Target trong route table | **S3, DynamoDB** | **Miễn phí** |
| **Interface Endpoint** (PrivateLink) | ENI với private IP | Hầu hết AWS services | Tính phí theo giờ + data |
| **VPC Endpoint Service** | Bạn là nhà cung cấp | App của bạn | Tính phí |

> 💡 **Công thức ghi nhớ:**
> "S3 hoặc DynamoDB, không qua internet, miễn phí" → **Gateway VPC Endpoint**
> "Các service khác (SQS, SNS, SSM...), không qua internet" → **Interface VPC Endpoint** (PrivateLink, có phí)
> "VPN" → vẫn qua internet, chỉ mã hóa — không giải quyết yêu cầu private
>
> Mẹo nhớ: **S3 + DynamoDB = Gateway (miễn phí)** — chỉ 2 service này thôi, nhớ kỹ!
