## Câu hỏi 69

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A startup launched a fleet of on-demand Amazon EC2 instances to host a massively multiplayer online role-playing game (MMORPG). The EC2 instances are configured with Auto Scaling and AWS Systems Manager.
>
> What can be used to **configure the EC2 instances without having to establish an RDP or SSH connection** to each instance?

---

### 🔤 DỊCH CHI TIẾT

Một startup chạy một nhóm EC2 instances để host game MMORPG (game nhập vai trực tuyến nhiều người chơi). Các instances được cấu hình với **Auto Scaling** và **AWS Systems Manager** (công cụ quản lý tập trung tài nguyên AWS).

**Câu hỏi:** Dùng gì để cấu hình EC2 instances **mà không cần mở kết nối RDP hoặc SSH** đến từng máy?

**Từ khóa then chốt:** "without RDP or SSH" + "AWS Systems Manager" đã được cài sẵn

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Không dùng RDP/SSH** → không kết nối trực tiếp vào từng instance
2. **Cấu hình từ xa** trên nhiều instances cùng lúc (fleet)
3. **AWS Systems Manager đã được cấu hình** → gợi ý dùng tính năng bên trong Systems Manager

**Từ khóa nhận dạng:** "no SSH/RDP" + "Systems Manager" → **Run Command**

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Run Command** (thuộc AWS Systems Manager)

- Cho phép **thực thi lệnh từ xa** trên một hoặc nhiều EC2 instances **không cần SSH/RDP** ✅
- Hoạt động qua **SSM Agent** (agent cài sẵn trên instance) giao tiếp với Systems Manager ✅
- Có thể chạy từ **Console, CLI, SDK, PowerShell** ✅
- Hỗ trợ chạy lệnh đồng loạt trên **toàn bộ fleet** cùng lúc ✅
- Không cần mở inbound port SSH (22) hay RDP (3389) → **bảo mật hơn** ✅

---

❌ **AWS App Runner**

- Dịch vụ managed để **deploy và chạy containerized web app** từ source code hoặc container image ❌
- Không có khả năng thực thi lệnh hay cấu hình EC2 instances ❌

---

❌ **EC2Config**

- Chỉ là **initialization service** cho EC2 Windows instances cũ ❌
- Xử lý các tác vụ khởi tạo cơ bản (set password admin, cấu hình ban đầu) ❌
- Không hỗ trợ thực thi lệnh tập trung trên nhiều instances ❌

---

❌ **AWS CodePipeline**

- Công cụ **CI/CD** — tự động hóa build, test, deploy ứng dụng ❌
- Không cho phép chạy lệnh cấu hình trực tiếp trên EC2 instances ❌

---

### Note / Pipeline đúng

**Yêu cầu:** Cấu hình fleet EC2 không cần SSH/RDP, đã có Systems Manager.

```
Admin → AWS Console / CLI / SDK
→ AWS Systems Manager Run Command
→ Gửi lệnh đến SSM Agent (chạy sẵn trên từng EC2)
→ SSM Agent thực thi lệnh (install, configure, update...)
→ Trả kết quả về Console
(Không cần mở port SSH 22 / RDP 3389)
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Service | Mục đích chính | Thay SSH/RDP? |
|---|---|---|
| **SSM Run Command** | Thực thi lệnh từ xa trên EC2 | ✅ Đúng mục đích |
| **App Runner** | Deploy containerized web app | ❌ |
| **EC2Config** | Khởi tạo Windows EC2 cũ | ❌ |
| **CodePipeline** | CI/CD pipeline | ❌ |

> 💡 **Công thức ghi nhớ:**
> "No SSH/RDP" + "Systems Manager" → **Run Command**
> Run Command hoạt động qua **SSM Agent** → không cần mở port → bảo mật hơn SSH
>
> Mẹo nhớ: **Systems Manager = remote control EC2 không cần bàn phím vật lý**
