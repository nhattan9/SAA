## Câu hỏi 75

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A client is hosting their company website on a cluster of web servers that are behind a public-facing Application Load Balancer (AWS ALB). The client also uses Amazon Route 53 to manage their public DNS.
>
> How should the client configure the **DNS zone apex record** to point to the load balancer?

---

### 🔤 DỊCH CHI TIẾT

Client đang dùng **ALB** (Application Load Balancer) phía trước cụm web server, và dùng **Route 53** để quản lý DNS công khai. Cần cấu hình **DNS zone apex record** (bản ghi DNS tại đỉnh vùng — ví dụ: `tutorialsdojo.com`, không phải `www.tutorialsdojo.com`) để trỏ đến load balancer.

**Từ khóa then chốt:** "**zone apex**" + "load balancer DNS name"

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Zone apex** (ví dụ: `example.com`) — **không thể dùng CNAME** theo chuẩn DNS
2. IP của ALB **thay đổi liên tục** → không thể dùng A record trỏ thẳng vào IP
3. Giải pháp: **Alias A record** của Route 53 → trỏ vào DNS name của ALB

**Điểm dễ nhầm:** Nhiều người nghĩ dùng CNAME cho load balancer — đúng với subdomain (`www.`), nhưng **sai với zone apex** (`example.com`).

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án C:** Create an **A record aliased** to the load balancer DNS name.

- **Alias A record** là tính năng đặc biệt của Route 53 — trỏ đến DNS name của AWS resource (ALB, CloudFront, S3...) ✅
- Route 53 tự động resolve DNS name của ALB ra IP → trả về IP cho client ✅
- **Dùng được tại zone apex** (`example.com`) ✅
- IP ALB thay đổi → Route 53 tự cập nhật → không cần sửa record ✅
- **Miễn phí** — không tính phí query cho Alias record trỏ đến AWS resource ✅

---

❌ **Đáp án A:** Create an **A record pointing to the IP address** of the load balancer.

- IP của ALB **thay đổi liên tục** khi scale hoặc update ❌
- Hardcode IP vào A record → IP đổi là website hỏng ❌

---

❌ **Đáp án D:** Create a **CNAME record** pointing to the load balancer DNS name.

- **CNAME không được phép tại zone apex** theo chuẩn DNS (RFC) ❌
- `example.com` không thể là CNAME — chỉ subdomain như `www.example.com` mới được ❌

---

❌ **Đáp án B:** Create an **alias for CNAME record** to the load balancer DNS name.

- Không có khái niệm "alias for CNAME" trong Route 53 ❌
- Alias chỉ dùng với **A record hoặc AAAA record**, không phải CNAME ❌

---

### Note / Pipeline đúng

**Yêu cầu:** Zone apex (`example.com`) → trỏ đến ALB.

```
User gõ: example.com
→ Route 53 nhận query
→ Tra Alias A record: "example.com" alias → ALB DNS name
→ Route 53 resolve ALB DNS name ra IP hiện tại
→ Trả IP về cho user
→ User kết nối đến ALB → web server
(IP ALB có thể đổi bất kỳ lúc nào → Route 53 tự xử lý)
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Record Type | Zone Apex? | Trỏ đến ALB? | Ghi chú |
|---|---|---|---|
| **Alias A record** | ✅ | ✅ | Đúng giải pháp |
| A record (IP cứng) | ✅ | ❌ | IP ALB thay đổi → không dùng |
| CNAME | ❌ | ✅ | Không dùng được ở zone apex |
| Alias CNAME | ❌ | ❌ | Không tồn tại trong Route 53 |

> 💡 **Công thức ghi nhớ:**
> **Zone apex** (`example.com`) → **chỉ dùng Alias A record**
> **Subdomain** (`www.example.com`) → có thể dùng CNAME hoặc Alias A record
> **IP của ALB thay đổi** → không bao giờ hardcode IP vào A record
>
> Mẹo nhớ: **Zone apex + AWS resource = Alias A record** — đây là combo chuẩn của Route 53
