## Câu hỏi 66

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company has a web application hosted in their on-premises infrastructure that they want to migrate to AWS cloud. Your manager has instructed you to ensure that there is **no downtime** while the migration process is on-going. Your team decided to divert **50% of the traffic** to the new application in AWS and the other **50% to the application hosted in their on-premises** infrastructure. Once the migration is over, a full diversion to AWS will be implemented. The company's VPC is connected to its on-premises network via an **AWS Direct Connect** connection.
>
> Which of the following are the possible solutions that you can implement? **(Select TWO.)**

---

### 🔤 DỊCH CHI TIẾT

Công ty muốn migrate (di chuyển) web app từ on-premises lên AWS mà **không có downtime** (không gián đoạn dịch vụ). Trong quá trình migrate, cần chia traffic (lưu lượng) theo tỷ lệ **50/50** giữa AWS và on-premises. VPC kết nối on-premises qua **AWS Direct Connect** (kết nối chuyên dụng tốc độ cao).

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Chia traffic theo tỷ lệ cụ thể (weighted)** — 50% AWS, 50% on-premises → cần cơ chế **weighted routing**, không phải failover
2. **Không downtime** → traffic phải được điều phối liên tục, không ngắt
3. **Kết nối on-premises** → dùng Direct Connect → on-premises có IP private → ALB target type `ip` có thể đăng ký on-premises IP

**Điểm dễ nhầm:** Failover routing ≠ weighted routing. Failover = active-passive (chỉ chuyển khi có sự cố), không chia tỷ lệ được.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án B:** Use **Route 53 with Weighted routing policy** to divert traffic 50% AWS / 50% on-premises.

- **Weighted routing** (định tuyến theo trọng số) cho phép chia traffic theo tỷ lệ tùy chỉnh giữa nhiều endpoints ✅
- Có thể trỏ một record sang AWS, một record sang on-premises IP ✅
- Dễ điều chỉnh dần tỷ lệ về 100% AWS khi migration xong ✅

---

✅ **Đáp án C:** Use **Application Load Balancer with Weighted Target Groups** to divert traffic 50% AWS / 50% on-premises.

- **ALB hỗ trợ Weighted Target Groups** — chia traffic theo trọng số giữa nhiều target groups ✅
- Target type `ip` cho phép đăng ký **on-premises IP** (qua Direct Connect/VPN) vào target group ✅
- Hỗ trợ **blue-green, canary, hybrid deployment** không cần nhiều load balancer ✅

---

❌ **Đáp án A:** Use **Route 53 with Failover routing policy** to divert 50% / 50%.

- **Failover routing** = active-passive → chỉ chuyển traffic khi primary gặp sự cố ❌
- **Không thể chia tỷ lệ 50/50** bằng Failover routing ❌

---

❌ **Đáp án D:** Use **Network Load Balancer with Weighted Target Groups**.

- **NLB không hỗ trợ Weighted Target Groups** ❌
- Chỉ **ALB** mới có tính năng này ❌

---

❌ **Đáp án E:** Use **AWS Global Accelerator** + on-premises dùng AnyCast static IP + kết nối qua Direct Connect Gateway.

- Global Accelerator chỉ hỗ trợ endpoint là **AWS resources** (NLB, ALB, EC2, Elastic IP) — không đăng ký on-premises trực tiếp ❌
- **Direct Connect Gateway** không tự tạo kết nối on-premises → VPC, chỉ dùng để kết nối nhiều VPC ở nhiều Region qua một Direct Connect ❌
- **AnyCast IP** của Global Accelerator là do AWS cấp, không phải on-premises tự có ❌

---

### Note / Pipeline đúng

**Yêu cầu:** Migration hybrid, no downtime, chia traffic 50/50, có Direct Connect.

```
Giải pháp 1 — Route 53 Weighted Routing:
  DNS query → Route 53
  → 50% trả về IP AWS (EC2/ALB)
  → 50% trả về IP on-premises
  Điều chỉnh weight dần → 100% AWS khi xong

Giải pháp 2 — ALB Weighted Target Groups:
  Client → ALB
  → Target Group A (weight 50): EC2 trên AWS
  → Target Group B (weight 50): on-premises IP (qua Direct Connect)
  Điều chỉnh weight dần → 100% AWS khi xong
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Giải pháp | Hỗ trợ weighted? | Hỗ trợ on-premises? | Ghi chú |
|---|---|---|---|
| Route 53 Weighted | ✅ | ✅ | Chia theo DNS |
| ALB Weighted Target Groups | ✅ | ✅ (target type `ip` qua Direct Connect) | Chia theo LB rule |
| Route 53 Failover | ❌ | ✅ | Chỉ active-passive |
| NLB Weighted Target Groups | ❌ | — | NLB không có tính năng này |
| Global Accelerator | ✅ weight | ❌ on-premises | Chỉ endpoint AWS |

> 💡 **Công thức ghi nhớ:**
> "Chia traffic theo tỷ lệ" → **Weighted** (Route 53 hoặc ALB Target Groups)
> "Chuyển khi có sự cố" → **Failover** (Route 53)
> "NLB" → **không có Weighted Target Groups** (chỉ ALB mới có)
> "ALB target type ip" → đăng ký được **on-premises IP** qua Direct Connect/VPN
