## Câu hỏi 23

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A game development company operates several VR and AR games that use various RESTful web APIs hosted in its on-premises data center, which currently sits behind a CDN for faster global delivery. Due to the unprecedented growth of the company, the management decided to migrate its system to AWS Cloud to scale out its resources as well as to minimize costs.
>
> Which of the following is the most cost-effective and scalable solution to meet the above requirement?

---

### 🔤 DỊCH CHI TIẾT

Một công ty phát triển game vận hành nhiều game VR/AR sử dụng các **RESTful web API** (giao diện lập trình ứng dụng theo chuẩn REST) được host tại data center nội bộ, hiện đang đứng sau một **CDN** (Content Delivery Network — mạng phân phối nội dung) để tăng tốc phân phối toàn cầu. Do tăng trưởng đột biến, công ty muốn migrate lên AWS để **scale out** (mở rộng tài nguyên) và **minimize costs** (tối thiểu hóa chi phí).

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Cost-effective nhất** — trả tiền ít nhất có thể, tránh trả cho tài nguyên nhàn rỗi.
2. **Scalable** — tự động mở rộng khi traffic tăng đột biến (game có thể có spike lớn).

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

---

✅ **Đáp án đúng (C):** **AWS Lambda** + **Amazon API Gateway**

- **AWS Lambda** là serverless compute — chỉ tính tiền khi function thực sự chạy (theo số request và thời gian thực thi), hoàn toàn **không tốn tiền khi không có traffic**. ✅
- **Amazon API Gateway** là dịch vụ quản lý RESTful API fully managed, tự động scale theo request mà không cần cấu hình gì thêm. ✅
- Kết hợp Lambda + API Gateway = kiến trúc **serverless** chuẩn cho RESTful API → vừa rẻ nhất, vừa scale vô hạn theo nhu cầu. ✅

---

❌ **Đáp án sai (A):** Static S3 web hosting + CloudFront

- **S3 static hosting** chỉ phục vụ file tĩnh (HTML, CSS, JS, image) — **không có khả năng compute**, không thể chạy logic API động.
- RESTful web API cần xử lý logic phía server → S3 không đáp ứng được.

---

❌ **Đáp án sai (B):** Amazon ECS + ECR + AWS Fargate

- ECS/Fargate dùng để chạy **Docker container** — phù hợp cho microservice phức tạp, nhưng **vẫn phải trả tiền theo giờ** kể cả khi container không xử lý request.
- Không cost-effective bằng Lambda cho workload API theo kiểu request/response.

---

❌ **Đáp án sai (D):** Spot Fleet EC2 + Elastic Fabric Adapter (EFA) + ALB

- **EFA** (Elastic Fabric Adapter) là network adapter dành cho **HPC** (High Performance Computing) và **low-latency clustering** — hoàn toàn không liên quan đến RESTful API thông thường.
- **Spot Fleet không có Auto Scaling** trong mô tả này → không đảm bảo scalable.
- EC2 dù là Spot vẫn tốn tiền liên tục khi chạy → kém cost-effective hơn Lambda.

---

> 📝 **Note:** Từ khóa quyết định ở đây là **"RESTful API + cost-effective + scalable"**
>
> **Pipeline đúng:** `Client request` → `Amazon API Gateway` → `AWS Lambda` (chỉ chạy khi có request, tự scale, trả tiền theo lần dùng)

---

### 📊 KẾT LUẬN & BÀI HỌC

| Yêu cầu / Tính năng | Service phù hợp |
|---|---|
| RESTful API serverless, cost-effective | **Lambda + API Gateway** |
| Host file tĩnh (HTML, image...) | **S3 Static Hosting** |
| Chạy Docker container | **ECS + Fargate** |
| HPC / low-latency network clustering | **EC2 + EFA** |
| CDN / phân phối nội dung toàn cầu | **CloudFront** |

💡 **Công thức ghi nhớ:**

> **"RESTful API + serverless"** = **API Gateway + Lambda** → chỉ trả tiền khi có request

> **"Static website"** = **S3** → không có compute, không chạy được API động

> **"EFA"** = tín hiệu nhận ra ngay là **HPC/clustering** → không liên quan đến web API thông thường

> **Lambda vs EC2/Fargate về chi phí:** Lambda = trả theo ms thực thi; EC2/Fargate = trả theo giờ chạy → Lambda rẻ hơn nhiều cho workload có traffic không đều
