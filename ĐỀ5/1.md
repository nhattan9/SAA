## Câu hỏi 61

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> Two target groups with three EC2 instances each are added to an Application Load Balancer. Port 80 is allowed in the security group. However, the instances are still showing "out of service" from the load balancer.
>
> What could be the root cause?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

**"Out of service" từ ALB** — đây là trạng thái health check fail. Security group đã đúng (port 80 open) → nguyên nhân còn lại là **health check configuration sai**.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng B:** **Health check configuration is not properly defined.**

- ALB định kỳ gửi health check requests đến instances → nếu config sai (sai path, sai port, sai protocol) → instances fail health check → bị đánh dấu **"out of service"**. ✅
- Security group đã mở port 80 → vấn đề không phải network → health check config là nguyên nhân hợp lý nhất. ✅
- Ví dụ: health check path set là `/health` nhưng app không có endpoint đó → trả về 404 → fail. ✅

---

❌ **Đáp án sai A (đáp án bạn chọn):** **Wrong subnet**

- Subnet sai sẽ ảnh hưởng đến routing/connectivity, nhưng không trực tiếp gây ra "out of service" status từ health check khi security group đã đúng.

---

❌ **Đáp án sai C:** **Wrong AMI**

- AMI sai có thể khiến ứng dụng không chạy đúng, nhưng đây là trường hợp đặc biệt — không phải nguyên nhân trực tiếp nhất.

---

❌ **Đáp án sai D:** **Wrong instance type**

- Instance type không ảnh hưởng trực tiếp đến health check status.

---

**📝 Note — Mô tả yêu cầu chính:** Instance "out of service" trong ALB = fail health check → kiểm tra health check config trước.

**Health check configuration cần kiểm tra:**

```
Target Group Health Check Settings:
├── Protocol: HTTP / HTTPS
├── Port: traffic port / custom port
├── Path: / hay /health hay /api/status...
├── Healthy threshold: số lần success liên tiếp
├── Unhealthy threshold: số lần fail liên tiếp
├── Timeout: thời gian chờ response
└── Interval: tần suất check
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Triệu chứng | Nguyên nhân phổ biến |
|---|---|
| **"Out of service" / Unhealthy** | Health check config sai (path, port, protocol) |
| **Connection timeout** | Security group chưa mở port |
| **502 Bad Gateway** | Instance down hoặc app crash |
| **Instances không nhận traffic** | Cross-zone LB tắt hoặc listener rule sai |

💡 **Công thức ghi nhớ:**

> **"Out of service trong ALB"** = fail health check → kiểm tra **health check configuration** trước tiên

> **Khi security group đã đúng** → nguyên nhân tiếp theo là **health check path/port/protocol sai**

> **Health check** phải trả về **HTTP 200** mới được tính là healthy → nếu path sai trả về 404/500 → fail
