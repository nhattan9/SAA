## Câu hỏi 78

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company runs an application in an AWS Auto Scaling group of EC2 instances across multiple AZs behind an ALB. On several occasions, some instances have been **automatically terminated** after failing HTTPS health checks, which results in the **loss of ephemeral logs** stored locally on the instance. A Solutions Architect must design a solution that ensures **application and server logs are collected reliably**, even if the Auto Scaling group **immediately terminates** the instance.
>
> What is the **EASIEST** way for the Architect to automate the log collection from the EC2 instances?

---

### 🔤 DỊCH CHI TIẾT

EC2 instances bị terminate đột ngột khi fail health check → **logs lưu trên instance bị mất** (ephemeral = tạm thời, mất khi instance tắt). Cần thu thập logs **trước khi terminate**, theo cách **dễ nhất**.

**Từ khóa then chốt:** "immediately terminates" + "easiest" + "logs before termination"

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Delay việc terminate** để kịp thu thập logs → **Lifecycle Hook** (Terminating → Terminating:Wait)
2. **Trigger hành động** khi instance sắp bị terminate → **EventBridge rule**
3. **Thu thập logs dễ nhất** → **CloudWatch Agent** (không cần viết script thủ công)

**Điểm dễ nhầm:**
- `Pending:Wait` = dành cho **scale-out** (launch), không phải terminate
- `Terminating:Wait` = dành cho **scale-in** (terminate) ✅
- SSM Run Command cần viết script thủ công → không phải "easiest"
- Step Functions không phù hợp để collect logs

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án B:** Lifecycle hook → **Terminating:Wait** + EventBridge rule (EC2 Instance-terminate Lifecycle Action) + Lambda + **CloudWatch Agent** push logs → resume termination.

- `Terminating:Wait` = đúng state để delay terminate ✅
- **EventBridge** bắt event terminate → trigger Lambda ✅
- **CloudWatch Agent** = cách đơn giản nhất để push logs lên CloudWatch Logs (không cần viết script) ✅
- Sau khi push xong → resume termination ✅

---

❌ **Đáp án A:** Lifecycle hook → Terminating:Wait + **AWS Step Functions** collect logs.

- **Step Functions** không phải công cụ thu thập logs từ EC2 ❌
- Phức tạp không cần thiết ❌

---

❌ **Đáp án C:** Lifecycle hook → **Pending:Wait** + EventBridge + SSM Automation script.

- **Pending:Wait** = state của **scale-out** (launch instance), không phải terminate ❌
- Dùng sai state hoàn toàn ❌

---

❌ **Đáp án D:** Lifecycle hook → Terminating:Wait + EventBridge (**EC2 Instance Terminate Successful**) + SSM Run Command script.

- **"EC2 Instance Terminate Successful"** = event khi instance **đã terminate xong** → quá muộn để collect logs ❌
- Phải dùng **"EC2 Instance-terminate Lifecycle Action"** = event khi instance **sắp** terminate ❌
- SSM Run Command cần viết script thủ công → không phải "easiest" ❌

---

### Note / Pipeline đúng

**Yêu cầu:** Collect logs trước khi instance bị terminate, cách dễ nhất.

```
Instance fail health check
→ Auto Scaling quyết định terminate
→ Lifecycle Hook can thiệp:
   Terminating → Terminating:Wait (tạm dừng terminate)
       │
       ▼
EventBridge bắt event "EC2 Instance-terminate Lifecycle Action"
       │
       ▼
Trigger Lambda function
       │
       ▼
Lambda ra lệnh CloudWatch Agent trên instance
push logs lên CloudWatch Logs
       │
       ▼
Logs đã được lưu an toàn ✅
       │
       ▼
Lambda gọi CompleteLifecycleAction → resume terminate
       │
       ▼
Instance bị terminate bình thường
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Lifecycle State | Dùng khi nào |
|---|---|
| **Pending:Wait** | Scale-out (instance đang được launch) |
| **Pending:Proceed** | Sau khi hoàn tất action lúc launch |
| **Terminating:Wait** | Scale-in (instance sắp bị terminate) ✅ |
| **Terminating:Proceed** | Sau khi hoàn tất action lúc terminate |
| **Terminated** | Instance đã bị xóa hoàn toàn |

| EventBridge Event | Ý nghĩa |
|---|---|
| **EC2 Instance-terminate Lifecycle Action** | Instance **sắp** terminate → dùng để collect logs ✅ |
| **EC2 Instance Terminate Successful** | Instance **đã** terminate xong → quá muộn ❌ |

> 💡 **Công thức ghi nhớ:**
> "Collect logs trước terminate" → **Lifecycle Hook (Terminating:Wait)** + **EventBridge** + **Lambda** + **CloudWatch Agent**
> **CloudWatch Agent** = easiest way to push logs (không cần script thủ công)
> **Pending:Wait** = launch, **Terminating:Wait** = terminate — đừng nhầm!
