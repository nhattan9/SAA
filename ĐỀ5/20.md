## Câu hỏi 81

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company troubleshoots operational issues by logging the AWS API call history of all AWS resources. The Solutions Architect must implement a solution to quickly identify the most recent changes made to resources, including **creation, modification, and deletion** of AWS resources. One of the requirements is that the **generated log files should be encrypted**.
>
> Which of the following is the most suitable approach to implement the encryption?

---

### 🔤 DỊCH CHI TIẾT

Công ty cần ghi lại **lịch sử API calls** để troubleshoot và theo dõi thay đổi tài nguyên AWS (tạo, sửa, xóa). Log files phải được **mã hóa**. Câu hỏi hỏi cách implement mã hóa phù hợp nhất.

**Từ khóa then chốt:** "API call history" → **CloudTrail**. "Encrypted logs" → cần biết CloudTrail có mã hóa mặc định không.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Ghi lại API call history** → **AWS CloudTrail** (dịch vụ duy nhất làm việc này)
2. **Log files phải được mã hóa** → CloudTrail **đã mã hóa mặc định** bằng S3 SSE
3. Không cần cấu hình thêm gì → **default settings là đủ**

**Bẫy:** Các đáp án kia đều đề xuất cấu hình thêm SSE — nhưng CloudTrail đã làm điều đó **tự động**.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án D:** Use **CloudTrail with its default settings**.

- CloudTrail **mặc định** mã hóa log files bằng **Amazon S3 SSE (AES-256)** ✅
- Log lưu vào **S3** và tự động được mã hóa ✅
- Không cần cấu hình thêm ✅
- Nếu muốn mã hóa mạnh hơn có thể thêm **AWS KMS key** (tuỳ chọn) ✅

---

❌ **Đáp án A:** Use CloudTrail + configure S3 bucket to use **SSE**.

- CloudTrail đã dùng SSE mặc định → cấu hình thêm là **thừa** ❌
- Không sai về kỹ thuật nhưng **không cần thiết** ❌

---

❌ **Đáp án B:** Use CloudTrail + configure **Amazon Glacier** to use SSE.

- CloudTrail lưu log vào **S3**, không phải Glacier ❌
- Sai về destination storage ❌

---

❌ **Đáp án C:** Use CloudTrail + configure S3 with SSE + **AES-128**.

- CloudTrail đã dùng SSE mặc định → thừa ❌
- **SSE-S3 chỉ dùng AES-256**, không có AES-128 ❌
- Sai cả về thuật toán mã hóa ❌

---

### Note / Pipeline đúng

**Yêu cầu:** Track API calls + mã hóa logs.

```
AWS API calls (Console/CLI/SDK)
→ CloudTrail tự động ghi lại
→ Log files lưu vào S3
→ S3 SSE (AES-256) mã hóa tự động ✅ (mặc định)
→ (Tùy chọn) thêm KMS key để kiểm soát mã hóa chặt hơn

Không cần cấu hình gì thêm cho mã hóa!
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Đặc điểm CloudTrail | Chi tiết |
|---|---|
| Lưu log ở đâu | **Amazon S3** (không phải Glacier) |
| Mã hóa mặc định | **S3 SSE (AES-256)** — tự động |
| Mã hóa nâng cao | AWS KMS key (tùy chọn thêm) |
| Thuật toán SSE-S3 | **AES-256** (không có AES-128) |
| Retention | Tùy chỉnh bằng S3 Lifecycle rules |

> 💡 **Công thức ghi nhớ:**
> **CloudTrail = mặc định đã mã hóa** bằng S3 SSE-AES-256
> **Không cần cấu hình thêm** cho encryption cơ bản
> **SSE-S3 = AES-256** — không phải AES-128, không bao giờ là AES-128
> CloudTrail lưu vào **S3**, không phải Glacier
