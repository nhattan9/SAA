## Câu hỏi 65

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company needs to accelerate the performance of its AI-powered medical diagnostic application by running its machine learning workloads on the edge of telecommunication carriers' 5G networks. The application must be deployed to a Kubernetes cluster and have role-based access control (RBAC) access to IAM users and roles for cluster authentication.
>
> Which of the following should the Solutions Architect implement to ensure single-digit millisecond latency for the application?

---

### 🔤 DỊCH CHI TIẾT

Công ty cần tăng tốc hiệu suất ứng dụng chẩn đoán y tế AI bằng cách chạy workload machine learning tại **edge** (biên) của mạng **5G** của các nhà mạng viễn thông. Ứng dụng phải:
- Triển khai trên **Kubernetes cluster** (cụm container orchestration)
- Có **RBAC** (Role-Based Access Control — kiểm soát truy cập theo vai trò) dùng IAM users và roles để xác thực cluster

**Câu hỏi:** Giải pháp nào đảm bảo độ trễ **single-digit millisecond** (dưới 10ms)?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Single-digit millisecond latency tại 5G edge** → cần hạ tầng AWS đặt ngay trong data center của nhà mạng 5G → chỉ có **AWS Wavelength Zones** đáp ứng
2. **Kubernetes cluster** → **Amazon EKS** (Elastic Kubernetes Service)
3. **RBAC với IAM users/roles** → cơ chế chuẩn của EKS là **aws-auth ConfigMap** (không cần cài thêm tool thủ công)

**Điểm dễ nhầm:** RBAC trong EKS không dùng Control Tower, không cần cài aws-iam-authenticator thủ công — chỉ cần apply **aws-auth ConfigMap**.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án D:** Launch the application to an Amazon EKS cluster. Create node groups in **AWS Wavelength Zones** for the EKS cluster via the Wavelength service. Apply the **aws-auth ConfigMap** to your cluster.

- **AWS Wavelength Zones** nhúng AWS compute vào data center của nhà mạng 5G → traffic không cần ra internet → latency cực thấp ✅
- **aws-auth ConfigMap** là cơ chế chính thức để map IAM users/roles → Kubernetes RBAC groups ✅
- Tự động được tạo khi tạo managed node group → không cần cài thủ công ✅

---

❌ **Đáp án A:** Launch EKS cluster. Configure **Amazon VPC CNI plugin** for Kubernetes to manage pod networking in AWS Wavelength Zones. Install **aws-iam-authenticator** to your cluster.

- **VPC CNI plugin** chỉ dùng để cấp IP từ VPC cho pods — đây là networking model mặc định, **không liên quan đến việc deploy tại Wavelength** ❌
- **Cài aws-iam-authenticator thủ công là thừa** — EKS đã tích hợp sẵn IAM Authenticator trên control plane ❌

---

❌ **Đáp án C:** Host EKS cluster. Set up node groups in AWS Wavelength Zones. Attach **AmazonECSConnectorAgentRole** to cluster. Use **AWS Control Tower** for RBAC access.

- **AmazonECS**ConnectorAgentRole → sai hoàn toàn: đây là role để kết nối **external Kubernetes cluster** vào AWS Console, không phải cho RBAC ❌
- **AWS Control Tower** → dùng để quản lý multi-account environment, **không cấp RBAC** cho EKS cluster ❌

---

❌ **Đáp án B:** Host EKS + run pods on **AWS Fargate**. Create node groups in Wavelength Zones. Add **AmazonEKSFargatePodExecutionRole** và ensure Fargate profile has **same IAM role** as EC2 node groups.

- Fargate profile **phải có IAM role RIÊNG BIỆT** so với EC2 node groups — **không được dùng chung** ❌
- Đây là lỗi cấu hình bảo mật nghiêm trọng ❌

---

### Note / Pipeline đúng

**Yêu cầu chính:** Chạy ML workload tại 5G edge với latency < 10ms, dùng EKS + RBAC bằng IAM.

```
Traffic từ thiết bị 5G
→ Vào Wavelength Zone (AWS compute trong data center nhà mạng)
   → Không cần hop ra internet → latency < 10ms ✅
→ EKS cluster với node groups trong Wavelength Zone
→ aws-auth ConfigMap map IAM User/Role → Kubernetes RBAC
→ Ứng dụng AI chạy tại edge với độ trễ cực thấp
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Yêu cầu | Giải pháp đúng |
|---|---|
| Latency < 10ms tại 5G edge | **AWS Wavelength Zones** |
| Kubernetes managed | **Amazon EKS** |
| RBAC với IAM | **aws-auth ConfigMap** |
| Pod networking trong VPC | VPC CNI plugin (mặc định, không cần cấu hình thêm) |
| Fargate IAM | Role **riêng biệt** với EC2 node group |

> 💡 **Công thức ghi nhớ:**
> "5G edge + latency < 10ms" → **AWS Wavelength Zones**
> "EKS RBAC với IAM" → **aws-auth ConfigMap** (tự động tạo, không cài thủ công)
> "Fargate profile IAM" ≠ "EC2 node group IAM" → **luôn tách biệt**
>
> Mẹo nhớ bẫy: **ECS**ConnectorAgentRole ≠ EKS → nhìn chữ **ECS** là loại ngay!
