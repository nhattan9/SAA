## Câu hỏi 42

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A solutions architect is designing a secure, cost-effective, and highly available storage solution. Requirements: preserve previous versions of files, comply with strict regulatory requirements mandating data retention in an immutable state for at least 3 years, then archive. Once archived, data will only be accessed once a year.
>
> How should the solutions architect build the solution?

---

### 🔤 DỊCH CHI TIẾT

Yêu cầu gồm 3 phần: (1) giữ **previous versions** của file khi có file mới upload, (2) dữ liệu phải ở trạng thái **immutable (không thể sửa/xóa)** trong ít nhất 3 năm theo regulatory compliance, (3) sau 3 năm chuyển sang **archive** với tần suất truy cập cực thấp (1 lần/năm).

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Bài toán có 4 yêu cầu kỹ thuật cụ thể:

1. **Versioning** → bật S3 Object Versioning để giữ previous versions.
2. **Immutable storage (WORM)** → S3 Object Lock để ngăn sửa/xóa trong retention period.
3. **Regulatory compliance nghiêm ngặt** → phải dùng **Compliance Mode**, không phải Governance Mode.
4. **Archive sau 3 năm, truy cập 1 lần/năm** → S3 Glacier Deep Archive (rẻ nhất).

---

### 🔑 PHÂN BIỆT: Compliance Mode vs Governance Mode

| | Compliance Mode | Governance Mode |
|---|---|---|
| **Ai có thể xóa/sửa?** | **Không ai** — kể cả root account | Một số users có permission đặc biệt |
| **Dùng cho** | Regulatory compliance nghiêm ngặt | Kiểm soát nội bộ, linh hoạt hơn |
| **Mức bảo vệ** | Cao nhất | Thấp hơn |

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: S3 Standard + Object Versioning + S3 Object Lock Compliance Mode + Lifecycle rule → Glacier Deep Archive sau 3 năm.**

- S3 Standard → durability cao nhất (11 9's), multi-AZ. ✅
- Versioning → giữ previous versions khi có file mới. ✅
- Object Lock **Compliance Mode** → WORM, **không ai có thể xóa hay sửa** kể cả root user — đáp ứng regulatory compliance nghiêm ngặt nhất. ✅
- Lifecycle rule → tự động chuyển sang Glacier Deep Archive sau 3 năm → cost-effective cho data truy cập 1 lần/năm. ✅

---

❌ **Đáp án sai 1: S3 Standard + Versioning + Lifecycle → Glacier Deep Archive sau 3 năm (thiếu Object Lock).**

- Versioning và lifecycle đúng hướng, nhưng **thiếu Object Lock** → dữ liệu vẫn có thể bị xóa hoặc sửa trong 3 năm. ✗
- Không đáp ứng yêu cầu **immutable state** theo regulatory compliance. ✗
- Đây là bẫy phổ biến — trông có vẻ đủ nhưng thiếu đúng 1 thành phần quan trọng nhất. ✗

---

❌ **Đáp án sai 2: S3 Standard + Object Lock Governance Mode (không có Versioning, không có Lifecycle).**

- **Governance Mode** cho phép một số users có quyền đặc biệt xóa lock → không đủ nghiêm ngặt cho regulatory compliance. ✗
- Thiếu Versioning → không giữ được previous versions. ✗
- Thiếu Lifecycle rule → không tự động archive sau 3 năm. ✗

---

❌ **Đáp án sai 3: S3 One Zone-IA + Versioning + Lifecycle → Glacier Deep Archive sau 3 năm.**

- S3 One Zone-IA chỉ lưu ở **1 AZ** → durability thấp hơn, không phù hợp cho compliance data quan trọng. ✗
- Nếu AZ đó gặp sự cố → có thể mất dữ liệu → vi phạm yêu cầu highly available. ✗
- Thiếu Object Lock Compliance Mode → không immutable. ✗

---

### 📊 KẾT LUẬN & BÀI HỌC

| Yêu cầu | Giải pháp |
|---|---|
| Giữ previous versions | **S3 Versioning** |
| Immutable, không ai xóa được kể cả root | **S3 Object Lock — Compliance Mode** |
| Kiểm soát nội bộ, một số admin có thể gỡ lock | **S3 Object Lock — Governance Mode** |
| Archive sau retention period, truy cập 1 lần/năm | **S3 Glacier Deep Archive + Lifecycle Rule** |
| Highly available, critical compliance data | **S3 Standard** (không dùng One Zone-IA) |

💡 **Công thức ghi nhớ:**

> "Regulatory compliance, immutable, không ai được xóa" → **Object Lock Compliance Mode**

> "Internal governance, linh hoạt hơn" → **Object Lock Governance Mode**

> **Object Lock yêu cầu Versioning phải được bật trước** — không bật Versioning thì không dùng được Object Lock

> "Archive dài hạn, truy cập cực ít" → **Glacier Deep Archive** (rẻ nhất)

> **Compliance Mode = WORM tuyệt đối** — không có ngoại lệ kể cả root account AWS.
