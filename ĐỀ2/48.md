## Câu hỏi 48

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> An organization stores financial records on-premises (almost out of space) and wants to move all records to cloud storage. All future records will also be stored in the cloud. Records must be prevented from being deleted or overwritten. Once migrated, records will be accessed less than once a month but must be available immediately if audited (S3 Standard-IA).
>
> Which solution meets the requirements?

---

### 🔤 DỊCH CHI TIẾT

Tổ chức cần: (1) **migrate toàn bộ** dữ liệu on-premises lên cloud, (2) **ngăn xóa/ghi đè** dữ liệu (immutable), (3) truy cập **ít hơn 1 lần/tháng** nhưng **sẵn sàng ngay lập tức** khi audit → S3 Standard-IA.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Bài toán có 3 yêu cầu cốt lõi:

1. **Migrate data từ on-premises lên cloud** → AWS DataSync (công cụ migration dữ liệu).
2. **Immutable — không thể xóa/ghi đè** → S3 Object Lock.
3. **Object Lock chỉ có trên S3** → không có trên EFS, EBS hay dịch vụ khác.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: AWS DataSync để migrate + Amazon S3 + enable Object Lock.**

- DataSync được thiết kế để **migrate lượng lớn dữ liệu** từ on-premises vào S3 — hỗ trợ initial full copy và incremental transfers sau đó. ✅
- S3 hỗ trợ **Object Lock** → WORM protection, ngăn xóa/ghi đè trong retention period. ✅
- S3 Standard-IA → available immediately, truy cập ít → đúng yêu cầu audit. ✅

---

❌ **Đáp án sai 1: AWS DataSync + Amazon EFS + Object Lock.**

- **Amazon EFS chỉ hỗ trợ file locking** (khóa file khi đang được dùng) — hoàn toàn khác Object Lock. ✗
- **S3 Object Lock là tính năng độc quyền của Amazon S3** — EFS không có. ✗
- DataSync đúng nhưng storage sai. ✗

---

❌ **Đáp án sai 2: Multi-Region Replication để transfer on-premises records vào S3 + Object Lock.**

- **Multi-Region Replication** là tính năng của S3 để **replicate objects giữa các S3 buckets** — không thể transfer từ on-premises vào S3. ✗
- Đây là bẫy thuật ngữ: "transfer" nghe giống migrate nhưng thực ra chỉ copy giữa S3 buckets đã có. ✗
- Cần dùng DataSync hoặc Snowball để đưa dữ liệu từ on-premises vào S3. ✗

---

❌ **Đáp án sai 3: AWS Storage Gateway + Amazon EBS + Object Lock.**

- **Amazon EBS không hỗ trợ Object Lock** — EBS là block storage cho EC2, không có tính năng immutable object. ✗
- Storage Gateway phù hợp hơn cho **hybrid cloud** (vừa on-premises vừa cloud) với ongoing access, không phải pure migration. ✗
- Sai cả storage service lẫn tính năng. ✗

---

### 📊 KẾT LUẬN & BÀI HỌC

| Yêu cầu | Giải pháp đúng |
|---|---|
| Migrate data từ on-premises vào AWS | **AWS DataSync** |
| Immutable, ngăn xóa/ghi đè | **S3 Object Lock** |
| Object Lock hỗ trợ trên service nào? | **Chỉ Amazon S3** |
| Truy cập ít, sẵn sàng ngay khi cần | **S3 Standard-IA** |
| Hybrid: vừa on-premises vừa cloud ongoing | **AWS Storage Gateway** |

💡 **Công thức ghi nhớ:**

> "Migrate on-premises data lên AWS" → **AWS DataSync** (full migration)

> "Immutable, không thể xóa/ghi đè" → **S3 Object Lock** — chỉ có trên **S3**, không có trên EFS hay EBS

> **DataSync vs Storage Gateway**: DataSync = **di chuyển** data (one-time/scheduled migration) | Storage Gateway = **hybrid access** (on-premises app vẫn dùng được cloud storage)

> **Multi-Region Replication** = copy giữa S3 buckets, không phải từ on-premises → S3

> **EFS file locking ≠ S3 Object Lock** — hai cơ chế hoàn toàn khác nhau, không thay thế được nhau.
