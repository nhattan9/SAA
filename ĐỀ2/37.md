## Câu hỏi 37

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A DevOps Engineer is planning a highly available and fault-tolerant architecture with an Elastic Load Balancer and Auto Scaling group across multiple AZs. The online accounting application requires path-based routing, host-based routing, and bi-directional streaming using gRPC.
>
> Which configuration will satisfy the given requirement?

---

### 🔤 DỊCH CHI TIẾT

Kiến trúc cần: HA/fault-tolerant trên nhiều AZ, hỗ trợ **path-based routing** (định tuyến theo URL path), **host-based routing** (định tuyến theo hostname), và **bi-directional streaming qua gRPC** (giao thức RPC hai chiều dùng HTTP/2).

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Bài toán có 3 yêu cầu kỹ thuật cụ thể:

1. **Path-based và host-based routing** → chỉ có ALB (Layer 7) mới hỗ trợ — NLB và GLB không có tính năng này.
2. **gRPC (bi-directional streaming)** → gRPC hoạt động ở **Layer 7 trên HTTP/2** — chỉ ALB hỗ trợ gRPC protocol version.
3. **Highly available, multi-AZ** → Auto Scaling group trải đều các AZ — tiêu chuẩn với mọi loại LB.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Cấu hình Application Load Balancer, chọn gRPC làm protocol version.**

- ALB hoạt động ở **Layer 7** → hỗ trợ đầy đủ path-based routing, host-based routing, HTTP header routing. ✅
- ALB hỗ trợ **gRPC protocol version** → route và load balance gRPC traffic giữa microservices và clients. ✅
- Không cần thay đổi infrastructure phía client hay service khi dùng gRPC qua ALB. ✅

---

❌ **Đáp án sai 1: Network Load Balancer + UDP listener.**

- NLB hoạt động ở **Layer 4 (TCP/UDP)** → không hiểu nội dung HTTP, không có path-based hay host-based routing. ✗
- NLB **không hỗ trợ gRPC**. ✗
- UDP listener càng không liên quan đến gRPC (gRPC dùng TCP/HTTP2). ✗

---

❌ **Đáp án sai 2: Gateway Load Balancer + GENEVE protocol port 6081.**

- GLB hoạt động ở **Layer 3 (Network) và Layer 4** — dùng để integrate third-party virtual network appliances (firewall, IDS/IPS). ✗
- GENEVE protocol là tunnel protocol của GLB để forward traffic đến appliances — hoàn toàn không liên quan gRPC. ✗
- gRPC là Layer 7 → GLB không thể xử lý. ✗

---

❌ **Đáp án sai 3: Network Load Balancer + AWS Global Accelerator.**

- NLB vẫn không hỗ trợ path-based/host-based routing hay gRPC. ✗
- Global Accelerator tối ưu routing qua AWS global network thay vì internet công cộng — không thêm tính năng Layer 7 cho NLB. ✗
- Kết hợp này không giải quyết bất kỳ yêu cầu kỹ thuật nào trong bài. ✗

---

### 📊 KẾT LUẬN & BÀI HỌC

| Tiêu chí | ALB | NLB | GLB |
|---|---|---|---|
| **Layer OSI** | Layer 7 (Application) | Layer 4 (Transport) | Layer 3+4 (Network+Transport) |
| **Path-based routing** | ✅ | ❌ | ❌ |
| **Host-based routing** | ✅ | ❌ | ❌ |
| **gRPC support** | ✅ | ❌ | ❌ |
| **Protocol** | HTTP, HTTPS, gRPC | TCP, UDP, TLS | GENEVE (port 6081) |
| **Use case** | Web app, microservices, API | TCP/UDP, ultra-low latency | Network appliances (firewall, IDS) |

💡 **Công thức ghi nhớ:**

> "Path-based routing, host-based routing" → chỉ **ALB**

> "gRPC, HTTP/2, WebSocket" → chỉ **ALB**

> "Ultra-low latency, TCP/UDP, static IP" → **NLB**

> "Firewall, IDS/IPS third-party appliance" → **GLB** + GENEVE protocol

> **gRPC = Layer 7** → bất kỳ câu hỏi nào nhắc đến gRPC → đáp án luôn là **ALB**
