## Câu hỏi 19

### 📋 NỘI DUNG CÂU HỎI

> *A GraphQL API hosted in an Amazon EKS cluster with AWS Fargate launch type. The API is connected to an Amazon DynamoDB table with DAX as its data store. A solutions architect must improve network security by preventing database calls from traversing the public internet. An automated cross-account backup for the DynamoDB table is also required for long-term retention.*

Có 2 yêu cầu chính:
1. **Ngăn traffic đến DynamoDB đi qua internet** *(prevent traversing public internet)*
2. **Backup tự động sang account AWS khác** *(cross-account backup)*

---

### 🧠 Hiểu vấn đề trước

**DynamoDB là public resource** — mặc định traffic từ VPC phải đi qua Internet Gateway mới đến được DynamoDB. Để giữ traffic trong mạng nội bộ AWS → cần **VPC Endpoint**.

DynamoDB có **2 loại endpoint:**

| | **Gateway Endpoint** | **Interface Endpoint** |
|---|---|---|
| Chi phí | **Miễn phí** | Tốn tiền |
| Cách hoạt động | Cập nhật **route table** | Tạo **ENI** trong subnet |
| Dùng cho | **S3 và DynamoDB** | Hầu hết các AWS service khác |

→ DynamoDB chỉ hỗ trợ **Gateway Endpoint**, không phải Interface Endpoint.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** Tạo **DynamoDB Gateway Endpoint** + gắn vào **route table** + dùng **AWS Backup** để copy sang account khác

- **Gateway Endpoint** → traffic đến DynamoDB đi trong mạng AWS, không qua internet ✅
- Gắn vào **route table** → đúng cách cấu hình Gateway Endpoint ✅
- **AWS Backup** → hỗ trợ **cross-account backup** cho DynamoDB ✅

---

**❌ Đáp án sai 1:** DynamoDB **Interface** Endpoint + Route Table + **PITR**

- DynamoDB **không có Interface Endpoint** → sai ngay từ đầu.
- **PITR (Point-in-Time Recovery)** chỉ khôi phục trong **cùng account và region**, không hỗ trợ cross-account backup.

---

**❌ Đáp án sai 2:** DynamoDB **Gateway** Endpoint + **NACL rule** + **on-demand backup**

- Gateway Endpoint đúng, nhưng thêm NACL rule là **không cần thiết** — Gateway Endpoint đã tự xử lý routing rồi.
- **DynamoDB on-demand backup không thể copy sang account khác** → phải dùng AWS Backup.

---

**❌ Đáp án sai 3:** DynamoDB **Interface** Endpoint + **AWS Network Firewall** + **Amazon Timestream**

- DynamoDB **không có Interface Endpoint** → sai.
- **Amazon Timestream** là database **time-series** cho IoT, không liên quan gì đến backup DynamoDB.
- Giải pháp hoàn toàn không phù hợp.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Yêu cầu | Giải pháp |
|---|---|
| Traffic DynamoDB không qua internet | **DynamoDB Gateway Endpoint** |
| Cấu hình Gateway Endpoint | Gắn vào **route table** |
| Cross-account backup DynamoDB | **AWS Backup** |
| Khôi phục về thời điểm cụ thể (cùng account) | **PITR** |

**💡 Công thức ghi nhớ:**
> **DynamoDB + S3** → dùng **Gateway Endpoint** *(miễn phí, gắn vào route table)*
>
> Các service khác → dùng **Interface Endpoint** *(tốn tiền, tạo ENI trong subnet)*
>
> **Cross-account backup DynamoDB** → chỉ có **AWS Backup** mới làm được, on-demand backup thông thường **không thể** copy sang account khác.
