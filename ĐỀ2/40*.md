## Câu hỏi 40

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company has an on-premises MySQL database that needs to be replicated in Amazon S3 as CSV files. The database will eventually be launched on Aurora Serverless and accessed through RDS Proxy. Once data has been fully copied, ongoing changes should be continually streamed into S3. The company wants a solution with little management overhead yet highly secure.
>
> Which ingestion pattern should a solutions architect take?

---

### 🔤 DỊCH CHI TIẾT

Công ty cần: (1) **copy toàn bộ** MySQL on-premises lên S3 dưới dạng CSV, (2) **liên tục stream ongoing changes** lên S3 sau khi copy xong, (3) **bảo mật cao**, (4) **ít operational overhead**. Sau này sẽ migrate lên Aurora Serverless với RDS Proxy.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Bài toán có 4 yêu cầu cốt lõi:

1. **Full load migration** → copy toàn bộ dữ liệu hiện tại.
2. **Change Data Capture (CDC)** → bắt tiếp ongoing changes sau khi full load xong.
3. **Output là CSV trong S3** → DMS hỗ trợ sẵn CSV format khi dùng S3 làm target.
4. **Highly secure + ít overhead** → dùng SSL/TLS cho DMS endpoint, không cần thêm dịch vụ phức tạp.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Tạo full load + CDC replication task bằng AWS DMS. Thêm CA certificate và tạo DMS endpoint với SSL.**

- AWS DMS hỗ trợ đồng thời **full load và CDC** trong một replication task — đúng cả 2 yêu cầu đầu. ✅
- Khi S3 là target, DMS mặc định xuất dữ liệu dưới dạng **CSV** (hoặc Parquet nếu muốn). ✅
- Bảo mật kết nối bằng **SSL/TLS** qua CA certificate upload thẳng vào DMS console — đơn giản, không cần dịch vụ thêm. ✅
- Fully managed service → **ít operational overhead** nhất trong các lựa chọn. ✅

---

❌ **Đáp án sai 1: AWS DMS full load only + tạo DMS endpoint SSL qua AWS Network Firewall.**

- **Full load only** không capture ongoing changes → thiếu CDC → vi phạm yêu cầu stream liên tục. ✗
- **AWS Network Firewall** không có chức năng tạo hay quản lý DMS endpoint với SSL — đây là tính năng của DMS console. ✗
- Network Firewall là dịch vụ lọc traffic tại VPC perimeter, không liên quan đến certificate management. ✗

---

❌ **Đáp án sai 2: Snowball Edge cluster migrate lên S3 + DataSync capture ongoing changes + custom KMS envelope key.**

- Snowball Edge cluster phù hợp cho **dữ liệu cực lớn (petabytes)** cần nhiều thiết bị vật lý — quá phức tạp cho bài toán này. ✗
- DataSync có thể sync files nhưng không phải công cụ tối ưu để capture database CDC. ✗
- Custom KMS envelope key tạo thêm complexity không cần thiết. ✗
- Tổng thể: nhiều dịch vụ, nhiều bước thủ công → **high operational overhead**. ✗

---

❌ **Đáp án sai 3: AWS SCT convert MySQL sang CSV + AWS Application Migration Service (MGN) capture ongoing changes.**

- **AWS SCT** (Schema Conversion Tool) chuyển đổi **database schema** (cấu trúc) từ engine này sang engine khác — không dùng để replicate hay export data. ✗
- **AWS MGN** (Application Migration Service) là dịch vụ **lift-and-shift** toàn bộ server/application (VM, physical) lên AWS — không phải database CDC tool. ✗
- Cả hai dịch vụ đều bị dùng sai mục đích hoàn toàn. ✗

---

### 📊 KẾT LUẬN & BÀI HỌC

| Dịch vụ | Vai trò thực sự |
|---|---|
| **AWS DMS** | Migrate database + CDC ongoing replication, hỗ trợ S3 CSV output |
| **AWS SCT** | Chuyển đổi database **schema** (structure), không phải data replication |
| **AWS MGN** | Lift-and-shift toàn bộ **server/application**, không phải database CDC |
| **AWS DataSync** | Sync files/objects giữa storage systems, không tối ưu cho database CDC |
| **Snowball Edge** | Bulk physical data transfer cho dữ liệu cực lớn |
| **AWS Network Firewall** | Lọc traffic VPC perimeter, không quản lý SSL certificate |

💡 **Công thức ghi nhớ:**

> "Migrate database + capture ongoing changes (CDC)" → **AWS DMS** (full load + CDC task)

> "DMS output sang S3" → mặc định **CSV format** (hoặc Parquet)

> "Bảo mật DMS connection" → **SSL/TLS qua CA certificate** upload vào DMS console — không cần thêm dịch vụ

> **SCT** = chuyển schema (structure) | **DMS** = migrate data + replication — hai công cụ bổ sung nhau, không thay thế nhau

> **AWS MGN** = di chuyển cả server/VM lên AWS, không phải database migration tool
