## Phân tích Câu hỏi 20

### Bối cảnh bài toán
Một công ty truyền thông kỹ thuật số cần:
- Chia sẻ **nội dung tĩnh** (static content) cho người dùng premium toàn cầu và các đối tác
- **Giảm chi phí máy chủ**
- **Bảo mật** khi phân phối dữ liệu
- **Độ trễ thấp** (low latency) trên toàn cầu

---

### Tại sao chọn Amazon S3 + Amazon CloudFront?

**Amazon S3** — Lưu trữ nội dung tĩnh
- Là dịch vụ lưu trữ object storage, lý tưởng cho **nội dung tĩnh** (ảnh, video, file media)
- Chi phí cực thấp so với việc tự vận hành server
- Độ bền cao (11 nines durability), sẵn sàng phục vụ lượng lớn file

**Amazon CloudFront** — CDN phân phối toàn cầu
- Là mạng phân phối nội dung (CDN) với **edge locations trên toàn thế giới**
- Cache nội dung tại các điểm gần người dùng nhất → **giảm độ trễ**
- Tích hợp trực tiếp với S3, **không tính phí data transfer** giữa S3 và CloudFront
- Hỗ trợ **signed URLs / signed cookies** để giới hạn truy cập cho premium users và đối tác (bảo mật)
- Tích hợp AWS Shield để chống DDoS

---

### Tại sao loại các đáp án còn lại?

| Dịch vụ | Lý do loại |
|---|---|
| **AWS Lambda** | Chạy code serverless, không dùng để lưu trữ hay phân phối nội dung tĩnh |
| **AWS Fargate** | Serverless compute cho container, đắt hơn S3 nhiều khi chỉ cần lưu static content |
| **AWS Global Accelerator** | Phù hợp cho các use case non-HTTP (gaming UDP, IoT, VoIP) hoặc cần static IP cố định. Không tích hợp trực tiếp với S3, không phải CDN |

---

### Tóm tắt kiến trúc tối ưu

```
[Users / Partners] → CloudFront (Edge Locations) → S3 (Origin)
```

- **S3** = kho lưu trữ chi phí thấp, bền vững
- **CloudFront** = CDN bảo mật, phân phối nhanh toàn cầu, tích hợp với S3 không tốn phí data transfer nội bộ

Đây là kiến trúc kinh điển và phổ biến nhất trên AWS cho bài toán phân phối nội dung tĩnh toàn cầu.
