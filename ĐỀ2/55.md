## Câu hỏi 55

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> An Intelligence Agency developed a missile tracking application hosted on both development and production AWS accounts. A junior developer only has access to the development account but has received temporary security clearance to access the production account with write access to EC2 and S3 only.
>
> Which allows the developer to issue short-lived access tokens that act as temporary security credentials to allow access to AWS resources?

---

### 🔤 DỊCH CHI TIẾT

Một cơ quan tình báo có ứng dụng theo dõi tên lửa chạy trên hai AWS accounts riêng biệt: **development** và **production**. Junior developer chỉ có quyền truy cập vào dev account. Developer đã được cấp **security clearance tạm thời** để vào production account với **quyền write-only vào EC2 và S3**.

Dịch vụ nào cho phép phát hành **short-lived access tokens** (token truy cập có thời hạn ngắn) đóng vai trò là **temporary security credentials** (thông tin xác thực bảo mật tạm thời) để truy cập AWS resources?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Từ khóa quyết định đáp án: **"short-lived access tokens"** + **"temporary security credentials"** + **"cross-account access"** → đây là đặc tả chính xác của **AWS STS (Security Token Service)**.

---

### 🔑 Cách hoạt động của STS trong bài toán

```
Dev Account                          Prod Account
┌─────────────────┐                 ┌─────────────────────┐
│ Developer Alice │                 │ IAM Role: WriteAccess│
│                 │  1. AssumeRole  │ (EC2 + S3 write)     │
│                 │ ──────────────→ │                      │
│                 │                 └─────────────────────┘
│                 │  2. STS trả về temporary credentials   │
│                 │ ←──────────────────────────────────── │
│                 │                                        │
│                 │  3. Dùng temp credentials để access    │
│                 │ ──────────────→ EC2, S3 trong Prod     │
└─────────────────┘
```

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: AWS Security Token Service (STS).**

- STS tạo **temporary security credentials** gồm: Access Key ID, Secret Access Key, và **Session Token** có thời hạn. ✅
- Developer gọi **AssumeRole** API → STS trả về credentials tạm thời → dùng để access Prod account. ✅
- Credentials hết hạn tự động sau thời gian định trước → đúng yêu cầu "temporary". ✅
- Hỗ trợ **cross-account access** — developer từ Dev account vào Prod account. ✅

---

❌ **Đáp án sai 1: Amazon Cognito + JSON Web Tokens (JWT).**

- Cognito dùng cho **user authentication của ứng dụng** (đăng nhập web/mobile app). ✗
- JWT là token dùng để xác thực phiên người dùng trong ứng dụng — không phải AWS resource credentials. ✗
- Không dùng để cấp quyền truy cập cross-account vào EC2 hay S3. ✗

---

❌ **Đáp án sai 2: AWS SSO (IAM Identity Center).**

- IAM Identity Center quản lý **workforce identity** và truy cập tập trung vào nhiều AWS accounts. ✗
- Phù hợp cho **đăng nhập console** và quản lý access dài hạn — không generate temporary tokens trực tiếp theo cơ chế AssumeRole. ✗
- Không phải dịch vụ phát hành temporary security credentials như STS. ✗

---

❌ **Đáp án sai 3: All of the given options are correct.**

- Chỉ STS mới có khả năng phát hành temporary security credentials. ✗
- Cognito và SSO không đáp ứng yêu cầu short-lived tokens cho AWS resource access. ✗

---

### 📊 KẾT LUẬN & BÀI HỌC

| Dịch vụ | Vai trò thực sự |
|---|---|
| **AWS STS** | Phát hành temporary security credentials, AssumeRole cross-account |
| **Amazon Cognito** | Authentication cho web/mobile app users, JWT tokens |
| **IAM Identity Center (SSO)** | Quản lý workforce identity, SSO vào nhiều accounts/apps |
| **IAM Users** | Long-term credentials (không phải temporary) |

💡 **Công thức ghi nhớ:**

> "Temporary credentials, short-lived tokens, cross-account access" → **AWS STS**

> **STS AssumeRole** = cơ chế chuẩn để truy cập cross-account trong AWS

> **Cognito** = authentication cho end users của app (không phải AWS credentials)

> **SSO/IAM Identity Center** = quản lý đăng nhập workforce, không generate STS tokens trực tiếp

> STS credentials gồm 3 thành phần: **Access Key ID + Secret Access Key + Session Token** — thiếu Session Token thì credentials không hợp lệ.
