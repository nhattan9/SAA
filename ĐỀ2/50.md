## Câu hỏi 50

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company has developed public APIs hosted in Amazon EC2 instances behind an Elastic Load Balancer. The APIs will be used by various clients from respective on-premises data centers. A Solutions Architect received a report that the web service clients can only access trusted IP addresses whitelisted on the web service's firewalls.
>
> What should be done to accomplish the above requirement?

---

### 🔤 DỊCH CHI TIẾT

Một công ty đã xây dựng **public APIs** (giao diện lập trình ứng dụng công khai) được host trên các EC2 instances đứng sau một Elastic Load Balancer. Các API này sẽ được dùng bởi nhiều **clients** (ứng dụng khách) từ các **on-premises data centers** (trung tâm dữ liệu nội bộ) tương ứng của họ.

Solutions Architect nhận được báo cáo rằng các web service clients (ứng dụng phía client gọi API) **chỉ có thể kết nối đến những IP addresses đã được whitelist** (danh sách IP tin cậy được cho phép) trên **firewall** của web service. Nói cách khác, firewall của client chỉ cho phép traffic đi ra đến một số IP cố định đã biết trước — nếu IP của server thay đổi, kết nối sẽ bị chặn.

Cần làm gì để đáp ứng yêu cầu này?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Bài toán có 2 yêu cầu cốt lõi:

1. **Load Balancer phải có static IP cố định** → clients có thể whitelist IP đó trên firewall của họ.
2. **Chỉ NLB mới hỗ trợ Elastic IP** → ALB không thể gán EIP trực tiếp.

---

### 🔑 KIẾN THỨC CỐT LÕI: EIP + Load Balancer

| Load Balancer | Có thể gán Elastic IP? | Static IP? |
|---|---|---|
| **ALB** (Application LB) | ❌ Không | ❌ IP thay đổi |
| **NLB** (Network LB) | ✅ Có | ✅ Static IP per AZ |
| **GLB** (Gateway LB) | ❌ Không | ❌ |

ALB dùng DNS name (dynamic IP) → không thể whitelist IP. NLB cấp static IP per AZ và có thể gán thêm Elastic IP → clients whitelist IP đó một lần, không bao giờ phải cập nhật lại.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Associate an Elastic IP address to a Network Load Balancer.**

- NLB hỗ trợ **static IP per Availability Zone** và cho phép gán **Elastic IP** vào. ✅
- Dùng tính năng **BYOIP (Bring Your Own IP)** → có thể dùng IP tin cậy sẵn có làm EIP → không cần cập nhật lại whitelist trên firewall của clients. ✅
- NLB hoạt động ở **Layer 4** → xử lý TCP/UDP, phù hợp cho API traffic với static IP. ✅

---

❌ **Đáp án sai 1: Associate an Elastic IP address to an Application Load Balancer.**

- **Không thể gán Elastic IP trực tiếp vào ALB** — đây là giới hạn kỹ thuật của AWS. ✗
- ALB dùng DNS name với dynamic IP — IP có thể thay đổi bất kỳ lúc nào. ✗
- Giải pháp thay thế nếu muốn dùng ALB: đặt NLB phía trước ALB, gán EIP vào NLB. ✗

---

❌ **Đáp án sai 2: Create a CloudFront distribution trỏ đến private IP của web servers.**

- CloudFront dùng DNS name với dynamic IP — không có static IP để whitelist. ✗
- CloudFront origin phải là public endpoint hoặc ALB, không phải private IP trực tiếp. ✗
- Không giải quyết được vấn đề static IP cho firewall whitelist. ✗

---

❌ **Đáp án sai 3: Configure General Purpose SSD volumes trên load balancer nodes để có static IP.**

- **General Purpose SSD (gp3)** là loại **EBS volume** (ổ đĩa cho EC2) — hoàn toàn không liên quan đến IP address hay load balancer. ✗
- EBS volume không có bất kỳ tính năng nào liên quan đến networking hay IP management. ✗
- Đây là đáp án nhiễu hoàn toàn vô lý về mặt kỹ thuật. ✗

---

### 📊 KẾT LUẬN & BÀI HỌC

| Yêu cầu | Giải pháp |
|---|---|
| Static IP cho Load Balancer | **NLB + Elastic IP** |
| Clients cần whitelist IP cố định | **NLB** (ALB không hỗ trợ) |
| Dùng IP sẵn có của công ty | **BYOIP feature** → gán làm EIP |

💡 **Công thức ghi nhớ:**

> "Static IP, whitelist IP, firewall rules" → **Network Load Balancer + Elastic IP**

> **ALB không thể gán EIP** — đây là quy tắc cứng, không có ngoại lệ

> **NLB = static IP per AZ** — mỗi AZ có một IP cố định, clients whitelist một lần dùng mãi

> Nếu cần ALB (Layer 7) + static IP → đặt **NLB trước ALB**, gán EIP vào NLB

> **EBS volume ≠ networking** — gp3, gp2, io1 là storage type, không liên quan gì đến IP address.
