## Câu hỏi 14

### 📋 NỘI DUNG CÂU HỎI

> *A company has a serverless application made up of AWS Amplify, Amazon API Gateway and a Lambda function. The application is connected to an Amazon RDS MySQL database instance inside a private subnet. There are times during peak loads when the database throws a "too many connections" error preventing the users from accessing the application.*

Công ty có ứng dụng **serverless** *(không cần quản lý server)* gồm Amplify + API Gateway + Lambda, kết nối đến **RDS MySQL**. Trong giờ cao điểm (**peak loads**), database báo lỗi **"too many connections"** *(quá nhiều kết nối)* khiến người dùng không truy cập được ứng dụng.

---

### 🧠 Hiểu vấn đề trước

Lambda có thể **scale lên hàng chục nghìn instances đồng thời** — mỗi Lambda instance mở **1 kết nối riêng** đến RDS → RDS bị quá tải kết nối → báo lỗi "too many connections".

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** Tạo **RDS Proxy** giữa Lambda và RDS

- **RDS Proxy** = lớp trung gian quản lý **connection pool** *(bể kết nối)* giữa Lambda và RDS.
- Thay vì mỗi Lambda instance mở 1 kết nối mới → tất cả Lambda instances **dùng chung** một pool kết nối có sẵn từ RDS Proxy.
- RDS chỉ cần duy trì số kết nối nhỏ với Proxy, còn Proxy lo việc phân phối cho hàng nghìn Lambda.
- Giải quyết **triệt để** bài toán "too many connections".

---

**❌ Đáp án sai 1:** Tăng **rate limit** của API Gateway

- **Rate limit** = giới hạn số request mỗi giây client có thể gửi.
- Tăng rate limit → cho phép **thêm request** vào hệ thống → càng thêm kết nối đến RDS → vấn đề càng **tệ hơn**, không giải quyết được gì.

---

**❌ Đáp án sai 2:** Tăng **concurrency limit** của Lambda

- **Concurrency limit** = số Lambda instances chạy đồng thời tối đa.
- Tăng limit → cho phép **thêm Lambda instances** → thêm kết nối đến RDS → vấn đề càng **tệ hơn**.

---

**❌ Đáp án sai 3:** Tăng **memory allocation** của Lambda

- Tăng memory giúp Lambda chạy **nhanh hơn**, không liên quan đến số lượng kết nối database.
- Không giải quyết được lỗi "too many connections" vì đây là vấn đề của **RDS**, không phải Lambda.

---

### 🎯 KẾT LUẬN & BÀI HỌC

**💡 Hình dung RDS Proxy đơn giản:**

```
Không có Proxy:
10,000 Lambda → 10,000 kết nối → RDS quá tải 💥

Có RDS Proxy:
10,000 Lambda → RDS Proxy → ~100 kết nối → RDS ổn định ✅
```

> Câu hỏi có từ **"too many connections"** + **Lambda** + **RDS** → đáp án gần như chắc chắn là **RDS Proxy**.
>
> **RDS Proxy** = giải pháp chuẩn khi Lambda scale lớn mà vẫn cần kết nối RDS hiệu quả.
