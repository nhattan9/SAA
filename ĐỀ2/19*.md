## Câu hỏi 20

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)
> *A solutions architect is designing a three-tier website that will be hosted on an Amazon EC2 Auto Scaling group fronted by an Internet-facing Application Load Balancer (ALB). The website will persist data to an Amazon Aurora Serverless DB cluster, which will also be used for generating monthly reports. The company requires a network topology that follows a layered approach to reduce the impact of misconfigured security groups or network access lists. Web filtering must also be enabled to automatically stop traffic to known malicious URLs and to immediately drop requests coming from blacklisted fully qualified domain names (FQDNs). Which network topology provides the minimum resources needed for the website to work?*

---

### 🔤 DỊCH CHI TIẾT

Một **solutions architect** *(kiến trúc sư giải pháp)* đang thiết kế một website **three-tier** *(3 tầng)* được **hosted** *(lưu trữ/chạy)* trên **Amazon EC2 Auto Scaling group** *(nhóm máy chủ tự động co giãn theo tải)* đứng sau một **Internet-facing Application Load Balancer** *(bộ cân bằng tải có kết nối ra internet)*. Website sẽ **persist data** *(lưu trữ dữ liệu lâu dài)* vào **Amazon Aurora Serverless DB cluster** *(cụm cơ sở dữ liệu tự động co giãn, không cần quản lý server)*, cũng được dùng để **generating monthly reports** *(tạo báo cáo hàng tháng)*.

Công ty yêu cầu một **network topology** *(cấu trúc mạng)* theo **layered approach** *(tiếp cận nhiều lớp — mỗi thành phần nằm ở một lớp riêng biệt)* để giảm tác động khi **misconfigured security groups** *(cấu hình sai security group)* hoặc **network access lists** *(danh sách kiểm soát truy cập mạng — NACL)*. **Web filtering** *(lọc web)* cũng phải được bật để tự động chặn traffic đến **malicious URLs** *(URL độc hại)* và ngay lập tức **drop** *(từ chối/loại bỏ)* các request đến từ **blacklisted FQDNs** *(Fully Qualified Domain Names — tên miền đầy đủ bị liệt vào danh sách đen)*.

Cấu trúc mạng nào cung cấp **minimum resources** *(tài nguyên tối thiểu cần thiết)* để website hoạt động?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Bài toán có **3 yêu cầu cốt lõi:**

1. **Kiến trúc 3 tầng theo layered approach** → mỗi thành phần phải nằm đúng lớp: ALB public, EC2 + Aurora private.
2. **Chặn malicious URLs và blacklisted FQDNs** → cần dịch vụ web filtering chuyên dụng, không phải ALB hay NAT Gateway.
3. **Minimum resources** → không thêm thành phần dư thừa không cần thiết như NAT Gateway khi EC2 không cần ra internet.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** ALB ở **public subnet** + EC2 & Aurora ở **private subnet** + **AWS Network Firewall** + **reroute traffic qua firewall endpoint**

- ALB ở public subnet → nhận traffic từ internet ✅
- EC2 + Aurora ở private subnet → đúng layered approach ✅
- **AWS Network Firewall** → chuyên chặn malicious URLs và blacklisted FQDNs ✅
- **Reroute traffic qua firewall endpoint** → đúng cách tích hợp Network Firewall vào VPC ✅
- Không có NAT Gateway không cần thiết → đúng minimum resources ✅

---

**❌ Đáp án sai 1:** ALB + EC2 & Aurora ở private subnet + **NAT Gateway chặn FQDN**

- **NAT Gateway** chỉ cho phép EC2 trong private subnet ra internet, **không có khả năng chặn FQDNs hay malicious URLs**.
- NAT Gateway không cần thiết ở đây → vi phạm "minimum resources".

---

**❌ Đáp án sai 2:** **Tất cả tài nguyên ở public subnet** + **host-based routing của ALB** lọc URL/FQDN

- Tất cả ở public subnet → **không có layered approach** → vi phạm yêu cầu bảo mật nhiều lớp.
- **ALB host-based routing** chỉ định tuyến traffic theo hostname, **không thể chặn malicious URLs hay FQDNs**.

---

**❌ Đáp án sai 3:** ALB + **NAT Gateway** ở public subnet + EC2 & Aurora ở private subnet + **tích hợp Network Firewall trực tiếp vào ALB**

- **NAT Gateway không cần thiết** → vi phạm "minimum resources".
- **Không thể tích hợp Network Firewall trực tiếp vào ALB** — WAF mới gắn được vào ALB, Network Firewall phải reroute toàn bộ VPC traffic.

---

### 📊 KẾT LUẬN & BÀI HỌC

| Yêu cầu | Giải pháp |
|---|---|
| Chặn malicious URLs, blacklisted FQDNs | **AWS Network Firewall** |
| Kiến trúc nhiều lớp (layered) | ALB public subnet, EC2 + DB private subnet |
| Tích hợp Network Firewall vào VPC | **Reroute traffic qua firewall endpoint** |
| Lọc SQL injection, XSS trên HTTP | **AWS WAF** (gắn trực tiếp vào ALB) |
| EC2 private cần ra internet | **NAT Gateway** |

**💡 Công thức ghi nhớ:**
> **"Malicious URLs, blacklisted FQDNs, domain filtering"** → **AWS Network Firewall**
>
> **"SQL injection, XSS"** → **AWS WAF**
>
> **NAT Gateway** = chỉ dùng khi EC2 private cần **ra internet**, không có tính năng bảo mật hay web filtering.
>
> **Network Firewall** không gắn trực tiếp vào ALB — phải **reroute toàn bộ VPC traffic qua firewall endpoint**.
