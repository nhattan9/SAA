## Câu hỏi 63

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A Solutions Architect is building a cloud infrastructure where EC2 instances require access to various AWS services (S3, Redshift). System administrators also need access to deploy and test changes.
>
> Which configuration should be used to ensure access is secured and not compromised? (Select TWO.)

---

### 🔤 DỊCH CHI TIẾT

Kiến trúc cần: (1) **EC2 instances truy cập AWS services** (S3, Redshift) một cách bảo mật, (2) **system administrators** cần truy cập để deploy và test thay đổi. Cần chọn 2 cấu hình đảm bảo bảo mật, không bị compromise (xâm phạm).

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Bài toán có 2 nhóm đối tượng cần bảo mật:

1. **EC2 instances** truy cập AWS services → dùng **IAM Role** (không dùng Access Keys).
2. **System administrators** truy cập AWS account → thêm **MFA** để bảo vệ tài khoản.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng 1: Assign IAM Role cho EC2 instance.**

- IAM Role cung cấp **temporary credentials tự động rotate** cho EC2 instance — không cần lưu Access Keys thủ công. ✅
- Application trên EC2 tự động lấy credentials từ **Instance Metadata Service** → gọi S3, Redshift API. ✅
- Credentials **không bao giờ bị hardcode** → không thể bị lộ nếu server bị xâm phạm. ✅
- AWS strongly recommend: **IAM Role cho EC2**, không phải Access Keys hay IAM User. ✅

---

✅ **Đáp án đúng 2: Enable Multi-Factor Authentication (MFA).**

- MFA thêm **lớp bảo vệ thứ hai** cho system administrators — ngoài username/password còn cần thiết bị MFA. ✅
- Ngay cả khi password bị lộ, attacker vẫn không thể đăng nhập nếu không có thiết bị MFA vật lý. ✅
- AWS best practice bắt buộc cho mọi tài khoản có quyền cao (admin, deploy...). ✅

---

❌ **Đáp án sai 1: Lưu AWS Access Keys trong EC2 instance.**

- Access Keys lưu trực tiếp trên EC2 → nếu server bị hack → **credentials bị lộ hoàn toàn**. ✗
- AWS khuyến cáo **không bao giờ** lưu Access Keys trên EC2 instance. ✗
- Phải dùng IAM Role thay thế — credentials được quản lý tự động và rotate thường xuyên. ✗

---

❌ **Đáp án sai 2: Assign IAM User cho mỗi EC2 Instance.**

- IAM User sinh ra để đại diện cho **người** (human), không phải machine/service. ✗
- Mỗi IAM User có long-term Access Keys → rủi ro bảo mật cao nếu lưu trên nhiều EC2. ✗
- IAM Role linh hoạt hơn, an toàn hơn, dễ quản lý hơn IAM User cho use case này. ✗

---

❌ **Đáp án sai 3: Lưu AWS Access Keys trong ACM (AWS Certificate Manager).**

- **ACM** là dịch vụ quản lý **SSL/TLS certificates** — không phải nơi lưu Access Keys. ✗
- ACM không có chức năng lưu trữ hay quản lý credentials. ✗
- Dịch vụ đúng để lưu secrets/credentials là **AWS Secrets Manager** hoặc **AWS Systems Manager Parameter Store**. ✗

---

### 📊 KẾT LUẬN & BÀI HỌC

| Yêu cầu | Giải pháp đúng | Giải pháp sai |
|---|---|---|
| EC2 truy cập AWS services | **IAM Role** | Access Keys hardcoded, IAM User |
| Bảo vệ admin account | **MFA** | Chỉ dùng password |
| Lưu secrets/credentials | Secrets Manager, Parameter Store | ACM, EC2 filesystem |

💡 **Công thức ghi nhớ:**

> **EC2 cần access AWS service** → luôn dùng **IAM Role**, không bao giờ hardcode Access Keys

> **IAM Role** = temporary credentials tự động rotate, không bao giờ expire nếu EC2 còn chạy

> **MFA** = bảo vệ human accounts (admins, developers) khỏi bị compromise dù mất password

> **IAM Role** = cho machines/services | **IAM User** = cho humans | Không hoán đổi hai loại này

> **ACM** = SSL/TLS certificate management → không phải secure storage cho Access Keys
