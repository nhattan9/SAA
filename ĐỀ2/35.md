## Câu hỏi 35

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> All objects uploaded to an Amazon S3 bucket must be encrypted for security compliance. The bucket will use server-side encryption with Amazon S3-Managed encryption keys (SSE-S3) to encrypt data using 256-bit AES-256 block cipher.
>
> Which of the following request headers must be used?

---

### 🔤 DỊCH CHI TIẾT

Tất cả objects upload lên S3 bucket phải được mã hóa để tuân thủ bảo mật. Bucket dùng **SSE-S3** (server-side encryption với khóa do AWS quản lý) mã hóa bằng AES-256. Request header nào phải được sử dụng?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Câu hỏi kiểm tra khả năng phân biệt **request headers của từng loại S3 server-side encryption**. Đây là bài toán thuần túy về thuộc tính kỹ thuật — cần nhớ đúng header tương ứng với từng encryption type.

---

### 🔑 KIẾN THỨC CỐT LÕI: Phân loại S3 Server-Side Encryption

| Loại | Ai quản lý khóa | Request Header |
|---|---|---|
| **SSE-S3** | AWS hoàn toàn | `x-amz-server-side-encryption` |
| **SSE-KMS** | AWS KMS (có audit trail) | `x-amz-server-side-encryption: aws:kms` |
| **SSE-C** | Khách hàng tự cung cấp khóa | 3 headers riêng (xem bên dưới) |

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: `x-amz-server-side-encryption`**

- Đây là header chuẩn để yêu cầu **SSE-S3** — AWS tự quản lý toàn bộ encryption keys. ✅
- Giá trị kèm theo: `x-amz-server-side-encryption: AES256`
- AWS tự động encrypt mỗi object bằng unique key, rồi encrypt chính key đó bằng master key luân phiên định kỳ. ✅
- Có thể enforce bắt buộc dùng header này qua **bucket policy** — từ chối mọi upload không có header này. ✅

---

❌ **Đáp án sai 1: `x-amz-server-side-encryption-customer-key`**

- Đây là header của **SSE-C** (customer-provided keys) — khách hàng tự cung cấp khóa mã hóa trong mỗi request. ✗
- SSE-C yêu cầu đồng thời 3 headers, không phải chỉ một. ✗
- Không liên quan đến SSE-S3. ✗

---

❌ **Đáp án sai 2: `x-amz-server-side-encryption-customer-algorithm`**

- Cũng là header của **SSE-C**, dùng để chỉ định thuật toán mã hóa (ví dụ: AES256) khi khách hàng tự cung cấp key. ✗
- Không phải header của SSE-S3. ✗

---

❌ **Đáp án sai 3: `x-amz-server-side-encryption-customer-key-MD5`**

- Header thứ ba của **SSE-C**, chứa MD5 checksum của encryption key để AWS xác minh tính toàn vẹn của key được gửi lên. ✗
- Không phải header của SSE-S3. ✗

---

### 📊 KẾT LUẬN & BÀI HỌC

| Encryption Type | Headers cần dùng |
|---|---|
| **SSE-S3** | `x-amz-server-side-encryption: AES256` |
| **SSE-KMS** | `x-amz-server-side-encryption: aws:kms` |
| **SSE-C** | `x-amz-server-side-encryption-customer-algorithm` + `x-amz-server-side-encryption-customer-key` + `x-amz-server-side-encryption-customer-key-MD5` |

💡 **Công thức ghi nhớ:**

> Header có chữ **"-customer-"** → luôn thuộc về **SSE-C** (customer-provided key)

> Header ngắn gọn **`x-amz-server-side-encryption`** (không có "-customer-") → **SSE-S3 hoặc SSE-KMS**

> **SSE-S3** = AWS lo tất cả, đơn giản nhất, không audit trail per-key

> **SSE-KMS** = AWS KMS quản lý key, có audit trail qua CloudTrail, kiểm soát chi tiết hơn

> **SSE-C** = khách hàng tự giữ key, AWS không lưu key — mất key là mất data vĩnh viễn
