## Câu hỏi 56

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> An insurance company utilizes SAP HANA for its ERP operations and needs to integrate it with the current AWS workload in the VPC by establishing a site-to-site VPN connection.
>
> What needs to be configured OUTSIDE of the VPC for an AWS Virtual Private Network connection to succeed?

---

### 🔤 DỊCH CHI TIẾT

Công ty bảo hiểm dùng **SAP HANA** (hệ thống ERP) on-premises và cần kết nối với workload trong AWS VPC qua **site-to-site VPN**. Cần cấu hình gì ở **phía ngoài VPC** (tức là phía on-premises) để kết nối VPN thành công?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Câu hỏi tập trung vào kiến thức về **kiến trúc Site-to-Site VPN** — cụ thể là phần cấu hình **phía on-premises (outside VPC)**, không phải phía AWS.

---

### 🔑 Kiến trúc Site-to-Site VPN

```
On-Premises Network                    AWS VPC
┌──────────────────────┐           ┌──────────────────┐
│  SAP HANA Server     │           │                  │
│                      │           │  EC2 Instances   │
│  Customer Gateway    │  VPN      │                  │
│  (physical device    │ ═════════ │  Virtual Private │
│   hoặc software)     │  tunnel   │  Gateway (VGW)   │
│                      │           │                  │
│  → Cần static        │           │  (gắn vào VPC)   │
│    public IP         │           │                  │
└──────────────────────┘           └──────────────────┘
   OUTSIDE VPC                         INSIDE VPC
```

**Phía AWS (inside VPC):** Virtual Private Gateway (VGW) gắn vào VPC + route table cập nhật.

**Phía on-premises (outside VPC):** Customer Gateway với **static public IP** — đây là yêu cầu bắt buộc.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: An Internet-routable IP address (static) của Customer Gateway's external interface.**

- **Customer Gateway** là thiết bị vật lý hoặc phần mềm phía on-premises đại diện cho đầu VPN của công ty. ✅
- Phải có **static public IP** (routable trên internet) để AWS biết kết nối đến đâu. ✅
- Không thể dùng dynamic IP vì AWS cần IP cố định để duy trì VPN tunnel ổn định. ✅
- Đây là thông tin duy nhất cần cấu hình **outside VPC** trong quá trình setup VPN. ✅

---

❌ **Đáp án sai 1: Gán EIP vào Virtual Private Gateway.**

- **Virtual Private Gateway (VGW)** nằm **trong VPC** (AWS side) — không phải outside VPC. ✗
- VGW không cần gán Elastic IP — AWS tự quản lý endpoint của VGW. ✗
- Không thể attach EIP vào VGW. ✗

---

❌ **Đáp án sai 2: Cấu hình main route table trong VPC để route traffic qua NAT instance.**

- Route table nằm **trong VPC** — không phải outside VPC. ✗
- NAT instance dùng để cho private subnet ra internet — không liên quan đến VPN connection. ✗

---

❌ **Đáp án sai 3: Dedicated NAT instance trong public subnet.**

- NAT instance **không cần thiết** cho site-to-site VPN. ✗
- Nằm **trong VPC**, không phải outside VPC. ✗
- VPN traffic đi qua VGW trực tiếp, không cần NAT. ✗

---

### 📊 KẾT LUẬN & BÀI HỌC

| Thành phần | Nằm ở đâu | Vai trò |
|---|---|---|
| **Customer Gateway** | Outside VPC (on-premises) | Đại diện thiết bị VPN phía công ty, cần static public IP |
| **Virtual Private Gateway (VGW)** | Inside VPC (AWS side) | Điểm kết thúc VPN tunnel phía AWS, gắn vào VPC |
| **VPN Tunnel** | Kết nối hai đầu | Mã hóa IPsec, traffic đi qua internet |
| **Route Table** | Inside VPC | Cấu hình để traffic đến on-premises đi qua VGW |

💡 **Công thức ghi nhớ:**

> "Site-to-site VPN outside VPC" → **Customer Gateway với static public IP**

> **Customer Gateway** = phía on-premises | **Virtual Private Gateway** = phía AWS/VPC

> VPN không cần NAT Gateway hay NAT instance — traffic đi thẳng qua VGW

> Static IP bắt buộc cho Customer Gateway — dynamic IP không dùng được vì AWS cần biết IP cố định để duy trì tunnel

> Sau khi setup VPN: cập nhật **route table trong VPC** để route traffic on-premises qua VGW — nhưng đây là cấu hình **inside VPC**, không phải outside.
