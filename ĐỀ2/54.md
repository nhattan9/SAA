## Câu hỏi 54

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company receives semi-structured and structured data stored in an S3 data lake with client-side encryption (CSE-Custom). The architect plans to use big data processing frameworks to analyze data and access it using various business intelligence tools and standard SQL queries.
>
> Which provides the MOST high-performing solution?

---

### 🔤 DỊCH CHI TIẾT

Công ty nhận **semi-structured** (dữ liệu bán cấu trúc — JSON, CSV, XML) và **structured data** (dữ liệu có cấu trúc — bảng, quan hệ) từ nhiều nguồn, lưu trong **S3 data lake** với mã hóa client-side. Cần dùng **big data processing frameworks** (Apache Hadoop, Spark...) để xử lý và phân tích dữ liệu thông qua **BI tools** và **standard SQL queries**.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Bài toán có 2 từ khóa quyết định đáp án:

1. **"Big data processing frameworks"** (Hadoop, Spark, Hive...) → **Amazon EMR**
2. **"Business intelligence tools + standard SQL queries"** → **Amazon Redshift** (data warehouse)

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Amazon EMR cluster + store processed data trong Amazon Redshift.**

- **Amazon EMR** là managed cluster platform chuyên chạy big data frameworks (Hadoop, Spark, Hive, Pig) → xử lý và transform dữ liệu từ S3 data lake. ✅
- Sau khi EMR xử lý (ETL), load dữ liệu vào **Amazon Redshift** — cloud data warehouse hàng đầu. ✅
- Redshift hỗ trợ **standard SQL** và tích hợp với mọi BI tools (Tableau, QuickSight, Power BI...). ✅
- Redshift xử lý **terabytes đến petabytes** với columnar storage và massively parallel query execution. ✅

---

❌ **Đáp án sai 1: AWS Batch + lưu vào S3.**

- AWS Batch dùng cho **batch computing workloads** (chạy jobs theo lô) — không phải big data frameworks. ✗
- Không cung cấp interactive SQL analytics hay BI integration. ✗
- S3 không phải data warehouse — không hỗ trợ standard SQL queries hay BI tools trực tiếp. ✗

---

❌ **Đáp án sai 2: Amazon Managed Service for Apache Flink Studio + Amazon DynamoDB.**

- Apache Flink chuyên xử lý **streaming data** (real-time) — không phải batch big data analytics. ✗
- **DynamoDB** là NoSQL key-value database — không hỗ trợ complex analytical SQL queries hay BI tools đầy đủ. ✗
- Cả hai dịch vụ đều không phù hợp với use case analytics/BI này. ✗

---

❌ **Đáp án sai 3: EC2 instance + Amazon EBS.**

- Single EC2 instance có computing capability hạn chế — không scale được cho big data. ✗
- Phải **tự cài đặt và maintain** Hadoop/Spark thủ công → operational overhead cao. ✗
- EBS không phải data warehouse, không hỗ trợ SQL analytics hay BI tools. ✗
- EMR đã giải quyết tất cả những vấn đề này tự động. ✗

---

### 📊 KẾT LUẬN & BÀI HỌC

| Dịch vụ | Vai trò thực sự | Use case |
|---|---|---|
| **Amazon EMR** | Managed big data frameworks (Hadoop, Spark, Hive) | ETL, big data processing |
| **Amazon Redshift** | Cloud data warehouse, columnar storage | SQL analytics, BI tools |
| **AWS Batch** | Batch computing jobs | Chạy jobs theo lô, không phải big data |
| **Apache Flink** | Stream processing | Real-time data, không phải batch analytics |
| **DynamoDB** | NoSQL key-value/document | High-speed read/write, không phải analytics |

💡 **Công thức ghi nhớ:**

> "Big data frameworks (Hadoop, Spark, Hive)" → **Amazon EMR**

> "Standard SQL queries + BI tools + data warehouse" → **Amazon Redshift**

> **EMR → Redshift** = pipeline chuẩn cho big data ETL → analytics

> **Apache Flink** = streaming (real-time) | **EMR/Spark** = batch (bulk processing) — không nhầm lẫn

> **DynamoDB** = fast NoSQL, không phải analytical database — không dùng cho BI/SQL analytics phức tạp
