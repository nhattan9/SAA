## Câu hỏi 41

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company needs to assess and audit all configurations in its AWS account. It must enforce strict compliance by tracking all configuration changes to S3 buckets. Publicly accessible S3 buckets should also be identified automatically to avoid data breaches.
>
> Which option will meet this requirement?

---

### 🔤 DỊCH CHI TIẾT

Công ty cần: (1) **audit và track mọi thay đổi cấu hình** S3 buckets, (2) **tự động phát hiện** S3 buckets có public access, (3) **enforce compliance** — tức là không chỉ ghi lại mà còn đánh giá xem cấu hình có vi phạm policy hay không.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Từ khóa quyết định đáp án: **"assess", "audit", "enforce compliance", "track configuration changes", "automatically identify"** → đây là đặc tả chính xác của **AWS Config**.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Use AWS Config to set up a rule in your AWS account.**

- AWS Config **liên tục monitor và ghi lại** cấu hình của AWS resources. ✅
- Có thể tạo **Config Rules** để tự động đánh giá cấu hình — ví dụ rule `s3-bucket-public-read-prohibited` tự động phát hiện bucket public. ✅
- Dashboard hiển thị **compliance status** — resource nào compliant, resource nào vi phạm. ✅
- Lưu **configuration history** chi tiết → audit trail đầy đủ cho mọi thay đổi cấu hình S3. ✅

---

❌ **Đáp án sai 1: Use AWS CloudTrail and review the event history.**

- CloudTrail ghi lại **API call history** (ai làm gì, lúc nào) — audit trail tốt nhưng chỉ là **passive logging**. ✗
- CloudTrail **không thể tạo rules** để enforce compliance hay tự động đánh giá cấu hình. ✗
- Không tự động phát hiện public bucket hay cảnh báo vi phạm policy. ✗

---

❌ **Đáp án sai 2: Use AWS Trusted Advisor to analyze your AWS environment.**

- Trusted Advisor cung cấp **best practice recommendations** (gợi ý tối ưu) theo 5 category: cost, performance, security, fault tolerance, service limits. ✗
- Không thể **định nghĩa custom rules** hay enforce compliance policy cụ thể. ✗
- Không track configuration changes hay lưu configuration history. ✗

---

❌ **Đáp án sai 3: Use AWS IAM to generate a credential report.**

- IAM credential report chỉ liệt kê **thông tin credentials của IAM users** (password age, MFA status, access key rotation...). ✗
- Hoàn toàn không liên quan đến S3 bucket configuration hay compliance tracking. ✗

---

### 📊 KẾT LUẬN & BÀI HỌC

| Dịch vụ | Vai trò thực sự | Enforce compliance? |
|---|---|---|
| **AWS Config** | Monitor + record cấu hình resources, đánh giá compliance qua rules | ✅ Có |
| **AWS CloudTrail** | Ghi lại API call history, ai làm gì lúc nào | ❌ Log only |
| **AWS Trusted Advisor** | Best practice recommendations | ❌ Gợi ý, không enforce |
| **AWS IAM Credential Report** | Liệt kê thông tin IAM user credentials | ❌ Không liên quan |
| **Amazon GuardDuty** | Phát hiện hành vi độc hại, threat detection | ❌ Security threats, không phải config compliance |

💡 **Công thức ghi nhớ:**

> "Assess, audit, track config changes, enforce compliance" → **AWS Config**

> "Ai làm gì, API call history, governance logging" → **AWS CloudTrail**

> "Best practice suggestions, cost optimization" → **AWS Trusted Advisor**

> **Config** = WHAT (cấu hình như thế nào, có đúng không?)

> **CloudTrail** = WHO/WHEN (ai thay đổi, lúc nào?)

> Hai dịch vụ bổ sung nhau: Config phát hiện cấu hình sai → CloudTrail truy vết xem ai đã thay đổi.
