## Câu hỏi 12

### 📋 NỘI DUNG CÂU HỎI

> *An application is hosted in AWS Fargate and uses RDS database in Multi-AZ Deployments configuration with several Read Replicas. A Solutions Architect was instructed to ensure that all of their database credentials, API keys, and other secrets are encrypted and rotated on a regular basis to improve data security. The application should also use the latest version of the encrypted credentials when connecting to the RDS database.*

Ứng dụng chạy trên **AWS Fargate** dùng **RDS Multi-AZ**. Yêu cầu: tất cả **database credentials** *(thông tin đăng nhập database)*, **API keys** và các **secrets** *(bí mật)* khác phải được **encrypted** *(mã hóa)* và **rotated** *(tự động thay đổi định kỳ)* thường xuyên. Ứng dụng phải luôn dùng **phiên bản mới nhất** của credentials khi kết nối vào RDS.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** Dùng **AWS Secrets Manager** để lưu, mã hóa và bật **automatic rotation**

- **Secrets Manager** được thiết kế chuyên biệt để **lưu trữ, mã hóa và xoay vòng** credentials, API keys, passwords.
- Hỗ trợ **automatic rotation** *(tự động thay đổi password định kỳ)* theo lịch tùy chỉnh — tích hợp sẵn với RDS.
- Ứng dụng gọi API đến Secrets Manager để lấy credentials mới nhất → không cần **hardcode** *(nhúng thẳng vào code)* password.
- Đây là dịch vụ **đúng mục đích nhất** cho bài toán này.

---

**❌ Đáp án sai 1:** Dùng **Systems Manager Parameter Store** với **SecureString**

- Parameter Store **có thể** lưu trữ secrets dạng mã hóa (SecureString), nhưng **không tự động rotate** credentials.
- Phải tự viết code/Lambda để xoay vòng — tốn công hơn Secrets Manager.
- Đáp án còn sai ở chỗ nói *"credentials are automatically rotated by default"* — điều này **hoàn toàn sai**.

---

**❌ Đáp án sai 2:** Dùng **AWS ACM** *(AWS Certificate Manager)*

- **ACM** chuyên quản lý **SSL/TLS certificates** *(chứng chỉ mã hóa kết nối HTTPS)*.
- Hoàn toàn **không dùng để lưu** database credentials hay API keys.

---

**❌ Đáp án sai 3:** Dùng **AWS KMS** *(Key Management Service)*

- **KMS** chuyên tạo và quản lý **encryption keys** *(khóa mã hóa)* — dùng để mã hóa dữ liệu.
- Không phải nơi để **lưu trữ** credentials hay secrets.
- Thực tế, Secrets Manager **dùng KMS bên dưới** để mã hóa secrets, nhưng KMS đứng một mình thì không đủ chức năng.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Dịch vụ | Dùng để làm gì |
|---|---|
| **AWS Secrets Manager** | Lưu, mã hóa, **tự động rotate** credentials/API keys |
| **SSM Parameter Store** | Lưu cấu hình và secrets, **không tự động rotate** |
| **AWS KMS** | Tạo và quản lý **khóa mã hóa**, không lưu secrets |
| **AWS ACM** | Quản lý **SSL/TLS certificates** cho HTTPS |

**💡 Công thức ghi nhớ:**
> Câu hỏi có từ **"rotate credentials", "encrypted secrets", "API keys", "database credentials"** → nghĩ ngay đến **AWS Secrets Manager**.
>
> **Secrets Manager vs Parameter Store:** Điểm khác biệt lớn nhất là Secrets Manager có **automatic rotation** tích hợp sẵn, còn Parameter Store thì không.
