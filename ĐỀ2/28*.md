## Câu hỏi 28

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company is running a dashboard application on a Spot EC2 instance inside a private subnet. The dashboard is reachable via a domain name that maps to the private IPv4 address of the instance's network interface. A solutions architect needs to increase network availability by allowing the traffic flow to resume in another instance if the primary instance is terminated.
>
> Which solution accomplishes these requirements?

---

### 🔤 DỊCH CHI TIẾT

Một công ty đang chạy dashboard application trên Spot EC2 instance trong private subnet. Dashboard có thể truy cập qua domain name ánh xạ đến private IPv4 address của network interface. Solutions Architect cần tăng network availability bằng cách đảm bảo traffic có thể tiếp tục chạy trên instance khác nếu instance chính bị terminated (Spot instance có thể bị AWS thu hồi bất kỳ lúc nào).

Giải pháp nào đáp ứng yêu cầu này?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Bài toán có 3 yêu cầu cốt lõi:

1. **Giữ nguyên IP address và domain name** khi chuyển sang instance khác → không cần thay đổi DNS.
2. **Instance trong private subnet** → không dùng Elastic IP (EIP chỉ dùng cho public).
3. **Primary ENI không thể tách ra** → phải dùng secondary ENI để di chuyển giữa các instance.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Tạo secondary ENI, gán private IPv4 cho domain name, attach vào primary instance. Nếu instance down → move secondary ENI sang instance khác.**

- Secondary ENI **có thể detach và reattach** giữa các EC2 instance trong cùng AZ. ✅
- ENI giữ nguyên private IP, MAC address khi di chuyển → traffic tự động flow sang instance mới mà **không cần cập nhật DNS hay route table**. ✅
- Phù hợp với private subnet (không cần EIP). ✅
- Đây là pattern chuẩn cho high availability với "hot standby instance". ✅

---

❌ **Đáp án sai 1: Dùng AWS Network Firewall để detach primary ENI và move sang instance mới.**

- **Primary ENI (eth0) của EC2 instance không thể detach** — đây là quy tắc cứng của AWS. ✗
- AWS Network Firewall chỉ dùng để **lọc traffic tại VPC perimeter**, không có khả năng quản lý ENI. ✗
- Hoàn toàn sai về cả kỹ thuật lẫn vai trò của dịch vụ. ✗

---

❌ **Đáp án sai 2: Dùng AWS Transfer for FTPS để disable source/destination check và reassociate primary ENI sang instance khác.**

- AWS Transfer for FTPS là dịch vụ **truyền file qua FTPS protocol**, không liên quan gì đến ENI management. ✗
- AWS Transfer chỉ hỗ trợ **Explicit FTPS mode**, không phải Implicit. ✗
- Disable source/destination check chỉ cho phép instance hoạt động như **gateway (vừa gửi vừa nhận)** — không liên quan đến failover. ✗
- Và như đã nói, primary ENI không thể detach dù bằng bất kỳ dịch vụ nào. ✗

---

❌ **Đáp án sai 3: Gán Elastic IP vào primary ENI, dùng AWS Transit Gateway để tự động move EIP sang instance phụ khi primary down.**

- Application trong **private subnet** không cần Elastic IP (EIP là public IP). ✗
- **AWS Transit Gateway** là dịch vụ kết nối VPC và on-premises qua hub trung tâm — không có khả năng tự động move EIP giữa các instance. ✗
- Hai dịch vụ được kết hợp sai mục đích hoàn toàn. ✗

---

### 📊 KẾT LUẬN & BÀI HỌC

| Khái niệm | Điều cần nhớ |
|---|---|
| **Primary ENI (eth0)** | Không thể detach khỏi EC2 instance |
| **Secondary ENI** | Có thể detach/reattach giữa các instance trong cùng AZ |
| **Khi move secondary ENI** | Giữ nguyên private IP, MAC address → không cần đổi DNS hay route table |
| **Elastic IP (EIP)** | Chỉ dùng cho public subnet, không cần thiết cho private subnet |
| **AWS Transit Gateway** | Kết nối VPC và on-premises, không quản lý ENI hay EIP |
| **AWS Network Firewall** | Lọc traffic tại VPC perimeter, không quản lý ENI |

💡 **Công thức ghi nhớ:**

> "Giữ nguyên private IP khi chuyển instance" → **Secondary ENI** (detach → reattach)

> "Giữ nguyên public IP khi chuyển instance" → **Elastic IP** (reassociate)

> Primary ENI = **không bao giờ detach được** — đây là điểm bẫy cực phổ biến trong đề thi.

> Secondary ENI di chuyển → traffic tiếp tục ngay, **không cần update DNS hay route table**.
