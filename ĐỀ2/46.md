## Câu hỏi 45

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company has two EC2 instances inside a VPC in the same Availability Zone but deployed to different subnets. One runs a database, the other runs a web application that connects to the database. What must be checked to ensure these two instances can communicate inside the VPC? (Select TWO.)

---

### 🔤 DỊCH CHI TIẾT

Hai EC2 instances trong cùng VPC, cùng AZ nhưng khác subnet. Một chạy database, một chạy web app cần kết nối đến database. Cần kiểm tra điều gì để đảm bảo chúng giao tiếp được với nhau?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Bài toán kiểm tra kiến thức về **2 lớp kiểm soát network trong VPC**:

1. **Network ACL (NACL)** → kiểm soát traffic ở cấp **subnet** — vì hai instances ở **khác subnet**.
2. **Security Group** → kiểm soát traffic ở cấp **instance** — cần allow đúng port/protocol.

---

### 🔑 NACL vs Security Group — Điểm khác biệt cốt lõi

| | Security Group | Network ACL |
|---|---|---|
| **Áp dụng cho** | EC2 instance (ENI) | Subnet |
| **Stateful/Stateless** | Stateful (response tự động được allow) | Stateless (phải allow cả inbound lẫn outbound) |
| **Default** | Deny all inbound, allow all outbound | Allow all traffic |
| **Khi nào cần kiểm tra?** | Luôn luôn | Khi instances ở **khác subnet** |

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng 1: Kiểm tra Security Groups có allow application host giao tiếp với database đúng port và protocol.**

- Security Group của DB instance phải có **inbound rule** cho phép traffic từ web app instance trên đúng port (ví dụ: MySQL port 3306). ✅
- Security Group của web app instance phải có **outbound rule** cho phép kết nối ra DB. ✅
- Đây là lớp kiểm soát cơ bản nhất, luôn phải kiểm tra. ✅

---

✅ **Đáp án đúng 2: Kiểm tra Network ACL có allow communication giữa hai subnets.**

- Vì hai instances ở **khác subnet**, traffic phải đi qua NACL của cả hai subnet. ✅
- NACL là **stateless** → phải có cả inbound lẫn outbound rule cho phép traffic theo cả hai chiều. ✅
- Nếu cùng subnet thì không cần kiểm tra NACL, nhưng bài này khác subnet → bắt buộc phải check. ✅

---

❌ **Đáp án sai 1: Kiểm tra xem hai instances có cùng instance class không.**

- Instance class (t3.micro, m5.large...) **không ảnh hưởng** đến khả năng giao tiếp mạng. ✗
- Hai instances khác class hoàn toàn có thể communicate với nhau. ✗

---

❌ **Đáp án sai 2: Kiểm tra default route có trỏ đến NAT instance hoặc Internet Gateway.**

- NAT Gateway và Internet Gateway dùng để **kết nối ra internet** — không cần thiết cho giao tiếp **nội bộ trong VPC**. ✗
- Hai instances trong cùng VPC communicate qua **local route** (tự động có trong mọi VPC route table). ✗

---

❌ **Đáp án sai 3: Đảm bảo hai EC2 instances ở cùng Placement Group.**

- Placement Group dùng để tối ưu **low-latency network performance** cho HPC workloads (cluster placement) hoặc **spread ra nhiều phần cứng** (spread placement). ✗
- Không ảnh hưởng đến khả năng kết nối cơ bản giữa hai instances. ✗

---

### 📊 KẾT LUẬN & BÀI HỌC

| Tình huống | Cần kiểm tra |
|---|---|
| Hai instances **cùng subnet** | Chỉ cần kiểm tra **Security Group** |
| Hai instances **khác subnet** | Kiểm tra **Security Group** + **Network ACL** |
| Instance cần ra **internet** | Kiểm tra **Internet Gateway + Route Table + Security Group** |
| Instance trong **private subnet** cần ra internet | Kiểm tra **NAT Gateway + Route Table** |

💡 **Công thức ghi nhớ:**

> **Security Group** = firewall cấp instance, **stateful** → chỉ cần allow một chiều

> **Network ACL** = firewall cấp subnet, **stateless** → phải allow cả inbound lẫn outbound

> Hai instances **khác subnet** → traffic phải qua NACL của cả 2 subnet → bắt buộc check NACL

> **NAT/IGW** = ra internet, không dùng cho internal VPC communication

> **Local route** trong VPC route table cho phép tất cả instances trong cùng VPC giao tiếp với nhau mà không cần thêm cấu hình routing.
