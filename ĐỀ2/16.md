## Câu hỏi 17

### 📋 NỘI DUNG CÂU HỎI

> *A company developed a meal planning application that provides meal recommendations for the week, as well as the food consumption of the users. The application resides on an Amazon EC2 instance, which requires access to various AWS services for its day-to-day operations. Which of the following is the best way to allow the EC2 instance to access the Amazon S3 bucket and other AWS services?*

Ứng dụng chạy trên **EC2 instance** cần truy cập vào **S3** và các dịch vụ AWS khác. Cách nào tốt nhất để cấp quyền truy cập?

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** Tạo **IAM Role** và gắn vào EC2 instance

- **IAM Role** = cấp quyền trực tiếp cho EC2 instance mà **không cần lưu trữ API credentials** ở bất kỳ đâu.
- EC2 instance tự động nhận **temporary credentials** *(thông tin xác thực tạm thời)* từ IAM Role → tự động làm mới → không bao giờ hết hạn lâu dài.
- Đây là **best practice** *(thực hành tốt nhất)* của AWS — an toàn, không lộ credentials.

---

**❌ Đáp án sai 1:** Dùng **BatchGetSecretValue** lấy **long-term API credentials** từ Secrets Manager và lưu vào EC2

- **Long-term credentials** *(thông tin xác thực dài hạn)* lưu trên EC2 rất nguy hiểm — nếu instance bị xâm phạm, credentials bị lộ hoàn toàn.
- IAM Role dùng **temporary credentials** an toàn hơn nhiều.
- Secrets Manager phù hợp để lưu **database passwords, API keys của bên thứ 3** — không phải để thay thế IAM Role.

---

**❌ Đáp án sai 2:** Lưu **API credentials** trong **bastion host**

- **Bastion host** = máy chủ trung gian dùng để SSH vào các server trong private subnet — không phải nơi lưu credentials.
- Lưu credentials ở đây vừa sai mục đích vừa không an toàn.

---

**❌ Đáp án sai 3:** Thêm **API Credentials** vào **Security Group**

- **Security Group** = tường lửa kiểm soát traffic vào/ra EC2 (mở/đóng port) — hoàn toàn **không liên quan** đến việc lưu trữ credentials.
- Đây là hiểu sai hoàn toàn về chức năng của Security Group.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Tình huống | Giải pháp |
|---|---|
| EC2 cần truy cập dịch vụ AWS | **IAM Role gắn vào EC2** |
| Lưu database password, API key bên thứ 3 | **AWS Secrets Manager** |
| Kiểm soát traffic vào/ra EC2 | **Security Group** |
| SSH vào server trong private subnet | **Bastion Host** |

**💡 Công thức ghi nhớ:**
> **EC2 cần truy cập AWS service** → luôn dùng **IAM Role**, không bao giờ hardcode hay lưu credentials trên instance.
>
> **IAM Role** = **không lưu credentials** + **tự động rotate** + **an toàn nhất** → đây là AWS best practice số 1 cho EC2.
