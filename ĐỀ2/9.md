## Câu hỏi 10

### 📋 NỘI DUNG CÂU HỎI

> *An e-commerce company is receiving a large volume of sales data files in .csv format from its external partners on a daily basis. These data files are then stored in an Amazon S3 Bucket for processing and reporting purposes. The company wants to create an automated solution to convert these .csv files into Apache Parquet format and store the output of the processed files in a new S3 bucket called "tutorialsdojo-data-transformed". This new solution is meant to enhance the company's data processing and analytics workloads while keeping its operating costs low.*

Công ty nhận file **.csv** từ đối tác hàng ngày, lưu vào S3. Muốn **tự động chuyển đổi** các file .csv sang định dạng **Apache Parquet** *(định dạng lưu trữ theo cột, tối ưu cho analytics, tiết kiệm dung lượng và nhanh hơn)* rồi lưu vào bucket mới. Yêu cầu: **ít operational overhead nhất** *(ít công quản lý vận hành nhất)* và **chi phí thấp**.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** Dùng **AWS Glue Crawler** + **AWS Glue ETL job**

- **AWS Glue Crawler** = tự động quét S3, phát hiện file mới và đọc **schema** *(cấu trúc dữ liệu)* của file .csv.
- **AWS Glue ETL job** = công việc **Extract-Transform-Load** *(trích xuất - chuyển đổi - tải)*, tự động chuyển .csv sang Parquet và lưu vào bucket đích.
- Có thể **schedule** *(lên lịch chạy tự động)* hàng ngày.
- AWS Glue là dịch vụ **fully managed** *(AWS quản lý hoàn toàn)* → không cần quản lý server, cluster → **ít overhead nhất**.
- Glue hỗ trợ **native Parquet format** → không cần cài thêm thư viện nào.

---

**❌ Đáp án sai 1:** Dùng **S3 Event Notification** + **Lambda** + **Apache Spark trên EMR cluster**

- **EMR cluster** *(Elastic MapReduce)* = cụm máy chủ lớn để xử lý big data → phải **tự quản lý, cấu hình, scale, monitor** → **tốn công và tốn tiền**.
- Với bài toán chuyển đổi file .csv đơn giản hàng ngày, dùng EMR là **over-engineering** *(dùng giải pháp quá phức tạp so với yêu cầu)*.

---

**❌ Đáp án sai 2:** Dùng **AWS Batch** + Bash script

- **AWS Batch** = quản lý công việc xử lý hàng loạt trong **Docker container** → phải cấu hình **compute environment, job queue, container** → phức tạp hơn nhiều.
- Phải tự viết Bash script để chuyển đổi định dạng → tốn công phát triển và bảo trì.
- Chi phí khó tối ưu hơn Glue.

---

**❌ Đáp án sai 3:** Dùng **EMRFS** + **EMR Serverless** với Apache Spark

- **EMRFS** *(EMR File System)* chỉ là implementation của **HDFS** *(hệ thống file phân tán)* để EMR đọc/ghi file từ S3 — không phải công cụ để **phát hiện file mới**.
- Không phải cách phù hợp để trigger khi có file mới upload lên S3.
- Dù EMR Serverless không cần quản lý cluster, vẫn phức tạp hơn Glue cho bài toán này.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Yêu cầu | Dịch vụ |
|---|---|
| Tự động phát hiện file mới trong S3 và đọc schema | **AWS Glue Crawler** |
| Chuyển đổi định dạng file (CSV → Parquet) | **AWS Glue ETL job** |
| Xử lý big data phức tạp, cần tùy chỉnh cao | **Amazon EMR** |
| Chạy tác vụ hàng loạt trong container | **AWS Batch** |

**💡 Công thức ghi nhớ:**
> Câu hỏi có từ **"convert data format", "ETL", "least operational overhead", "S3"** → nghĩ ngay đến **AWS Glue**.
>
> **AWS Glue = dịch vụ ETL fully managed của AWS** — không cần quản lý server, tích hợp sẵn với S3, hỗ trợ Parquet, CSV và hàng chục định dạng khác.
