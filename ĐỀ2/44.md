## Câu hỏi 44

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company owns a photo-sharing app storing user uploads on S3. It wants to use AI to automatically flag explicit/offensive images for review. Any communication with resources on its VPC must not traverse the public Internet.
>
> How can this be accomplished with the LEAST amount of effort?

---

### 🔤 DỊCH CHI TIẾT

Công ty cần: (1) **tự động phát hiện ảnh nhạy cảm/bạo lực bằng AI** thay vì dùng người, (2) **không để traffic đi qua internet công cộng** khi giao tiếp với resources trong VPC, (3) **least effort** — ưu tiên giải pháp có sẵn, không tự xây dựng.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Bài toán có 3 yêu cầu cốt lõi:

1. **AI image moderation out-of-the-box** → không cần train model → Amazon Rekognition (ready-to-use).
2. **Private connectivity, không qua internet** → Interface VPC Endpoint cho Rekognition.
3. **Least effort** → dịch vụ managed, không cần code custom logic.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Amazon Rekognition để detect ảnh + Interface VPC Endpoint cho Rekognition.**

- Rekognition có sẵn **Content Moderation API** — phát hiện nudity, violence, explicit content mà **không cần train model**. ✅
- **Interface VPC Endpoint** tạo kết nối private giữa VPC và Rekognition qua AWS PrivateLink — traffic hoàn toàn không qua internet. ✅
- Least effort: chỉ cần gọi API sẵn có + tạo VPC endpoint → không cần xây dựng gì thêm. ✅

---

❌ **Đáp án sai 1: AWS Lambda tạo custom image moderation logic + deploy trong public VPC.**

- Viết custom logic cho image moderation → **nhiều effort**, phức tạp hơn dùng Rekognition sẵn có. ✗
- Deploy Lambda trong **public VPC** → traffic đi qua internet → vi phạm yêu cầu bảo mật. ✗
- Sai cả về effort lẫn security requirement. ✗

---

❌ **Đáp án sai 2: Amazon Detective + AWS Audit Manager để đảm bảo traffic không qua internet.**

- **Amazon Detective** dùng để **điều tra security incidents** (phân tích root cause, suspicious activities) — không thể detect ảnh nhạy cảm. ✗
- **AWS Audit Manager** dùng để audit compliance và risk — không có khả năng kiểm soát hay chặn traffic network. ✗
- Cả hai dịch vụ đều bị dùng hoàn toàn sai mục đích. ✗

---

❌ **Đáp án sai 3: Amazon SageMaker để train image classification model + Amazon GuardDuty đảm bảo traffic private.**

- SageMaker yêu cầu **tự train và deploy machine learning model** → rất nhiều effort, mất thời gian. ✗
- **GuardDuty** là threat detection service — không có khả năng kiểm soát hay đảm bảo traffic private trong SageMaker. ✗
- Vi phạm yêu cầu "least amount of effort". ✗

---

### 📊 KẾT LUẬN & BÀI HỌC

| Dịch vụ | Vai trò thực sự |
|---|---|
| **Amazon Rekognition** | AI image/video analysis sẵn có: nhận diện object, khuôn mặt, **content moderation** |
| **Interface VPC Endpoint** | Kết nối private giữa VPC và AWS service qua PrivateLink, không qua internet |
| **Amazon SageMaker** | Build, train, deploy ML model tùy chỉnh — cần effort cao |
| **Amazon Detective** | Điều tra security incidents, không phải image analysis |
| **AWS Audit Manager** | Compliance auditing, không kiểm soát network traffic |
| **Amazon GuardDuty** | Threat detection, không kiểm soát network routing |

💡 **Công thức ghi nhớ:**

> "AI nhận diện ảnh nhạy cảm, violent content, nudity" → **Amazon Rekognition** (Content Moderation API)

> "Kết nối private giữa VPC và AWS service, không qua internet" → **Interface VPC Endpoint (PrivateLink)**

> **Rekognition vs SageMaker**: Rekognition = ready-to-use (least effort) | SageMaker = tự train model (high effort)

> "Least amount of effort" trong câu hỏi → luôn ưu tiên **managed service có sẵn tính năng** thay vì tự build

> **Detective** = điều tra sau sự cố | **GuardDuty** = phát hiện threats | **Rekognition** = phân tích hình ảnh — ba dịch vụ hoàn toàn khác nhau.
