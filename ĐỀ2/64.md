## Câu hỏi 64

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company has a static corporate website hosted in a standard Amazon S3 bucket and a new web domain name registered using Amazon Route 53. What are the prerequisites when routing traffic using Route 53 to a website hosted in an S3 bucket? (Select TWO.)

---

### 🔤 DỊCH CHI TIẾT

Công ty có website tĩnh trên S3 và domain đã đăng ký qua Route 53. Cần tích hợp hai dịch vụ này để website hoạt động. **Điều kiện tiên quyết** (prerequisites) nào cần có khi dùng Route 53 để route traffic đến website trên S3?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Câu hỏi kiểm tra 2 điều kiện bắt buộc (không thể thiếu) khi tích hợp Route 53 với S3 static website hosting.

---

### 🔑 2 Điều kiện bắt buộc của AWS

**Điều kiện 1:** S3 bucket name **phải trùng khớp** với domain name.
- Ví dụ: domain là `www.example.com` → bucket phải đặt tên là `www.example.com`
- Lý do: Route 53 alias record trỏ đến S3 website endpoint dựa trên tên bucket.

**Điều kiện 2:** Phải có **registered domain name**.
- Route 53 cần domain để tạo hosted zone và DNS records.
- Có thể đăng ký qua Route 53 hoặc registrar khác rồi cập nhật nameservers.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng 1: S3 bucket name phải trùng với domain name.**

- AWS yêu cầu **tên bucket = tên domain** để Route 53 Alias record hoạt động đúng. ✅
- Ví dụ: `www.tutorialsdojo.com` → bucket phải tên là `www.tutorialsdojo.com`. ✅
- Nếu tên không khớp → Route 53 không thể resolve đúng endpoint S3. ✅

---

✅ **Đáp án đúng 2: Phải có registered domain name.**

- Cần domain đã đăng ký để tạo **hosted zone** trong Route 53. ✅
- Không có domain → không thể tạo DNS records để trỏ về S3. ✅
- Domain có thể đăng ký qua Route 53 hoặc bất kỳ registrar nào khác. ✅

---

❌ **Đáp án sai 1: S3 bucket phải ở cùng region với hosted zone.**

- **Không có ràng buộc này** — S3 bucket và Route 53 hosted zone có thể ở bất kỳ region nào. ✗
- Route 53 là global service, không bị giới hạn bởi region. ✗

---

❌ **Đáp án sai 2: Phải enable CORS trên S3 bucket.**

- **CORS chỉ cần** khi web application ở một domain gọi resources từ **domain khác** (cross-origin). ✗
- Với static website hosting đơn giản, CORS không phải điều kiện bắt buộc. ✗
- CORS là tính năng tùy chọn, không phải prerequisite của Route 53 + S3 integration. ✗

---

❌ **Đáp án sai 3: Record set phải là type "MX".**

- **MX record** (Mail Exchange) chỉ dùng để chỉ định mail server nhận email cho domain. ✗
- Để route web traffic đến S3, phải dùng **A record (Alias)** trỏ đến S3 website endpoint. ✗
- Hoàn toàn sai loại DNS record. ✗

---

### 📊 KẾT LUẬN & BÀI HỌC

| Điều kiện | Bắt buộc? | Lý do |
|---|---|---|
| **Bucket name = domain name** | ✅ Bắt buộc | Route 53 Alias dựa trên tên bucket |
| **Registered domain name** | ✅ Bắt buộc | Cần để tạo hosted zone và DNS records |
| Same region với hosted zone | ❌ Không cần | Route 53 là global |
| CORS enabled | ❌ Không cần | Chỉ cần khi cross-origin requests |
| MX record | ❌ Sai | MX dùng cho email, không phải web |

💡 **Công thức ghi nhớ:**

> Route 53 → S3 static website: **bucket name = domain name** (bắt buộc tuyệt đối)

> **A record (Alias)** trỏ đến S3 website endpoint — không phải MX, không phải CNAME

> **MX** = email server | **A/Alias** = IP/AWS resource | **CNAME** = alias cho domain khác

> **CORS** = chỉ cần khi browser gọi API cross-domain, không phải prerequisite của Route 53 + S3

> Quy trình setup: Đăng ký domain → Tạo S3 bucket cùng tên → Bật static website hosting → Tạo Alias record trong Route 53 → Done.
