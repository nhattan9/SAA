## Câu hỏi 13

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company has stored 200 TB of backup files in Amazon S3. The files are in a vendor-proprietary format. The Solutions Architect needs to use the vendor's proprietary file conversion software to retrieve the files from their Amazon S3 bucket, transform the files to an industry-standard format, and re-upload the files back to Amazon S3. The solution must minimize the data transfer costs.
>
> Which of the following options can satisfy the given requirement?

---

### 🔤 DỊCH CHI TIẾT

Một công ty đã lưu trữ 200 TB file backup trên Amazon S3. Các file ở định dạng độc quyền của vendor. Solutions Architect cần dùng phần mềm chuyển đổi file của vendor để lấy file từ S3, chuyển đổi sang định dạng chuẩn công nghiệp, rồi upload lại lên S3. Yêu cầu: **phải tối thiểu hóa chi phí data transfer**.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Phần mềm conversion** phải được cài trên một compute resource nào đó — không thể chạy trực tiếp trên S3.
2. **Từ khóa cốt lõi: "minimize data transfer costs"** — chi phí data transfer trên AWS phát sinh khi dữ liệu đi ra ngoài AWS hoặc đi giữa các region khác nhau.
3. **Nguyên tắc**: dữ liệu di chuyển **trong cùng một region** thì miễn phí hoặc chi phí rất thấp.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Deploy EC2 instance cùng Region với S3**

- EC2 instance và S3 bucket cùng region → data transfer **trong cùng region = miễn phí**. ✅
- Cài phần mềm conversion trên EC2, đọc file từ S3, xử lý, rồi re-upload lại S3 — toàn bộ quá trình không phát sinh data transfer cost. ✅
- Đây là pattern chuẩn: **"keep compute gần data"** để tối thiểu chi phí. ✅

---

❌ **Đáp án sai 1: Deploy EC2 instance ở Region khác**

- Dữ liệu phải đi từ S3 (region A) sang EC2 (region B) để xử lý, rồi re-upload từ region B về region A.
- **Data transfer cross-region tốn phí** — với 200 TB, chi phí này cực kỳ lớn.
- Vi phạm trực tiếp yêu cầu "minimize data transfer costs".

---

❌ **Đáp án sai 2: Cài phần mềm conversion trên Amazon S3, dùng S3 Batch Operations**

- **S3 Batch Operations** chỉ hỗ trợ các operations tích hợp sẵn của S3 (copy, tag, restore, invoke Lambda...) — không thể cài phần mềm của vendor lên S3.
- S3 là object storage, không phải compute environment — không thể chạy phần mềm tùy ý trên đó.

---

❌ **Đáp án sai 3: Export data sang AWS Snowball Edge, cài phần mềm, rồi re-upload**

- Snowball Edge dùng để **migrate dữ liệu lớn vào/ra AWS** khi bandwidth mạng không đủ — không phải giải pháp cho bài toán này.
- Dữ liệu **đã có sẵn trên S3** rồi — export ra Snowball Edge rồi lại re-upload là thừa bước, tốn thời gian và chi phí.
- Chi phí sử dụng Snowball Edge cộng thêm công vận chuyển thiết bị vật lý là không cần thiết.

---

### 📊 KẾT LUẬN & BÀI HỌC

**Quy tắc Data Transfer Cost trên AWS:**

| Luồng dữ liệu | Chi phí |
|---|---|
| Internet → AWS (inbound) | **Miễn phí** |
| AWS → Internet (outbound) | **Tốn phí** |
| Cùng region: EC2 ↔ S3 | **Miễn phí** |
| Khác region: EC2 ↔ S3 | **Tốn phí** |
| Giữa 2 AZ khác nhau trong cùng region | Tốn phí nhỏ |

💡 **Công thức ghi nhớ:**

> **"Minimize data transfer costs"** → đặt compute (EC2) **cùng region** với data (S3)

> **"Dữ liệu đã có sẵn trên AWS"** → không cần Snowball Edge (Snowball chỉ dùng khi cần migrate vào/ra AWS với lượng lớn)

> **S3 Batch Operations** = thực hiện hàng loạt operations S3 tích hợp sẵn, **không thể chạy phần mềm tùy ý của vendor**

> **Cross-region EC2** = luôn phát sinh data transfer cost → tránh khi đề yêu cầu minimize cost
