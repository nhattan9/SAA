## Câu hỏi 19

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A tech company currently has an on-premises infrastructure. They are currently running low on storage and want to have the ability to extend their storage using the AWS cloud.
>
> Which AWS service can help them achieve this requirement?

---

### 🔤 DỊCH CHI TIẾT

Một công ty công nghệ hiện đang có hạ tầng on-premises. Họ đang gần hết dung lượng lưu trữ và muốn có khả năng **mở rộng storage bằng cách sử dụng AWS cloud**.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Từ khóa cốt lõi: **"extend on-premises storage using AWS cloud"** — cần một cầu nối giữa hạ tầng on-premises và storage trên AWS. Đây chính xác là use case được thiết kế cho **AWS Storage Gateway**.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: AWS Storage Gateway**

- Storage Gateway là một **software appliance** cài trên on-premises, kết nối trực tiếp với AWS cloud storage. ✅
- Cho phép on-premises applications sử dụng AWS storage (S3, Glacier, EBS) như thể đang dùng local storage. ✅
- Tích hợp liền mạch, có data security features, scalable và cost-effective. ✅
- Đây chính là giải pháp **hybrid storage** tiêu chuẩn của AWS cho on-premises. ✅

---

❌ **Đáp án sai 1: Amazon EC2**

- EC2 là **compute service** — cung cấp máy chủ ảo để chạy ứng dụng.
- Không phải storage service, không có khả năng mở rộng on-premises storage.

---

❌ **Đáp án sai 2: Amazon EBS (Elastic Block Storage)**

- EBS là **block storage** gắn liền với EC2 instances — chỉ dùng được khi có EC2 instance đang chạy.
- Không có khả năng kết nối trực tiếp với hạ tầng on-premises để mở rộng storage.

---

❌ **Đáp án sai 3: Amazon SQS**

- SQS là **message queuing service** — dùng để truyền message giữa các component trong hệ thống.
- Không liên quan đến storage hay mở rộng dung lượng lưu trữ.

---

### 📊 KẾT LUẬN & BÀI HỌC

**Các loại AWS Storage Gateway:**

| Loại | Dùng cho |
|---|---|
| **S3 File Gateway** | Mount S3 bucket như NFS/SMB file share trên on-premises |
| **FSx File Gateway** | Mount Amazon FSx for Windows File Server trên on-premises |
| **Volume Gateway** | Block storage iSCSI, backup volume lên S3/EBS snapshots |
| **Tape Gateway** | Thay thế băng từ vật lý (virtual tape library), lưu trên S3/Glacier |

**So sánh các giải pháp hybrid storage:**

| Tình huống | Giải pháp |
|---|---|
| Mở rộng on-premises storage lên cloud | **AWS Storage Gateway** |
| Di chuyển lượng data lớn vào AWS (offline) | **AWS Snowball / Snowmobile** |
| Kết nối on-premises network với AWS | **AWS Direct Connect / VPN** |
| Backup on-premises data lên AWS | **AWS Storage Gateway (Tape/Volume Gateway)** |

💡 **Công thức ghi nhớ:**

> **"Extend on-premises storage using AWS"** → **AWS Storage Gateway**

> **"Hybrid storage"** = on-premises + AWS cloud → **Storage Gateway**

> **EBS** = chỉ dùng được với EC2, không kết nối được on-premises trực tiếp

> **Storage Gateway** = cầu nối on-premises ↔ AWS cloud storage, transparent với ứng dụng đang chạy on-premises
