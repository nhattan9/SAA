## Câu hỏi 48

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A multinational bank is storing its confidential files in an S3 bucket. The security team recently performed an audit, and the report shows that multiple files have been uploaded without 256-bit Advanced Encryption Standard (AES) server-side encryption. For added protection, the encryption key must be automatically rotated every year. The solutions architect must ensure that there would be no other unencrypted files uploaded in the S3 bucket in the future.

---

### 🔤 DỊCH CHI TIẾT

Ngân hàng lưu file bí mật trên S3 nhưng phát hiện nhiều file upload **không có encryption AES-256**. Yêu cầu: **chặn upload không có encryption**, dùng **AES-256**, key phải **tự động rotate hàng năm**, và **least operational overhead**.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Ba yêu cầu cần đáp ứng đồng thời:

1. **AES-256** → phải dùng **SSE-S3** (không phải SSE-KMS).
2. **Chặn upload không có encryption** → cần **S3 Bucket Policy** với điều kiện header.
3. **Tự động rotate key hàng năm + least overhead** → SSE-S3 tự động rotate, không cần làm gì thêm.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: S3 Bucket Policy deny upload không có header AES256 + SSE-S3**

- **Bucket Policy** deny upload nếu request không có header:
  `"s3:x-amz-server-side-encryption": "AES256"` → chặn hoàn toàn upload không encrypt. ✅
- **SSE-S3** dùng **AES-256** — đúng yêu cầu. ✅
- SSE-S3 có **built-in automatic key rotation** — AWS tự quản lý, không cần thêm bước nào. ✅
- **Least operational overhead**: không cần tạo KMS key, không cần tự rotate. ✅

---

❌ **Đáp án sai 1: Bucket Policy + header aws:kms + S3 Object Lock**

- Header `aws:kms` → dùng **SSE-KMS**, không phải AES-256 SSE-S3 — sai yêu cầu encryption type.
- **S3 Object Lock** dùng để **prevent deletion/overwrite**, không liên quan đến encryption hay key rotation.

---

❌ **Đáp án sai 2: Customer-managed KMS key + manual key rotation**

- Thiếu **bucket policy** → không chặn được upload không có encryption.
- **Manual rotation** = operational overhead cao → vi phạm yêu cầu.

---

❌ **Đáp án sai 3: SCP + SSE-S3 + modify built-in rotation**

- **SCP (Service Control Policy)** dùng để control **nhiều AWS accounts trong Organization**, không phải để control S3 bucket cụ thể — sai service.
- **Không thể modify** built-in key rotation của SSE-S3 — AWS quản lý hoàn toàn, người dùng không can thiệp được.

---

### 📊 KẾT LUẬN & BÀI HỌC

**So sánh ba loại S3 Server-Side Encryption:**

| | SSE-S3 | SSE-KMS | SSE-C |
|---|---|---|---|
| Encryption algorithm | **AES-256** | AES-256 (qua KMS) | Do bạn tự cung cấp |
| Key quản lý bởi | **AWS** (hoàn toàn) | AWS KMS (bạn control) | Bạn tự quản lý |
| Header trong request | `"AES256"` | `"aws:kms"` | N/A |
| Auto key rotation | ✅ Tự động (AWS lo) | ✅ Nếu bật (customer-managed) | ❌ Bạn tự lo |
| Operational overhead | **Thấp nhất** | Trung bình | Cao nhất |

**Bucket Policy để enforce encryption:**

```json
{
  "Effect": "Deny",
  "Action": "s3:PutObject",
  "Condition": {
    "StringNotEquals": {
      "s3:x-amz-server-side-encryption": "AES256"
    }
  }
}
```

→ Bất kỳ upload nào không có header `AES256` đều bị từ chối.

**Key rotation theo loại KMS key:**

| Loại key | Auto rotation |
|---|---|
| **AWS managed key** (SSE-S3, SSE-KMS dùng AWS key) | ✅ Tự động hàng năm, không thể tắt |
| **Customer-managed key** | ❌ Tắt mặc định, phải bật thủ công |

💡 **Công thức ghi nhớ:**

> **"AES-256 + auto key rotation + least overhead"** → **SSE-S3** (AWS quản lý key hoàn toàn)

> **"Enforce encryption cho mọi upload"** → **S3 Bucket Policy** deny khi thiếu encryption header

> **SSE-S3** header = `"AES256"` | **SSE-KMS** header = `"aws:kms"` — hai cái khác nhau, không nhầm

> **SCP** = control AWS accounts trong Organization, **không phải** control S3 bucket

> **S3 Object Lock** = prevent deletion/overwrite — không liên quan encryption
