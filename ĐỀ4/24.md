## Câu hỏi 24

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A leading IT consulting company has an application which processes a large stream of financial data by an Amazon ECS Cluster then stores the result to a DynamoDB table. You have to design a solution to detect new entries in the DynamoDB table then automatically trigger a Lambda function to run some tests to verify the processed data.
>
> What solution can be easily implemented to alert the Lambda function of new entries while requiring minimal configuration change to your architecture?

---

### 🔤 DỊCH CHI TIẾT

Một ứng dụng xử lý dữ liệu tài chính qua ECS Cluster rồi lưu kết quả vào DynamoDB table. Cần thiết kế giải pháp **tự động phát hiện entry mới** trong DynamoDB và **trigger Lambda function** để verify dữ liệu — với **thay đổi cấu hình tối thiểu**.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Detect new entries trong DynamoDB** → cần cơ chế lắng nghe thay đổi trong bảng.
2. **Tự động trigger Lambda** khi có entry mới.
3. **Minimal configuration change** → ưu tiên giải pháp native, tích hợp sẵn, không cần thêm nhiều service.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Enable DynamoDB Streams**

- **DynamoDB Streams** capture mọi thay đổi (insert, update, delete) trong DynamoDB table dưới dạng stream records. ✅
- AWS Lambda **poll stream tự động** và invoke Lambda function ngay khi phát hiện record mới — không cần code thêm gì. ✅
- Chỉ cần enable DynamoDB Streams + associate stream ARN với Lambda function → **minimal configuration**. ✅
- Đây là tích hợp native giữa DynamoDB và Lambda — được thiết kế chính xác cho use case này. ✅

---

❌ **Đáp án sai 1: SNS từ ECS Cluster để invoke Lambda**

- Cần thêm SNS topic, cấu hình ECS gửi notification đến SNS, SNS trigger Lambda — **nhiều bước hơn cần thiết**.
- Vi phạm yêu cầu "minimal configuration change".
- ECS cluster không nhất thiết biết khi nào DynamoDB có entry mới — coupling không tốt.

---

❌ **Đáp án sai 2: CloudWatch Alarms trigger Lambda**

- CloudWatch Alarms monitor **service metrics** (CPU, memory, latency...) — không thể phát hiện thay đổi **dữ liệu** trong DynamoDB table.
- Không có metric nào trong CloudWatch tương ứng với "có entry mới trong bảng DynamoDB".

---

❌ **Đáp án sai 3: Systems Manager Automation**

- **Systems Manager Automation** dùng để tự động hóa các tác vụ **vận hành EC2 và AWS resources** (patch, backup, compliance...).
- Không có khả năng detect thay đổi dữ liệu trong DynamoDB table.

---

### 📊 KẾT LUẬN & BÀI HỌC

**DynamoDB Streams hoạt động như thế nào:**

```
DynamoDB Table
    │ (có entry mới / update / delete)
    ↓
DynamoDB Stream (lưu record thay đổi trong 24 giờ)
    │ (Lambda poll liên tục)
    ↓
Lambda Function tự động được invoke
    │
    ↓
Xử lý: verify data, gửi notification, audit log...
```

**Các use case phổ biến của DynamoDB Streams + Lambda:**

| Use case | Mô tả |
|---|---|
| Data validation | Verify dữ liệu mới như bài toán này |
| Audit trail | Copy mọi thay đổi sang S3/EFS làm log |
| Cross-region replication | Sync data sang DynamoDB table ở region khác |
| Trigger notifications | Gửi email/SMS khi có thay đổi quan trọng |
| Materialized views | Tự động cập nhật bảng aggregate |

**Bảng so sánh các cách trigger Lambda:**

| Trigger source | Dùng khi |
|---|---|
| **DynamoDB Streams** | Phát hiện thay đổi data trong DynamoDB |
| **S3 Events** | File mới được upload lên S3 |
| **SNS** | Nhận message từ publisher |
| **SQS** | Poll message từ queue |
| **API Gateway** | HTTP request từ client |
| **CloudWatch Events** | Scheduled trigger hoặc AWS service events |

💡 **Công thức ghi nhớ:**

> **"Detect new/changed item trong DynamoDB → trigger Lambda"** → **DynamoDB Streams**

> **CloudWatch Alarms** = monitor metrics, không monitor data changes

> **Systems Manager** = EC2/infrastructure operations, không liên quan DynamoDB data

> DynamoDB Streams lưu record thay đổi trong **24 giờ** — Lambda poll và xử lý trong window này
