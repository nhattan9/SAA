## Câu hỏi 51

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> The application tier hosted in an Auto Scaling group of EC2 instances is the only identified resource that needs to connect to the database. The Architect should ensure that the architecture complies with the best practice of granting least privilege.
>
> Which of the following changes should be made to the security group configuration?

---

### 🔤 PHÂN TÍCH SECURITY GROUP HIỆN TẠI (từ ảnh)

| Type | Protocol | Port | Source | Vấn đề |
|---|---|---|---|---|
| RDP | TCP | 3389 | 125.84.117.80/32 | ✅ Ổn — chỉ cho một IP cụ thể |
| MS SQL | TCP | 1433 | **0.0.0.0/0, ::/0** | ❌ Cho phép **toàn bộ internet** — cực kỳ nguy hiểm |

**Vấn đề**: MS SQL port 1433 đang mở cho **Anywhere (0.0.0.0/0)** — bất kỳ ai trên internet đều có thể kết nối vào database.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Chỉ application tier** mới được connect đến database qua port 1433.
2. Application tier dùng **Auto Scaling group** → số lượng instance thay đổi liên tục.
3. **Least privilege** → chỉ đúng source cần thiết mới được phép.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Đổi Source thành Security Group ID của application tier**

- Thay `0.0.0.0/0` bằng **Security Group ID** của application tier (ví dụ: `sg-0abc123`). ✅
- Bất kỳ EC2 instance nào thuộc security group đó đều được phép connect — **kể cả instance mới do Auto Scaling thêm vào**. ✅
- Không cần update thủ công khi scale in/out. ✅
- **Least privilege**: chỉ đúng application tier, không phải toàn internet. ✅

---

❌ **Đáp án sai 1: EC2 instance IDs của Auto Scaling group**

- Auto Scaling **liên tục thêm/bớt instance** → instance IDs thay đổi liên tục.
- Phải **update security group thủ công** mỗi lần scale → operational overhead cao, dễ sai sót.
- Không scalable.

---

❌ **Đáp án sai 2: Static AnyCast IP**

- **AnyCast IP** là tính năng của **AWS Global Accelerator** — không phải địa chỉ IP của EC2 instances.
- Không liên quan đến security group configuration.

---

❌ **Đáp án sai 3: Network ACL ID của application tier**

- **NACL ID không dùng được** làm source trong Security Group rule.
- NACL hoạt động ở **tầng subnet** → affect toàn bộ instances trong subnet, không chỉ application tier.
- Không đủ granular cho least privilege.

---

### 📊 KẾT LUẬN & BÀI HỌC

**Cấu hình đúng sau khi sửa:**

| Type | Protocol | Port | Source |
|---|---|---|---|
| RDP | TCP | 3389 | 125.84.117.80/32 (giữ nguyên) |
| MS SQL | TCP | 1433 | **sg-xxxx (App tier Security Group ID)** |

**Tại sao dùng Security Group ID thay vì IP:**

```
Auto Scaling group thêm instance mới:
  → Instance tự động thuộc App tier Security Group
  → Tự động được phép connect đến DB (port 1433)
  → Không cần update gì thêm ✅

Nếu dùng IP:
  → Instance mới có IP mới
  → Phải update DB security group thủ công ❌
```

💡 **Công thức ghi nhớ:**

> **"Chỉ cho phép một service/tier cụ thể"** → dùng **Security Group ID** làm source, không dùng IP

> **Security Group reference** = tự động apply cho mọi instance thuộc group đó → hoạt động tốt với Auto Scaling

> **NACL** = subnet level, không đủ granular → không dùng để restrict traffic cho từng service

> **0.0.0.0/0** trong inbound rule của database = lỗ hổng bảo mật nghiêm trọng → phải thay ngay bằng Security Group ID cụ thể
