## Câu hỏi 46

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company has a global news website hosted in a fleet of EC2 Instances. Lately, the load on the website has increased which resulted in slower response time for the site visitors. This issue impacts the revenue of the company as some readers tend to leave the site if it does not load after 10 seconds. The goal is to address the issue in a cost-effective manner.
>
> Which of the below services in AWS can be used to solve this problem? (Select TWO.)

---

### 🔤 DỊCH CHI TIẾT

Website tin tức toàn cầu đang bị **chậm** do lượng truy cập tăng. Người đọc từ khắp nơi trên thế giới, và nếu trang không load trong 10 giây họ sẽ rời đi. Cần giải pháp **cost-effective** để cải thiện response time.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Global users** → cần giảm latency cho người dùng ở xa server.
2. **News website** → phần lớn là **read-only content** (bài viết, hình ảnh) → có thể cache.
3. **Cost-effective** → không muốn deploy EC2 khắp nơi.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng 1: Amazon CloudFront với website là custom origin**

- **CloudFront** là CDN (Content Delivery Network) — phân phối content từ các **Edge Locations** gần người dùng nhất trên toàn cầu. ✅
- Người dùng ở châu Á lấy content từ Edge Location châu Á thay vì phải request về server ở Mỹ → giảm latency đáng kể. ✅
- Cache static content (HTML, CSS, JS, hình ảnh) tại Edge → giảm tải cho EC2 origin. ✅
- Cost-effective: trả theo lượng data transfer, không cần deploy infrastructure mới. ✅

---

✅ **Đáp án đúng 2: Amazon ElastiCache**

- **ElastiCache** là in-memory caching service (Redis hoặc Memcached) — lưu kết quả query database và dữ liệu thường xuyên truy cập vào RAM. ✅
- Thay vì query database mỗi lần request → lấy từ cache trong microseconds. ✅
- News website = nhiều người đọc cùng bài viết → cache rất hiệu quả, giảm tải database và tăng tốc response. ✅

---

❌ **Đáp án sai 1: Deploy website lên tất cả regions**

- Tốn chi phí cực lớn (EC2, database, networking ở mỗi region).
- Phức tạp về quản lý, đồng bộ data giữa các region.
- CloudFront + ElastiCache giải quyết được vấn đề với chi phí thấp hơn nhiều.

---

❌ **Đáp án sai 2: RDS Multi-AZ**

- **Multi-AZ** cung cấp **high availability và automatic failover** — không phải giải pháp để tăng read throughput hay giảm latency cho website.
- Standby instance trong Multi-AZ không phục vụ read requests.
- Muốn scale read thì phải dùng **Read Replicas**, không phải Multi-AZ.

---

❌ **Đáp án sai 3: AWS Storage Gateway**

- **Storage Gateway** là giải pháp **hybrid storage** — kết nối on-premises storage với AWS.
- Không có chức năng distribute content hay cải thiện website performance.

---

### 📊 KẾT LUẬN & BÀI HỌC

**Hai lớp caching cho web application:**

```
User (global)
    │
    ↓
[CloudFront Edge Location] ← Cache static content (HTML, images, CSS, JS)
    │ (cache miss → về origin)
    ↓
[EC2 Web Server]
    │
    ↓
[ElastiCache] ← Cache database query results
    │ (cache miss → query DB)
    ↓
[RDS Database]
```

**Phân biệt các giải pháp cải thiện performance:**

| Vấn đề | Giải pháp |
|---|---|
| Latency cao cho global users | **CloudFront** (CDN) |
| Database query chậm, lặp lại | **ElastiCache** (in-memory cache) |
| Database overload với read | **RDS Read Replicas** |
| Database HA/failover | **RDS Multi-AZ** |
| Scale compute | **Auto Scaling Group** |

💡 **Công thức ghi nhớ:**

> **"Global users + slow response"** → **CloudFront** (CDN giảm latency địa lý)

> **"Repetitive reads + database overload"** → **ElastiCache** (in-memory cache)

> **"RDS Multi-AZ"** = HA/failover, **không phải** read scaling hay performance improvement

> **News website** = read-heavy, static content nhiều → CloudFront + ElastiCache là combo hoàn hảo
