## Câu hỏi 54

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company is using an On-Demand EC2 instance to host a legacy web application that uses an Amazon Instance Store-Backed AMI. The web application should be decommissioned as soon as possible and hence, you need to terminate the EC2 instance.
>
> When the instance is terminated, what happens to the data on the root volume?

---

### 🔤 DỊCH CHI TIẾT

EC2 instance đang dùng **Instance Store-Backed AMI** (AMI có root volume là instance store). Khi **terminate** instance này, điều gì xảy ra với data trên root volume?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Câu hỏi kiểm tra kiến thức về **sự khác biệt giữa hai loại AMI** và đặc tính **ephemeral** của Instance Store.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Data is automatically deleted**

- **Instance Store** là **ephemeral storage** — data chỉ tồn tại trong suốt vòng đời của instance.
- Khi instance **terminate** → toàn bộ data trên instance store bị **xóa vĩnh viễn, không khôi phục được**. ✅
- Không có backup tự động, không có snapshot.

---

❌ **Đáp án sai 1: Data saved as EBS snapshot**

- Chỉ **EBS volume** mới có thể tạo snapshot.
- Instance Store không tích hợp với EBS snapshot — không có cơ chế tự động backup.

---

❌ **Đáp án sai 2: Data saved as EBS volume**

- Instance Store không chuyển đổi tự động thành EBS volume khi terminate.
- Đây là hai loại storage hoàn toàn khác nhau.

---

❌ **Đáp án sai 3: Data unavailable until restart**

- Instance Store-Backed instance **không thể restart sau khi terminate** — terminate là vĩnh viễn.
- Ngoài ra, ngay cả khi **stop** (không phải terminate) một instance có instance store, data cũng bị mất.

---

### 📊 KẾT LUẬN & BÀI HỌC

**Hai loại AMI và đặc tính của root volume:**

| | EBS-Backed AMI | Instance Store-Backed AMI |
|---|---|---|
| Root volume | Amazon EBS | Instance Store (local NVMe/SSD) |
| Khi **Stop** instance | Data giữ nguyên | **Không thể Stop** |
| Khi **Terminate** instance | EBS volume mặc định bị xóa (có thể giữ lại) | **Data xóa vĩnh viễn** |
| Khi **Reboot** | Data giữ nguyên | Data giữ nguyên |
| Persistence | ✅ Persistent | ❌ Ephemeral |
| Snapshot | ✅ Có thể tạo EBS snapshot | ❌ Không hỗ trợ |

**Vòng đời data của Instance Store:**

```
Instance RUNNING  → Data tồn tại ✅
Instance REBOOT   → Data tồn tại ✅
Instance STOP     → Không thể stop ❌
Instance TERMINATE → Data bị XÓA VĨNH VIỄN ❌
Hardware failure  → Data bị XÓA VĨNH VIỄN ❌
```

💡 **Công thức ghi nhớ:**

> **Instance Store = Ephemeral** — data chỉ sống khi instance đang chạy, terminate hoặc hardware fail → mất hết

> **EBS = Persistent** — data tồn tại độc lập với instance, terminate instance vẫn giữ được EBS (nếu cấu hình đúng)

> Instance Store-Backed AMI **không thể Stop** — chỉ có Reboot hoặc Terminate

> Dùng Instance Store khi: data có replication, cache, temp data — **không bao giờ** lưu data quan trọng trên Instance Store mà không backup
