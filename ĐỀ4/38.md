## Câu hỏi 38

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A food company bought 50 licenses of Windows Server to be used by developers when launching Amazon EC2 instances. The licenses are tied to the total CPU count of each virtual machine. The company wants to ensure that developers won't be able to launch new instances once the licenses are exhausted, and wants to receive notifications when all licenses are in use.
>
> Which of the following options is the recommended solution to meet the company's requirements?

---

### 🔤 DỊCH CHI TIẾT

Công ty có **50 license Windows Server** cho developer dùng khi launch EC2. License tính theo **số CPU**. Yêu cầu: **chặn không cho launch instance mới** khi hết license, và **gửi notification** khi license cạn kiệt.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Từ khóa dẫn đến đáp án ngay:

1. **Software license management** trên AWS → **AWS License Manager** — đây là service được thiết kế chính xác cho use case này.
2. **Enforce limit** — chặn launch khi hết license.
3. **Notification** → SNS.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: AWS License Manager + Enforce license limit + Amazon SNS**

- **AWS License Manager** quản lý software licenses từ các vendor (Microsoft, Oracle, SAP...) trên AWS và on-premises. ✅
- Cho phép định nghĩa **licensing rules** dựa trên số CPU, core, socket... ✅
- Tùy chọn **"Enforce license limit"** → tự động **block instance launch** khi vượt quá số license cho phép. ✅
- Tích hợp với **SNS** để gửi notification khi license sắp hoặc đã hết. ✅
- Đây là giải pháp **BYOL (Bring Your Own License)** chuẩn trên AWS. ✅

---

❌ **Đáp án sai 1: AWS Resource Access Manager (RAM)**

- **AWS RAM** dùng để **chia sẻ AWS resources** (subnet, Transit Gateway, RAM shares...) giữa các AWS accounts trong Organization.
- Không có chức năng track hay enforce software license limits.

---

❌ **Đáp án sai 2: AWS Systems Manager Fleet Manager**

- **Fleet Manager** là một capability của Systems Manager — dùng để **quản lý và monitor EC2 instances** từ xa (xem file, user, process...).
- Không có chức năng quản lý hay distribute software licenses.

---

❌ **Đáp án sai 3: AWS Certificate Manager + SQS**

- **AWS Certificate Manager** dùng để **quản lý SSL/TLS certificates** — hoàn toàn không liên quan đến software licensing.
- **SQS** là message queue — không phải notification service. Phải dùng **SNS** để gửi alerts.
- Hai lỗi sai đồng thời: sai service quản lý license và sai notification service.

---

### 📊 KẾT LUẬN & BÀI HỌC

**AWS License Manager — tính năng chính:**

| Tính năng | Mô tả |
|---|---|
| **License rules** | Định nghĩa license theo CPU, core, socket, instance count |
| **Enforce license limit** | Block EC2 launch khi vượt quá số license |
| **Dashboard** | Visibility vào license usage real-time |
| **BYOL support** | Tái sử dụng license on-premises lên AWS |
| **SNS integration** | Gửi alert khi license sắp hết hoặc hết |
| **Multi-account** | Track license across nhiều AWS accounts |

**Phân biệt các service hay bị nhầm:**

| Service | Thực chất là gì |
|---|---|
| **AWS License Manager** | Quản lý software licenses (Windows, Oracle...) |
| **AWS Certificate Manager** | Quản lý SSL/TLS certificates |
| **AWS RAM** | Chia sẻ AWS resources giữa accounts |
| **Systems Manager Fleet Manager** | Quản lý và monitor EC2 instances từ xa |

**Notification service đúng:**

| Cần | Dùng |
|---|---|
| Gửi email/SMS/alert | **SNS** |
| Truyền message giữa services | SQS |
| Workflow phức tạp | SWF / Step Functions |

💡 **Công thức ghi nhớ:**

> **"Software license management (Windows, Oracle...) trên AWS"** → **AWS License Manager**

> **"Enforce license limit + block launch khi hết"** → tính năng built-in của License Manager, chỉ cần enable

> **AWS Certificate Manager** ≠ License Manager — Certificate Manager là SSL/TLS, không phải software license

> **AWS RAM** = share AWS resources giữa accounts, không phải license tracking

> Notification = **SNS**, không phải SQS
