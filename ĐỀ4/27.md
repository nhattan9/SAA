## Câu hỏi 27

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company hosts all its applications on its data center on the US East Coast. Most of the workloads are legacy applications hosted on individual virtual machines running in Linux and Windows operating systems. The company plans to migrate all of its VM workloads to the AWS cloud using a "lift-and-shift" strategy. The company also wants to minimize downtime during the migration process.
>
> Which of the following options should the Solutions Architect implement for this scenario?

---

### 🔤 DỊCH CHI TIẾT

Công ty muốn migrate toàn bộ VM workloads từ on-premises lên AWS bằng chiến lược **"lift-and-shift"** (di chuyển nguyên trạng, không sửa đổi ứng dụng) và yêu cầu **minimize downtime** trong quá trình migration.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Lift-and-shift** — giữ nguyên ứng dụng, không cần refactor hay thay đổi code.
2. **Minimize downtime** — cần giải pháp **continuous replication** (liên tục đồng bộ) để cutover nhanh, không phải copy một lần rồi xong.
3. **VM workloads (Linux & Windows)** — cần tool chuyên biệt cho VM migration.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: AWS Application Migration Service (AWS MGN) + Replication Agent**

- Cài **AWS Replication Agent** trên từng on-premises VM → agent **liên tục replicate** server lên AWS theo thời gian thực. ✅
- AWS MGN tự động convert server để boot và chạy native trên AWS (không cần thay đổi ứng dụng). ✅
- Có thể launch **Test instances** trước để verify — cutover chỉ xảy ra khi sẵn sàng → **minimize downtime**. ✅
- Đây là service **chính thức được AWS recommend** cho lift-and-shift migration. ✅

---

❌ **Đáp án sai 1: Export VM → upload S3 → VM Import/Export**

- Phải **export VM thủ công**, upload lên S3, rồi import — quá trình này **tốn rất nhiều thời gian**.
- Nếu source VM tiếp tục thay đổi sau khi upload, bản image trên S3 sẽ **bị lỗi thời (outdated)**.
- **Không phải continuous replication** → downtime cao → vi phạm yêu cầu minimize downtime.

---

❌ **Đáp án sai 2: AWS Application Discovery Service**

- **Application Discovery Service** dùng để **thu thập thông tin** về on-premises infrastructure (inventory, dependencies) và theo dõi migration status từ Migration Hub.
- **Không có khả năng thực hiện migration thực sự** — chỉ là công cụ discovery và tracking.
- Dễ nhầm vì tên gọi có chữ "Application" giống AWS MGN.

---

❌ **Đáp án sai 3: AWS DataSync**

- **DataSync** được thiết kế để **transfer data/files** giữa on-premises storage và AWS storage (S3, EFS, FSx).
- Không có khả năng replicate hay migrate **Virtual Machines**.
- DataSync phù hợp cho migration dữ liệu, không phải migration VM workloads.

---

### 📊 KẾT LUẬN & BÀI HỌC

**Bảng so sánh các Migration services của AWS:**

| Service | Mục đích thực sự | Dùng khi |
|---|---|---|
| **AWS MGN** (Application Migration Service) | Lift-and-shift VM migration với continuous replication | Migrate VM lên EC2, minimize downtime |
| **AWS Application Discovery Service** | Inventory & dependency mapping on-premises | Lên kế hoạch migration, tracking |
| **AWS DataSync** | Transfer files/data giữa on-premises và AWS storage | Di chuyển dữ liệu, không phải VM |
| **VM Import/Export** | Import VM image một lần vào AWS | Migration đơn giản, chấp nhận downtime cao |
| **AWS DMS** | Database migration | Migrate database, không phải VM |
| **AWS Snowball** | Offline data transfer khối lượng lớn | Khi bandwidth không đủ cho online transfer |

**Quy trình AWS MGN:**

```
On-premises VMs
      │ (cài Replication Agent)
      ↓
Continuous Replication → AWS Staging Area (low-cost)
      │
      ↓ (khi sẵn sàng)
Launch Test Instances → Verify hoạt động đúng
      │
      ↓ (cutover)
Production EC2 Instances trên AWS
```

💡 **Công thức ghi nhớ:**

> **"Lift-and-shift VM migration + minimize downtime"** → **AWS MGN (Application Migration Service)**

> **AWS MGN** = continuous replication + test before cutover = minimize downtime

> **Application Discovery Service** = chỉ discover/track, **không migrate** được

> **DataSync** = di chuyển data/files, **không migrate VM**

> **VM Import/Export** = one-time import, downtime cao, dùng cho trường hợp đơn giản không cần minimize downtime
