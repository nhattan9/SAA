## Câu hỏi 40

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A hospital has a mission-critical application that uses a RESTful API powered by Amazon API Gateway and AWS Lambda. The medical officers upload PDF reports to the system which are then stored as static media content in an Amazon S3 bucket. The security team wants to improve its visibility when it comes to cyber-attacks and ensure HIPAA compliance. The company is searching for a solution that continuously monitors object-level S3 API operations and identifies protected health information (PHI) in the reports, with minimal changes in the existing Lambda function.
>
> Which of the following solutions will meet these requirements with the LEAST operational overhead?

---

### 🔤 DỊCH CHI TIẾT

Bệnh viện có ứng dụng dùng API Gateway + Lambda. Bác sĩ upload PDF reports lên S3. Yêu cầu: nhận dạng **PHI (Protected Health Information — thông tin y tế cần bảo vệ)** trong các PDF, **thay đổi tối thiểu** Lambda function hiện có, và **least operational overhead**.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **PDF → Extract text** → cần service đọc và trích xuất text từ PDF.
2. **Identify PHI** trong text y tế → cần service chuyên biệt về medical NLP (Natural Language Processing).
3. **Minimal changes** → tích hợp vào Lambda hiện có, không tạo thêm Lambda mới.
4. **HIPAA compliance** → service phải là HIPAA-eligible.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Amazon Textract + Amazon Comprehend Medical**

- **Amazon Textract** trích xuất text từ PDF/image — được thiết kế chính xác cho tài liệu dạng scanned PDF. ✅
- **Amazon Comprehend Medical** là NLP service chuyên về dữ liệu y tế — nhận dạng được **PHI** (tên bệnh nhân, tuổi, mã hồ sơ, địa chỉ...), bệnh lý, thuốc, liều lượng... ✅
- **HIPAA-eligible** — đáp ứng yêu cầu compliance. ✅
- Tích hợp vào **Lambda hiện có** qua API call — minimal changes. ✅
- Không cần tự build algorithm → least operational overhead. ✅

---

❌ **Đáp án sai 1: Textract + custom PHI detection algorithm trong Lambda**

- Custom-built algorithm = phải tự viết, tự train, tự maintain → **operational overhead cao**.
- Kém chính xác và không HIPAA-compliant tự động như Comprehend Medical.
- Vi phạm yêu cầu "least operational overhead".

---

❌ **Đáp án sai 2: Amazon Transcribe + CloudWatch Logs**

- **Amazon Transcribe** là **speech-to-text** service — chuyển âm thanh thành text, không đọc được PDF hay image.
- **CloudWatch Logs** không có khả năng detect PHI trong nội dung document.
- Cả hai service đều sai mục đích hoàn toàn.

---

❌ **Đáp án sai 3: "Amazon Textract Medical" + PII redaction + Comprehend API thường**

- **"Amazon Textract Medical"** không tồn tại — đây là service giả. Textract là một service, không có phiên bản Medical.
- **PII** (Personally Identifiable Information) khác với **PHI** (Protected Health Information) — PII rộng hơn, không đặc thù cho y tế.
- Dùng **Comprehend thường** thay vì **Comprehend Medical** sẽ thiếu khả năng nhận dạng thông tin y tế đặc thù.
- Tạo **Lambda mới** vi phạm yêu cầu "minimal changes to existing Lambda".

---

### 📊 KẾT LUẬN & BÀI HỌC

**Pipeline đúng cho bài toán này:**

```
Medical officer upload PDF → S3
                │
                ↓ (trigger Lambda)
        Amazon Textract
        (extract text từ PDF)
                │
                ↓
    Amazon Comprehend Medical
    (identify PHI, medical entities)
                │
                ↓
        Kết quả trả về / lưu trữ / alert
```

**Phân biệt các AWS AI/ML services hay nhầm:**

| Service | Thực chất làm gì |
|---|---|
| **Amazon Textract** | Trích xuất text từ PDF, image, form |
| **Amazon Comprehend** | NLP cho text thông thường (sentiment, entities, key phrases) |
| **Amazon Comprehend Medical** | NLP chuyên biệt cho **medical text**, nhận dạng PHI, bệnh lý, thuốc |
| **Amazon Transcribe** | **Speech-to-text** — chuyển audio thành text |
| **Amazon Rekognition** | Phân tích hình ảnh và video |
| **Amazon Translate** | Dịch ngôn ngữ |

**PII vs PHI — phân biệt quan trọng:**

| | PII | PHI |
|---|---|---|
| Là gì | Thông tin cá nhân (tên, địa chỉ, email...) | Thông tin sức khỏe cá nhân (bệnh án, thuốc...) |
| Phạm vi | Rộng — mọi ngành | Đặc thù cho ngành y tế |
| Compliance | GDPR, CCPA | **HIPAA** |
| AWS service | Comprehend (PII detection) | **Comprehend Medical** |

💡 **Công thức ghi nhớ:**

> **"Extract text từ PDF"** → **Amazon Textract**

> **"Identify PHI trong medical text + HIPAA"** → **Amazon Comprehend Medical**

> **"Speech/audio → text"** → Amazon Transcribe (không dùng cho PDF)

> **Comprehend** ≠ **Comprehend Medical** — Medical có khả năng nhận dạng thông tin y tế đặc thù mà Comprehend thường không có
