## Câu hỏi 25

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A web application runs on an Auto Scaling group of Amazon EC2 instances deployed across multiple Availability Zones and is fronted by an Application Load Balancer. To enhance security, an SSL/TLS solution is required, and a certificate has been obtained from a third-party certificate authority (CA).
>
> Which services can the SSL/TLS certificate be securely imported into for use with the application? (Select TWO.)

---

### 🔤 DỊCH CHI TIẾT

Một web application chạy trên Auto Scaling group với ALB. Công ty đã có sẵn SSL/TLS certificate từ một **third-party CA** (không phải certificate do AWS cấp). Cần import certificate này vào đúng service để sử dụng với ứng dụng.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. Certificate đến từ **third-party CA** — cần service hỗ trợ **import** certificate ngoài.
2. Certificate phải **tích hợp được với ALB** để terminate SSL/TLS.
3. Cần **secure storage** cho private key của certificate.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng 1: AWS Certificate Manager (ACM)**

- ACM hỗ trợ **import third-party certificate** (không chỉ cấp certificate mới). ✅
- Tích hợp trực tiếp với ALB, CloudFront, API Gateway — associate certificate dễ dàng. ✅
- Lưu trữ private key an toàn, managed service, không cần tự xử lý. ✅
- Đây là giải pháp **recommended** và hiện đại nhất của AWS cho SSL/TLS. ✅

---

✅ **Đáp án đúng 2: AWS IAM Certificate Store**

- IAM certificate store là **legacy feature** — ra đời trước ACM, vẫn còn hoạt động. ✅
- Hỗ trợ import third-party certificate và sử dụng với ALB, CloudFront. ✅
- Thiếu các tính năng hiện đại như auto-renewal, streamlined integration — nên dùng ACM thay thế nếu có thể. ✅
- Vẫn là đáp án đúng vì câu hỏi hỏi "có thể import vào đâu" — không yêu cầu giải pháp tốt nhất.

---

❌ **Đáp án sai 1: S3 bucket với versioning**

- S3 là **object storage** — lưu file được, nhưng không có cơ chế nào để ALB đọc certificate từ S3 và dùng cho SSL termination.
- Versioning chỉ giúp bảo toàn file, không làm cho certificate "hoạt động" được với load balancer.

---

❌ **Đáp án sai 2: S3 bucket với SSE-C**

- SSE-C (Server-Side Encryption with Customer-provided keys) chỉ mã hóa **data at rest** trong S3.
- Tương tự đáp án trên: S3 không tích hợp với ALB để deploy SSL/TLS certificate.

---

❌ **Đáp án sai 3: CSE-C (Client-Side Encryption)**

- Client-side encryption có nghĩa là bạn tự mã hóa trước khi upload — AWS không thể đọc nội dung.
- Nếu AWS không đọc được certificate thì ALB cũng không dùng được.
- Đây không phải managed certificate solution, chỉ là cách mã hóa file thông thường.

---

### 📊 KẾT LUẬN & BÀI HỌC

**So sánh ACM vs IAM Certificate Store:**

| Tiêu chí | ACM | IAM Certificate Store |
|---|---|---|
| Import third-party cert | ✅ | ✅ |
| Tích hợp ALB, CloudFront | ✅ | ✅ |
| Auto-renewal | ✅ (chỉ với cert do ACM cấp) | ❌ |
| Managed private key | ✅ | ✅ |
| Trạng thái | **Recommended** | Legacy |
| Khi nào dùng IAM store | Môi trường cũ chưa migrate sang ACM | — |

**Các AWS service tích hợp được với ACM:**

- Application Load Balancer (ALB)
- Network Load Balancer (NLB)
- CloudFront
- API Gateway
- Elastic Beanstalk

💡 **Công thức ghi nhớ:**

> **"Import third-party SSL/TLS certificate vào AWS"** → **ACM** (recommended) hoặc **IAM Certificate Store** (legacy)

> **S3** = object storage — không thể dùng để deploy certificate cho load balancer dù có encrypt

> **ACM** = modern, tích hợp trực tiếp với AWS services, hỗ trợ auto-renewal

> **IAM Certificate Store** = legacy nhưng vẫn valid — hay xuất hiện trong đề thi như đáp án đúng thứ hai bên cạnh ACM
