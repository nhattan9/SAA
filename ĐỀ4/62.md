## Câu hỏi 62

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company has an Application Load Balancer (ALB) that accepts HTTP and HTTPS traffic on ports 80 and 443, respectively. Recently, the company associated a new domain for its website, and they want to ensure that all HTTP traffic for this new domain is automatically redirected to HTTPS to improve security.
>
> Which ALB configuration should be done to satisfy the requirement?

---

### 🔤 DỊCH CHI TIẾT

ALB đang có **2 listener sẵn**:
- Port **80** → HTTP (đã có)
- Port **443** → HTTPS (đã có)

Yêu cầu: HTTP traffic vào port 80 phải được **tự động redirect sang HTTPS** (port 443).

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Điểm mấu chốt: **listener đã tồn tại** → không thể tạo duplicate listener trên cùng port. Chỉ cần **sửa listener port 80 hiện có** để redirect sang 443.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Configure existing HTTP listener (port 80) to redirect to port 443**

- Listener port 80 **đã tồn tại** → chỉ cần **thêm redirect rule** vào listener đó. ✅
- Rule: khi nhận HTTP request → redirect sang `https://` port 443. ✅
- Không cần tạo listener mới — đơn giản, ít thay đổi nhất. ✅

---

❌ **Đáp án sai 1: Configure existing listener port 443 + add redirect to HTTP port 80**

- **Ngược chiều** — đây là redirect HTTPS → HTTP, không phải HTTP → HTTPS.
- Đi ngược lại mục tiêu bảo mật.

---

❌ **Đáp án sai 2: Tạo NEW listener port 443 để redirect HTTP sang HTTPS**

- **Port 443 đã có listener rồi** — không thể tạo thêm listener trên cùng port trong ALB.
- ALB không cho phép **duplicate port**.

---

❌ **Đáp án sai 3: Tạo NEW HTTP listener port 80 + redirect sang 443**

- **Port 80 đã có listener rồi** — không thể tạo thêm listener trên cùng port.
- Cách làm đúng về mặt kỹ thuật, nhưng không thực hiện được vì port đã bị dùng.

---

### 📊 KẾT LUẬN & BÀI HỌC

**Tình trạng ban đầu và thay đổi cần làm:**

```
Trước:
Port 80  (HTTP listener)  → forward đến target group
Port 443 (HTTPS listener) → forward đến target group

Sau khi sửa:
Port 80  (HTTP listener)  → REDIRECT sang https://port 443  ← chỉ sửa cái này
Port 443 (HTTPS listener) → forward đến target group (giữ nguyên)
```

**Nguyên tắc ALB Listener:**

| Quy tắc | Chi tiết |
|---|---|
| Mỗi port chỉ có **1 listener** | Không tạo được duplicate port |
| Listener có thể có nhiều **rules** | Forward, Redirect, Fixed response, Authenticate |
| **Redirect action** | Chuyển request sang protocol/port/host/path khác |

**ALB Redirect rule — các tham số:**

```
Protocol: HTTPS
Port: 443
Status code: 301 (Permanent) hoặc 302 (Temporary)
```

💡 **Công thức ghi nhớ:**

> **"HTTP → HTTPS redirect trên ALB"** → **Sửa listener port 80 hiện có**, thêm redirect action sang port 443

> **Không tạo listener mới** nếu port đó đã có listener — ALB không cho phép duplicate port

> Redirect action nằm trên listener **nguồn** (port 80), không phải listener đích (port 443)

> **301** = Permanent redirect (SEO-friendly) | **302** = Temporary redirect
