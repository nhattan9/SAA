## Câu hỏi 20

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A startup launched a new FTP server using an On-Demand EC2 instance in a newly created VPC with default settings. The server should not be accessible publicly but only through the IP address 175.45.116.100 and nowhere else.
>
> Which of the following is the most suitable way to implement this requirement?

---

### 🔤 DỊCH CHI TIẾT

Một startup đã khởi chạy FTP server trên một EC2 instance trong một VPC mới tạo với **default settings**. Server chỉ được phép truy cập từ duy nhất một IP là **175.45.116.100**, không cho phép bất kỳ IP nào khác.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Ba kiến thức cần nắm để giải câu này:

1. **FTP dùng protocol TCP, port 20–21** — không phải UDP.
2. **CIDR /32** = chỉ đúng một IP duy nhất | **/0** = toàn bộ internet.
3. **VPC default settings** — Network ACL mặc định đã **allow all traffic** → không cần thêm rule NACL, chỉ cần cấu hình Security Group là đủ.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Security Group inbound rule — TCP, port 20–21, source 175.45.116.100/32**

- **Protocol TCP** — đúng, FTP dùng TCP. ✅
- **Port 20–21** — đúng, FTP control port (21) và data port (20). ✅
- **Source /32** — đúng, chỉ cho phép đúng một IP duy nhất. ✅
- VPC có default settings → Network ACL đã allow all → chỉ cần Security Group là đủ. ✅

---

❌ **Đáp án sai 1: Network ACL, TCP, source 175.45.116.100/0**

- **/0** nghĩa là toàn bộ internet — sai, phải dùng **/32** để chỉ định một IP cụ thể.
- Thêm nữa: VPC default settings đã có NACL allow all → không cần thêm NACL rule.

---

❌ **Đáp án sai 2: Network ACL, **UDP**, source 175.45.116.100/0**

- **Sai protocol**: FTP dùng TCP, không phải UDP.
- **Sai CIDR**: /0 là toàn bộ internet, không phải một IP.
- Hai lỗi sai đồng thời.

---

❌ **Đáp án sai 3: Security Group inbound rule — **UDP**, port 20–21, source 175.45.116.100/32**

- **Sai protocol**: FTP dùng TCP, không phải UDP.
- Source /32 là đúng, nhưng protocol sai nên toàn bộ rule không hoạt động.

---

### 📊 KẾT LUẬN & BÀI HỌC

**CIDR notation cần nhớ:**

| CIDR | Ý nghĩa |
|---|---|
| `x.x.x.x/32` | Đúng **một IP** duy nhất |
| `x.x.x.x/24` | Một subnet /24 (256 IPs) |
| `0.0.0.0/0` | **Toàn bộ internet** |

**Protocol của các dịch vụ phổ biến hay gặp trong đề thi:**

| Dịch vụ | Protocol | Port |
|---|---|---|
| FTP | TCP | 20, 21 |
| SSH | TCP | 22 |
| HTTP | TCP | 80 |
| HTTPS | TCP | 443 |
| RDP | TCP | 3389 |
| DNS | UDP (và TCP) | 53 |
| MySQL/Aurora | TCP | 3306 |
| PostgreSQL | TCP | 5432 |

**Security Group vs Network ACL — khi nào dùng cái nào:**

| Tiêu chí | Security Group | Network ACL |
|---|---|---|
| Áp dụng cho | EC2 instance | Toàn bộ subnet |
| Stateful/Stateless | Stateful (auto allow response) | Stateless (phải tạo cả inbound + outbound) |
| Default VPC | Deny all inbound, allow all outbound | Allow all inbound và outbound |
| Khi nào dùng | Kiểm soát traffic đến từng instance | Kiểm soát traffic ở cấp subnet |

💡 **Công thức ghi nhớ:**

> **FTP** = **TCP** port **20–21** (không phải UDP)

> **Một IP cụ thể** = dùng **/32** | **Toàn bộ internet** = dùng **/0**

> **VPC default settings** → NACL đã allow all → **chỉ cần Security Group**, không cần thêm NACL rule

> Security Group là **stateful** → chỉ cần tạo inbound rule, response traffic tự động được allow
