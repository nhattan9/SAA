## Câu hỏi 18

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company has a web application hosted in an On-Demand EC2 instance. You are creating a shell script that needs the instance's public and private IP addresses.
>
> What is the best way to get the instance's associated IP addresses which your shell script can use?

---

### 🔤 DỊCH CHI TIẾT

Một công ty có web application chạy trên On-Demand EC2 instance. Bạn đang viết một shell script cần lấy địa chỉ **public IP** và **private IP** của instance. Cách nào tốt nhất để script lấy được thông tin IP đó?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. Cần lấy IP từ **bên trong** EC2 instance (vì đang viết shell script chạy trên chính instance đó).
2. Cần giải pháp **đơn giản, không cần AWS CLI hay console** — chỉ dùng curl/wget từ bên trong instance.
3. Phân biệt **Instance Metadata** vs **User Data** — hai endpoint khác nhau, mục đích khác nhau.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Curl đến `http://169.254.169.254/latest/meta-data/`**

- Đây là địa chỉ **Instance Metadata Service (IMDS)** — một endpoint đặc biệt chỉ có thể truy cập từ bên trong EC2 instance. ✅
- Cung cấp thông tin về instance: private IP, public IP, instance ID, security groups, IAM role... ✅
- Ví dụ lấy private IP trong shell script:
  ```bash
  curl http://169.254.169.254/latest/meta-data/local-ipv4
  ```
- Ví dụ lấy public IP:
  ```bash
  curl http://169.254.169.254/latest/meta-data/public-ipv4
  ```
- Không cần credentials, không cần AWS CLI, chạy được ngay từ bên trong instance. ✅

---

❌ **Đáp án sai 1: CloudWatch metric**

- CloudWatch thu thập **performance metrics** (CPU, network bytes...) — không cung cấp IP address của instance.
- Không thể dùng trong shell script để lấy IP một cách đơn giản.

---

❌ **Đáp án sai 2: IAM**

- IAM dùng để quản lý **quyền truy cập và xác thực** — không liên quan đến việc lấy IP address.

---

❌ **Đáp án sai 3: Curl đến `http://169.254.169.254/latest/user-data/`**

- Endpoint này trả về **User Data** — là đoạn script hoặc configuration mà bạn **tự truyền vào** lúc launch instance (ví dụ: bootstrap script cài phần mềm).
- Không chứa IP address hay thông tin hệ thống của instance.
- Dễ nhầm vì cùng địa chỉ 169.254.169.254, chỉ khác đường dẫn cuối.

---

### 📊 KẾT LUẬN & BÀI HỌC

**Phân biệt hai endpoint quan trọng trên EC2:**

| Endpoint | Tên gọi | Chứa gì | Ví dụ dùng |
|---|---|---|---|
| `.../latest/meta-data/` | **Instance Metadata** | IP, instance ID, AZ, IAM role, security groups... | Shell script lấy IP, lấy IAM credentials |
| `.../latest/user-data/` | **User Data** | Script/config do bạn tự truyền vào lúc launch | Bootstrap script cài Apache, cấu hình ứng dụng lúc khởi động |

**Một số metadata path hay dùng:**

| Path | Trả về |
|---|---|
| `meta-data/local-ipv4` | Private IP |
| `meta-data/public-ipv4` | Public IP |
| `meta-data/instance-id` | Instance ID |
| `meta-data/placement/availability-zone` | AZ hiện tại |
| `meta-data/iam/security-credentials/` | IAM role credentials |

💡 **Công thức ghi nhớ:**

> **"Lấy thông tin của instance từ bên trong"** → `http://169.254.169.254/latest/meta-data/`

> **`meta-data`** = thông tin hệ thống của instance (IP, ID, AZ...)

> **`user-data`** = dữ liệu bạn tự truyền vào lúc launch (bootstrap script)

> Địa chỉ `169.254.169.254` là **link-local address** — chỉ accessible từ bên trong EC2 instance, không route ra ngoài internet
