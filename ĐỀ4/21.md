## Câu hỏi 21

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A startup plans to develop a multiplayer game that uses UDP as the protocol for communication between clients and game servers. The data of the users will be stored in a key-value store. As the Solutions Architect, you need to implement a solution that will distribute the traffic across a number of servers.
>
> Which of the following could help you achieve this requirement?

---

### 🔤 DỊCH CHI TIẾT

Một startup muốn phát triển game multiplayer sử dụng **UDP** làm giao thức giao tiếp giữa client và game server. Dữ liệu người dùng sẽ được lưu trong **key-value store**. Cần triển khai giải pháp phân phối traffic đến nhiều server.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Bài toán có hai yêu cầu độc lập, mỗi yêu cầu loại bỏ một số đáp án:

1. **UDP protocol** → chỉ **Network Load Balancer (NLB)** hỗ trợ UDP — ALB không hỗ trợ.
2. **Key-value store** → chỉ **DynamoDB** là key-value store — Aurora và RDS là relational database.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Network Load Balancer + Amazon DynamoDB**

- **NLB** hoạt động ở **Layer 4 (Transport Layer)** của mô hình OSI → hỗ trợ cả TCP lẫn **UDP**. ✅
- NLB xử lý UDP bằng **flow hash algorithm** dựa trên source IP, source port, destination IP, destination port → cùng một UDP flow luôn được route đến cùng một target. ✅
- **DynamoDB** là managed NoSQL database, hỗ trợ **key-value và document store model** → đúng yêu cầu. ✅

---

❌ **Đáp án sai 1: Application Load Balancer + DynamoDB**

- **ALB** hoạt động ở **Layer 7 (Application Layer)** → chỉ hỗ trợ HTTP/HTTPS, **không hỗ trợ UDP**.
- DynamoDB đúng, nhưng ALB sai → toàn bộ đáp án sai.

---

❌ **Đáp án sai 2: Network Load Balancer + Amazon Aurora**

- NLB đúng, nhưng **Aurora là relational database** — không phải key-value store.
- Bài toán yêu cầu key-value store → phải dùng DynamoDB.

---

❌ **Đáp án sai 3: Application Load Balancer + Amazon RDS**

- **ALB không hỗ trợ UDP** — sai.
- **RDS là relational database** — không phải key-value store — sai.
- Cả hai thành phần đều sai.

---

### 📊 KẾT LUẬN & BÀI HỌC

**Bảng so sánh ba loại Load Balancer của AWS:**

| Tiêu chí | Application LB (ALB) | Network LB (NLB) | Gateway LB (GWLB) |
|---|---|---|---|
| OSI Layer | Layer 7 | Layer 4 | Layer 3 |
| Protocol hỗ trợ | HTTP, HTTPS, WebSocket | TCP, UDP, TLS | IP |
| Use case | Web app, REST API, microservices | Game server, IoT, real-time, UDP | Firewall, IDS/IPS appliances |
| UDP support | ❌ | ✅ | ✅ |

**Bảng so sánh Database theo mô hình lưu trữ:**

| Database | Loại | Dùng khi |
|---|---|---|
| **DynamoDB** | Key-value + Document | Cần key-value store, low latency, scalable |
| **Amazon RDS** | Relational (SQL) | Cần structured data, joins, transactions |
| **Amazon Aurora** | Relational (SQL) | Tương tự RDS nhưng performance cao hơn |
| **Amazon Neptune** | Graph | Dữ liệu dạng đồ thị (social network) |
| **Amazon ElastiCache** | In-memory key-value | Caching, session store, cần tốc độ cực cao |

💡 **Công thức ghi nhớ:**

> **UDP traffic** → chỉ **Network Load Balancer** (NLB) — ALB không hỗ trợ UDP

> **"Key-value store"** → **DynamoDB** — không phải RDS, không phải Aurora

> **ALB** = Layer 7, HTTP/HTTPS only → web app, API

> **NLB** = Layer 4, TCP/UDP → game, IoT, real-time, high-performance

> Từ khóa nhận diện DynamoDB trong đề: **"key-value"**, **"NoSQL"**, **"low latency"**, **"millisecond"**, **"flexible schema"**
