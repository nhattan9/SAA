## Câu hỏi 37

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company has a team of developers that provisions its own resources on the AWS cloud. The developers use IAM user access keys to automate resource provisioning and application testing processes in AWS. To ensure proper security compliance, the security team wants to automate the process of deactivating and deleting any IAM user access key that is over 90 days old.
>
> Which solution will meet these requirements with the LEAST operational effort?

---

### 🔤 DỊCH CHI TIẾT

Các developer dùng **IAM user access keys** để tự động hóa việc provision resources. Security team muốn **tự động deactivate và delete** các access key đã tồn tại quá **90 ngày** — với ít operational effort nhất.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Phát hiện** access key nào đã quá 90 ngày (non-compliant) → cần service có thể đánh giá cấu hình IAM.
2. **Tự động xử lý** (deactivate + delete) khi phát hiện non-compliant → cần trigger action.
3. **Least operational effort** → ưu tiên managed rule có sẵn, không tự build custom.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: AWS Config managed rule + EventBridge + Lambda**

- **AWS Config** có sẵn managed rule **`access-keys-rotated`** — tự động kiểm tra access key nào chưa được rotate trong 90 ngày và đánh dấu **NON_COMPLIANT**. ✅
- **Amazon EventBridge** lắng nghe Config compliance events → khi phát hiện NON_COMPLIANT key → trigger Lambda. ✅
- **Lambda function** thực hiện deactivate và delete key tự động. ✅
- Dùng **managed rule có sẵn** → không cần tự viết logic kiểm tra → least operational effort. ✅

---

❌ **Đáp án sai 1: EventBridge filter IAM keys > 90 ngày + AWS Batch**

- **EventBridge không thể trực tiếp filter IAM access key dựa trên tuổi của key** — EventBridge không có thông tin về age của IAM keys.
- **AWS Batch** là service xử lý batch computing jobs phức tạp — overkill cho tác vụ đơn giản như delete access key.
- Phải dùng AWS Config để detect non-compliant keys trước.

---

❌ **Đáp án sai 2: Custom Config rule + EventBridge Scheduler + AWS Batch**

- **Custom Config rule** không cần thiết — đã có **managed rule** `access-keys-rotated` sẵn rồi.
- **AWS Batch** lại không phù hợp — Lambda đơn giản hơn nhiều cho tác vụ này.
- Cả hai điểm này đều tăng thêm complexity không cần thiết → vi phạm "least operational effort".

---

❌ **Đáp án sai 3: EventBridge filter IAM keys > 90 ngày + Lambda**

- Tương tự đáp án sai 1: **EventBridge không thể trực tiếp phát hiện age của IAM access keys**.
- Lambda là đúng, nhưng thiếu bước phát hiện — phải có **AWS Config** ở giữa.
- Thiếu Config → pipeline không hoạt động được.

---

### 📊 KẾT LUẬN & BÀI HỌC

**Pipeline đúng cho bài toán này:**

```
AWS Config Managed Rule
(access-keys-rotated — kiểm tra key > 90 ngày)
            │
            │ phát hiện NON_COMPLIANT
            ↓
    Amazon EventBridge
    (rule lắng nghe Config compliance event)
            │
            │ trigger
            ↓
    AWS Lambda Function
    (deactivate + delete access key)
```

**So sánh Custom rule vs Managed rule trong AWS Config:**

| | Managed Rule | Custom Rule |
|---|---|---|
| Do ai tạo | AWS cung cấp sẵn | Bạn tự tạo bằng Lambda |
| Effort | Thấp — chỉ cần enable | Cao — phải tự viết logic |
| Ví dụ | `access-keys-rotated`, `s3-bucket-public-read-prohibited` | Logic nghiệp vụ đặc thù |
| Khi nào dùng | Use case phổ biến đã có sẵn | Use case đặc thù không có managed rule |

**Một số AWS Config managed rules hay gặp trong đề thi:**

| Rule | Kiểm tra gì |
|---|---|
| `access-keys-rotated` | IAM access key chưa rotate trong N ngày |
| `iam-password-policy` | Password policy đủ yêu cầu không |
| `s3-bucket-public-read-prohibited` | S3 bucket có bị public read không |
| `encrypted-volumes` | EBS volumes có được encrypt không |
| `root-account-mfa-enabled` | Root account có bật MFA không |

💡 **Công thức ghi nhớ:**

> **"Tự động detect và xử lý IAM access key quá hạn"** → **AWS Config managed rule + EventBridge + Lambda**

> **EventBridge không biết age của IAM key** — phải đi qua **AWS Config** trước để detect non-compliant

> **Managed rule > Custom rule** khi use case đã có sẵn → least operational effort

> **AWS Batch** = batch computing phức tạp → overkill, dùng **Lambda** cho tác vụ đơn giản
