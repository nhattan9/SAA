## Câu hỏi 16

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A few easily reproducible files must be stored in AWS without concerns about storage capacity. During the first month, all files will be accessed frequently, but after that period, access will become infrequent. Older files will only be accessed by developers, and no specific retrieval time requirement is defined. However, files under a specific `tdojo-finance` prefix in the Amazon S3 bucket will be used for post-processing that requires millisecond retrieval time.
>
> Given these conditions, which of the following options would be the most cost-effective solution for the storage requirements?

---

### 🔤 DỊCH CHI TIẾT

Một số file **dễ tái tạo lại** (easily reproducible) cần được lưu trên AWS mà không lo về dung lượng. Trong tháng đầu tiên, tất cả file được truy cập thường xuyên, sau đó trở nên ít được truy cập. Các file cũ chỉ được developer truy cập, **không có yêu cầu cụ thể về thời gian retrieval**. Tuy nhiên, các file thuộc prefix `tdojo-finance` cần được dùng cho post-processing với yêu cầu **retrieval time tính bằng millisecond**.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Ba yếu tố cần phân tích kỹ:

1. **"Easily reproducible"** — nếu mất dữ liệu có thể tạo lại → không cần lưu ở nhiều AZ → **S3 One Zone-IA** đủ dùng, rẻ hơn Standard-IA.
2. **Prefix `tdojo-finance`** — cần millisecond retrieval → phải dùng storage class có thể truy cập ngay lập tức (không phải Glacier).
3. **Các file còn lại** — không có yêu cầu retrieval time cụ thể → có thể dùng **Glacier** (retrieval chậm hơn nhưng rẻ nhất).

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: S3 Standard (tháng đầu) → tdojo-finance prefix: S3 One Zone-IA | còn lại: Glacier**

- **S3 One Zone-IA** cho prefix `tdojo-finance`: truy cập millisecond ✅, chi phí thấp hơn Standard-IA ✅, an toàn vì file dễ tái tạo nếu AZ bị sự cố ✅.
- **Glacier** cho phần còn lại: không có yêu cầu retrieval time → Glacier là lựa chọn rẻ nhất ✅.
- Dùng **Lifecycle Policy** để tự động chuyển đổi sau 1 tháng ✅.
- Tổng thể là giải pháp **cost-effective nhất** trong tất cả các đáp án ✅.

---

❌ **Đáp án sai 1: tdojo-finance → S3 Standard-IA | còn lại → Glacier**

- **Standard-IA** lưu dữ liệu trên **nhiều AZ** → đắt hơn One Zone-IA.
- Vì file **dễ tái tạo**, không cần redundancy đa AZ → trả thêm tiền cho tính năng không cần thiết.
- Đây là đáp án mà bạn đã chọn — sai vì không tận dụng được One Zone-IA cho file easily reproducible.

---

❌ **Đáp án sai 2: Chuyển toàn bộ bucket sang S3 Standard-IA**

- Áp dụng một storage class cho toàn bộ bucket — không phân biệt được yêu cầu khác nhau giữa `tdojo-finance` prefix và các file còn lại.
- Standard-IA đắt hơn Glacier → các file không cần retrieval nhanh đáng lẽ nên ở Glacier để tiết kiệm hơn.
- Không tối ưu chi phí.

---

❌ **Đáp án sai 3: Chuyển toàn bộ bucket sang S3 Intelligent-Tiering**

- **Intelligent-Tiering** phù hợp khi access pattern **không thể đoán trước** — nhưng bài toán này access pattern **đã rõ ràng**: thường xuyên trong tháng đầu, ít dần sau đó.
- Intelligent-Tiering tính phí monitoring cho mỗi object → không cost-effective khi pattern đã biết trước.
- Không phân biệt được hai nhóm file có yêu cầu khác nhau.

---

### 📊 KẾT LUẬN & BÀI HỌC

**Bảng so sánh S3 Standard-IA vs S3 One Zone-IA:**

| Tiêu chí | S3 Standard-IA | S3 One Zone-IA |
|---|---|---|
| Lưu trữ | Nhiều AZ | Chỉ 1 AZ |
| Chi phí | Cao hơn | Thấp hơn ~20% |
| Retrieval time | Millisecond | Millisecond |
| Phù hợp khi | Data quan trọng, không thể tái tạo | Data dễ tái tạo hoặc có bản sao ở nơi khác |

**Timeline xử lý bài toán này:**

```
Tháng 1:      S3 Standard (tất cả file — truy cập thường xuyên)
                         ↓ Lifecycle Policy sau 30 ngày
Sau tháng 1:  tdojo-finance/ → S3 One Zone-IA (millisecond, rẻ)
              Còn lại       → Glacier (không cần retrieval nhanh, rẻ nhất)
```

💡 **Công thức ghi nhớ:**

> **"Easily reproducible"** → dùng **One Zone-IA** thay vì Standard-IA để tiết kiệm chi phí

> **"No specific retrieval time"** → dùng **Glacier** — rẻ nhất, không cần truy cập nhanh

> **"Millisecond retrieval"** → phải dùng Standard, Standard-IA, hoặc One Zone-IA — **không dùng Glacier**

> **"Access pattern đã biết trước"** → **không dùng Intelligent-Tiering** (Intelligent-Tiering chỉ dùng khi pattern không đoán được)
