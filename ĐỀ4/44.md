## Câu hỏi 44

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company is using the AWS Directory Service to integrate its on-premises Microsoft Active Directory (AD) domain with its Amazon EC2 instances via an AD connector. The IAM identities have the following identity-based policy attached. Which of the following BEST describes what the above resource policy does?

---

### 🔤 DỊCH CHI TIẾT

Câu hỏi cho một đoạn IAM policy và hỏi: policy này cho phép làm gì, và điều kiện giới hạn là gì?

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "DirectoryTutorialsDojo1234",
      "Effect": "Allow",
      "Action": ["ds:*"],
      "Resource": "arn:aws:ds:us-east-1:987654321012:directory/d-1234567890"
    }
  ]
}
```

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Cần đọc đúng cấu trúc ARN và phân biệt các thành phần:

> `arn:aws:ds:us-east-1:987654321012:directory/d-1234567890`

| Thành phần | Giá trị | Ý nghĩa |
|---|---|---|
| `arn:aws:ds` | ds | AWS Directory Service |
| `us-east-1` | us-east-1 | Region |
| `987654321012` | 987654321012 | **Account ID** |
| `directory/d-1234567890` | d-1234567890 | **Directory ID** |
| `Sid` | DirectoryTutorialsDojo1234 | **Statement ID** — chỉ là label, không phải resource ID |

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: Allow ds:* với directory ID là d-1234567890**

- `"Action": ["ds:*"]` = cho phép **tất cả Directory Service API calls**. ✅
- `"Resource": "arn:aws:...directory/d-1234567890"` = giới hạn chỉ áp dụng cho directory có ID **d-1234567890**. ✅
- Đây là cách đọc ARN chuẩn — phần sau dấu `/` cuối cùng là resource ID. ✅

---

❌ **Đáp án sai 1: Directory ID là DirectoryTutorialsDojo1234**

- `DirectoryTutorialsDojo1234` là giá trị của **`Sid`** (Statement ID) — chỉ là nhãn đặt tên cho statement, không có ý nghĩa kỹ thuật về resource.
- Không phải Directory ID.

---

❌ **Đáp án sai 2: Directory ID là 987654321012**

- `987654321012` là **AWS Account ID** — định danh tài khoản AWS, không phải Directory ID.
- Luôn nằm ở vị trí thứ 5 trong ARN.

---

❌ **Đáp án sai 3: Directory name là DirectoryTutorialsDojo1234**

- Tương tự đáp án sai 1: `DirectoryTutorialsDojo1234` là **Sid** (Statement ID), không phải directory name.

---

### 📊 KẾT LUẬN & BÀI HỌC

**Cấu trúc ARN (Amazon Resource Name) cần nhớ:**

```
arn : aws : service : region : account-id : resource-type/resource-id

arn : aws : ds  : us-east-1 : 987654321012 : directory/d-1234567890
 ↑          ↑        ↑            ↑                      ↑
prefix   service  region     account-id            resource-id
```

**Cấu trúc một IAM Policy Statement:**

```json
{
  "Sid":      "Tên tùy đặt — chỉ để đọc cho dễ, không ảnh hưởng gì",
  "Effect":   "Allow hoặc Deny",
  "Action":   "Service:API cần cho phép/chặn",
  "Resource": "ARN của resource cụ thể"
}
```

**Phân biệt các thành phần hay nhầm:**

| Tên | Vị trí trong ARN | Ví dụ |
|---|---|---|
| **Account ID** | Thành phần thứ 5 | `987654321012` |
| **Directory ID** | Sau `directory/` | `d-1234567890` |
| **Sid** | Field trong Statement | `DirectoryTutorialsDojo1234` — chỉ là label |
| **Region** | Thành phần thứ 4 | `us-east-1` |

💡 **Công thức ghi nhớ:**

> **`Sid`** = Statement ID — chỉ là **tên đặt cho statement**, không phải resource ID hay directory name

> **Account ID** = 12 chữ số, luôn ở **vị trí thứ 5** trong ARN

> **Resource ID** = phần **cuối cùng** của ARN, sau `resource-type/`

> Khi đọc ARN: đếm từ trái → `arn:aws:service:region:account:type/id`
