## Câu hỏi 50

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> An organization plans to use an AWS Direct Connect connection to establish a dedicated connection between its on-premises network and AWS. The organization needs to launch a fully managed solution that will automate and accelerate the replication of data to and from various AWS storage services.
>
> Which of the following solutions would you recommend?

---

### 🔤 DỊCH CHI TIẾT

Tổ chức sẽ dùng **AWS Direct Connect** (đường truyền vật lý chuyên dụng, không qua internet) để kết nối on-premises với AWS. Cần giải pháp **fully managed**, tự động và tăng tốc việc **replicate data** đến và từ các AWS storage services.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Direct Connect** → kết nối private, không qua internet → data transfer phải đi qua **service endpoint**, không phải internet.
2. **Automate and accelerate data replication** → AWS DataSync là giải pháp được thiết kế chính xác cho use case này.
3. **Fully managed** → không tự build custom solution.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: AWS DataSync agent qua service endpoint**

- **AWS DataSync** là fully managed service — tự động hóa và tăng tốc việc copy data giữa on-premises và AWS storage (S3, EFS, FSx...). ✅
- Deploy **DataSync agent** on-premises → agent kết nối với AWS qua **Direct Connect service endpoint** (không qua internet). ✅
- Hỗ trợ NFS, SMB, S3 API — linh hoạt với nhiều loại storage. ✅

---

❌ **Đáp án sai 1: DataSync agent qua Internet**

- Đề đã nói rõ sẽ dùng **Direct Connect** — đường truyền private, không qua internet.
- Dùng internet là không nhất quán với kiến trúc đã chọn và kém bảo mật hơn.

---

❌ **Đáp án sai 2: Storage Gateway File Gateway (SMB)**

- **File Gateway** dùng để **hybrid cloud storage** — cho phép on-premises app lưu file lên S3 qua giao thức NFS/SMB.
- Không phải giải pháp để **accelerate data replication** giữa nhiều AWS storage services.
- Hỗ trợ ít storage service target hơn DataSync.

---

❌ **Đáp án sai 3: Storage Gateway Tape Gateway**

- **Tape Gateway** thay thế băng từ vật lý (virtual tape library) → lưu backup lên S3/Glacier.
- Dùng cho **archiving/backup**, không phải accelerate data replication.

---

### 📊 KẾT LUẬN & BÀI HỌC

**AWS DataSync vs AWS Storage Gateway — phân biệt:**

| Tiêu chí | AWS DataSync | AWS Storage Gateway |
|---|---|---|
| Mục đích chính | **Migrate/replicate data** nhanh, tự động | **Hybrid storage** — on-premises dùng AWS storage như local |
| Use case | Migration, sync data định kỳ | Mở rộng on-premises storage lên cloud |
| Storage targets | S3, EFS, FSx, Snowcone | Tùy loại gateway (S3, Glacier...) |
| Tốc độ | **Tối ưu cho throughput cao** | Không tối ưu cho bulk transfer |
| Protocol | NFS, SMB, S3 API | NFS, SMB, iSCSI, virtual tape |

**Kết nối DataSync với AWS — hai cách:**

| Cách | Qua đâu | Dùng khi |
|---|---|---|
| **Qua Internet** | Public internet | Không có Direct Connect |
| **Qua Service Endpoint** | **Direct Connect / VPN** | Đã có Direct Connect, cần private/secure |

**Ba loại Storage Gateway:**

| Loại | Dùng cho |
|---|---|
| **File Gateway** | Lưu file lên S3 qua NFS/SMB |
| **Volume Gateway** | Block storage iSCSI, backup snapshot |
| **Tape Gateway** | Thay thế băng từ vật lý, archive lên Glacier |

💡 **Công thức ghi nhớ:**

> **"Automate + accelerate data replication to AWS storage"** → **AWS DataSync**

> **"Direct Connect + DataSync"** → data đi qua **service endpoint** (không phải internet)

> **Storage Gateway** = hybrid storage architecture, không phải bulk data migration/replication

> **DataSync** = di chuyển data | **Storage Gateway** = mở rộng storage on-premises lên cloud
