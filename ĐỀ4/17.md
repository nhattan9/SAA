## Câu hỏi 17

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> An organization uses a Microsoft SQL Server database to support its suite of applications. The organization plans to transition to an Amazon Aurora PostgreSQL database while minimizing application code modifications.
>
> Which combination of actions will achieve these objectives? (Select TWO.)

---

### 🔤 DỊCH CHI TIẾT

Một tổ chức đang dùng Microsoft SQL Server để hỗ trợ các ứng dụng của mình. Tổ chức muốn chuyển sang Amazon Aurora PostgreSQL trong khi **tối thiểu hóa việc chỉnh sửa code ứng dụng**.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Bài toán có hai thách thức cần giải quyết đồng thời:

1. **Chuyển đổi schema và migrate data** từ SQL Server sang Aurora PostgreSQL.
2. **Tối thiểu hóa thay đổi code ứng dụng** — ứng dụng đang viết bằng T-SQL (ngôn ngữ SQL của Microsoft), cần tiếp tục chạy được trên Aurora PostgreSQL mà không phải viết lại.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng 1: Bật Babelfish trên Aurora PostgreSQL**

- **Babelfish** là extension của Aurora PostgreSQL, hoạt động như một **lớp dịch thuật (translation layer)**. ✅
- Chuyển đổi **T-SQL** (ngôn ngữ SQL của Microsoft SQL Server) thành PostgreSQL — ứng dụng tiếp tục gửi câu query T-SQL như cũ, Babelfish tự xử lý phía sau. ✅
- Giải quyết trực tiếp yêu cầu "minimize application code modifications". ✅

---

✅ **Đáp án đúng 2: AWS SCT + AWS DMS**

- **AWS Schema Conversion Tool (SCT)**: chuyển đổi **database schema** từ SQL Server sang Aurora PostgreSQL (cấu trúc bảng, stored procedures, indexes...). ✅
- **AWS Database Migration Service (DMS)**: migrate **data** từ SQL Server sang Aurora PostgreSQL với downtime tối thiểu, hỗ trợ continuous replication. ✅
- Hai công cụ này bổ trợ nhau: SCT lo phần cấu trúc, DMS lo phần dữ liệu. ✅

---

❌ **Đáp án sai 1: AWS AppConfig**

- AppConfig dùng để **quản lý và deploy thay đổi cấu hình ứng dụng** một cách an toàn — ví dụ bật/tắt feature flag.
- Hoàn toàn không liên quan đến schema conversion, data migration hay SQL query compatibility.

---

❌ **Đáp án sai 2: Amazon Kinesis Data Streams**

- Kinesis dùng cho **real-time data ingestion và processing** — ví dụ stream log, clickstream, IoT data.
- Không có khả năng chuyển đổi schema hay đảm bảo SQL query compatibility giữa hai database engine.

---

❌ **Đáp án sai 3: AWS Glue để transform SQL queries**

- AWS Glue là **ETL service** — dùng để extract, transform, load dữ liệu cho mục đích analytics.
- Không được thiết kế để transform SQL queries giữa các database engine khác nhau.
- Dễ nhầm vì Glue có chữ "transform" nhưng transform ở đây là transform **data**, không phải transform **SQL syntax**.

---

### 📊 KẾT LUẬN & BÀI HỌC

**Bảng so sánh các công cụ Database Migration của AWS:**

| Service | Vai trò | Dùng khi |
|---|---|---|
| **AWS SCT** | Chuyển đổi database schema | Migrate từ engine này sang engine khác (SQL Server → PostgreSQL) |
| **AWS DMS** | Migrate và replicate data | Chuyển dữ liệu thực tế, hỗ trợ continuous replication |
| **Babelfish** | Dịch T-SQL → PostgreSQL | Muốn giữ nguyên code ứng dụng SQL Server khi chuyển sang Aurora PostgreSQL |

**Luồng migration SQL Server → Aurora PostgreSQL:**

```
Microsoft SQL Server
        │
        ├──[AWS SCT]──→ Chuyển đổi schema → Aurora PostgreSQL schema
        │
        ├──[AWS DMS]──→ Migrate data → Aurora PostgreSQL data
        │
        └──[Babelfish]── Ứng dụng tiếp tục gửi T-SQL → Babelfish dịch → PostgreSQL
```

💡 **Công thức ghi nhớ:**

> **"Minimize code changes khi migrate từ SQL Server"** → **Babelfish** (dịch T-SQL → PostgreSQL)

> **"Migrate database schema + data"** → **AWS SCT** (schema) + **AWS DMS** (data)

> **AWS Glue** = ETL cho analytics, không phải SQL query converter

> **Kinesis** = real-time streaming data, không phải database migration

> **AppConfig** = quản lý configuration ứng dụng, không liên quan đến database migration
