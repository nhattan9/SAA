## Câu hỏi 22

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company has a regional API Gateway in the us-east-2 region that serves as a proxy to a backend service. Clients connect to the service using the invoke URL of the API stage. To improve usability, the company wants to associate a custom domain name (api.tutorialsdojo.com) with the API. Moreover, the domain name must support HTTPS to ensure secure connections. The company has an existing hosted zone for its domain on Amazon Route 53.
>
> Which of the following would be the next step to achieve the company's objective?

---

### 🔤 DỊCH CHI TIẾT

Một công ty có **Regional API Gateway** ở region **us-east-2**. Client đang kết nối bằng invoke URL mặc định của API stage. Công ty muốn gắn **custom domain name** (`api.tutorialsdojo.com`) vào API và phải hỗ trợ **HTTPS**. Công ty đã có hosted zone trên Route 53.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Ba điểm mấu chốt cần nhớ:

1. **Regional API Gateway** → certificate ACM phải được request **cùng region với API** (us-east-2) — khác với Edge-optimized API Gateway (phải dùng us-east-1).
2. **Custom domain** → phải dùng **API Gateway Custom Domain Names feature**, không phải chỉ tạo CNAME trỏ đến invoke URL.
3. **Route 53** → dùng **Alias record** (không phải CNAME) để trỏ về API Gateway domain name.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: ACM certificate ở us-east-2 + Regional custom domain + Route 53 Alias record**

- Request **public certificate** từ ACM tại **us-east-2** — đúng region với API Gateway. ✅
- Tạo **Regional API Gateway Custom Domain Name**, associate với `api.tutorialsdojo.com` và ACM certificate. ✅
- Tạo **Alias record** trong Route 53 trỏ về API Gateway domain name. ✅
- Đây là quy trình chuẩn để gắn custom domain cho Regional API Gateway. ✅

---

❌ **Đáp án sai 1: ACM ở us-east-1 thay vì us-east-2**

- **us-east-1 là sai** — Regional API Gateway ở us-east-2 yêu cầu ACM certificate phải ở **cùng region (us-east-2)**.
- us-east-1 chỉ bắt buộc với **Edge-optimized API Gateway** (vì dùng CloudFront, mà CloudFront chỉ dùng certificate ở us-east-1).
- Đây là bẫy phổ biến: nhầm lẫn giữa Regional và Edge-optimized API Gateway.

---

❌ **Đáp án sai 2: Import certificate vào ACM us-east-2 + CNAME record trỏ invoke URL**

- Import certificate là hợp lệ, nhưng **tạo CNAME trỏ thẳng đến invoke URL là sai** — không dùng được Custom Domain Names feature của API Gateway theo cách này.
- Phải tạo **API Gateway Custom Domain Name** rồi mới tạo record trong Route 53 trỏ về đó.
- Ngoài ra, Route 53 **không cho phép tạo CNAME cho root domain** (apex domain) — phải dùng Alias record.

---

❌ **Đáp án sai 3: ACM Private Certificate + stage variables**

- **Private certificate** dùng cho internal network — không phù hợp cho API public trên internet.
- **Stage variables** không liên quan đến việc gắn custom domain.
- Hai thứ đều sai hoàn toàn với yêu cầu bài toán.

---

### 📊 KẾT LUẬN & BÀI HỌC

**ACM Certificate — region nào phụ thuộc vào loại API Gateway:**

| Loại API Gateway | ACM Certificate phải ở |
|---|---|
| **Regional** | Cùng region với API (ví dụ us-east-2) |
| **Edge-optimized** | Bắt buộc **us-east-1** (vì dùng CloudFront) |
| **Private** | Cùng region với API |

**Quy trình gắn custom domain cho Regional API Gateway:**

```
1. Request ACM public certificate (cùng region với API)
            ↓
2. Tạo API Gateway Custom Domain Name
   → Associate với custom domain + ACM certificate
            ↓
3. Tạo API Mapping (map domain về API stage)
            ↓
4. Tạo Alias record trong Route 53
   → Trỏ về API Gateway domain name
```

**Alias record vs CNAME record trong Route 53:**

| Tiêu chí | Alias record | CNAME record |
|---|---|---|
| Dùng cho root domain (apex) | ✅ Có | ❌ Không |
| Trỏ về AWS resource | ✅ Tối ưu | ⚠️ Được nhưng không tối ưu |
| Tính phí DNS query | Miễn phí với AWS resources | Tính phí |

💡 **Công thức ghi nhớ:**

> **Regional API Gateway** → ACM certificate **cùng region**

> **Edge-optimized API Gateway** → ACM certificate bắt buộc **us-east-1**

> Custom domain cho API Gateway → **API Gateway Custom Domain Names feature** + **Route 53 Alias record**

> **CNAME trỏ thẳng invoke URL** = sai — phải đi qua API Gateway Custom Domain Names feature

> **Private certificate** = internal network only, không dùng cho public API
