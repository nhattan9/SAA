Đáp án này gồm 3 từ khóa, giải thích từng cái:

---

**1. Modeling (Mô hình hóa)**

Trước khi tạo bất cứ thứ gì trên AWS, bạn **viết ra** toàn bộ hạ tầng trong file YAML:

```yaml
EC2 instance: loại t3.medium, region us-east-1
S3 bucket: tên "my-app-storage"
RDS database: MySQL, 20GB storage
VPC: CIDR 10.0.0.0/16
```

Đây là bước **"model"** — mô tả hệ thống trên giấy trước khi build thực tế. Giống như kiến trúc sư vẽ bản thiết kế trước khi xây nhà.

---

**2. Provisioning (Cung cấp/Tạo ra)**

Sau khi có file YAML, bạn chạy một lệnh → CloudFormation **tự động tạo ra tất cả** EC2, S3, RDS, VPC... đúng như mô tả trong file — không cần click thủ công trên console từng bước.

---

**3. Version-controlling (Quản lý phiên bản)**

File YAML là file text bình thường → có thể lưu trên **Git** như code phần mềm:

- Hôm nay bạn thêm một RDS instance → commit lên Git với message "add database"
- Tuần sau deploy nhầm, xóa mất database → quay lại Git, revert về version cũ → chạy lại CloudFormation → database được tạo lại y hệt

---

**Ví dụ thực tế giải thích cho manager:**

> Tưởng tượng bạn cần setup hệ thống giống nhau cho 3 môi trường: Dev, Staging, Production. Nếu làm thủ công trên console → phải click 3 lần, dễ sai. Với CloudFormation → chạy cùng 1 file YAML 3 lần → 3 môi trường giống hệt nhau, không sai sót.
