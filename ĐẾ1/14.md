# 1. Dịch câu hỏi

Một tổ chức cần một **persistent block storage volume** để hỗ trợ các workload quan trọng.

Dữ liệu backup sẽ được lưu trong một dịch vụ **object storage**, và sau 30 ngày sẽ được chuyển sang một dịch vụ **archival storage** để lưu trữ dài hạn.

Team đã cấu hình sẵn **Amazon EBS snapshot retention rule** để phục hồi theo từng thời điểm.

Cần làm gì để đáp ứng yêu cầu trên?

---

# 2. Đáp án đúng

✅ **Attach an EBS volume to your Amazon EC2 instance. Use Amazon S3 to store your backup data and configure a lifecycle policy to transition your objects to S3 Glacier Flexible Retrieval.**

Dịch:

> Gắn một EBS volume vào Amazon EC2 instance. Sử dụng Amazon S3 để lưu backup data và cấu hình lifecycle policy để chuyển object sang S3 Glacier Flexible Retrieval.

---

# 3. Keyword quan trọng của đề

## Keyword 1

# "persistent block storage volume"

=> cần storage dạng block và dữ liệu phải tồn tại lâu dài.

Phù hợp:

# Amazon EBS

---

## Keyword 2

# "mission-critical workloads"

=> workload quan trọng.

Không nên dùng storage tạm thời.

=> loại Instance Store.

---

## Keyword 3

# "backup data stored in object storage"

=> cần object storage.

Phù hợp:

# Amazon S3

---

## Keyword 4

# "after 30 days, transitioned to archival storage"

=> cần lifecycle policy để chuyển sang archive.

Phù hợp:

# S3 Lifecycle → S3 Glacier Flexible Retrieval

---

## Keyword 5

# "EBS snapshot retention rule"

Đây là bẫy.

EBS snapshot có thể dùng để point-in-time recovery, nhưng không dùng S3 Lifecycle để chuyển snapshot sang Glacier theo cách thông thường.

---

```text
EC2 Instance
    ↓
```

```text
Backup data
    ↓

    ↓ after 30 days lifecycle
S3 Glacier Flexible Retrieval
```

---

# 5. Vì sao EBS đúng?

Amazon EBS là:

# persistent block storage

Dùng cho EC2 workloads như:

* database
* application data
* file system
* mission-critical workload

Quan trọng:

```text
EC2 stop/start → data trên EBS vẫn còn
```

=> phù hợp persistent storage.

---

# 6. Vì sao S3 đúng cho backup?

Amazon S3 là:

# object storage

Phù hợp cho:

* backup
* restore
* archive
* data lake
* log storage

Backup data lưu dưới dạng object trong bucket.

---

# 7. Vì sao Glacier Flexible Retrieval đúng?

S3 Glacier Flexible Retrieval là:

# archival storage

Dùng cho dữ liệu:

* ít truy cập
* lưu dài hạn
* cần chi phí thấp
* vẫn có thể retrieve khi cần

Đề nói:

```text
after 30 days → archival storage
```

=> Glacier Flexible Retrieval phù hợp hơn S3 One Zone-IA.

---

# 8. Vì sao Instance Store sai?

Instance Store là:

# temporary block storage

Dữ liệu có thể mất khi:

* instance stop
* instance terminate
* underlying disk failure

Ngoài ra:

```text
Không thể attach instance store sau khi instance đã launch
```

=> không phù hợp mission-critical persistent workload.

---

# 9. Vì sao S3 One Zone-IA sai?

S3 One Zone-IA là:

# infrequent access storage

Không phải archival storage.

Ngoài ra nó chỉ lưu ở một AZ.

Không phù hợp bằng Glacier cho long-term retention.

---

# 10. Bẫy trong câu hỏi

Câu này có bẫy:

```text
The team has already configured an Amazon EBS snapshot retention rule
```

Nhiều người sẽ nghĩ:

```text
EBS snapshot nằm trong S3 nên dùng S3 Lifecycle chuyển Glacier
```

Nhưng sai.

EBS snapshot được AWS quản lý riêng.

Bạn không trực tiếp quản lý snapshot như object trong S3 bucket của bạn.

=> Không dùng S3 Lifecycle trên EBS snapshot được.

---

# 11. Phân biệt nhanh

| Requirement                           | Service                       |
| ------------------------------------- | ----------------------------- |
| Persistent block storage              | EBS                           |
| Temporary block storage               | Instance Store                |
| Object storage                        | S3                            |
| Archive storage                       | S3 Glacier Flexible Retrieval |
| Infrequent access, không phải archive | S3 One Zone-IA                |

---

# Chốt ngắn gọn

Giải pháp đúng gồm 3 phần:

```text

```
