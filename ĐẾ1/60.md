# Dịch đề

Một nền tảng trading cryptocurrency đang dùng:

```text
AWS Lambda + API Gateway
```

để build API.

Do các tin tức và tin đồn gần đây về việc:

* Bitcoin
* Ethereum
* các crypto khác

sắp tăng giá mạnh, nên dự kiến platform sẽ có:

* lượng visitor tăng đột biến
* nhiều user mới

trong vài ngày tới.

Hỏi:

# làm sao bảo vệ backend systems khỏi traffic spikes?

---

# Đáp án đúng

✅

```text
Enable throttling limits and result caching in API Gateway.
```

Dịch:

> Bật throttling limits và result caching trong API Gateway.

# 1. Throttling là gì?

Throttling:

# giới hạn request rate

Ví dụ:

```text
1000 requests/sec
```

Nếu vượt:

```text
HTTP 429 Too Many Requests
```

---

# API Gateway sẽ bảo vệ backend thế nào?

```text
Users
   ↓
API Gateway throttling
   ↓
Lambda
```

Khi traffic tăng mạnh:

* API Gateway chặn bớt request
* Lambda không bị overload
* database/backend phía sau được bảo vệ

---

# 2. Result Caching là gì?

API Gateway cache response.

Ví dụ:

User A gọi:

```text
GET /bitcoin-price
```

API Gateway cache kết quả.

---

## User B gọi tiếp

Không cần gọi Lambda nữa.

API Gateway trả cache luôn.

---

# Luồng có cache

```text
Users
   ↓
API Gateway Cache
   ↓ (cache miss only)
Lambda
```

---

# Vì sao cache cực quan trọng?

Trong crypto trading:

Rất nhiều user gọi cùng dữ liệu:

```text
BTC price
ETH price
market data
```

Nếu không cache:

```text
100,000 users
→ 100,000 Lambda invokes
```

Nếu cache:

```text
1 Lambda invoke
→ nhiều users dùng cache
```

=> giảm tải cực mạnh.

# Đây là combo tối ưu nhất.

---

# Vì sao CloudFront sai?

CloudFront chủ yếu:

# CDN content delivery

Nó giúp:

* giảm latency
* cache static content

Nhưng API Gateway đã có built-in caching phù hợp hơn cho API responses.

Đề focus:

```text
protect backend systems
```

=> API Gateway throttling + caching trực tiếp là best answer.

---

# Vì sao EC2 + ELB + Auto Scaling sai?

Sai vì:

* over-engineering
* mất tính serverless
* Lambda + API Gateway đã scale tốt

Không cần migrate architecture.

---

# Vì sao Move Lambda into VPC sai?

VPC chỉ là network isolation.

Không liên quan traffic spike protection.

---

# Bảng phân biệt cực hay thi

| Requirement                     | Service/Feature        |
| ------------------------------- | ---------------------- |
| Protect backend from API spikes | API Gateway throttling |
| Reduce repeated API calls       | API Gateway caching    |
| CDN/static content              | CloudFront             |
| Serverless compute              | Lambda                 |
| Scale EC2                       | Auto Scaling           |
