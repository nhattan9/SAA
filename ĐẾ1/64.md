# Dịch đề

Một website chia sẻ ảnh du lịch đang dùng Amazon S3 để phục vụ ảnh chất lượng cao cho khách truy cập.

Sau vài ngày, công ty phát hiện các website du lịch khác đang:

* link trực tiếp tới ảnh
* sử dụng ảnh của công ty

Điều này gây thiệt hại tài chính cho doanh nghiệp.

Hỏi cách hiệu quả nhất để giảm thiểu vấn đề này là gì?

---

# Đáp án đúng

✅

```text
Configure your S3 bucket to remove public read access and use pre-signed URLs with expiry dates.
```

Dịch:

> Cấu hình S3 bucket để xoá quyền public read và sử dụng pre-signed URLs có thời hạn hết hạn.

---

# Phân tích keyword

## Keyword 1

```text
other travel websites are linking to and using these photos
```

=> website khác đang hotlink ảnh trực tiếp từ S3.

---

## Keyword 2

```text
financial losses
```

=> công ty bị tốn tiền bandwidth/request do người khác dùng ảnh.

---

## Keyword 3 quan trọng nhất

```text
MOST effective
```

=> cần chặn truy cập public trực tiếp.

# Vấn đề đang xảy ra là gì?

Hiện tại ảnh trong S3 có thể đang public.

Ví dụ:

```text
https://bucket.s3.amazonaws.com/photo.jpg
```

Website khác chỉ cần copy link này rồi nhúng vào site của họ:

```html
<img src="https://bucket.s3.amazonaws.com/photo.jpg">
```

Kết quả:

* họ dùng ảnh miễn phí
* công ty bạn trả tiền S3 request/data transfer
* mất kiểm soát nội dung

---

# Pre-signed URL là gì?

Pre-signed URL là link truy cập S3 tạm thời.

Ví dụ:

```text
photo.jpg chỉ truy cập được trong 10 phút
```

Sau khi hết hạn:

```text
URL không còn dùng được
```

---

# Luồng đúng

```text
User vào website chính
        ↓
App kiểm tra quyền / tạo URL
        ↓
Generate pre-signed URL có hạn
        ↓
User xem ảnh
        ↓
URL hết hạn sau một thời gian
```

---

# Vì sao phải remove public read access?

Nếu bucket vẫn public thì ai cũng truy cập được ảnh.

Nên phải:

```text
Block Public Access
Remove public read ACL/policy
```

Sau đó chỉ cấp quyền bằng:

```text
pre-signed URL
```

---

# Vì sao đáp án đúng?

✅ Chặn website khác dùng link public lâu dài
✅ URL hết hạn nên không hotlink mãi được
✅ Không cần đổi service lưu ảnh
✅ Vẫn dùng S3 đúng mục đích static/object storage

---

# Vì sao các đáp án khác sai?

## ❌ Use CloudFront distributions

CloudFront chỉ giúp phân phối ảnh nhanh hơn.

Nếu ảnh vẫn public hoặc không có signed URL/cookie thì website khác vẫn hotlink được.

CloudFront một mình không giải quyết quyền truy cập.

---

## ❌ Store photos on Amazon WorkDocs

WorkDocs dùng cho cộng tác tài liệu nội bộ.

Không phù hợp để serve ảnh chất lượng cao cho website public.

---

## ❌ Block IP bằng NACL

Sai vì:

* không biết hết IP của website khác
* họ có thể đổi IP/CDN dễ dàng
* NACL áp dụng ở subnet/VPC, không phải cách kiểm soát object S3 hiệu quả

---

# Bảng nhớ nhanh

| Vấn đề                      | Giải pháp                             |
| --------------------------- | ------------------------------------- |
| S3 object bị hotlink        | Remove public access + pre-signed URL |
| Cần URL tạm thời            | Pre-signed URL                        |
| Cần CDN nhanh               | CloudFront                            |
| Cần private content qua CDN | CloudFront signed URL/cookie          |
| Chặn IP network-level       | NACL / Security Group                 |

---

# Mẹo đi thi AWS

Nếu đề có:

```text
S3 photos/files
other websites linking
unauthorized access
temporary access
expiry date
```

=> chọn:

```text
Private S3 + Pre-signed URLs
```

# Chốt

Cần biến ảnh từ public sang private, rồi chỉ cấp quyền xem tạm thời bằng:

```text
Pre-signed URL with expiration
```
