# Dịch đề

Trong môi trường production có sự cố: dữ liệu user lưu trong Amazon S3 bucket bị một Junior DevOps Engineer vô tình xoá.

Sau đó management yêu cầu cải thiện bảo mật và bảo vệ AWS resources.

Hỏi chọn 2 giải pháp nào để bảo vệ S3 objects khỏi:

* xoá nhầm
* bị ghi đè nhầm

---

# Đáp án đúng

## ✅ 1. Enable Versioning

Dịch:

> Bật S3 Versioning.

---

## ✅ 2. Enable Multi-Factor Authentication Delete

Dịch:

> Bật MFA Delete cho S3 bucket.

---

# Phân tích keyword

## Keyword 1

```text
accidentally deleted
```

=> bị xoá nhầm.

---

## Keyword 2

```text
protect S3 objects
```

=> bảo vệ object trong S3.

---

## Keyword 3

```text
from both accidental deletion and overwriting
```

=> chống cả:

* delete nhầm
* overwrite nhầm

---

# Vì sao cần Enable Versioning?

S3 Versioning lưu nhiều phiên bản của cùng một object.

Ví dụ object:

```text
user-data.json
```

Khi overwrite:

```text
version 1: dữ liệu cũ
version 2: dữ liệu mới
```

Nếu ghi đè nhầm, có thể restore lại version cũ.

---

# Khi delete thì sao?

Nếu bucket bật versioning, khi xoá object, S3 không xoá hẳn ngay.

Nó tạo:

```text
delete marker
```

Version cũ vẫn còn.

=> có thể khôi phục.

---

# Vì sao cần MFA Delete?

MFA Delete yêu cầu xác thực MFA khi:

* thay đổi trạng thái versioning
* xoá vĩnh viễn một object version

Tức là nếu ai đó muốn xoá hẳn dữ liệu, phải có:

```text
password/access key + MFA code
```

=> giảm rủi ro xoá nhầm hoặc xoá bởi người không đủ quyền.

---

# Combo này bảo vệ thế nào?

```text
Versioning
→ lưu lại các version cũ

MFA Delete
→ chặn xoá vĩnh viễn nếu không có MFA
```

=> bảo vệ khỏi:

✅ accidental deletion
✅ accidental overwrite
✅ permanent deletion

---

# Vì sao các đáp án khác sai?

## ❌ Pre-signed URL only

Pre-signed URL chỉ dùng để cấp quyền truy cập tạm thời vào object.

Nó không bảo vệ object khỏi bị xoá hoặc overwrite.

---

## ❌ Disallow S3 Delete using IAM bucket policy

Cách này chặn delete hoàn toàn.

Nghe có vẻ đúng, nhưng đề muốn bảo vệ khỏi xoá nhầm, không nhất thiết cấm mọi hành động delete.

Ngoài ra nó không xử lý tốt trường hợp overwrite object.

---

## ❌ Enable S3 Intelligent-Tiering

S3 Intelligent-Tiering chỉ giúp tối ưu chi phí lưu trữ theo tần suất truy cập.

Không liên quan bảo vệ delete/overwrite.

---

# Mẹo đi thi AWS

Nếu đề có:

```text
accidental deletion
accidental overwrite
recover objects
```

=> chọn:

```text
S3 Versioning
```

Nếu đề có:

```text
prevent permanent delete
extra authentication
```

=> chọn:

```text
MFA Delete
```

# Chốt

Đáp án là:

```text
Enable Versioning
Enable Multi-Factor Authentication Delete
```
