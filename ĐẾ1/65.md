# Dịch đề

Một công ty viễn thông muốn cấp quyền truy cập AWS Console cho developers.

Chính sách công ty yêu cầu:

* sử dụng identity federation
* role-based access control (RBAC)

Hiện tại:

```text
roles đã được assign thông qua groups trong corporate Active Directory
```

Hỏi cần dùng combination nào để developers có thể truy cập AWS Console?

(Chọn 2)

---

# Đáp án đúng

✅

```text
AWS Directory Service AD Connector
```

✅

```text
IAM Roles
```

---

# Phân tích keyword cực quan trọng

---

# Keyword 1

```text
identity federation
```

=> đây là keyword mạnh nhất.

=> nghĩ ngay:

# Active Directory ↔ AWS integration

---

# Keyword 2

```text
corporate Active Directory
```

=> công ty đã có:

# Microsoft AD on-premises

---

# Keyword 3

```text
roles are already assigned via groups
```

=> dùng:

# RBAC (Role-Based Access Control)

---

# Keyword 4

```text
AWS Console access
```

=> users cần login AWS Console bằng corporate credentials.

---

# Bối cảnh thực tế

Công ty đã có hệ thống user nội bộ:

```text
Active Directory
```

Ví dụ:

| User  | Group      |
| ----- | ---------- |
| John  | Developers |
| Alice | Admins     |

---

# Họ KHÔNG muốn:

❌ tạo IAM User riêng cho từng người.

Vì:

* khó quản lý
* duplicate accounts
* không centralized identity

---

# Họ muốn gì?

Developer dùng:

```text
username/password công ty
```

để login AWS Console.

---

# Giải pháp đúng

---

# 1. AWS Directory Service AD Connector

AD Connector là:

# cầu nối giữa AWS và on-premises Active Directory

---

# Nó KHÔNG lưu user

Nó chỉ:

```text
forward authentication request
```

về AD thật.

---

# Luồng hoạt động

```text
Developer
    ↓
AWS Login
    ↓
AD Connector
    ↓
Corporate Active Directory
```

---

# 2. IAM Roles

Sau khi authenticate bằng AD:

AWS sẽ map user/group sang:

# IAM Role

---

# Ví dụ

| AD Group   | IAM Role     |
| ---------- | ------------ |
| Developers | ReadOnlyRole |
| DevOps     | AdminRole    |

---

# Kết quả

Developer login bằng tài khoản công ty:

```text
john@company.com
```

AWS cấp temporary permissions qua:

```text
IAM Role
```

---

# Vì sao IAM Roles đúng?

Đề yêu cầu:

```text
role-based access control
```

IAM Role chính là RBAC trong AWS.

---

# Vì sao các đáp án khác sai?

---

# ❌ IAM Groups

IAM Groups chỉ dùng cho:

# IAM Users trong AWS

Nhưng đề yêu cầu:

```text
identity federation
```

=> không dùng IAM User/Group local AWS.

---

# ❌ Simple AD

Simple AD là:

# directory managed bởi AWS

Đề đã có:

```text
corporate Active Directory
```

=> dùng:

# AD Connector

để connect tới AD sẵn có.

Simple AD

Giống như:

tạo thêm 1 công ty clone

phải copy toàn bộ nhân viên qua.

Khi nào dùng Simple AD?

Simple AD dùng khi:

công ty chưa có Active Directory

và chỉ cần lightweight directory.

6. Khi nào dùng AD Connector?

Khi:

đã có corporate Active Directory

và muốn:

AWS dùng chung identity

---

# ❌ AWS Lambda

Lambda hoàn toàn không liên quan authentication/federation.

---

# So sánh cực quan trọng

| Service      | Purpose                           |
| ------------ | --------------------------------- |
| AD Connector | connect AWS ↔ existing AD         |
| Simple AD    | lightweight AWS-managed directory |
| IAM Role     | permissions / RBAC                |
| IAM Group    | group IAM users                   |
| Lambda       | serverless compute                |
