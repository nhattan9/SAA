## Ý chính của đề

Công ty cần:

* Encrypt data at rest
* Dễ tích hợp với AWS services → cần dùng KMS
* **Toàn quyền kiểm soát key**
* Có thể **xóa key material ngay lập tức**
* Có thể **audit độc lập ngoài CloudTrail**

=> Đây là keyword cực mạnh cho:

> **KMS Custom Key Store + CloudHSM**

---

# Đáp án đúng

✅ **Use AWS KMS to create a KMS key in a custom key store and store the key material in AWS CloudHSM**

---

# Vì sao đúng?

## 1. AWS KMS → tích hợp AWS services dễ dàng

Ví dụ:

* S3
* EBS
* RDS
* Lambda

đều hỗ trợ encrypt bằng KMS rất tốt.

---

## 2. Custom Key Store → bạn tự kiểm soát key

Thông thường:

* KMS key mặc định → AWS quản lý HSM
* Bạn không kiểm soát hoàn toàn

Nhưng với:

> **Custom Key Store**

thì KMS sẽ dùng:

> CloudHSM cluster của CHÍNH BẠN

=> bạn có full control.

---

## 3. CloudHSM → đáp ứng yêu cầu bảo mật cao

CloudHSM cho phép:

* Dedicated HSM riêng
* Single tenant
* Key không rời khỏi HSM
* Tự quản lifecycle

---

## 4. Có thể xóa key material ngay lập tức

Do bạn sở hữu CloudHSM cluster:

=> bạn có thể:

* revoke
* delete
* shutdown HSM

ngay lập tức mà không phụ thuộc AWS KMS.

---

## 5. Audit độc lập với CloudTrail

Đề nói:

> audit independently of AWS CloudTrail

CloudHSM hỗ trợ log/audit riêng.

=> phù hợp compliance cao.

---

# Vì sao các đáp án khác sai?

## ❌ AWS owned keys

AWS quản lý hoàn toàn.

Bạn không kiểm soát key.

---

## ❌ AWS managed keys

Vẫn là AWS quản lý.

Không đáp ứng:

* full control
* independent audit
* immediate removal

---

## ❌ Store key material in S3

S3 không phải HSM.

Không dùng để lưu cryptographic key material an toàn.

---
