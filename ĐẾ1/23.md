# 1. Dịch câu hỏi

Một công ty có kiến trúc cloud gồm cả:

* Linux EC2 instances
* Windows EC2 instances

Các instance này xử lý khối lượng lớn dữ liệu tài chính liên tục:

```text
24 giờ/ngày, 7 ngày/tuần
```

Để đảm bảo high availability, Solutions Architect cần tạo giải pháp monitor:

* memory utilization
* disk utilization

cho tất cả EC2 instances.

Giải pháp monitoring nào phù hợp nhất?

---

# 2. Đáp án đúng

✅ **Install the Amazon CloudWatch agent to all the EC2 instances that gather the memory and disk utilization data. View the custom metrics in the CloudWatch console.**

Dịch:

> Cài Amazon CloudWatch Agent trên tất cả EC2 instances để thu thập memory và disk utilization data. Sau đó xem các custom metrics trong CloudWatch console.

---

# 3. Keyword quan trọng của đề

## Keyword 1

# "Linux and Windows EC2 instances"

=> cần agent hỗ trợ cả Linux và Windows.

CloudWatch Agent hỗ trợ cả hai.

---

## Keyword 2

# "memory and disk utilization metrics"

Đây là keyword quan trọng nhất.

Mặc định CloudWatch KHÔNG có memory và disk space utilization cho EC2.

Muốn có phải cài:

# CloudWatch Agent

---

## Keyword 3

# "all instances"

=> cần triển khai agent trên toàn bộ EC2.

Có thể dùng Systems Manager để rollout, nhưng đáp án chính vẫn là CloudWatch Agent.

---

# 4. CloudWatch mặc định monitor EC2 được gì?

CloudWatch mặc định có các metric như:

* CPUUtilization
* NetworkIn / NetworkOut
* DiskReadOps / DiskWriteOps
* DiskReadBytes / DiskWriteBytes
* StatusCheckFailed

---

# 5. CloudWatch mặc định KHÔNG có gì?

CloudWatch mặc định không biết được bên trong OS như:

* RAM đang dùng bao nhiêu
* disk space còn bao nhiêu
* swap/page file
* process-level metrics

Vì các metric này nằm bên trong operating system.

---

# 6. Vì sao cần CloudWatch Agent?

CloudWatch Agent chạy bên trong EC2 instance.

Nó đọc thông tin OS-level rồi gửi lên CloudWatch.

Flow:

```text
EC2 Linux/Windows
      ↓
CloudWatch Agent
      ↓
Custom Metrics
      ↓
CloudWatch Console
```

---

# 7. Ví dụ dễ hiểu

Không cài agent:

```text
CloudWatch thấy CPU, network, disk I/O
```

Có cài agent:

```text
CloudWatch thấy thêm RAM usage, disk space usage
```

---

# 8. Vì sao đáp án đúng?

CloudWatch Agent:

* hỗ trợ Linux
* hỗ trợ Windows
* thu thập memory utilization
* thu thập disk utilization
* gửi custom metrics lên CloudWatch
* phù hợp monitor fleet EC2 lớn

---

# 9. Vì sao các đáp án khác sai?

## ❌ Default CloudWatch + SSM Inventory

Sai vì:

CloudWatch mặc định không có memory/disk utilization.

SSM Inventory chỉ thu thập metadata/configuration như:

* OS version
* installed applications
* packages
* instance metadata

Không dùng để monitor performance metrics.

---

## ❌ Enhanced Monitoring in EC2

Sai vì:

# Enhanced Monitoring là feature của RDS

Không phải EC2.

Đối với EC2, muốn memory/disk metrics thì dùng CloudWatch Agent.

---

## ❌ Amazon Inspector

Amazon Inspector dùng cho:

* security assessment
* vulnerability scanning
* network exposure analysis

Không dùng để monitor RAM/disk utilization.

---
