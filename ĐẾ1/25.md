# 1. Dịch câu hỏi

Một công ty dự định migrate workload từ on-premises lên AWS.

Kiến trúc hiện tại gồm:

* Microsoft SharePoint server
* Windows shared file storage

Solutions Architect cần chọn cloud storage solution:

* highly available
* tích hợp được với Active Directory
* hỗ trợ access control và authentication

Giải pháp nào đáp ứng yêu cầu?

---

# 2. Đáp án đúng

✅ **Create a file system using Amazon FSx for Windows File Server and join it to an Active Directory domain in AWS.**

Dịch:

> Tạo file system bằng Amazon FSx for Windows File Server và join nó vào Active Directory domain trên AWS.

---

# 3. Keyword quan trọng của đề

## Keyword 1

# "Microsoft SharePoint server"

SharePoint là hệ sinh thái Microsoft/Windows.

=> thường dùng SMB file share.

---

## Keyword 2

# "Windows shared file storage"

Đây là keyword rất mạnh.

Windows file sharing thường dùng:

# SMB protocol

---

## Keyword 3

# "integrated with Active Directory"

Cần:

* user authentication
* permission theo domain user/group
* access control kiểu Windows

=> cần storage support AD integration.

---

## Keyword 4

# "highly available"

Cần managed storage HA.

=> Amazon FSx for Windows File Server phù hợp.

---

# 4. Amazon FSx for Windows File Server là gì?

Là managed Windows file system trên AWS.

Nó cung cấp shared file storage qua:

# SMB protocol

và hỗ trợ:

* Microsoft Active Directory integration
* Windows ACL
* NTFS permissions
* user/group access control
* high availability
* Windows-native file sharing

---

# 5. Flow đúng

```text
Windows / SharePoint Server
        ↓ SMB
Amazon FSx for Windows File Server
        ↓
Joined to Active Directory
        ↓
Access control by AD users/groups
```

---

# 6. Vì sao FSx for Windows đúng?

Vì nó đáp ứng đủ yêu cầu:

* Windows shared file storage
* SMB protocol
* Active Directory authentication
* access control
* highly available
* phù hợp Microsoft workload như SharePoint

---

# 7. Vì sao EFS sai?

Amazon EFS là:

# Linux file system

Dùng protocol:

# NFS

Phù hợp cho Linux workloads.

Không phù hợp cho:

* Windows native file share
* SMB
* Active Directory integration kiểu Windows

=> đề yêu cầu Windows shared file storage nên EFS sai.

---

# 8. Vì sao S3 sai?

S3 là:

# object storage

Không phải Windows file system.

S3 không cung cấp:

* SMB native file share
* Windows ACL
* AD authentication trực tiếp cho file/folder

Mount S3 như file volume cũng không phải giải pháp chuẩn cho SharePoint.

---

# 9. Vì sao Storage Gateway NFS sai?

NFS chủ yếu dùng cho:

# Linux/Unix file sharing

Nhưng đề yêu cầu:

# Windows shared file storage

Windows nên dùng:

# SMB

Storage Gateway có SMB file share, nhưng option đưa ra là NFS nên sai.

---

# 10. Phân biệt nhanh

| Service                     | Phù hợp                              |
| --------------------------- | ------------------------------------ |
| FSx for Windows File Server | Windows shared file storage, SMB, AD |
| EFS                         | Linux shared file system, NFS        |
| S3                          | Object storage                       |
| Storage Gateway NFS         | Hybrid Linux file share              |
| Storage Gateway SMB         | Hybrid Windows file share            |

# 12. Chốt ngắn gọn

Yêu cầu chính là:

```text
Windows shared file storage + Active Directory
```

Giải pháp đúng:

```text
Amazon FSx for Windows File Server
        ↓
Join Active Directory
        ↓
Dùng SMB + Windows ACL để kiểm soát quyền
```
