Đáp án đúng: **API Gateway + AWS Lambda**

## 1. Keyword của đề

```text
low user activity most of the time
bursts of traffic within seconds
users around the globe access data using an API
```

Ý nghĩa:

* Bình thường traffic thấp → không nên chạy nhiều server tốn tiền
* Khi có announcement → traffic tăng đột ngột trong vài giây
* Cần API global cho user truy cập

=> Cần giải pháp **serverless, scale rất nhanh, trả tiền theo usage**.

## 2. Vì sao chọn API Gateway + Lambda?

**Amazon API Gateway** dùng để tạo API.

**AWS Lambda** dùng để xử lý request phía sau API.

Luồng sẽ là:

```text
User → API Gateway → Lambda → Amazon RDS
```

Lambda phù hợp vì:

* Không cần quản lý server
* Bình thường ít request thì gần như không tốn compute
* Khi traffic tăng đột ngột, Lambda scale nhanh trong vài giây
* Phù hợp workload kiểu bursty traffic

## 3. Vì sao không chọn EC2 / ECS / Elastic Beanstalk?

Các option kia đều dựa vào Auto Scaling để thêm compute.

Vấn đề là:

```text
Auto Scaling EC2 cần thời gian launch instance
```

Thường mất vài phút để:

* tạo EC2 mới
* boot OS
* chạy app
* register vào Load Balancer

Trong khi đề yêu cầu:

```text
bursts of traffic within seconds
```

=> EC2 / ECS / Elastic Beanstalk không tối ưu bằng Lambda.

## 4. Phân biệt nhanh

| Giải pháp                                    | Phù hợp không? | Lý do                                                 |
| -------------------------------------------- | -------------: | ----------------------------------------------------- |
| API Gateway + Lambda                         |         ✅ Đúng | Scale nhanh trong vài giây, serverless, tiết kiệm     |
| API Gateway + EC2 Auto Scaling               |          ❌ Sai | Scale chậm hơn, cần launch EC2                        |
