# 1. Dịch câu hỏi

Một công ty logistics muốn tự động hóa ứng dụng quản lý đơn hàng.

Công ty muốn sử dụng:

# SFTP file transfer

để upload các tài liệu business-critical.

Vì các file là confidential nên cần:

* mã hóa dữ liệu khi lưu trữ
* đảm bảo high availability
* mỗi file phải tự động bị xóa sau 1 tháng kể từ ngày tạo
* ít operational overhead nhất

Giải pháp nào nên triển khai?

---

# 2. Đáp án đúng

✅ **Create an Amazon S3 bucket with encryption enabled. Launch an AWS Transfer for SFTP endpoint to securely upload files to the S3 bucket. Configure an S3 lifecycle rule to delete files after a month.**

Dịch:

> Tạo một S3 bucket có bật encryption. Tạo AWS Transfer for SFTP endpoint để upload file an toàn vào S3 bucket. Cấu hình S3 Lifecycle rule để tự động xóa file sau 1 tháng.

---

# 3. Keyword quan trọng

## Keyword 1

# "SFTP file transfer"

=> cần hỗ trợ giao thức SFTP.

AWS managed service phù hợp:

# AWS Transfer Family / AWS Transfer for SFTP

---

## Keyword 2

# "confidential files"

=> cần encryption at rest.

S3 hỗ trợ:

* SSE-S3
* SSE-KMS

---

## Keyword 3

# "high availability"

=> nên dùng managed service.

S3 và AWS Transfer Family đều managed/highly available.

---

## Keyword 4

# "automatically deleted one month after creation"

=> cần lifecycle expiration.

S3 Lifecycle rule hỗ trợ tự động delete object sau số ngày nhất định.

---

## Keyword 5

# "least operational overhead"

Đây là keyword quan trọng nhất.

=> tránh tự dựng EC2, cron job, SFTP server.

# 5. Vì sao AWS Transfer for SFTP đúng?

AWS Transfer for SFTP là managed service cho phép client dùng SFTP truyền file vào AWS.

Bạn không cần:

* tự cài SFTP server
* patch server
* quản lý OS
* cấu hình HA thủ công

---

# 6. Vì sao S3 đúng?

S3 phù hợp vì:

* object storage managed
* highly available
* durable
* hỗ trợ encryption at rest
* hỗ trợ lifecycle expiration
* tích hợp trực tiếp với AWS Transfer for SFTP

---

# 7. Vì sao S3 Lifecycle đúng?

S3 Lifecycle có 2 loại action chính:

## Transition action

Chuyển object sang storage class khác.

Ví dụ:

```text
S3 Standard → S3 Standard-IA
```

---

## Expiration action

Tự động xóa object sau thời gian quy định.

Ví dụ:

```text
Delete object after 30 days
```

Đề yêu cầu xóa sau 1 tháng nên dùng:

# Expiration action

---

# 8. Vì sao các đáp án khác sai?

## ❌ EC2 + SFTP service + EFS + cron job

Giải pháp này có thể làm được nhưng sai vì:

```text
operational overhead cao
```

Bạn phải quản lý:

* EC2
* SFTP server
* OS patching
* scaling
* HA
* cron job
* monitoring

Trong khi đề yêu cầu ít operational overhead nhất.

---

## ❌ EFS + AWS Transfer for SFTP + EFS lifecycle delete

Sai vì:

EFS lifecycle policy không dùng để delete file sau 30 ngày.

EFS lifecycle chỉ chuyển file sang:

```text
EFS Infrequent Access
```

hoặc chuyển về Standard khi access.

Không phải retention/delete policy.

---

## ❌ S3 + AWS Transfer for SFTP + retention policy on SFTP server

Sai vì AWS Transfer for SFTP không có retention policy để auto delete file.

Việc xóa sau 30 ngày phải cấu hình ở:

# S3 Lifecycle rule
