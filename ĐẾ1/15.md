# 1. Dịch câu hỏi

Một nền tảng giao dịch tiền điện tử online được host trên AWS, sử dụng:

* Amazon ECS Cluster
* Amazon RDS với cấu hình Multi-AZ Deployments

Ứng dụng sử dụng RDS rất nhiều để xử lý các thao tác database phức tạp, gồm cả read và write.

Để duy trì:

* reliability
* availability
* performance

cần monitor kỹ cách các **process/thread khác nhau trên DB instance** sử dụng CPU, bao gồm:

* phần trăm CPU bandwidth
* tổng memory tiêu thụ bởi từng process

Giải pháp nào phù hợp nhất để monitor database đúng cách?

---

# 2. Đáp án đúng

✅ **Enable Enhanced Monitoring in RDS.**

Dịch:

> Bật Enhanced Monitoring trong Amazon RDS.

---

# 3. Keyword quan trọng của đề

## Keyword 1

# "different processes or threads"

Đây là keyword cực quan trọng.

Đề không chỉ hỏi CPU tổng thể.

Nó hỏi CPU/memory theo từng:

```text
process / thread
```

---

## Keyword 2

# "percentage of CPU bandwidth"

=> cần xem chi tiết CPU usage.

---

## Keyword 3

# "total memory consumed by each process"

=> cần xem memory theo từng process.

---

## Keyword 4

# "RDS instance"

Vì là RDS managed service, bạn không SSH trực tiếp vào DB server như EC2 để cài script tùy ý.

---

# 4. Vì sao Enhanced Monitoring đúng?

RDS Enhanced Monitoring cung cấp metric chi tiết ở cấp:

# Operating System level

Nó cho biết:

* CPU usage
* memory usage
* file system
* process list
* thread/process-level metrics

---

# 5. Flow hoạt động

```text
RDS DB Instance
      ↓
Enhanced Monitoring Agent
      ↓
CloudWatch Logs
      ↓
View detailed OS/process metrics
```

---

# 6. Enhanced Monitoring khác CloudWatch thường thế nào?

## CloudWatch thường

CloudWatch lấy metric từ:

```text
Hypervisor
```

Nó cho biết tổng quan như:

* CPUUtilization
* FreeStorageSpace
* DatabaseConnections
* ReadIOPS
* WriteIOPS

Nhưng không chi tiết từng process.

---

## Enhanced Monitoring

Enhanced Monitoring lấy metric từ:

```text
Agent chạy trên DB instance
```

Nó cho view chi tiết hơn:

* process nào dùng CPU nhiều
* process nào dùng memory nhiều
* OS-level metrics gần real-time

---

# 7. Ví dụ dễ hiểu

CloudWatch thường giống như:

```text
Tổng CPU DB đang dùng 80%
```

Nhưng bạn không biết process nào gây ra.

---

Enhanced Monitoring cho biết:

```text
postgres process A → CPU 45%
postgres process B → CPU 20%
system process → CPU 5%
```

=> biết rõ nguyên nhân bottleneck.

---

# 8. Vì sao các đáp án khác sai?

## ❌ Use Amazon CloudWatch to monitor CPU Utilization

CloudWatch xem được CPU tổng thể.

Nhưng đề cần:

```text
CPU và memory theo từng process/thread
```

CloudWatch thường không đủ chi tiết.

---

## ❌ Check CPU% and MEM% readily available in RDS console

Sai vì các metric CPU% và MEM% theo từng process không tự có sẵn mặc định trong RDS console.

Muốn xem chi tiết cần bật Enhanced Monitoring.

---

## ❌ Create a script to collect custom metrics

Sai vì RDS là managed database.

Bạn không có quyền truy cập OS như EC2 để cài script tùy ý lên DB instance.

Ngoài ra cách này phức tạp, không phải best practice.

---

# 9. Phân biệt nhanh

| Nhu cầu                    | Dùng gì              |
| -------------------------- | -------------------- |
| CPU tổng thể RDS           | CloudWatch           |
| DB connections             | CloudWatch           |
| IOPS / latency             | CloudWatch           |
| OS-level metrics           | Enhanced Monitoring  |
| Process/thread CPU, memory | Enhanced Monitoring  |
| Query-level performance    | Performance Insights |

---

# 10. Enhanced Monitoring vs Performance Insights

## Enhanced Monitoring

Dùng để xem:

```text
OS/process-level resource usage
```

Ví dụ:

* CPU của process
* memory của process
* OS load

---

## Performance Insights

Dùng để xem:

```text
database workload/query-level bottleneck
```

Ví dụ:

* query nào chậm
* wait event nào cao
* SQL nào gây tải

---
