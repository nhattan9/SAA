# 1. Dịch câu hỏi

Một công ty vừa migrate ứng dụng microservices sang:

# Amazon EKS

Trong quá trình migration, công ty cần đảm bảo tất cả dữ liệu cấu hình nhạy cảm và credentials như:

* database password
* API keys

được lưu trữ an toàn và được mã hóa bên trong:

# etcd key-value store của Amazon EKS cluster

Giải pháp nào phù hợp nhất?

---

# 2. Đáp án đúng

✅ **Enable secret encryption with a new AWS KMS key on an existing Amazon EKS cluster to encrypt sensitive data stored in the EKS cluster's etcd key-value store.**

Dịch:

> Bật secret encryption bằng một AWS KMS key mới trên EKS cluster hiện có để mã hóa dữ liệu nhạy cảm được lưu trong etcd key-value store của EKS cluster.

# 4. etcd là gì?

etcd là:

# key-value store của Kubernetes

Nó lưu trạng thái cluster và config như:

* Pods
* Services
* ConfigMaps
* Secrets
* cluster state

---

# 5. Kubernetes Secrets là gì?

Kubernetes Secrets dùng để lưu dữ liệu nhạy cảm như:

```text
DB_PASSWORD
API_KEY
TOKEN
```

Nhưng nếu không cấu hình encryption phù hợp, secrets có thể được lưu trong etcd ở dạng chưa được mã hóa bởi KMS.

---

# 6. Vì sao cần AWS KMS?

AWS KMS dùng để:

* tạo encryption key
* quản lý key
* encrypt/decrypt secrets
* kiểm soát quyền sử dụng key
* audit key usage

Trong EKS, KMS được tích hợp để mã hóa Kubernetes Secrets trước khi lưu vào etcd.

---

# 7. Flow đúng

```text
Kubernetes Secret
      ↓
EKS secret encryption
      ↓
AWS KMS key
      ↓
Encrypted data stored in etcd
```

---

# 8. Vì sao đáp án đúng?

Vì nó xử lý đúng requirement:

```text
encrypt sensitive data inside EKS etcd
```

Cụ thể:

* tạo KMS key mới
* bật secret encryption cho EKS cluster
* secrets được mã hóa at rest trong etcd
* bảo vệ database passwords/API keys

---

# 9. Vì sao AWS Secrets Manager sai?

AWS Secrets Manager rất tốt để:

* lưu secrets tập trung
* rotate secrets
* retrieve secrets runtime

Nhưng đề hỏi cụ thể:

```text
encrypted within the EKS cluster's etcd key-value store
```

Secrets Manager không mã hóa dữ liệu đã lưu trong etcd.

Nó là nơi lưu secret bên ngoài cluster.

---

# 10. Vì sao EBS volume encryption sai?

EBS encryption mã hóa:

```text
EBS volumes
```

của worker nodes.

Nhưng etcd của EKS control plane không nằm trong EBS volume của worker node mà bạn quản lý.

=> không giải quyết yêu cầu mã hóa secrets trong etcd.

---

# 11. Vì sao EBS CSI Driver sai?

EBS CSI Driver dùng để:

# cung cấp persistent volumes cho pods

Ví dụ app cần volume để lưu file/data.

Nhưng nó không liên quan đến:

```text
Kubernetes Secrets trong etcd
```

---
