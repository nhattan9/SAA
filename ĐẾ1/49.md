# Dịch đề

Một cơ quan chính phủ muốn lưu trữ các tài liệu thuế bảo mật trên AWS.

Do dữ liệu rất nhạy cảm, Solutions Architect phải:

1. Chỉ cho phép các request truy cập dữ liệu từ **một Amazon VPC cụ thể**
2. Ngăn file bị:

   * xoá
   * ghi đè
3. Đáp ứng mô hình lưu trữ:

# WORM (Write Once Read Many)

tức là:

```text
ghi 1 lần
đọc nhiều lần
không sửa/xoá được
```

Hỏi nên chọn kết hợp giải pháp nào? (Chọn 2)

---

# Đáp án đúng

## ✅ 1

```text
Configure an Amazon S3 Access Point for the S3 bucket to restrict data access to a particular VPC only.
```

Dịch:

> Cấu hình S3 Access Point để chỉ cho phép truy cập bucket từ một VPC cụ thể.

---

## ✅ 2

```text
Create a new Amazon S3 bucket with the S3 Object Lock feature enabled. Store the documents in the bucket and set the Legal Hold option for object retention.
```

Dịch:

> Tạo S3 bucket bật S3 Object Lock và dùng Legal Hold để giữ object.

# Keyword 1

```text
restrict access to a specific VPC only
```

# 1 VPC cụ thể

# Keyword 2 (QUAN TRỌNG NHẤT)

```text
prevent files from being deleted or overwritten
```

=> chống:

* delete
* overwrite

---

# Keyword 3

```text
WORM
```

=> nghĩ ngay:

# S3 Object Lock

---

# S3 Object Lock là gì?

S3 Object Lock cho phép:

# write once read many

Object sau khi lock:

✅ đọc được
❌ không xoá được
❌ không overwrite được

---

# Có 2 mode chính

| Mode       | Ý nghĩa                         |
| ---------- | ------------------------------- |
| Governance | user đặc biệt vẫn có thể bypass |
| Compliance | không ai bypass được            |

Ngoài ra còn có:

```text
Legal Hold
```

=> giữ object vô thời hạn cho legal/compliance.

---

# Vì sao dùng S3 Access Point?

S3 Access Point giúp:

# tạo endpoint riêng cho S3 bucket

Có thể:

* policy riêng
* network control riêng
* giới hạn theo VPC

---

# Luồng đúng

```text
Specific VPC
     ↓
S3 Access Point
     ↓
S3 Bucket (Object Lock enabled)
```

---

# Vì sao các đáp án khác sai?

---

# ❌ AWS Network Firewall

Sai vì:

Network Firewall dùng cho:

# VPC network traffic

Không attach trực tiếp vào S3 bucket để control access.

Để restrict S3 theo VPC:

=> dùng:

# S3 Access Point

---

# ❌ Glacier Instant Retrieval + Bucket Policy

Sai vì:

Glacier chỉ là:

# storage class

Không liên quan WORM.

Ngoài ra S3 Access Point tối ưu hơn bucket policy cho VPC restriction.

---

# ❌ Enable Object Lock but disable Versioning

Sai cực mạnh.

## Vì:

# Object Lock bắt buộc cần Versioning

Khi bật Object Lock:

✅ Versioning tự bắt buộc bật
❌ không thể disable

---

# Kiến thức cực hay thi

# S3 Object Lock REQUIREMENTS

| Requirement        | Needed     |
| ------------------ | ---------- |
| WORM               | ✅          |
| Versioning enabled | ✅ bắt buộc |
| Prevent delete     | ✅          |
| Prevent overwrite  | ✅          |

# Tổng hợp kiến thức câu này

| Requirement                      | AWS Service     |
| -------------------------------- | --------------- |
| WORM storage                     | S3 Object Lock  |
| Prevent delete/overwrite         | Object Lock     |
| Legal retention                  | Legal Hold      |
| Restrict access to one VPC       | S3 Access Point |
| Immutable storage                | Object Lock     |
| Private S3 endpoint style access | S3 Access Point |
