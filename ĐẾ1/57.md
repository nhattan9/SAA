# Dịch đề

Một công ty có web application sử dụng:

```text
IIS trên Windows Server
```

Application data đang được lưu trong:

```text
file share trên NAS ở on-premises data center
```

Công ty muốn migrate application và file share lên AWS để đạt:

```text
high availability
```

Hỏi nên dùng dịch vụ nào?

---

# Vì sao chọn Amazon FSx for Windows File Server?

Amazon FSx for Windows File Server là dịch vụ file system managed dành cho Windows.

Nó hỗ trợ:

* SMB protocol
* Windows NTFS
* Active Directory integration
* Windows ACLs
* file share cho nhiều EC2 cùng truy cập
* Multi-AZ deployment để high availability

---

# Luồng migrate hợp lý

```text
On-premises Windows file share / NAS
        ↓
Amazon FSx for Windows File Server
        ↓
EC2 Windows + IIS truy cập qua SMB
```

---

# SMB là gì?

SMB là protocol chia sẻ file phổ biến trong Windows.

Ví dụ trong Windows:

```text
\\server-name\share-folder
```

Đây chính là kiểu file share mà IIS/Windows app thường dùng.

---

# Vì sao các đáp án khác sai?

## ❌ Amazon EFS

EFS là shared file storage nhưng chủ yếu dùng cho Linux workloads.

Nó dùng protocol:

```text
NFS
```

Không phù hợp nhất cho Windows/IIS file share.

---

## ❌ Amazon EBS

EBS là block storage attach vào EC2.

Nó giống ổ đĩa:

```text
C:
D:
```

Nhưng không phải managed shared file system cho nhiều Windows servers qua SMB.

---

## ❌ AWS Storage Gateway

Storage Gateway dùng cho hybrid storage:

```text
on-premises ↔ AWS
```

Nó phù hợp khi vẫn giữ hệ thống on-prem và cần kết nối AWS.

Nhưng đề nói muốn migrate application và file share lên AWS, nên FSx Windows phù hợp hơn.
