# 1. Dịch câu hỏi

Solutions Architect cần đảm bảo rằng:

# EC2 instance chỉ được truy cập SSH từ IP:

```text
110.238.98.71
```

Cấu hình nào đáp ứng yêu cầu này?

---

# 2. Đáp án đúng

✅

```text
Security Group Inbound Rule:
Protocol – TCP
Port Range – 22
Source – 110.238.98.71/32
```

---

# 3. Keyword quan trọng

## Keyword 1

# "SSH connection"

SSH dùng:

# TCP port 22

Đây là kiến thức cực cơ bản rất hay thi.

---

## Keyword 2

# "only be accessed from this IP"

=> chỉ cho phép:

```text
110.238.98.71
```

---

## Keyword 3

# "/32"

Đây là keyword quan trọng.

CIDR:

```text
110.238.98.71/32
```

nghĩa là:

# chỉ đúng 1 IP duy nhất

---

# 4. Vì sao dùng Inbound Rule?

SSH là:

```text
client bên ngoài
→ kết nối vào EC2
```

=> đây là:

# inbound traffic

---

# Flow

```text
Your Laptop (110.238.98.71)
        ↓ SSH TCP 22
EC2 Instance
```

---

# 5. Vì sao dùng TCP?

SSH protocol chạy trên:

# TCP

KHÔNG phải UDP.

---

# SSH mặc định:

| Thành phần | Giá trị |
| ---------- | ------- |
| Protocol   | TCP     |
| Port       | 22      |

---

# 6. Vì sao không cần Outbound Rule?

Security Group là:

# stateful

Điều này rất quan trọng.

---

# Stateful nghĩa là gì?

Nếu inbound request được cho phép:

```text
Client → EC2
```

thì return traffic:

```text
EC2 → Client
```

được tự động cho phép.

Không cần explicit outbound rule.

---

# 7. Vì sao /32 quan trọng?

CIDR:

| CIDR      | Ý nghĩa       |
| --------- | ------------- |
| /32       | 1 IP duy nhất |
| /24       | cả network    |
| 0.0.0.0/0 | toàn Internet |

---

Đề yêu cầu:

```text
chỉ 1 IP
```

=> dùng:

# /32

---

# 8. Vì sao các đáp án khác sai?

---

## ❌ UDP Port 22

Sai vì:

SSH không dùng UDP.

SSH dùng:

# TCP

---

## ❌ Outbound Rule TCP 22

Sai vì:

Outbound rule kiểm soát:

```text
traffic đi RA khỏi EC2
```

Trong khi đề cần:

```text
client SSH vào EC2
```

=> phải là inbound.

---

## ❌ Outbound UDP 22 0.0.0.0/0

Sai cả 3 điểm:

* outbound thay vì inbound
* UDP thay vì TCP
* mở toàn Internet

---

# 9. Security Group vs NACL

| Feature    | Security Group | NACL         |
| ---------- | -------------- | ------------ |
| Level      | Instance level | Subnet level |
| Stateful   | ✅              | ❌            |
| Allow/Deny | Allow only     | Allow + Deny |

---

Đề này dùng:

# Security Group

---

# 10. Stateful cực kỳ hay thi

Ví dụ:

```text
Inbound:
TCP 22
Source: Your IP
```

là đủ.

Không cần:

```text
Outbound:
TCP 22
```

vì Security Group tự cho phép response traffic.

---

# 11. Cách nhớ exam

Nếu đề có:

```text
SSH access
```

=> nghĩ ngay:

# TCP 22

---

Nếu đề có:

```text
only one IP
```

=> nghĩ:

# /32 CIDR

---

Nếu đề hỏi:

```text
client access vào EC2
```

=> dùng:

# inbound rule

---

# 12. Chốt ngắn gọn

Yêu cầu:

```text
Chỉ IP 110.238.98.71 được SSH vào EC2
```

Cấu hình đúng:

```text
Inbound Rule
Protocol: TCP
Port: 22
Source: 110.238.98.71/32
```

Vì:

* SSH dùng TCP 22
* /32 = đúng 1 IP
* Security Group là stateful nên không cần outbound rule riêng
