# Dịch đề

Một công ty dược phẩm có tài nguyên chạy:

* trên hệ thống on-premises
* và trên AWS cloud

Công ty yêu cầu tất cả Software Architects:

* truy cập tài nguyên ở cả hai môi trường
* sử dụng chính tài khoản on-premises hiện có
* các credentials này đang được lưu trong Active Directory

Hỏi giải pháp nào phù hợp nhất?

---

# Đáp án đúng

✅

```text id="9ecr87"
Set up SAML 2.0-Based Federation by using a Microsoft Active Directory Federation Service.
```

Dịch:

> Thiết lập SAML 2.0 Federation bằng Microsoft Active Directory Federation Services (AD FS).

---

```text id="0pn9cg"
on-premises credentials
```

KHÔNG tạo user mới trên AWS.

## Keyword 2

```text id="57bryz"
stored in Active Directory
```

---

## Keyword 3 (QUAN TRỌNG NHẤT)

```text id="5c5gf9"
access AWS using existing enterprise login
```

=> đây là:

# Federation / SSO

---

# Federation là gì?

Federation nghĩa là:

# dùng hệ thống login có sẵn để đăng nhập AWS

Ví dụ:

```text id="w9i0th"
Company AD account
→ login AWS
```

Không cần tạo IAM User riêng.

---

# Vì sao dùng SAML 2.0?

AWS hỗ trợ:

# SAML 2.0 Federation

để kết nối với:

* Active Directory
* Azure AD
* Okta
* ADFS

---

# ADFS là gì?

ADFS:

```text id="vjp5ye"
Active Directory Federation Services
```

Là dịch vụ của Microsoft dùng để:

# làm Identity Provider (IdP)

cho SSO/Federation.

---

# Luồng hoạt động

```text id="s5uwfb"
User login bằng AD account
        ↓
ADFS xác thực
        ↓
SAML assertion
        ↓
AWS trust SAML provider
        ↓
User vào AWS Console/API
```

---

# Vì sao đây là giải pháp đúng?

Yêu cầu đề:

| Requirement               | Giải pháp         |
| ------------------------- | ----------------- |
| Dùng AD credentials       | ✅ SAML Federation |
| SSO                       | ✅                 |
| Không tạo IAM users riêng | ✅                 |
| Hybrid on-prem + AWS      | ✅                 |

# Vì sao các đáp án khác sai?

---

# ❌ IAM Users

Sai vì:

IAM User là:

# tài khoản AWS riêng biệt

Trong khi đề yêu cầu:

```text id="1h7dce"
use existing Active Directory credentials
```

=> không muốn tạo account mới.

---

# ❌ Web Identity Federation

Web Identity Federation dùng cho:

* Google
* Facebook
* Login with Amazon

Ví dụ mobile/web app login social.

KHÔNG dùng cho enterprise Active Directory.

---

## Keyword cực hay thi

| Federation Type         | Use case                     |
| ----------------------- | ---------------------------- |
| SAML Federation         | Enterprise AD/SSO            |
| Web Identity Federation | Google/Facebook/Login social |

---

# ❌ Amazon VPC

VPC chỉ là:

# network isolation

Không liên quan authentication/login.

---
