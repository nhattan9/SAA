# 1. Dịch câu hỏi

Một công ty học online đang host ứng dụng e-Learning viết bằng:

# Microsoft .NET

trên Windows Server trong on-premises data center.

Ứng dụng dùng:

# Oracle Database Standard Edition

làm backend database.

Công ty muốn migrate workload này lên AWS để tận dụng:

* high availability
* performance tốt hơn
* ít thay đổi code/development nhất
* môi trường dễ quản lý hơn

Cần chọn HAI giải pháp phù hợp.

---

# 2. Đáp án đúng

✅ **Rehost .NET application lên AWS Elastic Beanstalk Multi-AZ environment**

✅ **Migrate Oracle database sang Amazon RDS for Oracle Multi-AZ bằng AWS DMS**

---

# 3. Keyword quan trọng

## Keyword 1

# "Microsoft .NET application on Windows Server"

Ứng dụng .NET truyền thống chạy trên Windows/IIS.

=> phù hợp với:

# AWS Elastic Beanstalk for .NET

---

## Keyword 2

# "Oracle Database Standard Edition"

Database hiện tại là Oracle.

Nếu chuyển sang RDS for Oracle thì là:

# homogeneous migration

Oracle → Oracle.

---

## Keyword 3

# "minimize development changes"

Đây là keyword rất quan trọng.

=> không nên refactor nhiều.

Nên chọn:

# Rehost / Replatform nhẹ

Không chọn EKS/Fargate vì phải sửa app nhiều.

---

## Keyword 4

# "easier to manage"

=> nên dùng managed service.

* App: Elastic Beanstalk quản lý EC2, scaling, load balancing
* DB: RDS quản lý backup, patching, Multi-AZ

---

## Keyword 5

# "high availability"

=> cần Multi-AZ cho cả application và database.

---

# 4. Kiến trúc đúng

```text
On-premises .NET App
        ↓
AWS Elastic Beanstalk Multi-AZ
```

```text
On-premises Oracle DB
        ↓ AWS DMS
Amazon RDS for Oracle Multi-AZ
```

---

# 5. Vì sao Elastic Beanstalk đúng?

Elastic Beanstalk hỗ trợ .NET app chạy trên Windows/IIS.

Nó tự quản lý:

* EC2 provisioning
* Auto Scaling
* Load Balancer
* health monitoring
* deployment
* Multi-AZ environment

=> giảm operational overhead.

---

# 6. Vì sao RDS for Oracle Multi-AZ đúng?

RDS for Oracle là managed Oracle database.

Multi-AZ giúp:

```text
Primary DB ở AZ A
Standby DB ở AZ B
```

Khi primary lỗi:

```text
RDS tự động failover
```

=> high availability.

---

# 7. Vì sao AWS DMS đúng?

AWS DMS dùng để migrate database từ on-premises lên AWS.

Với Oracle → Oracle:

```text
homogeneous migration
```

Không cần đổi engine.

DMS giúp migrate với downtime thấp.

---

# 8. Vì sao không dùng AWS SCT?

AWS SCT dùng khi migrate khác engine.

Ví dụ:

```text
Oracle → PostgreSQL
Oracle → Aurora MySQL
SQL Server → PostgreSQL
```

Nhưng câu này là:

```text
Oracle → Oracle
```

=> không cần schema conversion.

---

# 9. Vì sao EKS Fargate sai?

Refactor sang .NET Core + chạy container serverless:

```text
thay đổi code/architecture nhiều
```

Trong khi đề yêu cầu:

```text
minimize development changes
```

=> sai.

---

# 10. Vì sao AWS MGN migrate Oracle server sang EC2 sai?

AWS MGN phù hợp lift-and-shift server.

Nhưng database Oracle nên dùng:

# AWS DMS → RDS for Oracle

vì RDS dễ quản lý hơn:

* backup
* patching
* monitoring
* Multi-AZ
* failover

Chạy Oracle trên EC2 thì phải tự quản lý nhiều hơn.

---

# 11. Vì sao ECS + EC2 worker nodes sai?

Có thể làm được, nhưng không tối ưu vì:

* phải containerize app
* quản lý ECS cluster/EC2 nodes
* thay đổi deployment nhiều hơn
* không đơn giản bằng Elastic Beanstalk

Đề muốn:

```text
ít thay đổi + dễ quản lý
```

=> Elastic Beanstalk tốt hơn.
