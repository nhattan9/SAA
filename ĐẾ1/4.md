# 1. Dịch câu hỏi

Một công ty có kiến trúc hybrid cloud kết nối:

* on-premises data center
* hạ tầng AWS cloud

Công ty cần:

* durable backup storage cho corporate documents lưu on-premises
* local cache để truy cập dữ liệu gần đây với độ trễ thấp
* giảm data egress charges

Yêu cầu thêm:

* tài liệu phải được lưu/truy xuất qua giao thức:

# SMB (Server Message Block)

* file phải truy cập được ngay trong vòng vài phút suốt 6 tháng
* sau đó archive thêm 10 năm để đáp ứng compliance

Giải pháp nào là tốt nhất và tiết kiệm chi phí nhất?

---

# 2. Đáp án đúng

✅ **Launch a new file gateway that connects to your on-premises data center using AWS Storage Gateway. Upload the documents to the file gateway and set up a lifecycle policy to move the data into Glacier for data archival.**

Dịch:

> Tạo File Gateway bằng AWS Storage Gateway kết nối với on-premises data center. Upload tài liệu qua File Gateway và dùng lifecycle policy để chuyển dữ liệu sang Glacier để archive.

# 4. AWS Storage Gateway là gì?

AWS Storage Gateway là service hybrid storage giữa:

```text
On-premises
    ↔
AWS Storage
```

---

# Có 3 loại chính

| Gateway        | Dùng cho gì          |
| -------------- | -------------------- |
| File Gateway   | SMB/NFS file storage |
| Volume Gateway | Block storage        |
| Tape Gateway   | Virtual tape backup  |

---

# 5. Vì sao File Gateway đúng?

File Gateway hỗ trợ:

* SMB
* NFS
* local cache
* lưu dữ liệu vào S3
* lifecycle sang Glacier

=> match toàn bộ requirement.

---

# 6. Flow hoạt động

```text
On-prem Users
      ↓ SMB
AWS Storage Gateway - File Gateway
      ↓
Local Cache
      ↓
Amazon S3
      ↓ Lifecycle
Amazon S3 Glacier
```

---

# 7. Vì sao local cache quan trọng?

File Gateway cache local các file truy cập gần đây.

=> lợi ích:

* truy cập nhanh
* giảm latency
* giảm download lại từ AWS
* giảm data transfer cost

---

# 8. Vì sao dùng Glacier?

Đề yêu cầu:

```text
archive thêm 10 năm
```

=> Glacier phù hợp cho:

* long-term archival
* compliance retention
* chi phí thấp

---

# 9. Vì sao không dùng Tape Gateway?

Đây là bẫy rất hay.

Tape Gateway phù hợp cho:

# backup/archive kiểu tape truyền thống

Nhưng KHÔNG phù hợp vì:

* không tối ưu SMB file access
* không có local cache như File Gateway
* retrieval không phù hợp workload active
* không phải file share system

---

# 10. Vì sao DataSync sai?

DataSync chỉ dùng để:

# transfer/sync dữ liệu

Nó không cung cấp:

* SMB file share
* local cache
* hybrid file access layer

---

# 11. Vì sao Direct Connect + EBS sai?

Sai vì:

## EBS không phù hợp

EBS là:

# block storage cho EC2

Không phù hợp làm hybrid SMB file storage.

---

## Operational complexity cao

Phải tự quản lý:

* EC2
* EBS
* snapshots
* backup flow

Trong khi File Gateway managed tốt hơn nhiều.
