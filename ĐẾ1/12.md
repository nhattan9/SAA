# 1. Dịch câu hỏi

Một công ty thương mại điện tử đang vận hành một web application có khả năng mở rộng cao và phụ thuộc vào cơ sở dữ liệu Amazon Aurora.

Khi số lượng người dùng tăng lên, read replica hiện tại gặp khó khăn trong việc xử lý lượng read traffic ngày càng tăng, gây ra hiện tượng nghẽn hiệu năng trong các giờ cao điểm.

Giải pháp nào sau đây sẽ giải quyết vấn đề với chi phí tối ưu nhất?

---

# 2. Đáp án đúng

✅ Use automatic scaling for the Aurora read replica using Aurora Auto Scaling.

---

# 3. Keyword quan trọng của đề

---

## Keyword 1

# "read replica faces difficulties"

=> bottleneck nằm ở:

# READ workload

Không phải write.

---

## Keyword 2

# "increasing read traffic"

=> cần:

# read scaling

---

## Keyword 3

# "during peak periods"

Đây là keyword QUAN TRỌNG.

Nghĩa là:

```text id="lq57b8"
traffic tăng theo thời điểm
```

Không phải lúc nào cũng cao.

=> workload fluctuating.

---

## Keyword 4

# "most cost-effective"

Đây là keyword QUAN TRỌNG NHẤT.

=> cần:

```text id="m4jzgw"
scale khi cần
giảm khi không cần
```

=> tránh overprovisioning.

---

# 4. Vấn đề hiện tại là gì?

Hiện tại:

```text id="vjlwmg"

   ↓
Aurora Reader
```

Khi peak traffic:

```text id="wrjcmw"
1 read replica không đủ tải
```

=> query chậm.

=> bottleneck.

# 6. Vì sao đây là giải pháp tốt nhất?

## ✅ Giải quyết đúng vấn đề

Vấn đề là:

# read bottleneck

Aurora Auto Scaling scale:

# read replicas

=> đúng requirement.

---

## ✅ Cost-effective

Bạn chỉ trả tiền khi cần thêm readers.

Không cần:

```text id="hsmum8"
always-on large cluster
```

=> tiết kiệm chi phí.

---

## ✅ Tự động

Không cần manual scaling.

---

# 7. Vì sao các đáp án khác sai?

---

## ❌ Increase the size of the Aurora DB cluster

Đây là:

# vertical scaling

Ví dụ:

```text id="7isn4w"
db.r6.large
   ↓
db.r6.4xlarge
```

---

# Vấn đề

Traffic chỉ cao:

```text id="50o5rc"
during peak periods
```

Nếu upscale permanent:

```text id="ck4vnp"
chi phí cao cả lúc idle
```

=> không cost-effective.

---

## ❌ Aurora Global Database

Aurora Global Database dùng cho:

* multi-region
* global low-latency reads
* disaster recovery

Đề KHÔNG nói:

* global users
* multi-region
* DR

=> overkill và đắt.

---

## ❌ Cross-region read replica

Cross-region replica phù hợp khi:

* read gần user region khác
* DR
* multi-region availability

Nhưng đề chỉ có:

# local read bottleneck

Ngoài ra còn phát sinh:

* inter-region replication cost
* complexity

=> không cost-effective.
