# Dịch đề

Một công ty dịch vụ tài chính muốn migrate ứng dụng trading từ hệ thống on-premises chạy Microsoft Windows Server lên AWS.

Yêu cầu:

* High Availability giữa nhiều Availability Zones
* Low-latency access tới block storage

Giải pháp nào phù hợp nhất?

---

# Đáp án đúng

✅

```text
Configure the trading application on Amazon EC2 Windows Server instances across two Availability Zones. Use Amazon FSx for NetApp ONTAP to create a Multi-AZ file system and access the data via iSCSI protocol.
```

Dịch:

> Cấu hình trading application trên các EC2 Windows Server trải dài trên 2 AZ.
> Sử dụng Amazon FSx for NetApp ONTAP tạo Multi-AZ file system và truy cập dữ liệu thông qua giao thức iSCSI.

---

# Phân tích keyword quan trọng

## Keyword 1

```text
Windows Server
```

=> workload Windows.

---

## Keyword 2

```text
high availability across multiple AZs
```

=> cần Multi-AZ.

---

## Keyword 3 (QUAN TRỌNG NHẤT)

```text
low-latency access to block storage
```

=> cần:

# BLOCK STORAGE

và latency thấp.

Đây là keyword quyết định đáp án.

# Vì sao chọn FSx for NetApp ONTAP + iSCSI?

## Amazon FSx for NetApp ONTAP hỗ trợ:

| Loại storage       | Có hỗ trợ? |
| ------------------ | ---------- |
| File storage       | ✅          |
| Block storage      | ✅          |
| iSCSI              | ✅          |
| Multi-AZ           | ✅          |
| Windows compatible | ✅          |

---

# iSCSI là keyword cực mạnh

iSCSI:

```text
Internet Small Computer Systems Interface
```

=> protocol dùng cho:

# block storage over network

Nó cho EC2 thấy storage giống như local disk.

---

# Trading application cần gì?

Trading system thường yêu cầu:

* latency cực thấp
* IOPS cao
* block storage
* HA

=> FSx ONTAP rất phù hợp.

---

# Multi-AZ giúp gì?

Nếu một AZ bị lỗi:

```text
AZ-A down
→ failover sang AZ-B
```

=> đảm bảo high availability.

---

# Vì sao các đáp án khác sai?

---

# ❌ Amazon FSx for Windows File Server

Sai vì:

FSx Windows chỉ hỗ trợ:

```text
SMB file storage
```

KHÔNG phải block storage.

---

## Keyword phân biệt cực quan trọng

| Service     | Type         |
| ----------- | ------------ |
| FSx Windows | File storage |
| FSx ONTAP   | File + Block |

---

# ❌ Amazon EFS

Sai vì:

EFS là:

```text
NFS file storage
```

Không phải block storage.

Ngoài ra EFS chủ yếu cho Linux.

Windows support không tối ưu.

---

## Keyword hay thi

| Service     | Protocol          |
| ----------- | ----------------- |
| EFS         | NFS               |
| FSx Windows | SMB               |
| FSx ONTAP   | NFS + SMB + iSCSI |

---

# ❌ Amazon S3

Sai vì:

S3 là:

# Object Storage

KHÔNG phải block storage.

Trading application cần:

```text
low-latency block storage
```

S3 không phù hợp.

---

# Phân biệt 3 loại storage cực hay thi

| Storage Type   | Ví dụ      |
| -------------- | ---------- |
| Block storage  | EBS, iSCSI |
| File storage   | EFS, FSx   |
| Object storage | S3         |

# Kiến thức cực quan trọng cho AWS SAA

## FSx for Windows File Server

* SMB only
* file storage
* Windows shared folder

---

## FSx for NetApp ONTAP

* file + block
* iSCSI support
* enterprise workloads
* HA Multi-AZ
* low latency

---

## EFS

* Linux/NFS
* shared file storage
* không phải block storage

---

## S3

* object storage
* không dùng cho transactional low-latency block workloads.
