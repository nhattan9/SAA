# Dịch đề

Một công ty đang dùng Amazon S3 để lưu dữ liệu frequently accessed.

Khi object được:

* tạo mới
* hoặc xoá

S3 bucket sẽ gửi event notification tới một Amazon SQS queue.

Solutions Architect cần tạo giải pháp để:

# notify cả development team và operations team

về các object được tạo hoặc xoá.

---

# Đáp án đúng

✅

```text id="y1x4os"
Create an Amazon SNS topic and configure two SQS queues to subscribe to the topic.
```

Dịch:

> Tạo một SNS Topic và cấu hình hai SQS queues subscribe vào topic đó.
> Cho phép S3 gửi notification vào SNS và cập nhật bucket để dùng SNS topic mới.

# SNS làm gì?

SNS:

# publish-subscribe messaging service

Khi SNS nhận message:

```text id="o4xq9h"
1 message
→ fanout
→ nhiều subscribers
```

# Vì sao các đáp án khác sai?

---

# ❌ Add another SQS queue directly

Sai vì:

S3 event notification:

# không thiết kế để fanout trực tiếp tới nhiều SQS queues

Chuẩn AWS là dùng SNS ở giữa.

---

# ❌ Two SQS queues poll SNS

Sai vì:

# SNS không phải polling service

SNS là:

```text id="l0g5th"
push-based
```

SQS phải:

# subscribe SNS topic

chứ không poll SNS.

---

# ❌ SNS FIFO topic for other team

Sai vì:

* không cần FIFO ordering
* overkill
* vẫn không giải quyết đúng fanout architecture

Ngoài ra S3 không thiết kế gửi tới nhiều SNS topics trực tiếp.
