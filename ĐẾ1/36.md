# 1. Dịch câu hỏi

Một ứng dụng ghi nhận dữ liệu thời tiết mỗi phút, đang chạy trên một fleet Amazon EC2 Spot Instances và sử dụng MySQL RDS database.

Hiện tại chỉ có:

```text
1 Amazon RDS instance chạy trong 1 Availability Zone
```

Database cần được cải thiện để đảm bảo high availability bằng cách:

```text
replicate dữ liệu đồng bộ sang một RDS instance khác
```

Dịch vụ/cấu hình nào trong RDS thực hiện synchronous data replication?

---

# 2. Đáp án đúng

✅ **RDS DB instance running as a Multi-AZ deployment**

Dịch:

> RDS DB instance chạy với cấu hình Multi-AZ deployment.

---

# 3. Keyword quan trọng

## Keyword 1

# "only one RDS instance running in one Availability Zone"

=> hiện tại database là single-AZ.

Nếu AZ đó lỗi:

```text
database down
```

---

## Keyword 2

# "ensure high availability"

=> cần HA cho database.

---

## Keyword 3

# "synchronous data replication"

Đây là keyword quan trọng nhất.

Trong RDS, replication đồng bộ là:

# Multi-AZ

---

# 4. Multi-AZ RDS hoạt động thế nào?

Khi bật Multi-AZ, RDS tạo:

```text
Primary DB ở AZ A
Standby DB ở AZ B
```

Dữ liệu từ Primary được replicate:

# synchronously

sang Standby.

---

# 5. Flow dễ hiểu

```text
AZ A
Primary RDS
   ↓ synchronous replication
AZ B
Standby RDS
```

Nếu Primary/AZ A lỗi:

```text
Standby ở AZ B được promote
```

=> app tiếp tục dùng database sau failover.

---

# 6. Vì sao Multi-AZ đúng?

Vì nó đáp ứng đúng 2 yêu cầu:

* high availability
* synchronous replication

Multi-AZ không phải để tăng read performance.

Nó chủ yếu để:

```text
failover / HA
```

---

# 7. Vì sao RDS Read Replica sai?

RDS Read Replica dùng:

# asynchronous replication

Nghĩa là dữ liệu replicate có thể bị delay.

Read Replica dùng để:

* scale read workload
* reporting
* analytics

Không phải giải pháp chính cho synchronous HA.

---

# 8. Multi-AZ vs Read Replica

| Tiêu chí                       | Multi-AZ          | Read Replica              |
| ------------------------------ | ----------------- | ------------------------- |
| Mục đích                       | High Availability | Read scaling              |
| Replication                    | Synchronous       | Asynchronous              |
| Failover                       | Automatic         | Thường cần manual promote |
| Standby có dùng để read không? | Không             | Có                        |
| AZ failure protection          | Có                | Không phải mục tiêu chính |

---

# 9. Vì sao DynamoDB Read Replica sai?

DynamoDB không có feature tên:

```text
Read Replica
```

DynamoDB dùng:

# Global Tables

để replicate multi-region.

Ngoài ra đề đang hỏi MySQL RDS.

---

# 10. Vì sao CloudFront sai?

CloudFront là CDN.

Dùng để:

* cache static/dynamic content
* edge delivery

Không liên quan database replication.

---

# 11. Cách nhớ exam

Nếu đề có:

```text
RDS
high availability
standby instance
synchronous replication
automatic failover
```

=> chọn:

# RDS Multi-AZ

Nếu đề có:

```text
read-heavy workload
offload reads
asynchronous replication
```

=> chọn:

# RDS Read Replica

---

# 12. Chốt ngắn gọn

Câu hỏi hỏi:

```text
RDS feature nào replicate đồng bộ?
```

Đáp án:

# RDS Multi-AZ

Vì:

```text
Primary DB
   ↓ synchronous replication
Standby DB ở AZ khác
```
