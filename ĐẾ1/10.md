# 1. Dịch câu hỏi

Một tổ chức y tế muốn xây dựng một hệ thống có thể dự đoán việc **lạm dụng đơn thuốc**.

Tổ chức này sẽ thu thập dữ liệu **real-time** từ nhiều nguồn khác nhau, trong đó có chứa:

# Personally Identifiable Information — PII

tức là thông tin định danh cá nhân.

Điều quan trọng là các thông tin nhạy cảm này phải được:

# anonymized trước khi được lưu vào NoSQL database

để xử lý tiếp.

Giải pháp nào đáp ứng yêu cầu?

---

# 2. Đáp án đúng

✅ **Ingest real-time data using Amazon Kinesis Data Stream. Use an AWS Lambda function to anonymize the PII, then store it in Amazon DynamoDB.**

Dịch:

> Thu thập dữ liệu real-time bằng Amazon Kinesis Data Streams. Dùng AWS Lambda để ẩn danh PII, sau đó lưu dữ liệu đã ẩn danh vào Amazon DynamoDB.

---

# 3. Keyword quan trọng của đề

---

## Keyword 1

# "real-time data from multiple sources"

=> cần dịch vụ ingest streaming real-time.

Phù hợp:

# Amazon Kinesis Data Streams

---

## Keyword 2

# "include Personally Identifiable Information"

=> dữ liệu chứa thông tin nhạy cảm.

Ví dụ PII:

* tên
* email
* số điện thoại
* địa chỉ
* mã bệnh nhân
* số bảo hiểm
* ngày sinh

---

## Keyword 3

# "anonymized prior to landing"

Đây là keyword QUAN TRỌNG NHẤT.

Nghĩa là:

```text
PII phải được ẩn danh TRƯỚC KHI lưu vào database/storage
```

Không được lưu raw PII vào S3, DynamoDB hay DB nào trước rồi mới xử lý.

---

## Keyword 4

# "NoSQL database"

=> nơi lưu cuối cùng phải là NoSQL.

Phù hợp:

# Amazon DynamoDB

Không phải Redshift.

---

# 4. Flow đúng

```text
Multiple data sources
        ↓
Amazon Kinesis Data Streams
        ↓
AWS Lambda anonymize PII
        ↓
Amazon DynamoDB
```

Ý nghĩa:

* Kinesis nhận streaming data real-time
* Lambda xử lý dữ liệu trên đường đi
* PII được anonymize trước
* DynamoDB chỉ nhận data đã sạch

---

# 5. Vì sao Kinesis Data Streams + Lambda + DynamoDB đúng?

## Kinesis Data Streams

Dùng để:

* ingest dữ liệu real-time
* nhận data từ nhiều nguồn
* xử lý streaming
* scale tốt

---

## Lambda

Dùng để:

* đọc record từ Kinesis
* transform data
* mask/remove/hash PII
* gửi data đã xử lý sang DynamoDB

---

## DynamoDB

Là:

# NoSQL database

Phù hợp để lưu dữ liệu đã xử lý để phân tích tiếp.

---

# 6. Ví dụ anonymize PII

Raw data:

```json
{
  "patient_name": "Nguyen Van A",
  "phone": "0912345678",
  "drug": "Medicine X"
}
```

Sau anonymize:

```json
{
  "patient_id_hash": "a8f92c...",
  "phone_masked": "091****678",
  "drug": "Medicine X"
}
```

DynamoDB chỉ lưu bản sau.

---

# 7. Vì sao các đáp án khác sai?

---

## ❌ S3 data lake + S3 trigger + Lambda

Sai vì flow là:

```text
Raw PII → S3 → Lambda anonymize → DynamoDB
```

Vấn đề:

```text
PII đã bị lưu vào S3 trước khi anonymize
```

Trong khi đề yêu cầu:

```text
anonymized prior to landing
```

=> sai requirement.

---

## ❌ DynamoDB Streams + Lambda

Sai vì flow là:

```text
Raw PII → DynamoDB → DynamoDB Streams → Lambda anonymize
```

Vấn đề:

```text
PII đã bị lưu vào DynamoDB trước
```

=> vi phạm yêu cầu.

Ngoài ra:

```text
AmazonDynamoDBFullAccess
```

là quyền quá rộng, không theo least privilege.

---

## ❌ Firehose + Redshift

Sai một phần vì Firehose có thể capture/transform stream.

Nhưng output là:

# Amazon Redshift

Redshift là:

```text
Data warehouse / relational analytics
```

Không phải NoSQL database.

Đề yêu cầu NoSQL database.

=> sai.

---

# 8. Kiến thức cần nhớ

---

# Kinesis Data Streams

Dùng khi đề có:

* real-time streaming
* multiple sources
* cần custom processing
* Lambda xử lý từng record

---

# Kinesis Data Firehose

Dùng khi đề có:

* delivery stream
* load data vào S3 / Redshift / OpenSearch
* ít custom logic hơn Kinesis Data Streams

---

# DynamoDB Streams

Dùng khi:

```text
data đã được ghi vào DynamoDB rồi
```

Sau đó mới trigger xử lý thay đổi.

Không phù hợp nếu cần xử lý trước khi lưu.

---

# 9. Cách nhớ exam

Nếu đề có:

```text
real-time data
PII
must anonymize before storage
NoSQL database
```

=> chọn:

# Kinesis Data Streams + Lambda + DynamoDB

---

# Chốt ngắn gọn

Đáp án đúng vì nó xử lý PII **trên đường đi** trước khi lưu.

```text
Kinesis → Lambda anonymize → DynamoDB
```

Các đáp án sai vì hoặc:

* lưu PII trước rồi mới xử lý
* hoặc lưu vào database không phải NoSQL.
