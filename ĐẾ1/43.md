Ta phân tích policy từng phần.

```json
{ 
 "Version": "2012-10-17", 
 "Statement": [ 
  { 
   "Effect": "Allow", 
   "Action": [ 
    "s3:Get*", 
    "s3:List*" 
   ], 
   "Resource": "*" 
  }, 
  { 
   "Effect": "Allow", 
   "Action": "s3:PutObject", 
   "Resource": "arn:aws:s3:::boracay/*" 
  } 
 ] 
}
```

---

# 1. Statement đầu tiên

```json
{
   "Effect": "Allow",
   "Action": [
      "s3:Get*",
      "s3:List*"
   ],
   "Resource": "*"
}
```

## Ý nghĩa

### `Effect: Allow`

=> cho phép.

---

### `s3:Get*`

Cho phép tất cả action bắt đầu bằng:

```text
Get
```

Ví dụ:

* GetObject
* GetBucketLocation
* GetObjectVersion

=> tức là:

# quyền READ

---

### `s3:List*`

Cho phép:

* ListBucket
* ListObjects

=> xem danh sách object trong bucket.

---

### `Resource: "*"`

Đây là keyword cực quan trọng.

```text
* = tất cả resource
```

=> không chỉ bucket `boracay`

=> mà:

# tất cả S3 buckets

mà account có quyền truy cập.

---

# Kết luận statement 1

User có thể:

✅ đọc object từ mọi S3 bucket
✅ list object trong bucket

---

# 2. Statement thứ hai

```json
{
   "Effect": "Allow",
   "Action": "s3:PutObject",
   "Resource": "arn:aws:s3:::boracay/*"
}
```

## Ý nghĩa

### `s3:PutObject`

=> upload / ghi object vào S3.

Ví dụ:

```text
upload file
copy file
overwrite file
```

---

### `arn:aws:s3:::boracay/*`

Chỉ áp dụng cho:

```text
bucket boracay
```

và tất cả object bên trong bucket đó.

---

# Kết luận statement 2

User có thể:

✅ upload object vào bucket boracay

---

# Các đáp án đúng

## ✅ 1

```text
An IAM user with this IAM policy is allowed to write objects into the boracay S3 bucket.
```

Đúng vì có:

```json
"s3:PutObject"
```

---

## ✅ 2

```text
An IAM user with this IAM policy is allowed to read objects from the boracay S3 bucket.
```

Đúng vì có:

```json
"s3:Get*"
```

và `Resource: "*"`.

---

## ✅ 3

```text
An IAM user with this IAM policy is allowed to read objects from all S3 buckets owned by the account.
```

Đúng vì:

```json
"Resource": "*"
```

=> mọi bucket.

---

# Các đáp án sai

## ❌ Read and DELETE objects

Sai vì không có:

```json
s3:DeleteObject
```

---

## ❌ Không được list objects

Sai vì có:

```json
s3:List*
```

---

## ❌ Change access rights

Sai vì không có quyền như:

* PutBucketPolicy
* PutBucketAcl
* PutObjectAcl

---

# Mẹo đọc IAM Policy khi đi thi

## Bước 1 — nhìn Action

| Action          | Ý nghĩa       |
| --------------- | ------------- |
| GetObject       | đọc           |
| PutObject       | upload/ghi    |
| DeleteObject    | xoá           |
| ListBucket      | xem danh sách |
| PutBucketPolicy | sửa quyền     |

---

## Bước 2 — nhìn Resource

| Resource                | Ý nghĩa             |
| ----------------------- | ------------------- |
| `*`                     | tất cả              |
| `arn:aws:s3:::bucket`   | bucket              |
| `arn:aws:s3:::bucket/*` | object trong bucket |

---

## Phân biệt rất hay ra đề

### Bucket ARN

```text
arn:aws:s3:::mybucket
```

=> bucket itself.

---

### Object ARN

```text
arn:aws:s3:::mybucket/*
```

=> file/object trong bucket.
