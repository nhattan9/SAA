# Dịch đề

Một mạng xã hội nổi tiếng đang chạy trên AWS và sử dụng Amazon DynamoDB làm database.

Có yêu cầu implement tính năng:

# follow

Trong đó:

* user có thể subscribe theo dõi user khác
* khi user đó cập nhật nội dung
* followers sẽ nhận email notification

Giải pháp nào phù hợp nhất?

# DynamoDB Streams là gì?

DynamoDB Streams là:

# log thay đổi dữ liệu theo thời gian

Nó ghi lại:

* INSERT
* UPDATE
* DELETE

trên DynamoDB table.

---

# Ví dụ

Nếu user đăng bài mới:

```text id="u0sq7d"
UPDATE item
```

DynamoDB Streams sẽ tạo stream record.

---

# Lambda trigger làm gì?

AWS Lambda tự động được gọi khi stream có dữ liệu mới.

Không cần polling thủ công.

---

# SNS làm gì?

Amazon SNS dùng để:

# gửi notification

Có thể gửi qua:

* email
* SMS
* HTTP
* Lambda

Ở đây:

```text id="5b0t97"
SNS → Email
```

---

# Vì sao đây là architecture chuẩn AWS?

Vì nó là:

# event-driven architecture

Các service hoạt động tự động theo event.

Rất scalable và serverless.

## Keyword cực hay thi

| Service          | Use case                  |
| ---------------- | ------------------------- |
| DynamoDB Streams | capture changes/events    |
| DAX              | caching/read acceleration |
| SNS              | notifications             |
| Lambda           | event processing          |

---

# Phân biệt DynamoDB Streams vs DAX

| Feature                      | Streams | DAX |
| ---------------------------- | ------- | --- |
| Capture INSERT/UPDATE/DELETE | ✅       | ❌   |
| Event-driven                 | ✅       | ❌   |
| Cache reads                  | ❌       | ✅   |
| Trigger Lambda               | ✅       | ❌   |
