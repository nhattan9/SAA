# 1. Dịch câu hỏi

Một công ty host website thương mại điện tử trên:

* Auto Scaling Group của Amazon EC2
* phía sau Application Load Balancer

Solutions Architect phát hiện website đang nhận rất nhiều request không hợp lệ từ bên ngoài, đến từ nhiều hệ thống khác nhau và IP thay đổi liên tục.

Để xử lý vấn đề hiệu năng, Solutions Architect cần triển khai giải pháp có thể chặn các request này, đồng thời ảnh hưởng tối thiểu đến traffic hợp lệ.

Giải pháp nào đáp ứng yêu cầu?

---

# 2. Đáp án đúng

✅ **Create a rate-based rule in AWS WAF and associate the web ACL to an Application Load Balancer.**

Dịch:

> Tạo rate-based rule trong AWS WAF và associate Web ACL đó với Application Load Balancer.

---

# 3. Keyword quan trọng của đề

## Keyword 1

# "high number of illegitimate external requests"

=> nhiều request xấu / request không hợp lệ.

---

## Keyword 2

# "multiple systems with frequently changing IP addresses"

Đây là keyword rất quan trọng.

IP thay đổi liên tục nên không thể block thủ công từng IP.

---

## Keyword 3

# "minimal impact on legitimate traffic"

Không được chặn nhầm user thật.

=> cần rule thông minh dựa trên rate, không phải deny cứng toàn bộ.

---

## Keyword 4

# "Application Load Balancer"

AWS WAF có thể gắn trực tiếp với ALB.

---

# 4. AWS WAF là gì?

AWS WAF là Web Application Firewall.

Nó bảo vệ application layer HTTP/HTTPS khỏi:

* request bất thường
* bot traffic
* SQL injection
* XSS
* request rate quá cao
* IP xấu

---

# 5. Rate-based rule là gì?

Rate-based rule là rule chặn dựa trên:

# số lượng request từ cùng một IP trong một khoảng thời gian

Ví dụ:

```text
Nếu 1 IP gửi hơn 2,000 requests trong 5 phút
→ block IP đó tạm thời
```

---

# 6. Vì sao Rate-based rule đúng?

Vì đề nói:

```text
nhiều request không hợp lệ
IP thay đổi thường xuyên
giảm ảnh hưởng user thật
```

Rate-based rule sẽ:

* tự theo dõi request rate theo IP
* block IP gửi quá nhiều request
* không cần biết trước IP
* user bình thường ít request nên không bị ảnh hưởng

---

# 7. Flow hoạt động

```text
Internet users / bots
        ↓
Application Load Balancer
        ↓
AWS WAF Web ACL
        ↓
Rate-based rule kiểm tra request rate
        ↓
EC2 Auto Scaling Group
```

Nếu IP request quá nhiều:

```text
AWS WAF → BLOCK
```

Nếu user request bình thường:

```text
AWS WAF → ALLOW
```

---

# 8. Vì sao các đáp án khác sai?

## ❌ Security Group custom rule

Security Group chỉ có thể:

```text
ALLOW traffic
```

Không có deny rule.

Ngoài ra Security Group không hỗ trợ:

* rate limiting
* HTTP inspection
* block theo số lượng request

=> không phù hợp.

---

## ❌ Network ACL

NACL có thể allow/deny ở subnet level.

Nhưng vấn đề là:

```text
IP thay đổi liên tục
```

NACL không phù hợp để rate limit request HTTP.

Ngoài ra NACL hoạt động ở network layer, không hiểu web request như WAF.

---

## ❌ Regular rule trong AWS WAF

Regular rule chỉ match theo điều kiện cố định như:

* IP set
* header
* URI path
* query string
* country
* SQLi/XSS pattern

Nhưng đề cần:

```text
chặn request gửi quá nhiều
```

=> phải dùng:

# Rate-based rule

---

# 9. Regular rule vs Rate-based rule

| Loại rule       | Dùng khi nào                  |
| --------------- | ----------------------------- |
| Regular rule    | Chặn pattern cố định          |
| Rate-based rule | Chặn IP gửi request quá nhiều |

---

# 10. Vì sao không block IP thủ công?

Vì đề nói:

```text
frequently changing IP addresses
```

Nếu block IP thủ công:

* bot đổi IP liên tục
* rule nhanh lỗi thời
* dễ ảnh hưởng user thật

Rate-based rule phù hợp hơn vì tự phát hiện IP nào đang spam.

---

# 11. Cách nhớ exam

Nếu đề có:

```text
ALB
many illegitimate requests
changing IP addresses
rate limit
minimal impact on legitimate users
```

=> chọn:

# AWS WAF Rate-based Rule

# Chốt ngắn gọn

Vấn đề là:

```text
request xấu nhiều, IP thay đổi liên tục
```

Giải pháp tối ưu:

```text
AWS WAF Rate-based Rule
    ↓
Associate Web ACL với ALB
    ↓
Tự block IP gửi quá nhiều request
```
