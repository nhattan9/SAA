# 1. Dịch câu hỏi

Một nền tảng giao dịch Forex thường xuyên xử lý và lưu trữ dữ liệu tài chính toàn cầu mỗi phút.

Hệ thống hiện đang chạy trong on-premises data center và sử dụng Oracle database.

Do sự cố hệ thống làm mát trong data center, công ty cần migrate gấp hạ tầng lên AWS để cải thiện hiệu năng application.

Solutions Architect cần đảm bảo:

* database được migrate đúng cách
* database vẫn available nếu database server bị lỗi trong tương lai
* tuân theo AWS Prescriptive Guidance cho database migration và high availability

Cần chọn kết hợp hành động nào? **Chọn HAI đáp án.**

# 4. AWS DMS là gì?

AWS Database Migration Service dùng để migrate database từ:

* on-premises → AWS
* AWS → AWS
* database này → database khác

DMS hỗ trợ migrate với downtime thấp.

Flow:

```text
On-prem Oracle
      ↓
AWS DMS
      ↓
Amazon RDS Oracle
```

---

# 5. Vì sao RDS Oracle Multi-AZ đúng?

Amazon RDS Multi-AZ tạo:

```text
Primary DB ở AZ A
Standby DB ở AZ B
```

Dữ liệu replicate đồng bộ.

Khi primary DB/server/AZ lỗi:

```text
RDS tự động failover sang standby
```

=> database vẫn available.

---

# 6. Kiến trúc đúng

```text
On-premises Oracle DB
        ↓ AWS DMS
Amazon RDS for Oracle - Multi-AZ
        ↓
Primary DB + Standby DB
```

---

# 7. Vì sao không cần AWS Schema Conversion Tool?

AWS SCT dùng khi migrate khác engine.

Ví dụ:

```text
Oracle → PostgreSQL
Oracle → Aurora MySQL
SQL Server → Aurora PostgreSQL
```

Nhưng câu này là:

```text
Oracle → Oracle
```

=> không cần convert schema.

---

# 8. Vì sao Aurora single instance sai?

Aurora single instance không phù hợp vì đề yêu cầu:

```text
available if database server failure
```

Single instance vẫn là điểm lỗi đơn.

Không đảm bảo high availability bằng Multi-AZ.

---

# 9. Vì sao RDS Oracle + RMAN sai?

RMAN là Oracle Recovery Manager.

Nhưng trên Amazon RDS:

* AWS quản lý backup/restore
* không dùng RMAN theo cách tự quản truyền thống

Ngoài ra RMAN là backup tool, không phải giải pháp HA tự động.

---

# 10. Phân biệt nhanh

| Requirement                         | Service                       |
| ----------------------------------- | ----------------------------- |
| Migrate DB on-premises lên AWS      | AWS DMS                       |
| Oracle → Oracle                     | Không cần SCT                 |
| Oracle → PostgreSQL/Aurora          | Cần AWS SCT                   |
| High availability cho RDS           | Multi-AZ                      |
| Read scaling                        | Read Replica                  |
| Backup/recovery Oracle truyền thống | RMAN, nhưng không phù hợp RDS |

---

# 11. Multi-AZ vs Backup

Multi-AZ dùng cho:

```text
High Availability
Automatic failover
```

Backup dùng cho:

```text
Restore data
Point-in-time recovery
```

Đề cần database vẫn available khi server failure.

=> chọn Multi-AZ.

---

# 12. Cách nhớ exam

Nếu đề có:

```text
on-premises database migration
minimal downtime
```

=> chọn:

# AWS DMS

Nếu đề có:

```text
same database engine
Oracle to Oracle
```

=> không cần SCT.

Nếu đề có:

```text
database server failure
high availability
automatic failover
```

=> chọn:

# RDS Multi-AZ
