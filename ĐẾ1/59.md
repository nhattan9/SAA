# Dịch đề

Công ty muốn launch một Amazon EC2 instance trong **private subnet** cho internal corporate web portal.

Vì lý do bảo mật, EC2 instance phải gửi dữ liệu tới:

* Amazon DynamoDB
* Amazon S3

thông qua **private endpoints**, không đi qua public Internet.

Hỏi giải pháp nào đáp ứng yêu cầu?

---

# Đáp án đúng

✅

```text
Use a DynamoDB VPC endpoint and an S3 VPC endpoint to route all access to these services via private endpoints.
```

Dịch:

> Sử dụng DynamoDB VPC Endpoint và S3 VPC Endpoint để route toàn bộ truy cập tới các service này qua private endpoints.

---

# Phân tích keyword

## Keyword 1

```text
EC2 instance in a private subnet
```

=> EC2 không có public IP, không muốn đi Internet.

---

## Keyword 2

```text
send data to DynamoDB and S3
```

=> cần truy cập AWS public services từ trong VPC.

---

## Keyword 3 quan trọng nhất

```text
private endpoints that don't pass through the public Internet
```

=> nghĩ ngay:

# VPC Endpoint

---

# VPC Endpoint là gì?

VPC Endpoint cho phép resources trong VPC truy cập AWS services bằng đường private.

Không cần:

* Internet Gateway
* NAT Gateway
* public IP
* VPN
* Direct Connect

Traffic sẽ đi trong:

```text
AWS private network
```

---

# Luồng đúng

```text
EC2 private subnet
      ↓
Route Table
      ↓
VPC Endpoint for S3 / DynamoDB
      ↓
Amazon S3 / DynamoDB
```

---

# Vì sao cần VPC Endpoint?

Nếu EC2 trong private subnet muốn gọi S3/DynamoDB mà không có endpoint, thường phải đi qua:

```text
Private subnet → NAT Gateway → Internet → S3/DynamoDB
```

Nhưng đề yêu cầu:

```text
don't pass through public Internet
```

=> dùng VPC Endpoint.

---

# Gateway Endpoint vs Interface Endpoint

Với câu này cần nhớ:

```text
S3 và DynamoDB hỗ trợ Gateway VPC Endpoint
```

Gateway Endpoint dùng bằng cách thêm route vào route table.

Ví dụ:

```text
Destination: prefix-list S3
Target: vpce-xxxx
```

---

# Vì sao các đáp án khác sai?

## ❌ Encryption at Rest

Sai vì encryption at rest chỉ mã hóa dữ liệu khi lưu.

Nó không quyết định traffic đi public hay private.

Ví dụ:

```text
S3 SSE-KMS
DynamoDB encryption
```

=> bảo vệ dữ liệu lưu trữ, không phải network routing.

---

## ❌ AWS Direct Connect

Direct Connect dùng để kết nối:

```text
On-premises data center ↔ AWS
```

Đề không nói có on-premises/hybrid.

Ngoài ra EC2 trong VPC truy cập S3/DynamoDB private thì dùng VPC Endpoint là đúng nhất.

---

## ❌ AWS VPN CloudHub

VPN CloudHub dùng cho kết nối nhiều remote sites với nhau qua AWS VPN.

Không dùng để tạo private endpoint cho EC2 truy cập S3/DynamoDB.

---

# Bảng nhớ nhanh

| Requirement                                  | Service                       |
| -------------------------------------------- | ----------------------------- |
| EC2 private subnet access S3 privately       | S3 VPC Endpoint               |
| EC2 private subnet access DynamoDB privately | DynamoDB VPC Endpoint         |
| Encrypt data at rest                         | SSE-KMS / DynamoDB encryption |
| Connect on-prem to AWS                       | Direct Connect / VPN          |
| Connect multiple remote sites                | VPN CloudHub                  |

---

# Mẹo đi thi AWS

Nếu đề có:

```text
private subnet
access S3/DynamoDB
without public Internet
private endpoint
```

=> chọn:

```text
VPC Endpoint
```

# Chốt

Kiến trúc đúng:

```text
EC2 private subnet
→ S3 Gateway VPC Endpoint
→ DynamoDB Gateway VPC Endpoint
```

Không cần NAT Gateway, Internet Gateway, VPN hay Direct Connect.
