# 1. Dịch câu hỏi

Một hệ thống y tế online được host trên AWS và lưu thông tin nhạy cảm PII của user trong Amazon S3 bucket.

Yêu cầu compliance rất nghiêm ngặt:

* master keys không bao giờ được gửi tới AWS
* dữ liệu chưa mã hóa cũng không bao giờ được gửi tới AWS

Solutions Architect nên dùng kỹ thuật mã hóa S3 nào?

---

# 2. Đáp án đúng

✅ **Use S3 client-side encryption with a client-side master key.**

Dịch:

> Dùng S3 client-side encryption với client-side master key.

---

# 3. Keyword quan trọng

## Keyword 1

# "unencrypted data should never be sent to AWS"

Đây là keyword rất quan trọng.

Nghĩa là dữ liệu phải được mã hóa **trước khi upload lên S3**.

=> cần:

# Client-side encryption

---

## Keyword 2

# "master keys should never be sent to AWS"

Đây là keyword quyết định.

Không được dùng KMS key do AWS quản lý hoặc biết KeyId.

=> cần:

# client-side master key

---

# 4. Client-side encryption là gì?

Client-side encryption nghĩa là:

```text
Mã hóa dữ liệu ở phía client/application trước khi gửi lên AWS
```

Flow:

```text
Application
   ↓ encrypt data locally
Encrypted data
   ↓ upload
Amazon S3
```

S3 chỉ nhận dữ liệu đã mã hóa.

---

# 5. Vì sao client-side master key đúng?

Vì master key nằm hoàn toàn ở phía client.

AWS không nhận:

* plaintext data
* master key

AWS chỉ lưu:

* encrypted object
* encrypted data key trong metadata

---

# 6. Flow đúng

```text
Plaintext PII
      ↓ encrypt locally
Client-side encryption
      ↓
Encrypted object
      ↓
Amazon S3
```

Master key:

```text
luôn nằm ở client
không gửi tới AWS
```

---

# 7. Cách nó hoạt động

Khi upload:

```text
1. Client tạo data key local
2. Data key mã hóa object
3. Master key local mã hóa data key
4. Upload encrypted object + encrypted data key metadata lên S3
```

Khi download:

```text
1. Client tải encrypted object
2. Client dùng master key local decrypt data key
3. Dùng data key decrypt object
```

---

# 8. Vì sao các đáp án khác sai?

## ❌ S3 client-side encryption with AWS KMS key

Vẫn là client-side encryption, nhưng dùng AWS KMS key.

Vấn đề:

```text
AWS KMS tham gia quản lý key
```

Bạn phải cung cấp KMS KeyId cho AWS/KMS.

Đề yêu cầu master key không được gửi/không phụ thuộc AWS.

=> không phù hợp.

---

## ❌ S3 server-side encryption with AWS KMS key

Sai vì server-side encryption nghĩa là:

```text
data gửi lên AWS trước
AWS mới mã hóa
```

Dữ liệu plaintext có thể đi tới AWS trước khi được mã hóa tại server side.

Ngoài ra KMS key do AWS KMS quản lý.

=> vi phạm cả 2 yêu cầu.

---

## ❌ S3 server-side encryption with customer-provided key — SSE-C

Nghe có vẻ đúng nhưng vẫn sai.

Vì với SSE-C:

```text
bạn gửi encryption key trong request tới S3
```

AWS dùng key đó để mã hóa object rồi discard key.

Nhưng đề nói:

```text
master keys should never be sent to AWS
```

=> SSE-C không phù hợp.
