# 1. Dịch câu hỏi 

Một công ty xử lý thanh toán muốn migrate ứng dụng on-premises của mình sang Amazon EC2.

Một IPv6 CIDR block đã được gắn vào Amazon VPC của công ty.

Chính sách bảo mật nghiêm ngặt yêu cầu:

* Production VPC chỉ được phép outbound communication qua IPv6 từ EC2 ra Internet
* Internet KHÔNG được phép chủ động initiate inbound IPv6 connection tới instance
* Kiến trúc mới cũng phải hỗ trợ:

  * traffic flow inspection
  * traffic filtering

Solutions Architect nên làm gì để đáp ứng yêu cầu này?

---

# 2. Đáp án đúng

✅

# Launch the EC2 instance to a private subnet and attach an Egress-Only Internet Gateway to the VPC to allow outbound IPv6 communication to the internet. Use AWS Network Firewall to set up the required rules for traffic inspection and traffic filtering.

---

# 3. Keyword quan trọng của đề

---

## Keyword 1

# "IPv6 CIDR block attached"

=> hệ thống đang dùng:

# IPv6

Đây là keyword rất quan trọng.

---

## Keyword 2

# "allow outbound communication over IPv6"

=> EC2 phải ra Internet được.

---

## Keyword 3

# "prevent the internet from initiating inbound IPv6 connection"

Đây là keyword QUAN TRỌNG NHẤT.

Nghĩa là:

```text
EC2 → Internet : ALLOW
Internet → EC2 : DENY
```

=> chỉ cho outbound IPv6.

---

## Keyword 4

# "traffic inspection and filtering"

=> cần firewall thực sự.

Không chỉ security group.

---

# 4. Egress-Only Internet Gateway là gì?

Đây là service thiết kế riêng cho:

# IPv6 outbound-only internet access

---

# Flow hoạt động

```text
EC2
  ↓ outbound IPv6
Egress-Only Internet Gateway
  ↓
Internet
```

---

# Quan trọng nhất

Internet KHÔNG thể:

```text
Internet
   ↓
EC2
```

khởi tạo connection ngược lại.

---

# 5. Vì sao cần Egress-Only IGW?

Trong IPv6:

# mọi IPv6 address đều public mặc định

Khác IPv4:

* IPv4 thường dùng private IP + NAT

IPv6 không NAT theo kiểu đó.

=> cần:

# Egress-Only Internet Gateway

để chặn inbound initiated traffic.

---

# 6. Vì sao EC2 phải ở private subnet?

Đề yêu cầu:

```text
Không cho Internet initiate inbound connection
```

=> EC2 không nên ở public subnet.

---

# Kiến trúc đúng

```text
Private Subnet
     ↓
EC2
     ↓
Egress-Only IGW
     ↓
Internet
```

---

# 7. AWS Network Firewall dùng để làm gì?

Đề yêu cầu thêm:

```text
traffic inspection
traffic filtering
```

=> cần firewall layer cao hơn.

---

# AWS Network Firewall hỗ trợ:

## ✅ Stateful inspection

Theo dõi flow/connections.

---

## ✅ Traffic filtering

Block/allow traffic theo rule.

---

## ✅ IPS (Intrusion Prevention)

Phát hiện exploit/network attacks.

---

## ✅ Domain filtering

Block domain xấu.

# 9. Vì sao các đáp án khác sai?

---

## ❌ NAT Gateway

NAT Gateway chủ yếu cho:

# IPv4 outbound internet access

Đúng là NAT Gateway có NAT64 cho IPv6 → IPv4 translation.

Nhưng:

```text
Không phải giải pháp chuẩn để chặn inbound IPv6
```

Đề đang hỏi specifically:

# outbound-only IPv6

=> phải dùng:

# Egress-Only IGW

---

## ❌ Internet Gateway + Public Subnet

IGW cho phép:

```text
2 chiều
```

Internet có thể initiate inbound connection.

=> vi phạm requirement.

---

## ❌ Traffic Mirroring

Traffic Mirroring chỉ:

# copy network packets

để analyze.

Không phải firewall.

Không filtering/block traffic.

---

## ❌ AWS PrivateLink

PrivateLink dùng để:

# private access tới AWS services/VPC services

Không dùng để outbound internet IPv6.

---

## ❌ GuardDuty

GuardDuty là:

# threat detection service

Nó:

* detect suspicious activity
* alert security findings

Nhưng KHÔNG:

* filter traffic
* inspect/block traffic inline

---

# 10. Phân biệt cực quan trọng

| -------------------- | ------------------------------ |
