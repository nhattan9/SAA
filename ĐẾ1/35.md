# 1. Dịch câu hỏi 

Một công ty lưu trữ hồ sơ y tế đang có kế hoạch lưu dữ liệu clinical trial nhạy cảm trong Amazon S3 với:

# object-level versioning enabled

Solutions Architect cần đảm bảo:

* không object nào có thể bị overwrite hoặc delete
* trong vòng 1 năm
* kể cả root user AWS account cũng không được phép thay đổi object

Giải pháp bảo mật nhất là gì?

---

# 2. Đáp án đúng

✅ **Enable S3 Object Lock in compliance mode with a retention period of one year.** 

Dịch:

> Bật S3 Object Lock ở compliance mode với retention period là 1 năm.

---

# 3. Keyword quan trọng

## Keyword 1

# "no object can be overwritten or deleted"

=> cần:

# WORM storage

(Write Once Read Many)

---

## Keyword 2

# "for one year"

=> cần:

# retention period = 1 year

---

## Keyword 3

# "root user must also be restricted"

Đây là keyword QUAN TRỌNG NHẤT.

Nếu ngay cả root user cũng không bypass được:

=> phải dùng:

# Compliance Mode

---

# 4. S3 Object Lock là gì?

S3 Object Lock cho phép object hoạt động theo mô hình:

# WORM (Write Once Read Many)

Nghĩa là:

```text
ghi 1 lần
không được sửa/xóa trong thời gian lock
```

---

# 5. Object Lock có 2 mode

| Mode            | Root user bypass được? |
| --------------- | ---------------------- |
| Governance Mode | ✅ Có thể               |
| Compliance Mode | ❌ Không thể            |

---

# 6. Governance Mode là gì?

Governance Mode bảo vệ object khỏi phần lớn user.

Nhưng:

```text
user có special permission
hoặc root user
```

vẫn có thể override/delete object.

Ví dụ cần permission:

```text
s3:BypassGovernanceRetention
```

---

# 7. Compliance Mode là gì?

Compliance Mode mạnh hơn rất nhiều.

Khi object bị lock:

```text
Không ai xóa/sửa được
KỂ CẢ ROOT USER
```

---

# Đây chính là requirement của đề.

---

# 8. Retention Period là gì?

Retention period là:

# thời gian object bị khóa

Ví dụ:

```text
1 year
```

Trong khoảng thời gian đó:

```text
object không thể overwrite/delete
```

---

# 9. Legal Hold là gì?

Legal Hold cũng khóa object.

Nhưng:

# không có thời hạn tự động

Nó tồn tại:

```text
cho tới khi ai đó remove legal hold
```

---

# Vì sao legal hold không phù hợp?

Đề yêu cầu rõ:

```text
1 năm
```

=> cần:

# retention period

Không phải legal hold.

---

# 10. Flow đúng

```text
S3 Bucket
    ↓
Enable Versioning
    ↓
Enable Object Lock
    ↓
Compliance Mode
    ↓
Retention Period = 1 year
```

---

# 11. Vì sao đáp án đúng?

Vì nó đáp ứng đủ:

* chống overwrite
* chống delete
* khóa đúng 1 năm
* root user cũng không bypass được
* phù hợp compliance/regulatory workloads

---

# 12. Vì sao các đáp án khác sai?

---

## ❌ Governance Mode + Retention Period

Sai vì:

```text
root user vẫn bypass được
```

Trong khi đề yêu cầu:

```text
root user cũng bị chặn
```

---

## ❌ Governance Mode + Legal Hold

Sai vì:

* governance mode không đủ mạnh
* legal hold không có thời hạn 1 năm

---

## ❌ Compliance Mode + Legal Hold

Sai vì:

Compliance mode đúng.

Nhưng:

```text
legal hold không có retention period
```

Không thể set:

```text
legal hold = 1 year
```

---

# 13. Phân biệt cực quan trọng

| Feature          | Có thời hạn? | Root bypass được? |
| ---------------- | ------------ | ----------------- |
| Governance Mode  | ✅            | ✅                 |
| Compliance Mode  | ✅            | ❌                 |
| Legal Hold       | ❌            | phụ thuộc quyền   |
| Retention Period | ✅            | tùy mode          |

---

# 14. Cách nhớ exam

Nếu đề có:

```text
WORM
compliance
regulatory
root user cannot delete
immutable storage
```

=> chọn:

# S3 Object Lock – Compliance Mode

---

Nếu đề có:

```text
temporary protection
admins may override
```

=> nghĩ:

# Governance Mode
