# 1. Dịch câu hỏi

Một Solutions Architect đang host một website trong một Amazon S3 bucket tên là:

```text
tutorialsdojo
```

Người dùng truy cập website bằng URL:

```text
http://tutorialsdojo.s3-website-us-east-1.amazonaws.com
```

Một yêu cầu mới được đưa ra:

* thêm JavaScript vào webpage
* JavaScript sẽ thực hiện các authenticated HTTP GET requests tới cùng bucket
* sử dụng S3 API endpoint:

```text
tutorialsdojo.s3.amazonaws.com
```

Tuy nhiên khi test:

```text
web browser chặn JavaScript request
```

Giải pháp nào phù hợp nhất?

---

# 2. Đáp án đúng

✅ **Enable Cross-origin resource sharing (CORS) configuration in the bucket.**

Dịch:

> Bật cấu hình CORS trên S3 bucket.

---

# 3. Keyword quan trọng của đề

## Keyword 1

# "JavaScript makes HTTP GET requests"

=> browser đang gọi API từ frontend.

---

## Keyword 2

# "same bucket using S3 API endpoint"

Đây là keyword QUAN TRỌNG.

Website load từ:

```text
tutorialsdojo.s3-website-us-east-1.amazonaws.com
```

Nhưng API request gửi tới:

```text
tutorialsdojo.s3.amazonaws.com
```

---

# 4. Vì sao browser block?

Mặc dù cùng bucket:

```text
tutorialsdojo
```

Nhưng browser nhìn thấy:

```text
Origin A:
tutorialsdojo.s3-website-us-east-1.amazonaws.com
```

và:

```text
Origin B:
tutorialsdojo.s3.amazonaws.com
```

là:

# khác origin

=> browser security policy chặn.

---

# 5. Đây chính là bài toán gì?

# Cross-Origin Request

---

# Browser có security policy gì?

# Same-Origin Policy (SOP)

Browser mặc định:

```text
Không cho JavaScript gọi request sang origin khác
```

để tránh:

* XSS
* data theft
* malicious requests

---

# 6. CORS là gì?

# Cross-Origin Resource Sharing

CORS cho phép server nói với browser:

```text
"Tôi cho phép origin này access resource của tôi"
```

---

# 7. Flow hiện tại

## ❌ Không có CORS

```text
Website Origin
    ↓ JS Request
S3 API Origin
    ↓
Browser BLOCK
```

---

## ✅ Có CORS

```text
Website Origin
    ↓ JS Request
S3 API Origin
    ↓
S3 trả Access-Control-Allow-Origin
    ↓
Browser cho phép
```

---

# 8. CORS config trong S3

Ví dụ:

```json
[
  {
    "AllowedOrigins": [
      "http://tutorialsdojo.s3-website-us-east-1.amazonaws.com"
    ],
    "AllowedMethods": [
      "GET"
    ],
    "AllowedHeaders": [
      "*"
    ]
  }
]
```

---

# 9. Vì sao các đáp án khác sai?

---

## ❌ Cross-Zone Load Balancing

Cross-Zone LB là tính năng của:

* ELB
* ALB
* NLB

Không liên quan S3.

---

## ❌ Cross-account access

Cross-account access dùng IAM permission giữa AWS accounts.

Không liên quan browser blocked requests.

---

## ❌ Cross-Region Replication (CRR)

CRR dùng để:

```text
replicate object giữa các region
```

Không liên quan JavaScript/browser/origin.

---

# 10. Phân biệt cực quan trọng

| Khái niệm            | Dùng cho gì                       |
| -------------------- | --------------------------------- |
| CORS                 | Browser cross-origin requests     |
| CRR                  | Replicate S3 objects giữa regions |
| Cross-account access | IAM permission giữa accounts      |
| Cross-zone LB        | ELB traffic balancing             |

# 12. Cách hiểu siêu ngắn

Website load từ:

```text
Domain A
```

JavaScript gọi:

```text
Domain B
```

Browser sẽ hỏi:

```text
"Domain B có cho phép Domain A gọi không?"
```

CORS chính là câu trả lời:

```text
"YES, ALLOWED"
```
