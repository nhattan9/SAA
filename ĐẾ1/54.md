# Dịch đề

Một công ty đang dùng:

```text
API Gateway + AWS Lambda
```

cho web service của một online web portal.

Portal này mỗi ngày có hàng trăm nghìn client truy cập.

Công ty sắp công bố một sản phẩm mới rất nổi bật, nên dự kiến portal sẽ nhận lượng truy cập cực lớn từ toàn cầu.

Hỏi:

```text
Làm sao bảo vệ backend systems và applications khỏi traffic spikes?
```

# Throttling trong API Gateway là gì?

Throttling nghĩa là:

# giới hạn số request được phép đi qua API Gateway

Ví dụ:

```text
Rate limit: 1,000 requests/second
Burst limit: 2,000 requests
```

Nếu request vượt quá giới hạn:

```text
API Gateway trả về HTTP 429 Too Many Requests
```

Backend như Lambda, database, service phía sau sẽ không bị “đập” quá tải.

---

# Vì sao đáp án này đúng?

API Gateway đứng trước backend:

```text
Users toàn cầu
   ↓
API Gateway throttling
   ↓
Lambda / backend systems
```

Khi traffic tăng đột ngột, API Gateway sẽ chặn bớt request vượt ngưỡng.

=> bảo vệ:

* Lambda khỏi concurrency spike
* database khỏi quá tải
* backend service khỏi nghẽn

---

# Vì sao không chọn “API Gateway tự scale nên không cần làm gì”?

API Gateway đúng là managed service và có thể scale.

Nhưng đề hỏi:

```text
protect backend systems
```

Nếu API Gateway cứ cho toàn bộ request đi qua Lambda/backend, backend vẫn có thể quá tải.

Vì vậy vẫn cần:

```text
throttling limits
```

---

# Vì sao các đáp án khác sai?

## ❌ Deploy Multi-AZ in API Gateway with Read Replica

Sai vì:

* API Gateway không có khái niệm Read Replica.
* Read Replica là của database như RDS.
* Multi-AZ cũng thường nói cho RDS/EC2/database, không phải cách bảo vệ API Gateway traffic spike.

---

## ❌ Manually upgrade EC2 instances being used by API Gateway

Sai vì:

API Gateway là fully managed service.

Bạn không quản lý EC2 underlying của API Gateway.

Không có chuyện “upgrade EC2 instance của API Gateway”.

---

## ❌ API Gateway automatically scale, do nothing

Sai vì:

Tự scale chỉ giúp API Gateway nhận request tốt hơn.

Nhưng muốn bảo vệ backend thì phải giới hạn request.

# Bảng nhớ nhanh

| Requirement                            | Solution                    |
| -------------------------------------- | --------------------------- |
| Protect backend from too many requests | API Gateway throttling      |
| Return error when too many requests    | HTTP 429                    |
| Cache response                         | API Gateway caching         |
| Auth users                             | Cognito / Lambda authorizer |
| Protect from DDoS                      | AWS Shield / WAF            |
| Backend compute                        | Lambda                      |

# Chốt

API Gateway có thể scale, nhưng để **bảo vệ backend khỏi traffic tăng đột biến**, cần cấu hình:

```text
Rate limit + Burst limit
```

trong API Gateway throttling.
