# Dịch đề

Một công ty muốn query dữ liệu nằm ở nhiều AWS accounts từ một central data lake.

Mỗi account có một Amazon S3 bucket riêng chứa dữ liệu phục vụ business function riêng.

Yêu cầu:

* query dữ liệu tập trung
* phân quyền truy cập theo user roles
* giảm tối đa:

  * management overhead
  * chi phí

Giải pháp nào phù hợp nhất?

---

# AWS Lake Formation là gì?

Lake Formation là service giúp:

# build data lake tập trung

trên Amazon S3.

Nó hỗ trợ:

✅ centralized data lake
✅ cross-account data sharing
✅ fine-grained permissions
✅ governance
✅ data catalog integration

# Vì sao Lake Formation phù hợp?

## 1. Multi-account data lake

Lake Formation hỗ trợ:

# cross-account sharing

rất tốt.

---

## 2. Role-based access

Có thể phân quyền:

* theo table
* theo column
* theo IAM role/user

---

## 3. Low operational overhead

Không cần:

❌ custom Lambda sync
❌ duplicate data
❌ manual ETL phức tạp

---

## 4. Cost-effective

Không cần copy dữ liệu nhiều lần giữa accounts.

Lake Formation + Glue Catalog hoạt động trực tiếp trên S3.

# Vì sao các đáp án khác sai?

---

# ❌ Lambda + EventBridge copy data

Sai vì:

* phải code custom
* khó maintain
* nhiều operational overhead
* tốn cost do duplicate data

Đề yêu cầu:

```text id="13vjbc"
minimize overhead and costs
```

---

# ❌ Amazon Data Firehose

Firehose dùng cho:

# streaming ingestion

Ví dụ:

* logs
* clickstream
* realtime events

Không tối ưu cho:

# centralized governed data lake

Ngoài ra còn tốn cost vì duplicate/move data.

---

# ❌ AWS Control Tower

Control Tower dùng cho:

# quản trị multi-account environment

Ví dụ:

* landing zone
* governance
* guardrails

KHÔNG phải service để:

# build/query centralized data lake
