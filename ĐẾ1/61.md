# Dịch đề

Một công ty đang thiết kế banking portal sử dụng:

```text
Amazon ElastiCache for Redis
```

làm distributed session management.

Yêu cầu bảo mật:

* bảo vệ session data
* Cloud Engineers phải authenticate trước khi chạy Redis commands
* đặc biệt là các lệnh như `MULTI EXEC`
* hệ thống phải yêu cầu nhập password
* dùng long-lived credentials

Hỏi nên làm gì?

---

# Đáp án đúng

✅

```text
Authenticate the users using Redis AUTH by creating a new Redis Cluster with both the --transit-encryption-enabled and --auth-token parameters enabled.
```

Dịch:

> Xác thực user bằng Redis AUTH bằng cách tạo Redis Cluster mới với cả `--transit-encryption-enabled` và `--auth-token`.

## Keyword 1

```text
Redis commands
```

=> đang nói trực tiếp đến command trong Redis.

```text
must authenticate before executing Redis commands
```

=> phải xác thực trước khi chạy command.

---

## Keyword 3 quan trọng nhất

```text

```

=> Redis AUTH.

```text
long-lived credentials
```

=> không phải IAM token vì IAM token có thời hạn ngắn.

# Redis AUTH là gì?

Redis AUTH là cơ chế yêu cầu client nhập password trước khi được chạy command Redis.

Ví dụ:

```text
AUTH my-password
```

Sau khi AUTH thành công mới được chạy:

```text
GET session:123
SET session:123 value
MULTI
EXEC
```

---

# Vì sao cần `--auth-token`?

Trong ElastiCache Redis, muốn bật Redis AUTH thì khi tạo cluster/replication group phải cấu hình:

```text
--auth-token <password>
```

Auth token chính là password Redis.

---

# Vì sao cần `--transit-encryption-enabled`?

Redis AUTH trên ElastiCache yêu cầu bật encryption in transit.

Nếu không bật encryption in transit, password có thể truyền qua network không an toàn.

Nên combo đúng là:

```text
--transit-encryption-enabled
--auth-token
```

# Vì sao các đáp án khác sai?

## ❌ AtRestEncryptionEnabled

At-rest encryption chỉ mã hóa dữ liệu khi lưu trữ.

Nó không bắt user nhập password trước khi chạy Redis command.

---

## ❌ Enable in-transit encryption only

In-transit encryption chỉ mã hóa dữ liệu khi truyền trên network.

Nó chưa có authentication bằng password.

Cần thêm:

```text
--auth-token
```

---

## ❌ IAM authentication token

Sai vì Redis command authentication không dùng IAM token kiểu này.

Ngoài ra IAM token thường có thời hạn ngắn, không phù hợp yêu cầu:

```text
long-lived credentials
```

---

# Bảng nhớ nhanh

| Requirement                    | Chọn                           |
| ------------------------------ | ------------------------------ |
| Redis yêu cầu password         | Redis AUTH                     |
| Password cho ElastiCache Redis | `--auth-token`                 |
| Bảo vệ data trên đường truyền  | `--transit-encryption-enabled` |
| Bảo vệ data khi lưu            | At-rest encryption             |
| IAM token cho Redis command    | Không phù hợp trong câu này    |

---

# Mẹo đi thi AWS

Nếu đề có:

```text
ElastiCache Redis
password
AUTH
execute Redis commands
MULTI EXEC
```

=> chọn:

```text
Redis AUTH + auth-token + transit encryption
```

# Chốt

Để Cloud Engineers phải nhập password trước khi chạy Redis commands:

```text
Create Redis cluster with:
--transit-encryption-enabled
--auth-token
```
