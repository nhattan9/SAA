# 1. Dịch câu hỏi

Một ứng dụng tài chính gồm:

* Auto Scaling Group của Amazon EC2
* Application Load Balancer
* MySQL RDS instance chạy Multi-AZ

Để bảo vệ dữ liệu bí mật của khách hàng, cần đảm bảo rằng Amazon RDS database chỉ có thể được truy cập bằng:

# authentication token

token này phải gắn với:

# profile credentials của EC2 instances

Hành động nào cần thực hiện để đáp ứng yêu cầu này?

---

# 2. Đáp án đúng

✅ **Enable the IAM DB Authentication.**

Dịch:

> Bật IAM Database Authentication cho RDS.

---

# 3. Keyword quan trọng của đề

## Keyword 1

# "MySQL RDS"

IAM DB Authentication hỗ trợ:

* MySQL
* PostgreSQL

=> phù hợp.

---

## Keyword 2

# "authentication token"

Đây là keyword rất mạnh.

RDS IAM DB Authentication dùng:

```text
temporary authentication token
```

thay vì password.

---

## Keyword 3

# "profile credentials of EC2 instances"

EC2 có thể dùng:

```text
IAM Role / Instance Profile
```

để generate token connect RDS.

---

## Keyword 4

# "only accessible using authentication token"

=> không dùng DB password truyền thống.

=> chọn IAM DB Authentication.

---

# 4. IAM DB Authentication là gì?

IAM DB Authentication cho phép application connect tới RDS bằng:

# IAM authentication token

thay vì dùng password database.

---

# 5. Flow hoạt động

```text
EC2 Instance
   ↓ dùng IAM Role / Instance Profile
Generate auth token
   ↓
Connect tới RDS MySQL
   ↓
RDS verify token bằng IAM
```

---

# 6. Token này có gì đặc biệt?

Authentication token:

* được RDS generate theo request
* dùng AWS Signature Version 4
* có thời hạn ngắn

```text
Token lifetime = 15 phút
```

=> giảm rủi ro lộ password lâu dài.

---

# 7. Vì sao đáp án đúng?

Vì đề yêu cầu:

```text
RDS chỉ truy cập bằng token dựa trên EC2 profile credentials
```

IAM DB Authentication đáp ứng đúng:

* không cần lưu DB password trong app
* dùng IAM Role của EC2
* generate token tạm thời
* quản lý quyền qua IAM
* hỗ trợ MySQL RDS

---

# 8. Ví dụ dễ hiểu

Thay vì app lưu:

```text
DB_USER=app_user
DB_PASSWORD=SuperSecret123
```

App sẽ dùng IAM Role của EC2 để tạo token:

```text
mysql -h rds-endpoint \
  -u app_user \
  -p <IAM_AUTH_TOKEN>
```

Token này hết hạn nhanh.

---

# 9. Vì sao các đáp án khác sai?

## ❌ Create IAM Role and assign it to EC2

Chưa đủ.

IAM Role chỉ cấp quyền cho EC2.

Nhưng RDS vẫn chưa biết dùng IAM token nếu chưa bật:

# IAM DB Authentication

=> phải enable IAM DB Authentication.

---

## ❌ IAM + STS

STS tạo temporary AWS credentials.

Nhưng RDS MySQL không dùng STS token trực tiếp để login DB.

RDS cần:

# IAM DB Authentication token

=> STS không phải đáp án chính.

---

## ❌ Configure SSL

SSL chỉ mã hóa traffic:

```text
EC2 ↔ RDS
```

Nó bảo vệ data in transit.

Nhưng không giải quyết yêu cầu:

```text
login bằng authentication token từ IAM
```

---

# 10. Phân biệt nhanh

| Yêu cầu                   | Giải pháp                   |
| ------------------------- | --------------------------- |
| Encrypt connection        | SSL/TLS                     |
| Login RDS bằng token IAM  | IAM DB Authentication       |
| EC2 có quyền AWS          | IAM Role / Instance Profile |
| Temporary AWS credentials | STS                         |
| High availability DB      | Multi-AZ                    |
