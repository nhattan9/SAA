# 1. Dịch câu hỏi

Một công ty có 3 DevOps engineers đang xử lý các quy trình phát triển phần mềm và quản lý hạ tầng.

Một trong các engineer đã vô tình xóa một file được lưu trữ trên Amazon S3, gây gián đoạn dịch vụ.

DevOps engineers có thể làm gì để ngăn việc này xảy ra lần nữa?

---

# 2. Đáp án đúng

✅ Enable S3 Versioning and Multi-Factor Authentication Delete on the bucket.

---

# 3. Phân tích yêu cầu

Đề có keyword rất quan trọng:

---

## Keyword 1

# "accidentally deleted a file"

### Ý nghĩa

Đây là:

# accidental deletion

(tức xóa nhầm)

Không phải:

* hacker
* malware
* permission issue

---

## Keyword 2

# "prevent this from happening again"

### Ý nghĩa

Cần:

* chống xóa nhầm
* hoặc recover được file nếu bị xóa

---

# 4. Vì sao đáp án đúng?

---

# S3 Versioning là gì?

S3 Versioning cho phép:

# lưu nhiều version của cùng 1 object

Ví dụ:

```text id="ed8z9d"
file.txt (v1)
file.txt (v2)
file.txt (v3)
```

---

## Nếu ai đó xóa file?

Thực tế:

```text id="n4l31d"
AWS không xóa hẳn version cũ
```

Nó chỉ thêm:

```text id="cvn0ez"
Delete Marker
```

=> bạn vẫn recover được file.

---

# Ví dụ dễ hiểu

Không bật versioning:

```text id="4r1zhy"
Delete file
   ↓
Mất luôn
```

---

Có versioning:

```text id="6djlwm"
Delete file
   ↓
Version cũ vẫn còn
   ↓
Restore được
```

---

# MFA Delete là gì?

MFA Delete yêu cầu:

# xác thực MFA khi delete object version

Ví dụ:

```text id="6n0m70"
Delete object permanently
    ↓
Nhập MFA code
```

=> giảm nguy cơ xóa nhầm cực mạnh.

---

# MFA Delete bảo vệ gì?

Nó yêu cầu MFA khi:

* permanently delete object version
* thay đổi versioning state

---

# 5. Flow hoạt động

```text id="mf2g7w"
User delete object
      ↓
Versioning giữ version cũ
      ↓
Muốn xóa hẳn
      ↓
Cần MFA
```

---

# 6. Vì sao các đáp án khác sai?

---

## ❌ Signed URL

Signed URL dùng để:

# kiểm soát truy cập tạm thời

Ví dụ:

```text id="32kk9g"
Cho download file trong 10 phút
```

Không liên quan:

* chống delete
* recover file

---

## ❌ IAM bucket policy disable delete

Nghe có vẻ đúng nhưng sai requirement.

Đề cần:

```text id="5f2v8w"
prevent accidental deletion
```

Không phải:

```text id="gkp7m9"
cấm delete hoàn toàn
```

Ngoài ra hệ thống thật vẫn cần delete hợp lệ.

---

## ❌ S3 Infrequent Access

S3 IA chỉ là:

# storage class tiết kiệm chi phí

Không giúp:

* chống delete
* restore file

---
