# 1. Dịch câu hỏi

Một công ty dự định triển khai một web application trên một Auto Scaling Group gồm các Amazon EC2 instances.

Ứng dụng này sẽ được người dùng trên toàn cầu sử dụng để upload và lưu trữ nhiều loại file khác nhau.

Dựa trên xu hướng sử dụng:

> các file cũ hơn 2 năm phải được chuyển sang một storage class khác.

Solutions Architect cần xây dựng một giải pháp:

* tiết kiệm chi phí
* có khả năng mở rộng cao (scalable)
* vẫn đảm bảo durability và high availability cho các file cũ.

Giải pháp nào dưới đây đáp ứng yêu cầu? (Chọn HAI đáp án.)

---

# 2. Phân tích yêu cầu

Đề có nhiều keyword rất quan trọng.

---

## Keyword 1

# "users globally upload and store files"

### Ý nghĩa

User toàn cầu upload file liên tục.

=> cần storage:

* massively scalable
* highly durable
* HA cao
* phù hợp object storage

=> keyword mạnh cho:

# Amazon S3

---

## Keyword 2

# "files older than 2 years"

### Ý nghĩa

Đề đang nói về:

# lifecycle management

Tức là:

```text id="f3d6wj"
File mới → storage thường
File cũ → storage rẻ hơn
```

---

## Keyword 3

# "stored in a different storage class"

### Ý nghĩa

Không phải delete file.

Mà là:

# chuyển storage class

=> cần:

# Lifecycle Policy

---

## Keyword 4

# "cost-effective"

### Ý nghĩa

Cần tối ưu chi phí.

File cũ ít truy cập hơn.

=> nên chuyển sang:

* S3 Standard-IA
* S3 Glacier

---

## Keyword 5

# "scalable"

### Ý nghĩa

Storage phải scale rất lớn.

=> S3 phù hợp nhất.

---

## Keyword 6

# "durability and high availability"

### Ý nghĩa

File vẫn phải an toàn dù là file cũ.

S3 có:

```text id="8d18gb"
11 nines durability
```

=> cực phù hợp.

# 3. Đáp án đúng

---

## ✅ Use Amazon S3 + Lifecycle → S3 Standard-IA

```text id="q6wy32"
S3
  ↓ lifecycle after 2 years
S3 Standard-IA
```

### Vì sao đúng?

S3 Standard-IA:

* rẻ hơn S3 Standard
* dành cho file ít truy cập
* vẫn highly available
* vẫn durable

---

## ✅ Use Amazon S3 + Lifecycle → S3 Glacier

```text id="v4n2x8"
S3
  ↓ lifecycle after 2 years
S3 Glacier
```

### Vì sao đúng?

Glacier:

* cực rẻ
* phù hợp archive lâu dài
* vẫn durable

Đề nói:

```text id="31d53g"
old files
```

=> Glacier rất hợp.

---

# 4. Vì sao các đáp án khác sai?

---

## ❌ Amazon EFS + EFS-IA

Bẫy nằm ở:

```text id="b55mw6"
after 2 years
```

EFS lifecycle policy chỉ hỗ trợ tối đa:

```text id="p0x8m6"
365 days
```

Trong khi:

```text id="yuz91v"
2 years = 730 days
```

=> không đáp ứng yêu cầu.

---

## ❌ Amazon EBS + DLM snapshots

EBS là:

# block storage

Phù hợp:

* attach vào EC2
* OS disk
* database disk

Không phù hợp:

* global file upload
* massively scalable object storage

Ngoài ra:

* đắt hơn S3
* khó share nhiều EC2
* multi-attach phức tạp

---

## ❌ RAID 0 + EBS

RAID chỉ là:

# kỹ thuật gộp disk

Không giải quyết:

* lifecycle transition
* storage class
* scalable object storage

Ngoài ra:

# RAID 0 không có redundancy

```text id="czukls"
1 disk hỏng → mất data
```

=> không phù hợp durability cao.

# 5. Kiến thức cần nhớ

---

# S3 Lifecycle Policy

Cho phép:

```text id="j6tq7p"
Tự động chuyển object sang storage class khác
```

Ví dụ:

```text id="y1x5mw"
30 ngày → Standard-IA
90 ngày → Glacier
```

---

# Mapping nhanh S3 storage classes

| Storage Class        | Use case              |
| -------------------- | --------------------- |
| S3 Standard          | truy cập thường xuyên |
| S3 Standard-IA       | ít truy cập           |
| S3 Glacier           | archive lâu dài       |
| Glacier Deep Archive | lưu cực lâu, cực rẻ   |
