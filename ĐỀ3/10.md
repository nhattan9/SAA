## Câu hỏi 10

### 📋 NỘI DUNG CÂU HỎI

> A company currently has an Augment Reality (AR) mobile game that has a serverless backend. It is using a DynamoDB table which was launched using the **AWS CLI** to store all the user data and a Lambda function to pull the data from DynamoDB. The game is being used by **millions of users each day** to read and store data.

Một công ty có game AR với backend serverless, dùng DynamoDB (tạo bằng **AWS CLI**) và Lambda. Game có hàng triệu người dùng mỗi ngày.

**Câu hỏi:** Thiết kế ứng dụng như thế nào để **cải thiện hiệu năng, tăng khả năng scale và giữ chi phí thấp**? (Chọn HAI đáp án)

---

### 🔑 Từ khóa quyết định đáp án

> "millions of users" + "performance" + "scalable" + "costs low" + "launched using AWS CLI" (Auto Scaling **không được bật mặc định**)

→ Cần: **DAX** (tăng tốc DynamoDB) + **Auto Scaling bật thủ công** + **API Gateway caching** + **DynamoDB global replication**.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng 1:** Enable DynamoDB Accelerator (DAX) and ensure that the Auto Scaling is enabled and increase the maximum provisioned read and write capacity.

- **DAX** là in-memory cache cho DynamoDB — tăng tốc từ milliseconds lên **microseconds**, giảm tải trực tiếp lên DynamoDB.
- **Auto Scaling** giúp tự động điều chỉnh read/write capacity theo traffic — tiết kiệm chi phí hơn so với đặt cố định cao.
- Quan trọng: DynamoDB tạo bằng **AWS CLI mặc định không bật Auto Scaling** — phải bật thủ công.

---

**✅ Đáp án đúng 2:** Use API Gateway in conjunction with Lambda and turn on the caching on frequently accessed data and enable DynamoDB global replication.

- **API Gateway caching** giúp cache response của các request thường xuyên — giảm số lần gọi Lambda và DynamoDB, giảm chi phí và tăng tốc độ phản hồi.
- **DynamoDB global replication** (Global Tables) giúp phân tán dữ liệu gần người dùng ở nhiều region — cải thiện hiệu năng cho game có người dùng toàn cầu.
- Lambda tự động scale không cần cấu hình thêm.

---

**❌ Đáp án sai 1:** Configure CloudFront with DynamoDB as the origin; cache using ElastiCache.

- **CloudFront không thể tích hợp trực tiếp với DynamoDB** làm origin — hai dịch vụ này không tương thích theo cách đó.
- CloudFront dùng để serve static content (HTML, CSS, image), không phải API/database query.

---

**❌ Đáp án sai 2:** Use AWS IAM Identity Center for SSO. Manually set provisioned capacity to higher RCU and WCU.

- **IAM Identity Center** là dịch vụ quản lý access vào nhiều AWS account — không liên quan đến hiệu năng hay scalability của ứng dụng.
- **Đặt capacity cố định cao** tốn chi phí 24/7 dù traffic thấp — không tiết kiệm chi phí như yêu cầu.

---

**❌ Đáp án sai 3:** Auto Scaling is enabled by default; enable DAX to improve from milliseconds to microseconds.

- **SAI** về mặt kỹ thuật: DynamoDB tạo bằng **AWS CLI không tự động bật Auto Scaling**.
- Auto Scaling chỉ được bật mặc định khi tạo DynamoDB qua **AWS Management Console**.
- Đây là bẫy tinh vi trong đề thi — phân biệt CLI vs Console.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Từ khóa trong đề | Giải pháp phù hợp |
|---|---|
| Tăng tốc DynamoDB, milliseconds → microseconds | **DynamoDB DAX** |
| Scale tự động theo traffic, tiết kiệm chi phí | **DynamoDB Auto Scaling** |
| Cache API response, giảm tải Lambda/DynamoDB | **API Gateway Caching** |
| Người dùng toàn cầu, giảm latency theo vùng | **DynamoDB Global Tables** |
| CloudFront + DynamoDB | ❌ Không tương thích trực tiếp |

💡 **Bài học quan trọng từ câu này:**
> **DynamoDB tạo bằng AWS CLI → Auto Scaling KHÔNG được bật mặc định** — phải bật thủ công. Chỉ khi tạo qua Console mới được bật sẵn. Đây là chi tiết hay bị bỏ qua trong đề thi SAA-C03.
>
> **DAX** chỉ hiệu quả cho **read-heavy workload** (đọc nhiều) — phù hợp với game có hàng triệu người chơi đọc dữ liệu liên tục.
