## Câu hỏi 24

### 📋 NỘI DUNG CÂU HỎI

> A company plans to use a **durable storage service** to store on-premises database backups to the AWS cloud. They need a service that can **store and retrieve objects through standard file storage protocols** for quick recovery.

Công ty cần backup dữ liệu on-premises lên AWS cloud. Yêu cầu: dùng **giao thức file storage chuẩn** (NFS/SMB) để lưu và truy xuất dữ liệu, lưu trên storage **bền vững (durable)**.

**Câu hỏi:** Giải pháp nào đáp ứng yêu cầu?

---

### 🔑 Từ khóa quyết định đáp án

> "standard file storage protocols" + "store and retrieve objects" + "durable storage" + "on-premises to AWS"

→ Cần bridge từ on-premises (dùng NFS/SMB) sang AWS S3 (durable) → **AWS Storage Gateway File Gateway**.

---

### 🧠 Kiến thức nền: AWS Storage Gateway

AWS Storage Gateway có 3 loại:

| Loại | Giao thức | Lưu trữ tại | Dùng khi |
|---|---|---|---|
| **File Gateway** | **NFS / SMB** (file protocols) | Amazon S3 | Backup file từ on-premises lên S3 qua giao thức file chuẩn |
| **Volume Gateway** | iSCSI (block protocol) | Amazon S3 (ẩn) / EBS Snapshots | Backup block storage, disaster recovery |
| **Tape Gateway** | iSCSI VTL | S3 / S3 Glacier | Thay thế băng từ vật lý (tape backup) |

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** Use the AWS Storage Gateway **File Gateway** to store all the backup data in Amazon S3.

- File Gateway hiển thị S3 bucket như một **NFS mount point hoặc SMB file share** — ứng dụng on-premises không cần thay đổi gì.
- Dữ liệu được lưu vào **Amazon S3** — durable, highly available.
- Hỗ trợ **standard file storage protocols** (NFS/SMB) đúng yêu cầu.
- Cache locally để truy cập nhanh (low-latency).

---

**❌ Đáp án sai 1:** Use AWS Storage Gateway **Volume Gateway** + Amazon S3 API.

- Volume Gateway dùng giao thức **iSCSI (block protocol)**, không phải file storage protocols (NFS/SMB).
- **Không thể truy cập trực tiếp bằng Amazon S3 API** — Volume Gateway ẩn S3 phía sau, người dùng không thấy.
- Sai giao thức và sai cách truy cập.

---

**❌ Đáp án sai 2:** Use Amazon EBS volumes attached to EC2.

- EBS là **block storage** gắn vào EC2 — không hỗ trợ NFS/SMB trực tiếp.
- EBS **không durable** bằng S3 (single AZ, không có 11 9s durability).
- Phải quản lý EC2 instance → tăng operational overhead.

---

**❌ Đáp án sai 3:** Use AWS Snowball Edge to backup to S3 Glacier.

- **Snowball Edge** là thiết bị vật lý dùng để **di chuyển dữ liệu lớn** (petabyte) — không phải để backup thường xuyên.
- Không hỗ trợ **standard file storage protocols** để store và retrieve objects.
- Không tích hợp trực tiếp với S3 Glacier.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Từ khóa trong đề | Giải pháp |
|---|---|
| "standard file protocols" (NFS/SMB) + on-premises + S3 | **File Gateway** |
| Block storage, iSCSI, disaster recovery | **Volume Gateway** |
| Thay thế tape backup vật lý | **Tape Gateway** |
| Di chuyển dữ liệu lớn (petabyte) một lần | **AWS Snowball** |
| File system dùng chung nhiều EC2 | **Amazon EFS** |

💡 **Công thức ghi nhớ:**
> "Standard file storage protocols" (NFS/SMB) + on-premises → S3 = **File Gateway**.
>
> File Gateway = **cầu nối** biến S3 thành ổ mạng (NFS/SMB) mà on-premises có thể dùng như ổ đĩa thông thường — không cần thay đổi ứng dụng hiện có.
