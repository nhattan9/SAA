## Câu hỏi 23

### 📋 NỘI DUNG CÂU HỎI

> A Solutions Architect is unable to connect to the **newly deployed EC2 instance via SSH** using a home computer. However, the Architect was able to **successfully access other existing instances** in the VPC without any issues.

Architect không SSH được vào **EC2 mới** từ máy tính ở nhà. Nhưng các EC2 cũ trong cùng VPC vẫn truy cập bình thường.

**Câu hỏi:** Cần kiểm tra và sửa điều gì để khôi phục kết nối?

---

### 🔑 Từ khóa quyết định đáp án

> "newly deployed" + "SSH" + "other instances work fine"

→ Vấn đề chỉ xảy ra với **instance mới** — các instance cũ bình thường → NACL của VPC không phải vấn đề (NACL áp dụng cho toàn subnet). Vấn đề nằm ở **Security Group của instance mới** chưa được cấu hình đúng.

---

### 🧠 Kiến thức nền

| Protocol | Port | Dùng để |
|---|---|---|
| **SSH** | **22** | Kết nối remote vào **Linux** EC2 |
| **RDP** | **3389** | Kết nối remote vào **Windows** EC2 |

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** Configure the Security Group of the EC2 instance to permit ingress traffic over **port 22** from your IP.

- SSH dùng **port 22** — cần mở inbound rule cho port này trong Security Group.
- Instance mới được tạo có thể gắn Security Group mặc định không có rule cho port 22.
- Các instance cũ hoạt động bình thường vì Security Group của chúng **đã có** rule port 22.
- Chỉ cần thêm inbound rule: `TCP port 22 from <your IP>` vào Security Group của instance mới.

---

**❌ Đáp án sai 1:** Configure the Network ACL to permit ingress traffic over port 22.

- Các instance **cũ trong cùng subnet** vẫn SSH được bình thường → NACL đang cho phép port 22 rồi.
- NACL áp dụng cho **toàn subnet** — nếu NACL chặn thì không instance nào SSH được cả.
- Vấn đề là **instance-level** (Security Group), không phải **subnet-level** (NACL).

---

**❌ Đáp án sai 2:** Configure the Security Group to permit ingress traffic over **port 3389**.

- **Port 3389 là RDP** — dùng để kết nối vào **Windows** EC2.
- SSH dùng **port 22**, không phải 3389.
- Mở port 3389 không giải quyết được vấn đề SSH.

---

**❌ Đáp án sai 3:** Use Amazon Data Lifecycle Manager.

- **Amazon Data Lifecycle Manager** quản lý vòng đời của EBS Snapshots và AMI — hoàn toàn không liên quan đến kết nối mạng.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Tình huống | Kiểm tra |
|---|---|
| Không SSH được vào **1 instance** cụ thể | **Security Group** của instance đó |
| Không SSH được vào **tất cả instance** trong subnet | **Network ACL** của subnet |
| Không SSH được dù SG và NACL đúng | Public IP, Route Table, Internet Gateway |

| Giao thức | Port | OS |
|---|---|---|
| **SSH** | **22** | Linux |
| **RDP** | **3389** | Windows |
| HTTP | 80 | Web |
| HTTPS | 443 | Web (bảo mật) |
| MySQL | 3306 | Database |

💡 **Công thức ghi nhớ:**
> "Chỉ instance MỚI bị lỗi, instance cũ OK" → vấn đề ở **Security Group của instance mới** (không phải NACL).
>
> SSH = **port 22** (Linux). RDP = **port 3389** (Windows). Nhớ hai port này, hay xuất hiện trong đề thi!
