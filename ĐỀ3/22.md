## Câu hỏi 22

### 📋 NỘI DUNG CÂU HỎI

> A company is building an automation tool for generating custom reports on its AWS usage. The company must be able to **programmatically access** and **forecast usage costs** on specific services.

Công ty đang xây dựng công cụ tự động để tạo báo cáo AWS usage. Yêu cầu: **truy cập dữ liệu chi phí theo chương trình (programmatically)** và **dự báo chi phí** cho các dịch vụ cụ thể.

**Câu hỏi:** Giải pháp nào đáp ứng yêu cầu với **ít operational overhead nhất**?

---

### 🔑 Từ khóa quyết định đáp án

> "programmatically access" + "forecast usage costs" + "least operational overhead"

→ Cần API có thể truy cập và dự báo chi phí theo chương trình → **AWS Cost Explorer API**.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** Use the AWS Cost Explorer API with pagination to programmatically retrieve the usage cost-related data.

- **Cost Explorer API** cho phép truy vấn dữ liệu chi phí và usage **theo chương trình** — không cần thao tác thủ công.
- Hỗ trợ **forecast** (dự báo) chi phí tương lai.
- **Pagination** giúp xử lý tập dữ liệu lớn hiệu quả.
- Fully managed, không cần setup hạ tầng thêm → **least operational overhead**.

---

**❌ Đáp án sai 1:** Utilize downloadable AWS Cost Explorer .csv files + AWS Budgets.

- Tải file .csv là thao tác **thủ công** — không phải programmatic access.
- Phải download, parse, xử lý file → **nhiều operational overhead**.
- Không đáp ứng yêu cầu "programmatically access".

---

**❌ Đáp án sai 2:** Configure AWS Budgets to send data via Amazon SNS.

- **AWS Budgets** chỉ gửi **thông báo khi vượt ngưỡng** budget — không cung cấp dữ liệu chi phí chi tiết.
- SNS chỉ là kênh gửi notification — không phải truy vấn dữ liệu.
- Không đáp ứng yêu cầu truy cập và dự báo chi phí chi tiết.

---

**❌ Đáp án sai 3:** Generate AWS Budgets reports via Amazon SQS.

- **AWS Budgets** chỉ giúp set budget và alert khi vượt ngưỡng — không cho phép truy vấn dữ liệu chi phí chi tiết theo service.
- **SQS** là message queue — không phải công cụ báo cáo chi phí.
- Kết hợp 2 dịch vụ không phù hợp này không giải quyết được yêu cầu.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Dịch vụ | Chức năng chính | Dùng khi |
|---|---|---|
| **AWS Cost Explorer API** | Truy vấn, phân tích, dự báo chi phí **theo chương trình** | Cần programmatic access vào cost data |
| AWS Cost Explorer (Console) | Visualize chi phí qua giao diện đồ họa | Phân tích thủ công, không cần automation |
| AWS Budgets | Set ngưỡng, gửi alert khi vượt budget | Theo dõi và cảnh báo chi phí |
| AWS Cost & Usage Report | Báo cáo chi tiết nhất, xuất ra S3 | Phân tích sâu, data warehousing |

💡 **Công thức ghi nhớ:**
> "Programmatically" + "forecast costs" + "specific services" → **AWS Cost Explorer API**.
>
> Phân biệt nhanh: **Budgets = cảnh báo khi vượt ngưỡng**. **Cost Explorer = phân tích và dự báo chi phí chi tiết**. Hai dịch vụ này hay bị nhầm lẫn trong đề thi.
