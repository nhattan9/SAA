## Câu hỏi 35

### 📋 NỘI DUNG CÂU HỎI

> A Solutions Architect is designing a highly available environment with EC2 instances in an Auto Scaling Group. One condition requires **data stored on root EBS volumes to be preserved** if an instance terminates.

EC2 instances trong Auto Scaling Group. Yêu cầu: khi instance bị terminate, **dữ liệu trên root EBS volume phải được giữ lại**.

**Câu hỏi:** Cần làm gì để đáp ứng yêu cầu?

---

### 🔑 Từ khóa quyết định đáp án

> "root EBS volume" + "preserved" + "instance terminates"

→ Mặc định root EBS volume bị **xóa khi instance terminate** → cần thay đổi thuộc tính **DeleteOnTermination = False**.

---

### 🧠 Kiến thức nền: EBS DeleteOnTermination

| Loại EBS Volume | Hành vi mặc định khi terminate |
|---|---|
| **Root volume** | **Bị xóa** (DeleteOnTermination = **True**) |
| **Additional volumes** (gắn thêm) | **Giữ lại** (DeleteOnTermination = **False**) |

→ Muốn giữ lại root volume: đổi `DeleteOnTermination = False` trên root volume.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** Set the value of **DeleteOnTermination** attribute of the EBS volumes to **False**.

- Thay đổi thuộc tính này → root volume **không bị xóa** khi instance terminate.
- Có thể cấu hình lúc launch instance qua Console hoặc CLI.
- Đơn giản, trực tiếp, đúng mục đích.

---

**❌ Đáp án sai 1:** Use AWS DataSync to replicate root volume data to Amazon S3.

- **AWS DataSync** hoạt động với NFS, SMB, S3, EFS, FSx — **không hỗ trợ EBS volumes**.
- Không giải quyết được yêu cầu preserve root EBS data.

---

**❌ Đáp án sai 2:** Configure ASG to suspend the health check process.

- Suspend health check → ASG **không thể phát hiện và thay thế** instance unhealthy.
- Làm giảm tính **high availability** — ngược lại với mục tiêu thiết kế.
- Không liên quan đến việc preserve EBS data.

---

**❌ Đáp án sai 3:** Enable Termination Protection for all EC2 instances.

- **Termination Protection** chỉ ngăn instance bị terminate **thủ công** qua Console.
- **Không ngăn** Auto Scaling terminate instance khi scale in hoặc replace unhealthy instance.
- Không giải quyết được yêu cầu preserve EBS data.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Thuộc tính | Mặc định | Ý nghĩa |
|---|---|---|
| Root EBS `DeleteOnTermination` | **True** (xóa khi terminate) | Đổi thành False để giữ lại |
| Additional EBS `DeleteOnTermination` | **False** (giữ lại) | Mặc định đã an toàn |

| Tính năng | Chức năng thực sự |
|---|---|
| **DeleteOnTermination = False** | Giữ root EBS sau khi instance terminate |
| **Termination Protection** | Ngăn terminate thủ công qua Console (không ngăn ASG) |
| **ASG suspend health check** | Tắt tính năng replace unhealthy instance |

💡 **Công thức ghi nhớ:**
> **Root EBS mặc định = bị xóa khi terminate** → muốn giữ lại → **DeleteOnTermination = False**.
>
> **Termination Protection ≠ giữ EBS data** — Termination Protection chỉ bảo vệ instance khỏi bị tắt thủ công, không bảo vệ dữ liệu. Đây là bẫy hay gặp trong đề thi!
