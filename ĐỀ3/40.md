## Câu hỏi 40

### 📋 NỘI DUNG CÂU HỎI

> A company stores financial reports in S3. They need to **track new objects added, objects removed**, and **versioned objects permanently deleted**. Must publish notifications to a **queue** (post-processing) and an **SNS topic** (notify Operations team).

Cần theo dõi: **thêm object mới**, **xóa object**, và **xóa vĩnh viễn versioned object**. Gửi thông báo đến **SQS** và **SNS**.

**Câu hỏi:** Giải pháp nào phù hợp nhất?

---

### 🔑 Từ khóa quyết định đáp án

> "new objects added" + "removed" + "versioned object permanently deleted" + "queue" + "SNS topic"

→ Cần đúng **event types** của S3 + đúng **destination** (SQS + SNS, không phải Amazon MQ).

---

### 🧠 Kiến thức nền: S3 Event Notification

**S3 Event Types quan trọng:**

| Event Type | Khi nào trigger |
|---|---|
| `s3:ObjectCreated:*` | Khi object **mới được tạo/upload** |
| `s3:ObjectRemoved:Delete` | Khi object bị **xóa hoàn toàn** (kể cả xóa vĩnh viễn versioned object) |
| `s3:ObjectRemoved:DeleteMarkerCreated` | Khi **delete marker được tạo** cho versioned object (chỉ đánh dấu xóa, chưa xóa thật) |

**S3 Event Destinations được hỗ trợ:**
- ✅ **Amazon SNS**
- ✅ **Amazon SQS**
- ✅ **AWS Lambda**
- ❌ **Amazon MQ** — **KHÔNG được hỗ trợ**

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** SNS + SQS. Events: `s3:ObjectCreated:*` + `s3:ObjectRemoved:Delete`

- **SQS** → queue cho post-processing ✅
- **SNS** → notify Operations team ✅
- `s3:ObjectCreated:*` → track tất cả object mới được tạo ✅
- `s3:ObjectRemoved:Delete` → track object bị xóa VÀ versioned object bị xóa vĩnh viễn ✅

---

**❌ Đáp án sai 1:** SNS + SQS. Events: `s3:ObjectCreated:*` + `s3:ObjectRemoved:DeleteMarkerCreated`

- `DeleteMarkerCreated` chỉ trigger khi **delete marker được tạo** — tức là chỉ đánh dấu xóa trên versioned object, **không track việc xóa vĩnh viễn**.
- Không đủ để track "permanently deleted versioned object".

---

**❌ Đáp án sai 2:** SNS + **Amazon MQ**. Events: `s3:ObjectAdded:*` + `s3:ObjectRemoved:*`

- **Amazon MQ không được S3 hỗ trợ** làm event destination.
- **`s3:ObjectAdded:*`** không tồn tại — tên đúng là `s3:ObjectCreated:*`.

---

**❌ Đáp án sai 3:** SNS + **Amazon MQ**. Events: `s3:ObjectCreated:*` + `DeleteMarkerCreated`

- **Amazon MQ không được hỗ trợ** bởi S3 event notifications.
- `DeleteMarkerCreated` không đủ để track xóa vĩnh viễn.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| S3 Event | Ý nghĩa |
|---|---|
| `s3:ObjectCreated:*` | Object mới được tạo (upload, copy, multipart...) |
| `s3:ObjectRemoved:Delete` | Object bị **xóa thật sự** (kể cả permanently delete versioned object) |
| `s3:ObjectRemoved:DeleteMarkerCreated` | Chỉ tạo delete marker — **chưa xóa thật**, chỉ đánh dấu |

| S3 Event Destination | Hỗ trợ? |
|---|---|
| Amazon SNS | ✅ |
| Amazon SQS | ✅ |
| AWS Lambda | ✅ |
| Amazon MQ | ❌ |

💡 **Công thức ghi nhớ:**
> S3 Event destinations chỉ có **3 loại: SNS, SQS, Lambda** — Amazon MQ không được hỗ trợ, đây là bẫy hay gặp.
>
> `ObjectCreated` ≠ `ObjectAdded` — tên đúng là **ObjectCreated**.
> `ObjectRemoved:Delete` = xóa thật. `ObjectRemoved:DeleteMarkerCreated` = chỉ đánh dấu xóa (chưa xóa thật sự).
