## Câu hỏi 49

### 📋 NỘI DUNG CÂU HỎI

> A web-based application must extract specific data from a large CSV file stored in Amazon S3. The application requires filtering or transforming the data dynamically before returning it, ensuring that only the needed data is returned without downloading the entire object.

Ứng dụng cần lọc/transform dữ liệu từ file CSV lớn trên S3 một cách động, chỉ trả về dữ liệu cần thiết mà không tải toàn bộ file.

---

### 🔑 Từ khóa quyết định đáp án

> **"filter or transform dynamically"** + **"without downloading entire object"** + **"S3 Object Lambda"** → cần xác định Lambda function xử lý dựa trên tham số nào để định danh chính xác file cần đọc.

---

### 🧠 Kiến thức nền: S3 Object Lambda

S3 Object Lambda cho phép gắn một Lambda function vào S3 GET request. Khi ứng dụng gọi GET object, Lambda function sẽ chặn request, xử lý/biến đổi dữ liệu, rồi trả về kết quả đã được lọc — tất cả diễn ra trong thời gian thực, không cần tạo bản sao dữ liệu mới.

| Thành phần | Vai trò |
|---|---|
| Bucket name | Xác định nơi lưu trữ (container) |
| Object key | Xác định chính xác file cụ thể trong bucket |
| Lambda function | Xử lý, lọc, transform dữ liệu on-the-fly |

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án đúng: S3 Object Lambda với Lambda function xử lý dựa trên bucket name và object key**

Bucket name xác định bucket chứa file. Object key (tên/đường dẫn file) xác định chính xác file CSV cần đọc. Hai tham số này kết hợp là đủ và cần thiết để Lambda truy cập và xử lý đúng file.

---

❌ **Bucket name + object metadata**

Metadata chỉ chứa thông tin mô tả về object (kích thước, ngày tạo, content-type...), không liên quan đến nội dung dữ liệu bên trong file. Không dùng để lọc dữ liệu CSV.

---

❌ **Bucket name + object tags**

Tags dùng để phân loại, quản lý chi phí, hoặc kiểm soát quyền truy cập. Không liên quan đến nội dung file hay logic lọc dữ liệu động.

---

❌ **Chỉ bucket name**

Bucket name chỉ xác định được container, không đủ để truy cập một file cụ thể khi bucket có thể chứa hàng nghìn file. Không thể xử lý đúng file nếu thiếu object key.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Tham số | Ý nghĩa | Cần thiết? |
|---|---|---|
| Bucket name | Xác định bucket | ✅ |
| Object key | Xác định file cụ thể | ✅ |
| Object metadata | Thông tin mô tả file | ❌ Không đủ |
| Object tags | Nhãn phân loại | ❌ Không liên quan |

💡 **Công thức ghi nhớ:**

> Để truy cập bất kỳ object nào trong S3, luôn cần đủ 2 thứ: **bucket name** (nơi chứa) + **object key** (tên file). Thiếu một trong hai đều không đủ để định danh chính xác file.
>
> S3 Object Lambda = giải pháp transform data **on-the-fly** khi GET request, không cần tạo bản sao mới.
