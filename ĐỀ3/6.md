## Câu hỏi 6

### 📋 NỘI DUNG CÂU HỎI

> A company hosted a web application on a Linux Amazon EC2 instance in the public subnet that uses a non-default network ACL. The instance uses a default security group and has an attached Elastic IP address. The network ACL is configured to block all inbound and outbound traffic. The Solutions Architect must allow incoming traffic on port 443 to access the application from any source.

Một công ty host ứng dụng web trên EC2 (Linux) trong public subnet, sử dụng **non-default Network ACL** đang chặn toàn bộ traffic vào và ra. Instance dùng **default security group** và có gắn Elastic IP. Solutions Architect cần cho phép traffic vào trên **port 443** từ mọi nguồn.

**Câu hỏi:** Kết hợp các bước nào sẽ đáp ứng yêu cầu này? (Chọn HAI đáp án)

---

### 🔑 Từ khóa quyết định đáp án

> "non-default Network ACL" (stateless) + "default Security Group" (stateful) + "port 443" + "Linux instance" + "ephemeral ports"

→ Cần hiểu rõ sự khác biệt giữa **Security Group (stateful)** và **Network ACL (stateless)** để chọn đúng.

---

### 📚 Kiến thức nền cần nhớ

**Security Group — Stateful:**
- Khi cho phép inbound → outbound response **tự động được cho phép**, không cần tạo thêm rule outbound.
- Chỉ cần tạo **1 inbound rule** là đủ.

**Network ACL — Stateless:**
- Inbound và outbound **hoàn toàn độc lập** — phải tạo rule cho **cả hai chiều** thủ công.
- Khi client kết nối vào server (port 443), server phản hồi qua **ephemeral port** ngẫu nhiên (32768–65535 trên Linux).
- Do đó phải mở cả **inbound port 443** và **outbound ephemeral ports 32768–65535**.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng 1:** In the Security Group, add a new rule to allow TCP connection on port 443 from source 0.0.0.0/0.

- Security Group là **stateful** → chỉ cần thêm inbound rule cho port 443, outbound response tự động được cho phép.
- Default security group không có sẵn rule inbound cho port 443 → cần thêm.

---

**✅ Đáp án đúng 2:** In the Network ACL, update the rule to allow inbound TCP on port 443 from 0.0.0.0/0 AND outbound TCP on port 32768–65535 to 0.0.0.0/0.

- Network ACL là **stateless** → phải cấu hình cả hai chiều thủ công.
- **Inbound port 443:** Nhận request từ client.
- **Outbound port 32768–65535:** Phản hồi về client qua ephemeral port (Linux kernel dùng range này).

---

**❌ Đáp án sai 1:** In the Security Group, create a new rule to allow TCP on port 443 to destination 0.0.0.0/0.

- Đây là **outbound rule** — cho phép EC2 kết nối ra ngoài Internet, không phải nhận kết nối vào.
- Default security group **đã có sẵn outbound rule cho phép tất cả traffic** → tạo thêm là thừa.
- Sai chiều: đề yêu cầu allow **incoming** traffic, không phải outgoing.

---

**❌ Đáp án sai 2:** In the Network ACL, allow both inbound AND outbound TCP on port 443.

- Thiếu outbound **ephemeral ports (32768–65535)**. Server phản hồi client không qua port 443 mà qua ephemeral port ngẫu nhiên.
- Mở outbound port 443 không giúp ích gì cho việc phản hồi request từ client.

---

**❌ Đáp án sai 3:** In the Network ACL, update the rule to allow only outbound TCP on port 32768–65535.

- Chỉ mở outbound ephemeral ports mà **không mở inbound port 443** → request không vào được server.
- Thiếu một nửa: cần cả inbound 443 lẫn outbound 32768–65535.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Thành phần | Stateful/Stateless | Cần cấu hình |
|---|---|---|
| Security Group | **Stateful** | Chỉ cần inbound rule — outbound tự động |
| Network ACL | **Stateless** | Phải tạo cả inbound lẫn outbound thủ công |

| Chiều traffic | Port |
|---|---|
| Client → Server (inbound) | **443** |
| Server → Client (outbound response) | **Ephemeral: 32768–65535** (Linux) |

💡 **Công thức ghi nhớ:**
> **Security Group = stateful** → chỉ cần 1 chiều.
> **Network ACL = stateless** → phải mở **2 chiều**: inbound port dịch vụ (443) + outbound ephemeral ports (32768–65535).
>
> Ephemeral ports là các port tạm thời mà client dùng để nhận phản hồi — **không bao giờ mở outbound port 443 để phản hồi client**, vì phản hồi đi qua ephemeral port, không phải port 443.
