## Câu hỏi 14

### 📋 NỘI DUNG CÂU HỎI

> A Solutions Architect for a global news company is configuring a fleet of EC2 instances in a subnet in a VPC with an Internet gateway attached. All existing EC2 instances can be accessed from the Internet. The architect launches **another subnet** and deploys an EC2 instance in it, however, the architect is **not able to access the EC2 instance from the Internet**.

Một VPC đã có Internet Gateway. Các EC2 trong subnet cũ truy cập Internet bình thường. Architect tạo **subnet mới**, deploy EC2 vào đó nhưng **không thể truy cập từ Internet**.

**Câu hỏi:** Đâu là nguyên nhân có thể? (Chọn HAI đáp án)

---

### 🔑 Từ khóa quyết định đáp án

> "new subnet" + "cannot access from Internet" + "Internet gateway already attached"

→ Internet Gateway đã có sẵn, vấn đề nằm ở **subnet mới** — cần kiểm tra 2 thứ: **Public IP** và **Route Table**.

---

### 🧠 Checklist khi EC2 không truy cập được từ Internet

Theo thứ tự cần kiểm tra:
1. EC2 có **Public IP hoặc Elastic IP** không?
2. **Route Table** của subnet có route đến Internet Gateway không?
3. **Security Group** có cho phép inbound traffic không?
4. **Network ACL** có chặn traffic không?

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng 1:** The Amazon EC2 instance does not have a public IP address associated with it.

- Subnet mới tạo mặc định **không tự động gán Public IP** cho EC2 — phải bật tính năng "Auto-assign public IPv4" cho subnet hoặc gán thủ công khi launch.
- Không có Public IP → không thể reach từ Internet dù route table đúng.

---

**✅ Đáp án đúng 2:** The route table is not configured properly to send traffic to the Internet through the Internet gateway.

- Subnet mới khi tạo **được gắn với main route table** của VPC — nếu main route table không có route đến Internet Gateway thì subnet đó là **private subnet**.
- Cần thêm route: `0.0.0.0/0 → igw-xxxxxxxx` vào route table của subnet mới.
- Đây là nguyên nhân phổ biến nhất khi tạo subnet mới mà quên cấu hình route.

---

**❌ Đáp án sai 1:** The Amazon EC2 instance does not have an attached Elastic Fabric Adapter (EFA).

- **EFA** (Elastic Fabric Adapter) là network adapter chuyên dụng cho **HPC và Machine Learning** — tăng tốc ứng dụng tính toán cao.
- Hoàn toàn không liên quan đến khả năng truy cập Internet của EC2.

---

**❌ Đáp án sai 2:** The Amazon EC2 instance is not a member of the same Auto Scaling group.

- **Auto Scaling Group** quản lý việc scale in/out số lượng instance — không ảnh hưởng gì đến kết nối Internet.

---

**❌ Đáp án sai 3:** The route table is not configured properly to send traffic through the **customer gateway (CGW)**.

- **Customer Gateway (CGW)** dùng trong **VPN connection** để kết nối on-premises với AWS — không dùng để kết nối Internet.
- Gateway đúng cho Internet phải là **Internet Gateway (IGW)**, không phải CGW.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Kiểm tra | Vấn đề | Giải pháp |
|---|---|---|
| Public IP | EC2 không có public IP | Bật auto-assign IP cho subnet hoặc gán Elastic IP |
| Route Table | Không có route `0.0.0.0/0 → IGW` | Thêm route đến Internet Gateway |
| Security Group | Chặn inbound port cần thiết | Thêm inbound rule phù hợp |
| Network ACL | Chặn traffic | Thêm allow rule cho inbound/outbound |

| Gateway | Dùng khi |
|---|---|
| **Internet Gateway (IGW)** | Kết nối EC2 ra **Internet** |
| **Customer Gateway (CGW)** | Kết nối **on-premises** qua VPN |
| **Virtual Private Gateway (VGW)** | Phía AWS của VPN connection |
| **NAT Gateway** | Private subnet ra Internet (1 chiều) |

💡 **Công thức ghi nhớ:**
> EC2 không truy cập được từ Internet → kiểm tra ngay **2 thứ**: **Public IP** và **Route Table có route đến IGW** không.
>
> Subnet mới tạo luôn đi kèm **main route table** — nếu main route table không có route IGW thì subnet đó mặc định là **private**.
