## Câu hỏi 21

### 📋 NỘI DUNG CÂU HỎI

> A FinTech startup deployed an application on an Amazon EC2 instance with **attached Instance Store volumes** and an **Elastic IP address**. The server is stopped from 6 PM to 8 AM for cost efficiency using Lambda.

EC2 instance có gắn **Instance Store volumes** và **Elastic IP**. Instance bị **stop và start** mỗi ngày.

**Câu hỏi:** Điều gì sẽ xảy ra khi EC2 bị stop và start lại? (Chọn HAI đáp án)

---

### 🔑 Từ khóa quyết định đáp án

> "Instance Store volumes" + "stop and start" + "Elastic IP"

→ Cần nhớ hành vi của **Instance Store** và **Elastic IP** khi EC2 bị stop/start.

---

### 🧠 Kiến thức nền cần nhớ

**Instance Store:**
- Là storage **vật lý gắn trực tiếp** vào host server của EC2.
- Dữ liệu **bị xóa hoàn toàn** khi instance bị **stop, terminate, hoặc host thay đổi**.
- Chỉ giữ nguyên khi **reboot** (khởi động lại mà không đổi host).

**Elastic IP (EIP):**
- EIP **vẫn gắn với instance** dù stop/start — không bị tách ra.
- EIP chỉ bị tính phí khi **không gắn vào instance đang chạy**.

**Underlying Host:**
- Khi EC2 bị stop, AWS **có thể chuyển instance sang host vật lý khác**.
- Instance có thể ở lại host cũ nếu host không có vấn đề, nhưng **không đảm bảo**.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng 1:** All data on the attached instance-store devices will be lost.

- Instance Store là storage **tạm thời**, gắn với host vật lý.
- Khi EC2 stop → host có thể thay đổi → Instance Store của host cũ **bị xóa sạch**.
- Đây là đặc tính cơ bản nhất của Instance Store — **không bao giờ dùng Instance Store để lưu dữ liệu quan trọng**.

---

**✅ Đáp án đúng 2:** The underlying host for the instance is possibly changed.

- Khi stop một EC2, AWS **có thể di chuyển** instance sang host vật lý khác.
- Không đảm bảo ở lại host cũ — đó là lý do dữ liệu Instance Store bị mất.
- Từ "possibly" (có thể) là đúng — không phải lúc nào cũng đổi host, nhưng **có khả năng**.

---

**❌ Đáp án sai 1:** The ENI (Elastic Network Interface) is detached.

- **ENI vẫn gắn** với instance khi stop — không bị tháo ra.
- Private IP của ENI cũng được giữ nguyên.

---

**❌ Đáp án sai 2:** The Elastic IP address is disassociated with the instance.

- **EIP vẫn gắn** với EC2-VPC instance khi stop/start — không bị tháo.
- EIP chỉ bị tháo tự động với **EC2-Classic** (loại cũ, hiện không còn dùng).

---

**❌ Đáp án sai 3:** There will be no changes.

- Sai hoàn toàn — có nhiều thay đổi: host có thể thay đổi, Instance Store bị xóa, public IP (không phải EIP) có thể thay đổi.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Thành phần | Khi Stop/Start | Khi Reboot |
|---|---|---|
| **Instance Store data** | ❌ **Bị xóa** | ✅ Giữ nguyên |
| **EBS data** | ✅ Giữ nguyên | ✅ Giữ nguyên |
| **Elastic IP** | ✅ Vẫn gắn | ✅ Vẫn gắn |
| **Public IP (không phải EIP)** | ❌ Thay đổi | ✅ Giữ nguyên |
| **Private IP** | ✅ Giữ nguyên | ✅ Giữ nguyên |
| **ENI** | ✅ Vẫn gắn | ✅ Vẫn gắn |
| **Underlying Host** | ⚠️ Có thể thay đổi | ✅ Giữ nguyên |

💡 **Công thức ghi nhớ:**
> **Instance Store** = "bộ nhớ tạm" — stop hoặc terminate là **mất sạch**. Chỉ dùng cho cache, temp files, không bao giờ dùng cho dữ liệu quan trọng.
>
> **EIP** = gắn cố định, stop/start không mất. **Public IP thường** = thay đổi mỗi lần stop/start → dùng EIP nếu cần IP cố định.
