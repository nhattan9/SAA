## Câu hỏi 28

### 📋 NỘI DUNG CÂU HỎI

> A new online banking platform uses **microservices architecture** with **Kubernetes**. The new solution should: **remove the need to provision and manage servers**, let you **pay for resources per application**, and **improve security through application isolation by design**.

Nền tảng ngân hàng dùng **Kubernetes**, kiến trúc microservices. Yêu cầu: **serverless** (không quản lý server), **trả tiền theo từng app**, **bảo mật qua application isolation**.

**Câu hỏi:** Giải pháp nào phù hợp nhất?

---

### 🔑 Từ khóa quyết định đáp án

> "Kubernetes" + "no provision/manage servers" + "pay per application" + "application isolation by design"

→ Đây là mô tả chính xác của **AWS Fargate trên Amazon EKS**.

---

### 🧠 Kiến thức nền

| Dịch vụ | Dùng cho | Serverless? |
|---|---|---|
| **Amazon EKS** | Chạy **Kubernetes** managed trên AWS | ❌ (cần quản lý node) |
| **Amazon ECS** | Chạy **Docker containers** (không phải Kubernetes) | ❌ (cần quản lý node) |
| **AWS Fargate** | Serverless compute engine cho **ECS và EKS** | ✅ Fully serverless |
| **EKS + Fargate** | Kubernetes **không cần quản lý server** | ✅ Fully serverless |

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** Use AWS Fargate on Amazon EKS with Service Auto Scaling.

- **EKS** = managed Kubernetes → đúng yêu cầu "dùng Kubernetes".
- **Fargate** = serverless compute → không cần provision/manage server.
- **Pay per application** → Fargate tính phí theo resource của từng pod/task, không trả tiền server nhàn rỗi.
- **Application isolation by design** → mỗi pod chạy trong **kernel riêng biệt** → bảo mật tốt hơn.
- **Service Auto Scaling** → tự động scale theo nhu cầu.

---

**❌ Đáp án sai 1:** Use Amazon ECS to run the Kubernetes cluster on AWS Fargate.

- **ECS không chạy Kubernetes** — ECS dùng cho Docker containers với task definition riêng.
- Muốn dùng Kubernetes trên AWS phải dùng **EKS**, không phải ECS.
- Đây là lỗi nhầm lẫn ECS và EKS — hay gặp trong đề thi.

---

**❌ Đáp án sai 2:** Deploy an Amazon EKS Cluster on AWS Outposts.

- **AWS Outposts** là rack server vật lý AWS đặt tại **on-premises data center** của khách hàng.
- Phải tự duy trì phần cứng on-premises → **không phải serverless**.
- **Amazon AppFlow** dùng để tích hợp SaaS bên thứ ba (Salesforce, SAP...) vào AWS — không liên quan đến Kubernetes pods.

---

**❌ Đáp án sai 3:** Host the application in Amazon EMR Serverless.

- **Amazon EMR** dùng cho **big data processing** (Hadoop, Spark) — không phải cho Kubernetes microservices.
- Cần cấu hình rất nhiều để chạy Kubernetes trên EMR — không phù hợp và phức tạp không cần thiết.
- EBS fast snapshot restore không liên quan đến yêu cầu bài toán.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Từ khóa | Dịch vụ |
|---|---|
| **Kubernetes** + serverless | **EKS + Fargate** |
| **Docker containers** + serverless | **ECS + Fargate** |
| Kubernetes on-premises / hybrid | EKS + AWS Outposts |
| Big data, Hadoop, Spark | Amazon EMR |
| Tích hợp SaaS bên thứ ba | Amazon AppFlow |

💡 **Công thức ghi nhớ:**
> **Kubernetes = EKS** (không phải ECS).
> **Serverless containers = Fargate** (dùng được cho cả ECS lẫn EKS).
> **EKS + Fargate** = Kubernetes serverless hoàn toàn — không cần quản lý node, trả tiền theo pod, isolation theo kernel.
>
> Nhớ: **ECS ≠ EKS** — ECS dùng Docker task definition, EKS dùng Kubernetes manifest. Đây là bẫy hay gặp nhất trong đề thi SAA-C03!
