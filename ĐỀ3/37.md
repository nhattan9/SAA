## Câu hỏi 37

### 📋 NỘI DUNG CÂU HỎI

> A company wants to upload confidential data from on-premises to Amazon S3. The encryption **must be done BEFORE sending to S3**. The company must **manage and rotate the encryption keys** themselves and **control who can access those keys**.

Công ty cần mã hóa dữ liệu **trước khi gửi lên S3** (không để S3 tự mã hóa). Phải **tự quản lý, rotate và kiểm soát** encryption keys.

**Câu hỏi:** Phương pháp nào đáp ứng yêu cầu? (Chọn HAI đáp án)

---

### 🔑 Từ khóa quyết định đáp án

> "encrypt BEFORE sending to S3" + "manage and rotate keys" + "control who can access keys"

→ Phải dùng **Client-Side Encryption** (mã hóa phía client trước khi upload). Server-Side Encryption bị loại vì S3 mã hóa sau khi nhận data.

---

### 🧠 Kiến thức nền: S3 Encryption Options

**Server-Side Encryption (SSE)** — S3 mã hóa SAU khi nhận data:
| Loại | Key quản lý bởi |
|---|---|
| SSE-S3 | AWS tự quản lý |
| SSE-KMS | AWS KMS (bạn kiểm soát được) |
| SSE-C | Bạn tự cung cấp key |

**Client-Side Encryption (CSE)** — Bạn mã hóa TRƯỚC khi upload:
| Loại | Key quản lý bởi |
|---|---|
| **CSE với AWS KMS key** | AWS KMS — bạn kiểm soát, rotate, audit |
| **CSE với Client-Side Master Key** | Bạn tự quản lý hoàn toàn |

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng 1:** Set up Client-Side Encryption with **AWS KMS key**.

- Data được mã hóa **tại máy client** trước khi gửi lên S3 → đúng yêu cầu.
- KMS key do bạn **kiểm soát, rotate, và audit** → đúng yêu cầu quản lý key.
- Có thể kiểm soát **ai được dùng key** qua IAM policy và KMS key policy.

---

**✅ Đáp án đúng 2:** Set up Client-Side Encryption using a **client-side master key**.

- Data được mã hóa **tại máy client** trước khi gửi lên S3 → đúng yêu cầu.
- Key do **bạn tự quản lý hoàn toàn** — rotate, lưu trữ, kiểm soát access.
- AWS không bao giờ biết key này — bảo mật tối đa.

---

**❌ Đáp án sai 1:** Set up Client-Side Encryption with **S3 managed encryption keys**.

- **S3 managed key** là key do **AWS tự quản lý** — bạn không thể tự rotate hay kiểm soát.
- Không đáp ứng yêu cầu "must spend time managing and rotating keys".
- Về mặt kỹ thuật, S3 managed key dùng cho **Server-Side Encryption**, không phải Client-Side.

---

**❌ Đáp án sai 2:** Set up Server-Side Encryption with keys stored in a separate S3 bucket.

- **Server-Side Encryption** → S3 mã hóa SAU khi nhận data — sai yêu cầu "encrypt before sending".
- **Lưu encryption key trong S3 bucket** là **rủi ro bảo mật nghiêm trọng** — không được khuyến nghị.

---

**❌ Đáp án sai 3:** Set up Server-Side Encryption (SSE) with Amazon EC2 key pair.

- **EC2 key pair** dùng để **SSH vào EC2**, không phải để mã hóa S3 data.
- Không thể dùng EC2 key pair để mã hóa S3 bucket.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Yêu cầu | Giải pháp |
|---|---|
| Mã hóa **trước khi gửi** lên S3 | **Client-Side Encryption** |
| Mã hóa sau khi lên S3, AWS quản lý key | SSE-S3 |
| Mã hóa sau khi lên S3, bạn kiểm soát key qua KMS | SSE-KMS |
| Tự quản lý key hoàn toàn, AWS không biết | CSE với Client-Side Master Key |
| Quản lý key qua KMS, có audit trail | CSE với AWS KMS key |

💡 **Công thức ghi nhớ:**
> "Encrypt BEFORE sending to S3" → **Client-Side Encryption** (CSE).
> "Encrypt AFTER reaching S3" → **Server-Side Encryption** (SSE).
>
> "Manage and rotate keys yourself" → **KMS key** (có kiểm soát) hoặc **Client-Side Master Key** (tự quản lý hoàn toàn). Không phải S3 managed key (AWS tự lo).
