## Câu hỏi 33

### 📋 NỘI DUNG CÂU HỎI

> You are automating the creation of EC2 instances in your VPC using a Python script to trigger Amazon EC2 API to request **50 EC2 instances** in a single Availability Zone. However, after **20 successful requests**, subsequent requests **failed**.

Bạn dùng script Python gọi EC2 API để tạo 50 instances trong 1 AZ. Sau 20 requests thành công, các requests tiếp theo bị thất bại.

**Câu hỏi:** Nguyên nhân là gì và cách giải quyết?

---

### 🔑 Từ khóa quyết định đáp án

> "20 successful" + "subsequent requests failed" + "50 instances"

→ AWS có **giới hạn số vCPU cho On-Demand instances theo region** — sau khi đạt giới hạn thì request mới bị từ chối.

---

### 🧠 Kiến thức nền: AWS Service Limits

AWS đặt **giới hạn mặc định** cho số lượng tài nguyên bạn có thể tạo để tránh việc vô tình hoặc cố ý dùng quá nhiều tài nguyên:

| Loại instance | Giới hạn | Phạm vi |
|---|---|---|
| **On-Demand Instances** | Theo **vCPU** | Theo **Region** |
| Reserved Instances | 20 instances | Theo Region |
| Spot Instances | Dynamic limit | Theo Region |

**Quan trọng:**
- Giới hạn tính theo **Region**, không phải theo AZ.
- Giới hạn có thể **tăng lên** bằng cách submit form yêu cầu lên AWS.
- Account mới thường có giới hạn **thấp hơn** account cũ.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** There is a vCPU-based On-Demand Instance limit per region. Submit the limit increase form to AWS and retry once approved.

- Sau 20 instances thành công → đã đạt giới hạn vCPU On-Demand của region đó.
- **Không cần đổi region hay AZ** — chỉ cần **xin tăng limit** với AWS.
- AWS sẽ xem xét và approve trong vài giờ đến vài ngày.

---

**❌ Đáp án sai 1:** Maximum 20 instances per AZ. Select a different AZ.

- **Giới hạn tính theo Region, không phải AZ** — chuyển AZ vẫn cùng region → vẫn bị limit.
- Đổi AZ không giải quyết được vấn đề.

---

**❌ Đáp án sai 2:** Maximum 20 instances per region. Select a different region.

- Giới hạn đúng là theo region, nhưng **không cần đổi region** — có thể xin tăng limit.
- Đổi region là giải pháp không hiệu quả, tạo phức tạp không cần thiết.

---

**❌ Đáp án sai 3:** There was an issue with the Amazon EC2 API. Just resend the requests.

- Không phải lỗi API — đây là **service limit** của AWS, resend vẫn thất bại.
- Phải xin tăng limit trước mới có thể tạo thêm instances.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Giới hạn EC2 | Phạm vi | Giải pháp khi đạt limit |
|---|---|---|
| **vCPU-based On-Demand limit** | **Theo Region** | Submit limit increase request lên AWS |
| Reserved Instances | Theo Region | Submit limit increase request |
| Spot Instances | Dynamic, theo Region | Submit limit increase request |

💡 **Công thức ghi nhớ:**
> EC2 limit = **per Region** (không phải per AZ, không phải per account global).
>
> Khi bị **throttle hoặc limit** trong AWS → **không đổi region, không đổi AZ** → **submit Service Limit Increase request** lên AWS Support.
>
> Đây áp dụng cho nhiều dịch vụ khác: Lambda concurrent executions, RDS instances, VPC per region... đều có thể xin tăng limit.
