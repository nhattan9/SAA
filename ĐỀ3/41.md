## Câu hỏi 41

### 📋 NỘI DUNG CÂU HỎI

> A company requires all AWS resources to be **tagged with a standard naming convention** for better access control. The solutions architect must implement a solution that **checks for untagged AWS resources**.

Công ty yêu cầu tất cả AWS resources phải được **gắn tag theo chuẩn**. Cần giải pháp **kiểm tra resources không có tag** hoặc tag không đúng chuẩn.

**Câu hỏi:** Giải pháp nào cần **ít effort nhất** để triển khai?

---

### 🔑 Từ khóa quyết định đáp án

> "check for untagged resources" + "least amount of effort"

→ Cần giải pháp **managed, sẵn có, không cần code** → **AWS Config rule** có sẵn managed rule `required-tags`.

---

### 🧠 Kiến thức nền: AWS Config

**AWS Config** là dịch vụ theo dõi và đánh giá **cấu hình AWS resources** theo thời gian.

- Có sẵn hàng trăm **managed rules** — chỉ cần bật lên, không cần code.
- Rule **`required-tags`**: Kiểm tra xem resource có tag được chỉ định không → báo cáo non-compliant nếu thiếu tag.
- Tự động scan liên tục, báo cáo compliance dashboard.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** Use an AWS Config rule to detect non-compliant tags.

- AWS Config có sẵn **managed rule `required-tags`** — chỉ cần enable và chỉ định tag key cần check.
- **Không cần viết code**, không cần setup infrastructure.
- Tự động detect và report resources không có tag → **least effort**.

---

**❌ Đáp án sai 1:** Use Service Control Policies (SCP).

- **SCP** là guardrail giới hạn quyền tối đa trong AWS Organizations.
- SCP **không có khả năng detect hoặc report** non-compliant tags.
- SCP kiểm soát **action** (ai được làm gì), không kiểm soát **tag compliance**.

---

**❌ Đáp án sai 2:** Create a Lambda function + EventBridge schedule.

- Giải pháp này hoạt động được nhưng **cần viết code**, setup Lambda, cấu hình EventBridge.
- Tốn nhiều effort hơn so với bật sẵn AWS Config managed rule.
- Không phải "least amount of effort".

---

**❌ Đáp án sai 3:** Use tag policies in Organizations + S3 Object Lock.

- **Tag policies** giúp standardize naming convention nhưng **không report resources thiếu tag**.
- **S3 Object Lock** (WORM model) hoàn toàn không liên quan đến tag compliance.
- Giải pháp sai mục đích.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Dịch vụ | Dùng khi |
|---|---|
| **AWS Config** | Detect và report **cấu hình không tuân thủ** (non-compliant resources) |
| **SCP** | Giới hạn **quyền tối đa** trong Organizations |
| **Tag Policies** | Standardize **tên tag** nhưng không detect thiếu tag |
| **Lambda + EventBridge** | Custom compliance check — cần code |

💡 **Công thức ghi nhớ:**
> "Check compliance" + "least effort" → **AWS Config managed rules** — chỉ cần enable, không cần code.
>
> **AWS Config** = "thanh tra cấu hình" — liên tục kiểm tra resources có tuân thủ quy định không. Có sẵn hàng trăm managed rules cho các use case phổ biến: tag compliance, encryption enabled, MFA enabled, public access blocked...
