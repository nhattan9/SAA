## Câu hỏi 26

### 📋 NỘI DUNG CÂU HỎI

EC2 instance host web application. Security Group hiện tại:

| Type | Protocol | Port | Source |
|---|---|---|---|
| SSH | TCP | 22 | 0.0.0.0/0 |

Route Table:

| Destination | Target | Status |
|---|---|---|
| 10.0.0.0/27 | local | Active |
| 0.0.0.0/0 | igw-b51618cc | Active |

Tình trạng: **SSH vào được bình thường**, nhưng **không mở được web app trên Chrome**.

**Câu hỏi:** Làm gì để khắc phục?

---

### 🔑 Từ khóa quyết định đáp án

> "SSH works" (Route Table OK) + "cannot connect via Chrome browser" (HTTP/HTTPS không hoạt động)

→ Route Table đã đúng (SSH đã đi qua được). Vấn đề: Security Group **chỉ có rule SSH (port 22)**, thiếu rule **HTTP (port 80)** cho Chrome.

---

### 🧠 Phân tích logic

**SSH hoạt động → Route Table không có vấn đề gì** vì traffic đã đi từ Internet qua IGW vào EC2 được.

**Chrome không kết nối được** → Chrome dùng **HTTP (port 80)** hoặc **HTTPS (port 443)** — Security Group hiện tại **không có rule nào cho port 80 hoặc 443** → bị chặn.

```
Security Group hiện tại:
✅ Port 22 (SSH)   → cho phép
❌ Port 80 (HTTP)  → không có rule → bị chặn
❌ Port 443 (HTTPS)→ không có rule → bị chặn
```

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** In the Security Group, add an Inbound HTTP rule.

- Thêm rule: **TCP port 80 from 0.0.0.0/0** → Chrome kết nối được qua HTTP.
- Không cần đụng vào Route Table vì Route Table đã đúng.

---

**❌ Đáp án sai 1:** In the Security Group, remove the SSH rule.

- Xóa SSH rule chỉ làm **mất khả năng SSH** — không giải quyết vấn đề Chrome.
- Ngược lại với mục tiêu.

---

**❌ Đáp án sai 2:** In the Route table, add: 0.0.0.0 → igw-b51618cc.

- Route `0.0.0.0/0 → igw-b51618cc` **đã tồn tại** trong Route Table rồi.
- Thêm vào là trùng lặp, không có tác dụng.
- SSH đã hoạt động → Route Table không phải vấn đề.

---

**❌ Đáp án sai 3:** In the Route table, add: 10.0.0.0/27 → local.

- Route `10.0.0.0/27 → local` **đã tồn tại** trong Route Table rồi.
- Không giải quyết được vấn đề kết nối web.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Triệu chứng | Nguyên nhân | Giải pháp |
|---|---|---|
| SSH OK, web không vào được | Security Group thiếu port 80/443 | Thêm inbound HTTP/HTTPS rule |
| Không SSH được, không web được | Route Table thiếu route IGW | Thêm route `0.0.0.0/0 → IGW` |
| Một số user vào được, một số không | Security Group source IP bị giới hạn | Sửa source thành `0.0.0.0/0` |

💡 **Công thức ghi nhớ:**
> Khi troubleshoot EC2 connectivity: **SSH được → Route Table OK** → vấn đề ở **Security Group hoặc ứng dụng**.
>
> **Chrome = HTTP (port 80) hoặc HTTPS (port 443)** — nếu Security Group không có rule này thì không bao giờ vào được web app dù Route Table đúng hoàn toàn.
