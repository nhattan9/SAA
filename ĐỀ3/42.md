## Câu hỏi 42

### 📋 NỘI DUNG CÂU HỎI

> A company runs a multi-tier web application on AWS. Application variables such as **DB hostnames, environment settings, product keys, and database passwords** must be stored **securely with encryption** for security compliance.

Công ty cần lưu trữ **các biến cấu hình ứng dụng** (DB hostname, password, product key...) một cách **bảo mật, có mã hóa**, với **chi phí thấp nhất**.

**Câu hỏi:** Giải pháp nào **tiết kiệm chi phí nhất**?

---

### 🔑 Từ khóa quyết định đáp án

> "application variables" + "secure with encryption" + "most cost-effective"

→ Cần lưu config/parameters bảo mật → **AWS Systems Manager Parameter Store (SecureString)** — miễn phí cho standard parameters.

---

### 🧠 Kiến thức nền: Parameter Store vs Secrets Manager

| | **Parameter Store (SecureString)** | **Secrets Manager** |
|---|---|---|
| **Chi phí** | **Miễn phí** (standard tier) | ~$0.40/secret/month |
| **Mã hóa** | ✅ KMS | ✅ KMS |
| **Auto rotation** | ❌ | ✅ (tự động rotate password) |
| **Dùng cho** | Config, parameters, app variables | Secrets cần rotate tự động |
| **Phù hợp với bài này** | ✅ **Cost-effective** | ❌ Đắt hơn không cần thiết |

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

**✅ Đáp án đúng:** Store values as **SecureString** in AWS Systems Manager Parameter Store + KMS encryption.

- **SecureString** = parameter được mã hóa bằng KMS → bảo mật.
- **Standard tier miễn phí** → cost-effective nhất.
- Ứng dụng truy xuất parameter khi khởi động qua API.
- Phù hợp cho DB hostname, password, product key — không cần auto-rotation.

---

**❌ Đáp án sai 1:** Store values in **AWS Secrets Manager** + BatchGetSecretValue.

- Secrets Manager bảo mật tốt, hỗ trợ auto-rotation — nhưng **tốn ~$0.40/secret/month**.
- Với nhiều variables, chi phí tăng đáng kể.
- Auto-rotation là tính năng thừa nếu các biến ít thay đổi.
- **Không cost-effective** so với Parameter Store.

---

**❌ Đáp án sai 2:** Store values in **S3 file** + enable encryption.

- Phải **upload lại file mỗi lần thay đổi** một giá trị → không tiện lợi.
- Không phải best practice cho configuration management.
- Khó version control và audit từng parameter.

---

**❌ Đáp án sai 3:** Store values in **AWS Systems Manager OpsCenter**.

- **OpsCenter** là công cụ quản lý **operational issues và incidents** — không phải datastore cho key-value pairs.
- Không được thiết kế để lưu application parameters.
- Sai mục đích sử dụng.

---

### 🎯 KẾT LUẬN & BÀI HỌC

| Tình huống | Dịch vụ |
|---|---|
| Lưu config/params **tiết kiệm**, có mã hóa | **Parameter Store (SecureString)** |
| Lưu secrets cần **auto-rotate** (DB password tự động đổi) | **Secrets Manager** |
| Quản lý incidents và operations | AWS Systems Manager OpsCenter |
| Lưu file cấu hình tĩnh | S3 (nhưng không recommended cho sensitive data) |

💡 **Công thức ghi nhớ:**
> **Parameter Store** = miễn phí, mã hóa tốt, dùng cho config/params ít thay đổi.
> **Secrets Manager** = tốn tiền hơn nhưng có **auto-rotation** — dùng khi cần tự động xoay vòng mật khẩu.
>
> Khi đề hỏi "most cost-effective" + "store application variables/config" → **Parameter Store SecureString**.
> Khi đề hỏi "automatic rotation" + "database credentials" → **Secrets Manager**.
