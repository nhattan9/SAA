## Câu hỏi 96

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company developed a financial analytics web application hosted in a **Docker container** using **MEAN stack** (MongoDB, Express.js, AngularJS, and Node.js). You want to easily **port** that web application to AWS Cloud which can **automatically handle** all the tasks such as **balancing load, auto-scaling, monitoring, and placing your containers** across your cluster.
>
> Which of the following services can be used to fulfill this requirement?

---

### 🔤 DỊCH CHI TIẾT

Ứng dụng web chạy trong **Docker container**, dùng **MEAN stack**. Cần migrate lên AWS và muốn dịch vụ **tự động xử lý tất cả**: load balancing, auto-scaling, monitoring, container placement — **không cần cấu hình thủ công**.

**Từ khóa then chốt:** "automatically handle" + "Docker container" + "load balancing + auto-scaling + monitoring" → **Elastic Beanstalk**

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Docker container** → cần service hỗ trợ Docker
2. **Automatically handle tất cả** (LB, scaling, monitoring) → không muốn cấu hình thủ công
3. **Dễ port** (migrate nhanh) → upload code/container là xong

**Điểm dễ nhầm:** ECS cũng chạy Docker nhưng phải **tự cấu hình** LB, scaling, monitoring → không "automatic" như Elastic Beanstalk.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **AWS Elastic Beanstalk**

- Hỗ trợ deploy ứng dụng từ **Docker containers** ✅
- **Tự động** xử lý: capacity provisioning, load balancing, scaling, health monitoring ✅
- Chỉ cần **upload code/container** → Beanstalk lo phần còn lại ✅
- Tích hợp sẵn với VPC, RDS, IAM ✅
- **Least effort** để deploy Docker app lên AWS ✅

---

❌ **Amazon ECS**

- Chạy được Docker containers ✅
- **Nhưng** Auto Scaling, Load Balancing, Monitoring **không tự động bật** ❌
- Phải **tự cấu hình** từng thứ thủ công ❌
- Phù hợp khi cần kiểm soát chi tiết hơn, không phải "automatically handle all" ❌

---

❌ **AWS CloudFormation**

- Infrastructure as Code — deploy hạ tầng qua templates ❌
- **Bạn phải tự viết template** kết nối các AWS resources ❌
- Không tự động handle LB, scaling, monitoring ❌

---

❌ **AWS Compute Optimizer**

- Phân tích workload và **đề xuất** resource tối ưu ❌
- Không deploy hay chạy containers ❌
- Chỉ là công cụ recommendation ❌

---

### Note / Pipeline đúng

**Yêu cầu:** Docker app → AWS, tự động mọi thứ, ít effort nhất.

```
Developer
    │ upload Docker container
    ▼
AWS Elastic Beanstalk
    │ tự động setup:
    ├── Load Balancer (ALB) ✅
    ├── Auto Scaling Group ✅
    ├── CloudWatch Monitoring ✅
    ├── EC2 instances chạy containers ✅
    └── Health checks ✅
    │
    ▼
Ứng dụng MEAN stack chạy trên AWS ✅
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Service | Docker? | Tự động LB/Scaling/Monitoring? | Effort |
|---|---|---|---|
| **Elastic Beanstalk** | ✅ | ✅ Tự động hoàn toàn | Thấp nhất |
| **ECS** | ✅ | ❌ Cần tự cấu hình | Trung bình |
| **EKS** | ✅ | ❌ Cần tự cấu hình | Cao |
| **CloudFormation** | ✅ | ❌ Cần tự viết template | Cao |
| **Compute Optimizer** | ❌ | ❌ | — |

> 💡 **Công thức ghi nhớ:**
> "Docker + automatically handle everything + least effort" → **Elastic Beanstalk**
> "Docker + full control + manual config" → **ECS hoặc EKS**
> **Elastic Beanstalk** = "PaaS của AWS" — upload code/container, AWS lo hết phần còn lại
