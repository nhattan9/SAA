## Câu hỏi 49

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> An online shopping platform has been deployed to AWS using **Elastic Beanstalk**. They simply uploaded their **Node.js application**, and Elastic Beanstalk automatically handles the details of **capacity provisioning, load balancing, scaling, and application health monitoring**. Since the entire deployment process is automated, the DevOps team is not sure where to get the **application log files** of their shopping platform.
>
> In Elastic Beanstalk, where does it store the **application files** and **server log files**?

---

### 🔤 DỊCH CHI TIẾT

- **Elastic Beanstalk**: dịch vụ triển khai ứng dụng web tự động (Platform as a Service)
- **Node.js application**: ứng dụng viết bằng Node.js
- **capacity provisioning**: cấp phát capacity (sức chứa) — tạo EC2, RDS, ...
- **load balancing**: cân bằng tải
- **application health monitoring**: giám sát sức khỏe ứng dụng
- **application files**: file ứng dụng (mã nguồn, package.json, ...)
- **server log files**: file log từ server (application logs, system logs)

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Application files** lưu ở đâu?
2. **Server log files** lưu ở đâu (có bao nhiêu cách)?
3. Elastic Beanstalk có những option nào cho log files?

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

❌ **A. Application files ở S3. Server log files ở Glacier hoặc CloudWatch Logs.**

- Application files ✅ S3 đúng
- Server log files ❌ không hỗ trợ **trực tiếp** Glacier (chỉ có thể via S3 lifecycle policy)

❌ **B. Application files ở S3. Server log files chỉ có thể ở EBS volumes của EC2.**

- Application files ✅ S3 đúng
- Server log files ❌ không chỉ ở EBS — có thể ở S3 hoặc CloudWatch Logs

✅ **C. Application files ở S3. Server log files có thể optionally ở S3 hoặc CloudWatch Logs.**

- Application files ✅ S3 — Elastic Beanstalk tự động upload app files lên S3
- Server log files ✅ **2 options:**
  - **S3**: Elastic Beanstalk có thể copy log files từ EC2 lên S3 mỗi giờ (cần config)
  - **CloudWatch Logs**: CloudWatch Logs agent trên EC2 streaming logs real-time
- Đúng hoàn toàn

❌ **D. Application files ở S3. Server log files ở CloudTrail hoặc CloudWatch Logs.**

- Application files ✅ S3 đúng
- Server log files ❌ CloudTrail là dùng để audit **API calls**, không phải application logs

---

**📌 Note — Pipeline lưu trữ trong Elastic Beanstalk:**

```
Elastic Beanstalk Environment
│
├─ Application Files
│   ↓ tự động upload
│   → Amazon S3 bucket
│
└─ Server Log Files (2 options)
   ├─ Option 1: CloudWatch Logs Agent
   │  ↓ streaming real-time
   │  → CloudWatch Logs (CloudWatch Insights query)
   │  ✅ Real-time monitoring, alarming
   │
   └─ Option 2: S3 (Manual config)
      ↓ copy mỗi giờ
      → Amazon S3 bucket
      ✅ Long-term storage, archive to Glacier
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Thành phần | Lưu trữ | Mục đích | Cấu hình |
|---|---|---|---|
| **Application Files** | S3 (bắt buộc) | Versioning, deployment history | Tự động |
| **Server Logs → CloudWatch** | CloudWatch Logs | Real-time monitoring, metrics, alarms | Enable CloudWatch Logs integration |
| **Server Logs → S3** | S3 bucket | Long-term storage, compliance | Configure environment |
| **CloudTrail** | ❌ Không dùng cho app logs | Audit API calls | Khác mục đích |
| **EBS Volumes** | Local EC2 storage | ⚠️ Mất khi EC2 terminate | Không recommended |
| **Glacier** | Chỉ qua S3 Lifecycle | Archiving | S3 → Glacier transition |

> 💡 **Công thức ghi nhớ:**
> **"Elastic Beanstalk → App files đi S3 (bắt buộc) → Logs đi CloudWatch hoặc S3 (tuỳ chọn)"**

> 🔑 **Keyword bẫy:**
> ❌ **"Only EBS"** → Sai, vì EBS lưu local mất khi terminate
> ❌ **"Direct Glacier"** → Sai, phải qua S3 lifecycle
> ❌ **"CloudTrail"** → Sai, CloudTrail dùng audit API, không phải app logs
> ✅ **"Optionally S3 or CloudWatch"** → Đúng! Hai cách khác nhau, tuỳ yêu cầu
