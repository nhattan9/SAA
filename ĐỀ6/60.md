## Câu hỏi 61

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A **software development company** has **hundreds of Amazon EC2 instances** with **multiple Application Load Balancers (ALBs) across multiple AWS Regions**. The **public applications hosted in their EC2 instances are accessed on their on-premises network**. The company needs to **reduce the number of IP addresses** that it needs to **regularly whitelist on the corporate firewall device**.
>
> Which of the following approach can be used to fulfill this requirement?

---

### 🔤 DỊCH CHI TIẾT

Một **software development company** (công ty phát triển phần mềm) có **hundreds of Amazon EC2 instances** (hàng trăm instance EC2) với **multiple Application Load Balancers (ALBs) across multiple AWS Regions** (nhiều ALB ở nhiều AWS Region). **Public applications hosted in their EC2 instances are accessed on their on-premises network** (các ứng dụng công cộng được lưu trữ trong EC2 instances được truy cập từ mạng tại chỗ). Công ty cần **reduce the number of IP addresses** (giảm số lượng địa chỉ IP) mà nó cần **regularly whitelist on the corporate firewall device** (thường xuyên whitelist trên thiết bị tường lửa công ty). Phương pháp nào sau đây có thể được sử dụng để đáp ứng yêu cầu này?

---

### 🎯 PHÂN TÍCH YÀU CẦU CHÍNH

1. **Multiple ALBs across multiple regions**: ALB ở nhiều region
2. **Reduce IP addresses to whitelist**: giảm số IP cần whitelist
3. **On-premises network access**: on-prem cần access cloud apps
4. **Static entry point**: cần một điểm vào cố định

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

❌ **A. Launch NLB with Elastic IP + set ALBs as targets**

- Elastic IP: ✅ Static IP
- ❌ KHÔNG thể route across multiple regions (NLB region-scoped)
- ❌ KHÔNG solution cho multi-region ALBs

✅ **B. Use AWS Global Accelerator + create endpoint group per region + associate ALBs**

- **AWS Global Accelerator**:
  - ✅ **2 static IP addresses** (fixed entry point)
  - ✅ **Create endpoint group for each region**
  - ✅ **Associate ALBs from each region**
  - ✅ Can add/remove endpoints without changing IPs
  - ✅ Perfect for multi-region setup
- ✅ On-prem firewall chỉ cần whitelist **2 static IPs** thay vì hàng chục ALB IPs

❌ **C. Global Accelerator + multiple endpoints + associate EC2 private IPs**

- ❌ KHÔNG nên associate EC2 instances, nên associate **ALBs**
- ❌ Private IPs không accessible từ on-prem

❌ **D. Lambda function to track ALB IPs + update firewall hourly**

- ❌ **Manual, overhead cao**: Lambda monitoring + hourly updates
- ❌ **Inefficient**: phải update firewall thường xuyên
- ❌ **AWS Global Accelerator** sẵn sàng, không cần custom solution

---

**📌 Note — AWS Global Accelerator Architecture:**

```
On-premises network
     ↓ Whitelist 2 static IPs
AWS Global Accelerator
     ├─ Static IP: 1.2.3.4 ✅
     └─ Static IP: 1.2.3.5 ✅
     
     ↓ Route intelligently to regions
     
     Endpoint Group 1 (us-east-1)
     └─ ALB: us-east-1.load...
     
     Endpoint Group 2 (eu-west-1)
     └─ ALB: eu-west-1.load...
     
     Endpoint Group 3 (ap-southeast-1)
     └─ ALB: ap-southeast-1.load...

Benefit: Only 2 static IPs, instead of 3+ ALB IPs!
```

### 📊 KẾT LUẬN & BÀI HỌC

| Approach | Static IPs | Multi-region? | Overhead? | Best for |
|---|---|---|---|---|
| **Global Accelerator** | ✅ 2 static | ✅ Yes | ✅ Low | ✅ Multi-region access |
| NLB + EIP | ✅ 1 static | ❌ No (region) | ⚠️ Medium | Single region |
| Lambda tracking | ❌ Dynamic | ✅ Yes | ❌ High | NOT recommended |
| Direct ALB IPs | ❌ Multiple | ✅ Yes | ❌ Very high | Manual whitelist |

| Feature | Requirement? | Global Accelerator? |
|---|---|---|
| **Reduce IPs to whitelist** | ✅ Yes | ✅ **2 static IPs** |
| **Multiple regions** | ✅ Yes | ✅ **Endpoint groups** |
| **Associate ALBs** | ✅ Yes | ✅ **Yes** |
| **Fixed entry point** | ✅ Yes | ✅ **Static IPs** |

> 💡 **Công thức ghi nhớ:**
> **"Multi-region ALBs + reduce IP whitelist → AWS Global Accelerator"**
> **"1 Global Accelerator = 2 static IPs + multiple endpoint groups"**

> ❌ **"NLB với EIP"** → KHÔNG cross-region

> ✅ **"Global Accelerator + endpoint groups"** → Perfect for multi-region
> ✅ **"2 static IPs"** → Only whitelist these, not ALB IPs
