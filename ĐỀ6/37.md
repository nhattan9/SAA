## Câu hỏi 37

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A computer animation film studio runs a web application on an Amazon EC2 instance. It uploads 5 GB video objects to an Amazon S3 bucket. Video uploads take longer than expected, impacting the application's performance.
>
> Which method will help improve the performance of the application?

---

### 🔤 DỊCH CHI TIẾT

Một studio làm phim hoạt hình chạy web application trên EC2. Studio upload các video object **5 GB** lên Amazon S3. Việc upload **mất quá nhiều thời gian**, ảnh hưởng đến hiệu năng ứng dụng.

Phương pháp nào giúp cải thiện hiệu năng?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Vấn đề là upload chậm lên S3** — không phải vấn đề disk I/O hay network adapter của EC2.
2. **Object size = 5 GB** — rất lớn, vượt ngưỡng 100 MB mà AWS khuyến nghị dùng Multipart Upload.

> ⚠️ Điểm mấu chốt: Upload S3 là **network-bound** (phụ thuộc băng thông mạng), không phải **disk-bound**. Giải pháp phải nhắm vào tối ưu **quá trình upload lên S3**, không phải tối ưu disk hay network adapter của EC2.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

---

✅ **Đáp án đúng (C):** **S3 Multipart Upload API**

- Chia file 5 GB thành nhiều phần nhỏ → upload **song song (parallel)** → tổng thời gian giảm đáng kể. ✅
- Nếu một phần bị lỗi → chỉ cần **retry phần đó**, không phải upload lại toàn bộ. ✅
- AWS khuyến nghị dùng Multipart Upload khi object **≥ 100 MB**. ✅

---

❌ **Đáp án sai (D):** Enable **Enhanced Networking với ENA** trên EC2

- ENA cải thiện **network throughput tổng thể** của EC2 — đúng là giúp mạng nhanh hơn.
- Nhưng vẫn upload **1 file 5 GB liên tục** → vẫn chậm dù mạng nhanh hơn.
- Không giải quyết được gốc rễ vấn đề bằng cách chia nhỏ và upload song song.

---

❌ **Đáp án sai (B):** **Amazon CloudFront** + HTTP POST

- **CloudFront là CDN** — dùng để phân phối nội dung **đến end user nhanh hơn (download)**, không phải tăng tốc upload lên S3.
- HTTP POST qua CloudFront không phải cơ chế tối ưu upload S3.

---

❌ **Đáp án sai (A):** **EBS Provisioned IOPS + LVM stripe** trên EBS-optimized instance

- Tối ưu **disk I/O** — phù hợp cho database hay workload đọc/ghi ổ đĩa nhiều.
- Upload S3 là **network-bound, không phải disk-bound** → tối ưu disk không giúp gì cho tốc độ upload S3.

---

### Note / Pipeline đúng

Yêu cầu: Upload file 5 GB lên S3 nhanh hơn.

```
File video 5 GB trên EC2
→ S3 Multipart Upload API chia thành nhiều phần (VD: 100 x 50 MB)
→ Upload các phần song song (parallel) cùng lúc
→ Tổng thời gian giảm đáng kể so với upload tuần tự
→ Nếu 1 phần lỗi → retry phần đó, không ảnh hưởng phần khác
→ S3 ghép lại thành object hoàn chỉnh 5 GB
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Vấn đề | Giải pháp đúng |
|---|---|
| Upload file lớn (≥ 100 MB) lên S3 chậm | **S3 Multipart Upload API** |
| Network throughput EC2 thấp | **ENA / Enhanced Networking** |
| Phân phối content đến user nhanh hơn | **CloudFront** |
| Disk I/O chậm trên EC2 | **EBS Provisioned IOPS** |

💡 **Công thức ghi nhớ:**

> **"Upload file lớn lên S3 + chậm"** → **S3 Multipart Upload** — chia nhỏ, upload song song

> **"S3 upload là network-bound"** → tối ưu disk (EBS/LVM) **không giúp ích gì**

> ⚠️ AWS khuyến nghị: file **≥ 100 MB** → dùng **Multipart Upload**, không upload single part
