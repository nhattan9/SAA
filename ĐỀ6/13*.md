## Câu hỏi 97

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> An application is hosted in an Auto Scaling group of EC2 instances and a **Microsoft SQL Server on Amazon RDS**. There is a requirement that all **in-flight data** between your web servers and RDS should be **secured**.
>
> Which of the following options is the MOST suitable solution? **(Select TWO.)**

---

### 🔤 DỊCH CHI TIẾT

Cần bảo mật **in-flight data** (dữ liệu đang truyền — data in transit) giữa EC2 và RDS SQL Server. Không phải data at rest (dữ liệu đang lưu trữ).

**Từ khóa then chốt:** "in-flight data" = **data in transit** → cần **SSL/TLS encryption**

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **In-flight = data in transit** → dùng **SSL** để mã hóa kết nối
2. Có **2 cách** bật SSL cho RDS SQL Server:
   - **Force SSL** cho tất cả connections (server-side)
   - **Import SSL certificate** và cấu hình từng client (client-side)

**Bẫy:** TDE = mã hóa **data at rest** (không phải in-flight) → sai yêu cầu.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án A:** Download **Amazon RDS Root CA certificate** → import vào servers → configure app dùng **SSL** để encrypt connection.

- Client-side SSL → từng EC2 instance import cert và dùng SSL ✅
- Mã hóa **data in transit** giữa EC2 và RDS ✅

---

✅ **Đáp án E:** Force all connections dùng SSL bằng cách set **rds.force_ssl = true** → reboot DB instance.

- Server-side force SSL → mọi connection đều phải dùng SSL ✅
- Transparent với client (client không cần làm gì thêm) ✅
- Sau khi set parameter → phải **reboot** để có hiệu lực ✅

---

❌ **Đáp án B:** Specify **TDE** (Transparent Data Encryption) option.

- **TDE = mã hóa data at rest** (data đang lưu trên disk) ❌
- Không liên quan đến **in-flight data** ❌
- Sai yêu cầu hoàn toàn ❌

---

❌ **Đáp án C:** Configure security groups to only allow **port 443**.

- Security group chỉ giới hạn port, không mã hóa data ❌
- RDS SQL Server dùng **port 1433**, không phải 443 ❌
- Không đủ để bảo mật in-flight data ❌

---

❌ **Đáp án D:** Enable **IAM DB authentication**.

- IAM DB authentication chỉ hỗ trợ **MySQL và PostgreSQL** ❌
- **Không hỗ trợ SQL Server** ❌
- Ngoài ra IAM DB auth giải quyết **authentication** (xác thực), không phải encryption ❌

---

### Note / Pipeline đúng

**Yêu cầu:** Mã hóa data in transit EC2 → RDS SQL Server.

```
Cách 1 (Force SSL - server side):
rds.force_ssl = true → reboot RDS
→ Mọi connection đến RDS BẮT BUỘC dùng SSL
→ Client không cần cấu hình gì

Cách 2 (Client SSL certificate):
Download RDS Root CA cert
→ Import vào EC2 instances
→ Configure app: ssl=true, sslcert=path/to/cert
→ Connection EC2 → RDS được mã hóa SSL
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Kỹ thuật | Bảo vệ gì | Dùng khi nào |
|---|---|---|
| **SSL/TLS (rds.force_ssl)** | Data in transit | ✅ In-flight data |
| **SSL Certificate (client)** | Data in transit | ✅ In-flight data |
| **TDE** | Data at rest | ❌ Không phải in-flight |
| **Encryption at rest** | Data at rest | ❌ Không phải in-flight |
| **IAM DB Auth** | Authentication | MySQL/PostgreSQL only |

> 💡 **Công thức ghi nhớ:**
> **"In-flight / in-transit"** → **SSL/TLS**
> **"At rest / stored"** → **TDE, KMS, SSE**
> **rds.force_ssl = true** → force SSL cho toàn bộ connection → cần **reboot**
> **TDE** = mã hóa disk, không phải network → đừng nhầm với SSL
