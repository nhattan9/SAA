## Câu hỏi 103

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A fast food company uses an Auto Scaling group of EC2 instances with an **Application Load Balancer**. A new routing logic is being considered:
> - Requests with URLs matching `/api/android` → **Android-Target-Group**
> - Requests with URLs matching `/api/ios` → **iOS-Target-Group**
>
> How can this change be implemented in AWS?

---

### 🔤 DỊCH CHI TIẾT

Cần định tuyến request đến các target group khác nhau dựa trên **URL path** (`/api/android` hoặc `/api/ios`). Đây là **path-based routing** (định tuyến theo đường dẫn URL).

**Từ khóa then chốt:** "/api/android", "/api/ios" = **URL path** → **Path conditions** trên ALB

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Routing dựa trên URL path** → **Path-based routing** (path conditions)
2. Đang dùng **ALB** → ALB hỗ trợ path-based routing ✅ (không cần thay)
3. **Host-based routing** = routing theo domain name (khác với path)

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án D:** Use **path conditions** to define rules that forward requests to different target groups based on the **URL in the request**.

- ALB hỗ trợ **path-based routing** ✅
- Tạo listener rules:
  - `/api/android*` → Android-Target-Group ✅
  - `/api/ios*` → iOS-Target-Group ✅
- Không cần thay đổi loại LB ✅

---

❌ **Đáp án B:** Use **host conditions** based on hostname in host header.

- **Host-based routing** = routing theo **domain name** (VD: `android.example.com` vs `ios.example.com`) ❌
- Bài hỏi routing theo **URL path** (`/api/android`) không phải hostname ❌

---

❌ **Đáp án C:** Replace ALB with **Gateway Load Balancer** + path conditions.

- **GLB không hỗ trợ path-based routing** ❌
- GLB dùng để deploy network appliances (firewall, IDS...) ❌
- Không cần thay ALB ❌

---

❌ **Đáp án A:** Replace ALB with **Network Load Balancer** + host conditions.

- **NLB không hỗ trợ path-based routing** ❌
- NLB hoạt động ở Layer 4 (TCP/UDP), không đọc được URL path ❌
- Lại dùng host conditions thay vì path conditions → sai 2 lần ❌

---

### Note / Pipeline đúng

**Yêu cầu:** Route theo URL path đến các target groups khác nhau.

```
Client request
    │
    ▼
Application Load Balancer (Layer 7)
    │
    ├── Listener Rule 1:
    │   Path: /api/android*
    │   → Forward → Android-Target-Group ✅
    │
    ├── Listener Rule 2:
    │   Path: /api/ios*
    │   → Forward → iOS-Target-Group ✅
    │
    └── Default Rule:
        → Forward → Default-Target-Group
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Routing Type | Dựa trên | Load Balancer |
|---|---|---|
| **Path-based** | URL path (`/api/android`) | **ALB** |
| **Host-based** | Domain (`android.example.com`) | **ALB** |
| **Header-based** | HTTP headers | **ALB** |
| **IP-based** | Source IP | **ALB** |
| **TCP/UDP port** | Port number | **NLB** |
| Network appliance | Packet-level | **GLB** |

> 💡 **Công thức ghi nhớ:**
> **URL path** (`/api/...`) → **Path-based routing** → **ALB**
> **Domain name** (`android.example.com`) → **Host-based routing** → **ALB**
> **NLB** = Layer 4, không đọc URL → không có path/host routing
> **GLB** = deploy network appliances, không có path routing
>
> Mẹo nhớ: Mọi routing thông minh (path, host, header, method) → **ALB** (Layer 7)
