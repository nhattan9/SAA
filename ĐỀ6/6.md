## Câu hỏi 90

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A tech company is having an issue whenever they try to connect to the newly created EC2 instance using a **Remote Desktop connection** from a computer. The instance has a public IP and the Internet gateway and route tables are in place.
>
> What else should he do to resolve this issue?

---

### 🔤 DỊCH CHI TIẾT

Công ty không kết nối được vào EC2 qua **Remote Desktop (RDP)**. Đã có: public IP, Internet Gateway, route tables. Cần tìm nguyên nhân còn lại.

**Từ khóa then chốt:** "Remote Desktop" = **RDP** = **port 3389**

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Checklist kết nối EC2 từ internet:
1. ✅ Public IP — đã có
2. ✅ Internet Gateway — đã có
3. ✅ Route tables — đã có
4. ❌ **Security Group** chưa mở port RDP (3389) → đây là nguyên nhân

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án A:** Adjust the security group to allow **inbound traffic on port 3389** from the company's IP address.

- **RDP** dùng **TCP/UDP port 3389** ✅
- Security Group mặc định không mở port 3389 ✅
- Chỉ cho phép IP của công ty → bảo mật hơn ✅

---

❌ **Đáp án C:** Adjust the security group to allow inbound traffic on **port 22**.

- **Port 22 = SSH** → dùng cho Linux instances, kết nối bằng terminal ❌
- **Không phải RDP** (Remote Desktop) ❌

---

❌ **Đáp án B:** Restart the EC2 instance.

- Instance mới tạo, không có lý do gì để restart ❌
- Vấn đề là networking/security group, không phải instance ❌

---

❌ **Đáp án D:** Create a new instance.

- Tương tự — vấn đề không phải ở instance ❌
- Tạo instance mới cũng sẽ gặp vấn đề tương tự nếu không mở port ❌

---

### Note / Pipeline đúng

```
Checklist kết nối EC2 từ internet:

1. Public IP ✅ (đã có)
2. Internet Gateway ✅ (đã có)
3. Route Table: 0.0.0.0/0 → IGW ✅ (đã có)
4. Security Group: inbound port 3389 ❌ → CẦN MỞ
5. Network ACL: cho phép port 3389 (thường mặc định allow all)

Fix: Security Group → Add Inbound Rule
Type: RDP | Protocol: TCP | Port: 3389 | Source: Company IP
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Protocol | Port | Dùng cho |
|---|---|---|
| **RDP** | **3389** | Windows EC2 — Remote Desktop |
| **SSH** | **22** | Linux EC2 — Terminal |
| **HTTP** | **80** | Web traffic |
| **HTTPS** | **443** | Web traffic mã hóa |
| **MySQL** | **3306** | Database |

> 💡 **Công thức ghi nhớ:**
> **Windows EC2 + Remote Desktop** → mở port **3389** trong Security Group
> **Linux EC2 + SSH** → mở port **22** trong Security Group
> Khi không kết nối được EC2 → kiểm tra **Security Group trước**, không phải restart hay tạo mới instance
