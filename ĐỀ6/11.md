## Câu hỏi 95

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A cryptocurrency company wants to go global with its international money transfer app. The database of the app must be highly available in multiple regions.
>
> What are the **benefits of adding Multi-AZ deployments in Amazon RDS**? **(Select TWO.)**

---

### 🔤 DỊCH CHI TIẾT

Câu hỏi hỏi về **lợi ích của RDS Multi-AZ**. Đây là câu hỏi kiến thức thuần túy về tính năng Multi-AZ.

**Từ khóa then chốt:** "Multi-AZ" → **durability** + **availability** (không phải performance)

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

RDS Multi-AZ mang lại **2 lợi ích chính:**
1. **Enhanced durability** — chịu được lỗi component hoặc AZ outage
2. **Increased availability** — giảm downtime khi maintenance (OS patch, scaling...)

**Điểm bẫy:** Multi-AZ **không cải thiện performance** và standby instance ở **cùng region** (khác AZ), không phải khác region.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án A:** **Provides enhanced database durability** in the event of a DB instance component failure or an AZ outage.

- Multi-AZ tự động failover sang standby khi primary fail ✅
- Standby ở AZ khác → nếu cả AZ bị down vẫn còn standby ✅
- **Durability** = dữ liệu không bị mất ✅

---

✅ **Đáp án D:** **Increased database availability** in the case of system upgrades like OS patching or DB instance scaling.

- Khi AWS patch OS hoặc scale instance → failover sang standby → ứng dụng tiếp tục chạy ✅
- Giảm downtime trong maintenance windows ✅
- **Availability** = luôn sẵn sàng phục vụ ✅

---

❌ **Đáp án B:** Provides **SQL optimization**.

- Multi-AZ không có bất kỳ tính năng SQL optimization nào ❌
- SQL optimization là việc của query design, indexes... ❌

---

❌ **Đáp án C:** Creates primary DB + synchronously replicates to standby in **a different region**.

- **SAI về địa lý:** standby ở **cùng region, khác AZ** ❌
- Nếu muốn khác region → cần **RDS Read Replica cross-region** ❌
- Multi-AZ = cùng region, khác AZ ❌

---

❌ **Đáp án E:** **Significantly increases the database performance**.

- Multi-AZ **không tăng performance** ❌
- Standby instance **không nhận read traffic** (chỉ Aurora mới có read replica nhận traffic)
- Muốn tăng performance → dùng **Read Replicas** ❌

---

### Note / Pipeline đúng

**RDS Multi-AZ hoạt động như thế nào:**

```
Primary DB (AZ-1) ──synchronous replication──► Standby DB (AZ-2)
      │                                              │
      │  (cùng Region, khác AZ)                     │
      │                                              │
Ứng dụng kết nối vào DNS endpoint (không đổi)        │
      │                                              │
Primary fail ──────────────────────────────────────► Auto Failover
                                               Standby promoted to Primary
                                               DNS tự động trỏ sang
                                               (~1-2 phút) ✅
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Tính năng | Multi-AZ | Read Replica |
|---|---|---|
| **Mục đích** | High availability | Read scaling |
| **Replication** | Synchronous | Asynchronous |
| **Nhận traffic đọc** | ❌ (standby idle) | ✅ |
| **Tăng performance** | ❌ | ✅ |
| **Failover tự động** | ✅ | ❌ (manual promote) |
| **Khác region** | ❌ (cùng region) | ✅ (cross-region) |

> 💡 **Công thức ghi nhớ:**
> **Multi-AZ** = **durability + availability** (không phải performance)
> **Multi-AZ** = cùng Region, **khác AZ** (không phải khác Region)
> **Read Replica** = tăng performance đọc
> Bẫy hay gặp: "Multi-AZ tăng performance" → **SAI**
