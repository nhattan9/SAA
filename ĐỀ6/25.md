## Câu hỏi 25

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A Solutions Architect created a brand new IAM User with a default setting using AWS CLI. This is intended to be used to send API requests to Amazon S3, DynamoDB, Lambda, and other AWS resources of the company's cloud infrastructure.
>
> Which of the following must be done to allow the user to make API calls to the AWS resources?

---

### 🔤 DỊCH CHI TIẾT

Một Solutions Architect tạo mới một **IAM User** (người dùng IAM) với cài đặt mặc định bằng **AWS CLI**. User này dùng để gửi **API requests** (yêu cầu qua giao diện lập trình) đến S3, DynamoDB, Lambda và các tài nguyên AWS khác.

Cần làm gì để user này có thể thực hiện API calls?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **IAM User tạo bằng AWS CLI với default setting** — đây là chi tiết mấu chốt: user tạo bằng CLI mặc định **không có credentials nào cả**.
2. **Mục đích là gọi API** (programmatic access) — cần loại credentials phù hợp với API calls, không phải console login.

> ⚠️ Điểm dễ nhầm: Nhiều người nghĩ chỉ cần gán IAM Policy là đủ quyền → sai, vì **quyền (permission) ≠ credentials (thông tin xác thực)**. Thiếu credentials thì có quyền cũng không gọi được API.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

---

✅ **Đáp án đúng (A):** **Create a set of Access Keys** for the user and attach the necessary permissions.

- **Access Keys** (gồm Access Key ID + Secret Access Key) là credentials dùng để **ký các programmatic requests** đến AWS CLI, SDK, và API. ✅
- IAM User tạo bằng CLI mặc định **không có credentials nào** → bắt buộc phải tạo Access Keys thủ công. ✅
- Kết hợp thêm IAM Policy để cấp quyền → đủ điều kiện gọi API. ✅

---

❌ **Đáp án sai (B) — Đáp án bạn chọn:** Assign an IAM Policy to the user.

- IAM Policy cấp **quyền** (permission) — xác định user được làm gì.
- Nhưng nếu không có **credentials** (Access Keys), user vẫn **không thể xác thực** để gọi API dù có đủ quyền.
- Thiếu một nửa: có quyền nhưng không có chìa khóa để vào.

---

❌ **Đáp án sai (C):** Do nothing — IAM User đã có thể gọi API.

- Hoàn toàn sai. IAM User tạo bằng **AWS CLI với default setting** không có bất kỳ credentials nào theo mặc định.
- Khác với tạo bằng AWS Console — Console bắt buộc bạn phải chọn ít nhất password hoặc access keys khi tạo.

---

❌ **Đáp án sai (D):** Enable Multi-Factor Authentication (MFA).

- MFA là lớp bảo mật bổ sung cho việc **đăng nhập console**, không phải credentials cho API calls.
- Bật MFA không tạo ra Access Keys → vẫn không gọi được API.

---

> 📝 **Note:** Từ khóa quyết định: **"created using AWS CLI + default setting + API calls"**
>
> **Pipeline đúng:** `Tạo IAM User (CLI)` → `Tạo Access Keys` → `Gán IAM Policy` → `Gọi API thành công`

---

### 📊 KẾT LUẬN & BÀI HỌC

| Mục đích | Credentials cần có |
|---|---|
| Gọi API / CLI / SDK (programmatic) | **Access Keys** (Key ID + Secret Key) |
| Đăng nhập AWS Management Console | **Password** |
| Bảo mật thêm cho console login | **MFA** |
| Xác định user được làm gì | **IAM Policy** (không phải credentials) |

💡 **Công thức ghi nhớ:**

> **"API calls / programmatic access"** → cần **Access Keys**

> **"Console login"** → cần **Password**

> **IAM Policy** = quyền hạn (WHAT you can do) ≠ **Credentials** = thông tin xác thực (WHO you are)

> ⚠️ IAM User tạo bằng **AWS CLI** = **không có credentials mặc định** → phải tạo thủ công. Tạo bằng **Console** = bắt buộc chọn password hoặc access keys ngay lúc tạo.
