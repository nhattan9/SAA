## Câu hỏi 105

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A multinational company is building a data analytics platform with **HPC workloads** which requires a **scalable, POSIX-compliant** storage service. The data needs to be stored **redundantly across multiple AZs** and allows **concurrent connections from thousands of EC2 instances** hosted on multiple Availability Zones.
>
> Which AWS storage service is the most suitable?

---

### 🔤 DỊCH CHI TIẾT

Cần storage cho HPC với các đặc điểm:
- **POSIX-compliant** (chuẩn file system Linux)
- **Redundant across multiple AZs** (dữ liệu lưu ở nhiều AZ)
- **Concurrent connections** từ **hàng nghìn EC2** ở nhiều AZ cùng lúc

**Từ khóa then chốt:** "concurrent connections" + "multiple EC2" + "multiple AZs" + "POSIX" → **EFS**

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Concurrent access từ nhiều EC2** → loại EBS (chỉ gắn 1 EC2)
2. **Multiple AZs** → loại EBS (gắn với 1 AZ)
3. **POSIX-compliant** → EFS hoặc FSx for Lustre
4. **Redundant across AZs** → EFS (tự động multi-AZ)

**Phân biệt với câu hỏi FSx for Lustre:** Bài này không đề cập "natively work with S3" → EFS là đủ.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Amazon Elastic File System (EFS)**

- **POSIX-compliant** file system ✅
- **Concurrent access** từ hàng nghìn EC2 instances cùng lúc ✅
- Tự động **redundant across multiple AZs** ✅
- **Scalable** tự động từ GB đến PB, không cần provision trước ✅
- Consistent performance cho tất cả instances ✅

---

❌ **Amazon EBS Volumes**

- Chỉ gắn được **1 EC2 instance** tại một thời điểm ❌
- Gắn với **1 AZ cụ thể** ❌
- Không hỗ trợ concurrent access từ nhiều EC2 ❌

---

❌ **Amazon S3**

- **Không phải file system** — không POSIX-compliant ❌
- Không hỗ trợ file system interface (open/read/write/close) ❌
- Dùng HTTP API (PUT/GET), không phải POSIX ❌

---

❌ **Amazon ElastiCache**

- **In-memory cache** (Redis, Memcached) ❌
- Không phải storage service ❌
- Không lưu trữ persistent data ❌

---

### Note / Pipeline đúng

**Yêu cầu:** Shared storage cho HPC, nhiều EC2, nhiều AZ.

```
AZ-1                    AZ-2                    AZ-3
EC2-1 ─┐               EC2-5 ─┐               EC2-9 ─┐
EC2-2 ─┤               EC2-6 ─┤               EC2-10─┤
EC2-3 ─┤── mount ───► EFS ◄── mount ──┤       EC2-11─┤
EC2-4 ─┘               EC2-7 ─┘               EC2-12─┘
                        EC2-8 ─┘

EFS tự động replicate data across AZs ✅
Tất cả EC2 mount cùng 1 EFS → đọc/ghi đồng thời ✅
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Storage | Concurrent Access? | Multi-AZ? | POSIX? | Scalable? |
|---|---|---|---|---|
| **EFS** | ✅ Hàng nghìn EC2 | ✅ | ✅ | ✅ Auto |
| **EBS** | ❌ 1 EC2 | ❌ 1 AZ | ✅ | ❌ Manual |
| **S3** | ✅ | ✅ | ❌ | ✅ |
| **FSx Lustre** | ✅ | ❌ (1 AZ default) | ✅ | ✅ |
| **ElastiCache** | ✅ | ✅ | ❌ | ✅ |

> 💡 **Công thức ghi nhớ:**
> "Concurrent access + multiple EC2 + multiple AZ + POSIX" → **EFS**
> "HPC + S3 natively + POSIX" → **FSx for Lustre**
> **EBS** = 1 EC2, 1 AZ → không share được
> Khi thấy **"concurrent connections from multiple EC2"** → **luôn nghĩ EFS trước**
