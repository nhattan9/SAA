## Câu hỏi 104

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company launched an EC2 instance to host a network monitoring system and used **CloudWatch to monitor, store, and access log files** of the instance.
>
> Which of the following provides an **automated way to send log data to CloudWatch Logs** from the Amazon EC2 instance?

---

### 🔤 DỊCH CHI TIẾT

Cần **tự động gửi log** từ EC2 instance lên **CloudWatch Logs**. Câu hỏi hỏi tool/agent nào làm việc này.

**Từ khóa then chốt:** "automated" + "send log data" + "EC2 → CloudWatch Logs"

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Để gửi log từ EC2 lên CloudWatch Logs cần có **agent** chạy trên instance để:
- Đọc log files trên EC2
- Push log lên CloudWatch Logs tự động

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **CloudWatch Logs agent**

- Agent cài trên EC2, tự động **push log files lên CloudWatch Logs** ✅
- Gồm 3 thành phần:
  - **Plugin AWS CLI** — push data lên CloudWatch Logs
  - **Script (daemon)** — chạy nền liên tục
  - **Cron job** — đảm bảo daemon luôn chạy ✅
- **Automated** — cài một lần, tự động gửi log ✅

---

❌ **CloudTrail with log file validation**

- CloudTrail = ghi lại **API calls** đến AWS resources ❌
- Log file validation = xác minh log không bị giả mạo ❌
- Không gửi EC2 application logs lên CloudWatch ❌

---

❌ **AWS Transfer for SFTP**

- Dịch vụ **SFTP managed** để transfer files vào/ra **S3** ❌
- Không liên quan đến EC2 log shipping ❌

---

❌ **CloudTrail Processing Library**

- **Java library** để xử lý CloudTrail logs ❌
- Không gửi log từ EC2 lên CloudWatch ❌

---

### Note / Pipeline đúng

**Yêu cầu:** EC2 tự động gửi logs lên CloudWatch Logs.

```
EC2 Instance
├── Application logs (/var/log/app.log)
├── System logs (/var/log/messages)
│
├── CloudWatch Logs Agent (chạy nền)
│   ├── Đọc log files theo config
│   ├── Push lên CloudWatch Logs tự động
│   └── Cron job đảm bảo agent luôn chạy
│
▼
CloudWatch Logs
├── Log Groups
├── Log Streams
└── Có thể tạo Metric Filters, Alarms...
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Tool | Dùng để làm gì |
|---|---|
| **CloudWatch Logs Agent** | Push EC2 logs → CloudWatch Logs ✅ |
| **CloudWatch Unified Agent** | Logs + metrics từ EC2 (phiên bản mới hơn) |
| **CloudTrail** | Track API calls đến AWS resources |
| **CloudTrail Processing Library** | Xử lý CloudTrail logs (Java library) |
| **AWS Transfer for SFTP** | Transfer files vào/ra S3 qua SFTP |

> 💡 **Công thức ghi nhớ:**
> "Send EC2 logs → CloudWatch" → **CloudWatch Logs Agent**
> Agent gồm: **Plugin CLI + Daemon + Cron job**
> **CloudTrail** = API calls tracking, không phải EC2 application logs
> Phiên bản mới hơn: **CloudWatch Unified Agent** (gửi cả logs lẫn custom metrics)
