## Câu hỏi 111

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company has an application that uses multiple EC2 instances located in various AWS regions such as US East (Ohio), US West (N. California), and EU (Ireland). The manager instructed the Solutions Architect to set up a **latency-based routing** to route incoming traffic for www.tutorialsdojo.com to all the EC2 instances across all AWS regions.
>
> Which of the following options can satisfy the given requirement?

---

### 🔤 DỊCH CHI TIẾT

- **latency-based routing** (định tuyến dựa trên độ trễ): tự động chọn region nào có độ trễ thấp nhất để phục vụ người dùng
- **EC2 instances located in various AWS regions**: các máy chủ ảo EC2 nằm ở nhiều region khác nhau — Ohio, N. California, Ireland
- **route incoming traffic**: điều hướng lưu lượng truy cập đến (inbound traffic) đến đúng nơi

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Multi-region**: EC2 instances nằm ở **nhiều AWS Region** khác nhau
2. **Latency-based routing**: phải chọn region có **độ trễ thấp nhất** với người dùng
3. **Domain**: www.tutorialsdojo.com → đây là yêu cầu DNS-level routing

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **A. Use Route 53 to distribute the load to the multiple EC2 instances across all AWS Regions.**

- Route 53 hỗ trợ **Latency-Based Routing Policy** — tạo nhiều record cùng tên (www.tutorialsdojo.com) nhưng mỗi record trỏ đến một region khác nhau
- Route 53 tự động đo độ trễ → chọn record có độ trễ thấp nhất → trả về IP của EC2 tương ứng
- Đây là **cách duy nhất** có thể định tuyến ở cấp độ DNS qua nhiều region

❌ **B. Use a Network Load Balancer to distribute the load to the multiple EC2 instances across all AWS Regions.**

- NLB hoạt động **trong một region duy nhất**, không native cross-region
- NLB không có latency-based routing policy
- Ghi chú: NLB có hỗ trợ target IP ở VPC peered khác region, nhưng đề không đề cập VPC peering

❌ **C. Use AWS DataSync to distribute the load to the multiple EC2 instances across all AWS Regions.**

- **AWS DataSync** là dịch vụ **chuyển dữ liệu** (data transfer), không phải load balancing hay traffic routing
- Hoàn toàn sai use case

❌ **D. Use an Application Load Balancer to distribute the load to the multiple EC2 instances across all AWS Regions.**

- ALB hoạt động **trong một region duy nhất**
- Không hỗ trợ cross-region routing hay latency-based routing theo kiểu DNS

---

**📌 Note — Pipeline đúng:**

```
User ở Europe
     ↓ DNS query → www.tutorialsdojo.com
Route 53 (Latency-Based Routing Policy)
     ↓ đo latency → EU (Ireland) thấp nhất
Trả về IP của EC2 ở EU (Ireland)
     ↓
User kết nối trực tiếp đến EC2 Ireland
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Service | Routing scope | Latency-based? | Cross-region? |
|---|---|---|---|
| **Route 53** | DNS (global) | ✅ Có | ✅ Có |
| **ALB** | HTTP/HTTPS trong 1 region | ❌ Không | ❌ Không |
| **NLB** | TCP/UDP trong 1 region | ❌ Không | ❌ Không |
| **AWS DataSync** | Data transfer tool | ❌ Không | N/A |

> 💡 **Công thức ghi nhớ:**
> **"Cross-region routing → Route 53"**
> Load Balancer (ALB/NLB) = **trong 1 region**
> Route 53 = **global DNS, biết region nào gần user nhất**

> 🔑 **Latency-Based Routing** = tạo nhiều Route 53 record cùng tên → mỗi record gắn với 1 AWS Region → Route 53 tự động chọn region có latency thấp nhất khi nhận DNS query
