## Câu hỏi 65

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A **company is using Amazon S3** to store **frequently accessed data**. The **S3 bucket is shared with external users** that will **upload files regularly**. A **Solutions Architect** needs to implement a solution that will **grant the bucket owner full access to all uploaded objects** in the S3 bucket.
>
> What action should be done to achieve this task?

---

### 🔤 DỊCH CHI TIẾT

Một công ty sử dụng **Amazon S3** để lưu trữ **frequently accessed data** (dữ liệu được truy cập thường xuyên). **S3 bucket is shared with external users** (S3 bucket được chia sẻ với người dùng bên ngoài) sẽ **upload files regularly** (tải lên tệp thường xuyên). **Solutions Architect** cần triển khai một giải pháp sẽ **grant the bucket owner full access to all uploaded objects** (cấp quyền truy cập toàn bộ cho chủ sở hữu bucket đối với tất cả các đối tượng được tải lên) trong S3 bucket. **What action should be done to achieve this task?** (Hành động nào nên được thực hiện để hoàn thành nhiệm vụ này?)

---

### 🎯 PHÂN TÍCH YÀU CẦU CHÍNH

1. **External users upload files**: người dùng bên ngoài upload file
2. **Bucket owner needs full access**: chủ bucket cần full access
3. **Grant full control**: cấp quyền điều khiển toàn bộ
4. **ACL**: Access Control List (danh sách kiểm soát truy cập)

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

❌ **A. Enable Requester Pays feature**

- Requester Pays: ✅ Người tải lên trả phí
- ❌ KHÔNG liên quan đến quyền truy cập
- ❌ KHÔNG grant bucket owner access

✅ **B. Create bucket policy requiring object ACL = bucket-owner-full-control**

- **Bucket policy**: ✅ Enforce rules cho uploads
- **bucket-owner-full-control**: ✅ Bucket owner có full access
- ✅ External user upload → forced to set ACL
- ✅ **Exact solution cho requirement**

❌ **C. Server access logging + IAM policy**

- Server access logging: ✅ Record requests
- ❌ Logging KHÔNG grant access
- ❌ IAM policy sai (phải bucket policy)
- ❌ Phải là object's ACL (không phải bucket's ACL)

❌ **D. Create CORS configuration**

- CORS: ✅ Cross-Origin Resource Sharing
- ❌ Chỉ cho phép cross-origin access
- ❌ KHÔNG liên quan đến ownership

---

**📌 Note — S3 Object Ownership Pipeline:**

```
TRƯỚC (KHÔNG có bucket policy):

External user uploads file
     ↓
S3 Object created
     ├─ **Owner = External user account** ❌
     ├─ Bucket owner = KHÔNG full access
     └─ Bucket owner phải request access

Bucket owner view object:
     ❌ DENIED (owned by external user)


SAU (CÓ bucket policy):

External user uploads file
     ↓ (bucket policy enforcement)
     ↓ User MUST set ACL: bucket-owner-full-control
     
S3 Object created
     ├─ **Owner = Bucket owner account** ✅
     ├─ Bucket owner = FULL access
     └─ Bucket owner có quyền delete, modify, ...

Bucket owner view object:
     ✅ ALLOWED (fully controlled)
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Feature | Purpose | Solve requirement? |
|---|---|---|
| **Bucket policy + ACL** | Enforce ownership rules | ✅ **Yes** |
| Requester Pays | Charge requester | ❌ No |
| Server access logging | Track requests | ❌ No |
| CORS configuration | Cross-origin access | ❌ No |
| IAM policy | User permissions | ❌ No (wrong type) |

| ACL Setting | Object owner | Bucket owner access? |
|---|---|---|
| **bucket-owner-full-control** | Bucket owner | ✅ **FULL** |
| public-read | External user | ❌ No (limited) |
| private | External user | ❌ No (private) |
| aws-exec-read | AWS internal | ⚠️ Limited |

> 💡 **Công thức ghi nhớ:**
> **"External user upload + need bucket owner access → Bucket policy + bucket-owner-full-control ACL"**

> 🔑 **Keyword bẫy:**
> ❌ **"Requester Pays"** → Payment, KHÔNG ownership
> ❌ **"Server access logging"** → Logging, KHÔNG access control
> ❌ **"CORS"** → Cross-origin, KHÔNG ownership
> ❌ **"IAM policy"** → Wrong type (phải bucket policy)
> ✅ **"Bucket policy"** → Enforce rules cho bucket
> ✅ **"bucket-owner-full-control"** → Full control ACL
