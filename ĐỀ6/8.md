## Câu hỏi 8

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company is planning to deploy a **High Performance Computing (HPC) cluster** in its VPC that requires a scalable, high-performance file system. The storage service must be optimized for efficient workload processing, and the data must be accessible via a **fast and scalable file system interface**. It should also **work natively with Amazon S3** that enables you to easily process your S3 data with a **high-performance POSIX interface**.
>
> Which of the following is the MOST suitable service?

---

### 🔤 DỊCH CHI TIẾT

HPC cluster cần:
- **High-performance file system** (hệ thống file hiệu năng cao)
- **Fast + scalable** file system interface
- **Natively work with S3** (tích hợp trực tiếp với S3)
- **POSIX interface** (chuẩn giao tiếp file system của Linux/Unix)

**Từ khóa then chốt:** "HPC" + "natively with S3" + "POSIX" → **FSx for Lustre**

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **HPC workload** → cần file system chuyên dụng cho HPC
2. **Natively work with S3** → EFS và FSx for Windows không có tính năng này
3. **POSIX interface** → chuẩn Linux → loại FSx for Windows (dùng SMB)

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Amazon FSx for Lustre**

- Chuyên biệt cho **HPC, ML, video processing, financial modeling** ✅
- **Natively integrate với S3** → S3 objects hiển thị như files, write results back to S3 ✅
- **POSIX-compliant** → Linux/Unix compatible ✅
- Sub-millisecond latency, hàng trăm GB/s throughput, triệu IOPS ✅
- Lustre = Linux + Cluster file system ✅

---

❌ **Amazon FSx for Windows File Server**

- Dùng cho **Windows-based applications** ❌
- Giao thức **SMB** (không phải POSIX) ❌
- **Không natively work với S3** ❌

---

❌ **Amazon EFS (Elastic File System)**

- Cloud-native file system cho **Linux** (NFS protocol) ✅ Linux
- **Không natively work với S3** ❌
- Phù hợp general purpose, không tối ưu cho HPC ❌

---

❌ **Amazon EBS (Elastic Block Storage)**

- **Block storage** gắn vào 1 EC2 instance ❌
- Không phải **scalable file system** ❌
- Không shared, không dùng cho HPC cluster ❌

---

### Note / Pipeline đúng

**Yêu cầu:** HPC cluster + S3 natively + POSIX + high performance.

```
S3 bucket (data source)
    │ natively linked
    ▼
FSx for Lustre file system
    │ POSIX interface (NFS-like)
    ▼
HPC Cluster (EC2 instances)
→ Đọc/ghi dữ liệu tốc độ cao (sub-ms latency)
→ Kết quả ghi ngược lại S3 tự động ✅
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Service | Use case | Protocol | Native S3? | POSIX? |
|---|---|---|---|---|
| **FSx for Lustre** | HPC, ML, big data | POSIX/NFS | ✅ | ✅ |
| **FSx for Windows** | Windows apps | SMB | ❌ | ❌ |
| **EFS** | General Linux shared storage | NFS | ❌ | ✅ |
| **EBS** | Single EC2 block storage | N/A | ❌ | ❌ |

> 💡 **Công thức ghi nhớ:**
> **HPC + S3 natively + POSIX** → **FSx for Lustre**
> **Windows + SMB** → **FSx for Windows**
> **Linux + shared + general purpose** → **EFS**
> **Single EC2 + block** → **EBS**
>
> Mẹo nhớ: **Lustre = Linux + Cluster** — tên nói lên tất cả, sinh ra cho HPC trên Linux
