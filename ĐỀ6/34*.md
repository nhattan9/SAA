## Câu hỏi 34

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> An e-commerce company plans to optimize its disaster recovery configuration using AWS Cloud to minimize operational disruptions during outages or major system maintenance for its on-premises Microsoft SQL Server-based application. The objective is to achieve a recovery point objective (RPO) of 60 seconds or less and a recovery time objective (RTO) of 1 hour.
>
> Which of the following is the MOST cost-effective solution for this scenario?

---

### 🔤 DỊCH CHI TIẾT

Công ty e-commerce muốn tối ưu cấu hình **DR** (Disaster Recovery — phục hồi thảm họa) trên AWS cho ứng dụng **Microsoft SQL Server on-premises**. Mục tiêu:
- **RPO** (Recovery Point Objective — lượng dữ liệu tối đa có thể mất) ≤ **60 giây**
- **RTO** (Recovery Time Objective — thời gian tối đa để khôi phục) ≤ **1 giờ**
- Giải pháp phải **most cost-effective** (tiết kiệm chi phí nhất).

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **RPO ≤ 60 giây** — replication phải **liên tục**, không thể backup theo giờ hay ngày.
2. **RTO ≤ 1 giờ** — phục hồi trong 1 giờ là đủ, không cần instant failover.
3. **Most cost-effective** — RTO 1 giờ cho phép dùng **pilot light** (tài nguyên tối thiểu ở AWS) thay vì chạy hệ thống đầy đủ 24/7.

> ⚠️ Điểm mấu chốt: RPO 60s bắt buộc phải có **continuous replication**, nhưng RTO 1h **không cần** warm standby hay active/active đắt tiền → **pilot light là đủ và rẻ nhất**.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

---

✅ **Đáp án đúng (A):** **Pilot light** dùng **AWS Elastic Disaster Recovery (AWS DRS)**

- **AWS DRS** thực hiện **continuous block-level replication** → RPO tính bằng **giây** (< 60s). ✅
- Khi có disaster → launch recovery instances → RTO **5–20 phút** (tốt hơn yêu cầu 1h). ✅
- **Pilot light** = chỉ duy trì staging area (storage rẻ + minimal compute), **không chạy server đầy đủ** → **chi phí thấp nhất**. ✅

---

❌ **Đáp án sai (B):** **Warm standby** dùng Amazon RDS for SQL Server + AWS DMS CDC

- Đáp ứng được RPO và RTO ✅ nhưng RDS instance phải **chạy liên tục 24/7** dù chưa cần dùng → **chi phí cao hơn DRS pilot light**.

---

❌ **Đáp án sai (C):** AWS Storage Gateway + EBS fast snapshot restore

- Storage Gateway là giải pháp **backup hybrid**, không có **continuous replication**.
- **Không có automated failover** → cần can thiệp thủ công → không đảm bảo RPO 60s và RTO 1h.

---

❌ **Đáp án sai (D):** SQL Server Enterprise **Always On** + multi-site active/active

- Đáp ứng RPO và RTO ✅ nhưng cần **nhiều replication server chạy liên tục** → **chi phí đắt nhất** trong 4 lựa chọn.
- Đề bài yêu cầu **most cost-effective** → loại ngay.

---

### Note / Pipeline đúng

Yêu cầu: RPO ≤ 60s, RTO ≤ 1h, chi phí thấp nhất cho SQL Server on-premises.

```
On-premises SQL Server
→ Cài AWS DRS agent trên server nguồn
→ Continuous block-level replication → Staging area trên AWS
   (storage rẻ + minimal compute = chi phí thấp, không chạy server đầy đủ)
→ Khi có disaster:
   → Launch recovery instances từ snapshot gần nhất
   → RPO = vài giây | RTO = 5–20 phút
→ Sau khi khắc phục: failback về on-premises qua DRS Failback Client
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| DR Strategy | RPO | RTO | Chi phí | Ghi chú |
|---|---|---|---|---|
| **Backup & Restore** | Giờ | Giờ–Ngày | Thấp nhất | RPO/RTO không nghiêm ngặt |
| **Pilot Light (AWS DRS)** | Giây | Phút | Thấp ✅ | RPO nghiêm ngặt, RTO vừa phải |
| **Warm Standby** | Giây | Phút | Trung bình | RDS chạy 24/7 → đắt hơn |
| **Active/Active** | ~0 | ~0 | Cao nhất | Không chịu được bất kỳ downtime |

💡 **Công thức ghi nhớ:**

> **"RPO vài giây + cost-effective"** → **AWS DRS Pilot Light** — continuous replication, staging area rẻ, không chạy server đầy đủ

> **"Warm standby"** = RDS chạy liên tục → đắt hơn DRS pilot light dù cùng đáp ứng RPO/RTO

> **"Always On active/active"** = đắt nhất → chỉ dùng khi zero downtime tuyệt đối

> ⚠️ **RPO** = mất bao nhiêu **data** | **RTO** = mất bao nhiêu **thời gian** để recover — hai chỉ số độc lập, cần đọc kỹ cả hai trước khi chọn đáp án
