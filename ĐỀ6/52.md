## Câu hỏi 53

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> An online stock trading system is hosted in AWS and uses an **AWS Auto Scaling group** of **Amazon EC2 instances**, an **Amazon Relational Database Service (Amazon RDS) database**, and an **Amazon ElastiCache for Redis**. To improve the **data security of the in-memory data store**, it is **required for users to enter a password** before being granted permission to execute **Redis commands**.
>
> Which of the following should be done to meet the above requirement?

---

### 🔤 DỊCH CHI TIẾT

Một hệ thống giao dịch chứng chỉ trực tuyến được lưu trữ trên AWS và sử dụng Auto Scaling group của Amazon EC2 instances, Amazon RDS database, và Amazon ElastiCache for Redis. Để cải thiện bảo mật dữ liệu của in-memory data store, người dùng phải nhập mật khẩu trước khi được cấp quyền thực hiện các Redis commands. Điều nào sau đây nên được thực hiện để đáp ứng yêu cầu trên?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **In-memory data store security**: bảo mật dữ liệu trong bộ nhớ
2. **Password-based authentication**: yêu cầu mật khẩu trước khi chạy Redis commands
3. **Redis AUTH**: giải pháp nên dùng

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

❌ **A. None of the above**

- Sai, vì có đáp án đúng

❌ **B. Create new Redis replication group + set AtRestEncryptionEnabled = true**

- At-Rest Encryption: ✅ Bảo mật dữ liệu lúc nằm yên trong bộ nhớ
- ❌ NHƯNG không enforce password-based access
- Cần **Redis AUTH**

✅ **C. Authenticate users using Redis AUTH by creating new Redis Cluster + --transit-encryption-enabled + --auth-token**

- **--auth-token**: ✅ Yêu cầu user nhập password trước execute Redis commands
- **--transit-encryption-enabled**: ✅ Encrypt dữ liệu lúc truyền tải
- Đây là **đầy đủ** cho yêu cầu password-based authentication

❌ **D. Create new Redis replication group + enable AWS KMS managed keys (CSE-KMS)**

- CSE-KMS: client-side encryption, ❌ KHÔNG enforce Redis authentication
- Không enforce password-based access

❌ **E. Enable in-transit encryption for Redis replication groups**

- In-transit encryption: ✅ Bảo mật dữ liệu lúc truyền tải
- ❌ NHƯNG không enforce password → KHÔNG đáp ứng requirement

---

**📌 Note — Redis AUTH pipeline:**

```
User request → Redis Cluster
     ↓ --auth-token enabled
     ↓ User must enter password
     ↓ Redis AUTH verification
     ├─ ✅ Password correct → execute Redis command
     └─ ❌ Password wrong → reject
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Encryption/Auth Type | Password Required? | Protect at-rest? | Protect in-transit? | Đáp ứng yêu cầu? |
|---|---|---|---|---|
| **Redis AUTH (--auth-token)** | ✅ **Yes** | ❌ No | ✅ Yes | ✅ **Yes** |
| At-Rest Encryption | ❌ No | ✅ Yes | ❌ No | ❌ No |
| In-Transit Encryption | ❌ No | ❌ No | ✅ Yes | ❌ No |
| CSE-KMS | ❌ No | ✅ Yes | ❌ No | ❌ No |
| **Auth + Transit encryption** | ✅ **Yes** | ❌ No | ✅ Yes | ✅ **Yes** |

> 💡 **Công thức ghi nhớ:**
> **"Redis password authentication → --auth-token"**
> **"Protect data in-transit → --transit-encryption-enabled"**
> **"Protect data at-rest → AtRestEncryptionEnabled"**

> 🔑 **Keyword bẫy:**
> ❌ **"At-Rest Encryption"** → Bảo mật dữ liệu nằm yên, KHÔNG phải authentication
> ❌ **"In-Transit Encryption only"** → Bảo mật lúc truyền tải, KHÔNG phải password
> ❌ **"KMS"** → Client-side encryption, KHÔNG liên quan Redis AUTH
> ✅ **"--auth-token"** → **Enforce password-based access**
> ✅ **"--transit-encryption-enabled"** → Bảo mật lúc truyền tải
