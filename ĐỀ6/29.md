## Câu hỏi 29

---

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company is looking for a way to analyze the calls between customers and service agents. Each conversation is transcribed, JSON-formatted, and saved to an Amazon S3 bucket. The company's solutions architect is tasked to design a solution for extracting and visualizing sentiments from the transcribed files.
>
> Which solution meets the requirements while minimizing the amount of operational overhead?

---

### 🔤 DỊCH CHI TIẾT

Công ty muốn phân tích các cuộc gọi giữa khách hàng và nhân viên. Mỗi cuộc hội thoại đã được **transcribe** (chuyển giọng nói thành văn bản), định dạng JSON, lưu trên S3. Yêu cầu thiết kế giải pháp để **extract** (trích xuất) và **visualize** (trực quan hóa) **sentiments** (cảm xúc/thái độ — tích cực, tiêu cực, trung tính) từ các file này, với **minimum operational overhead**.

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Sentiment analysis** từ văn bản — cần AI/ML service chuyên về NLP, không cần tự train model.
2. **Visualize kết quả** — cần dashboard phù hợp với dữ liệu đã index vào OpenSearch.
3. **Minimum operational overhead** — ưu tiên managed service có sẵn, tránh tự build model hay cấu hình phức tạp.

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

---

✅ **Đáp án đúng (A):** **Amazon Comprehend** + Amazon OpenSearch + **OpenSearch Dashboard**

- **Amazon Comprehend** là managed NLP service — phân tích sentiment (positive/negative/neutral/mixed) từ văn bản JSON mà **không cần train model**. ✅
- Index kết quả vào **Amazon OpenSearch** cluster để lưu trữ và tìm kiếm. ✅
- **OpenSearch Dashboard** là công cụ visualize **built-in** của OpenSearch — không cần cấu hình thêm, tích hợp tự nhiên, overhead thấp nhất. ✅

---

❌ **Đáp án sai (B):** **Amazon Textract** + OpenSearch + Amazon Managed Grafana

- **Amazon Textract** dùng để **extract text từ ảnh/PDF scan** (PNG, JPEG, TIFF, PDF) — **không có khả năng sentiment analysis**.
- File đã ở dạng JSON text sẵn rồi → dùng Textract là sai mục đích.
- **Grafana** phù hợp cho **time-series metrics** (CPU, memory, nhiệt độ...) — không phải cho sentiment data. Tích hợp với OpenSearch cần cấu hình thêm → overhead cao hơn.

---

❌ **Đáp án sai (C):** **Amazon SageMaker** (train custom NLP model) + OpenSearch + OpenSearch Dashboard

- **SageMaker** có thể làm được, nhưng **tự train NLP model** tốn thời gian, công sức, expertise về ML — operational overhead rất cao.
- Comprehend đã có sẵn sentiment analysis, không cần reinvent the wheel.

---

❌ **Đáp án sai (D):** Amazon Comprehend + OpenSearch + **Amazon Managed Grafana**

- Comprehend là đúng ✅, nhưng dùng **Grafana** để visualize sentiment data từ OpenSearch cần **cấu hình data source riêng** trong Grafana workspace → thêm bước, thêm overhead.
- **OpenSearch Dashboard** tích hợp trực tiếp với OpenSearch cluster → đơn giản hơn, không cần cấu hình thêm.

---

### Note / Pipeline đúng

Yêu cầu: Phân tích sentiment từ transcript JSON trên S3, visualize kết quả, tối thiểu overhead.

```
Transcript JSON files lưu trên Amazon S3
→ Amazon Comprehend analysis job
   (phân tích sentiment: positive/negative/neutral/mixed)
→ Index sentiment + transcript → Amazon OpenSearch cluster
→ OpenSearch Dashboard
   (visualize trực tiếp, built-in, không cần cấu hình thêm)
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Yêu cầu / Tính năng | Service phù hợp |
|---|---|
| Sentiment analysis, NLP từ văn bản | **Amazon Comprehend** |
| Extract text từ ảnh/PDF scan | **Amazon Textract** |
| Train custom ML model | **Amazon SageMaker** |
| Lưu trữ & tìm kiếm dữ liệu phân tích | **Amazon OpenSearch** |
| Visualize dữ liệu OpenSearch (built-in) | **OpenSearch Dashboard** |
| Visualize time-series metrics | **Amazon Managed Grafana** |

💡 **Công thức ghi nhớ:**

> **"Sentiment / NLP / key phrase / entity"** từ văn bản → **Amazon Comprehend** (không cần train)

> **"Extract text từ ảnh, scan, PDF"** → **Amazon Textract** (≠ sentiment analysis)

> **"OpenSearch Dashboard"** = built-in, zero config → luôn ít overhead hơn Grafana khi data đã ở OpenSearch

> ⚠️ **Grafana** giỏi time-series metrics, **không phải** lựa chọn tối ưu cho NLP/sentiment data
