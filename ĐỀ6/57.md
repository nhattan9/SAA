## Câu hỏi 58

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A **web application**, which is **hosted in the on-premises data center** and uses a **MySQL database**, must be **migrated to AWS Cloud**. You need to ensure that the **network traffic to and from your RDS database instance is encrypted using SSL**. For **improved security**, you have to use the **profile credentials specific to your EC2 instance** to **access your database**, instead of a **password**.
>
> Which of the following should you do to meet the above requirement?

---

### 🔤 DỊCH CHI TIẾT

Một **web application** (ứng dụng web) được **hosted in the on-premises data center** (lưu trữ trong trung tâm dữ liệu tại chỗ) và sử dụng **MySQL database** (cơ sở dữ liệu MySQL) phải được **migrated to AWS Cloud** (di chuyển sang AWS Cloud). Bạn cần đảm bảo rằng **network traffic to and from your RDS database instance is encrypted using SSL** (lưu lượng mạng đến và từ RDS database instance được mã hóa bằng SSL). Vì **improved security** (bảo mật cải tiến), bạn phải sử dụng **profile credentials specific to your EC2 instance** (thông tin xác thực của hồ sơ cụ thể cho EC2 instance) để **access your database** (truy cập cơ sở dữ liệu) thay vì **password** (mật khẩu). Bạn nên làm điều gì sau đây để đáp ứng yêu cầu trên?

---

### 🎯 PHÂN TÍCH YÀU CẦU CHÍNH

1. **Network traffic encrypted with SSL**: lưu lượng mạng mã hóa
2. **Use EC2 instance profile credentials**: NOT password → thông tin xác thực từ profile EC2
3. **RDS MySQL**: cơ sở dữ liệu MySQL trên RDS

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

❌ **A. Configure your RDS database to enable encryption**

- RDS encryption at-rest: ✅ Bảo mật dữ liệu nằm yên (storage)
- ❌ KHÔNG encrypt network traffic
- ❌ KHÔNG cho phép sử dụng profile credentials

❌ **B. Launch mysql client using --ssl-ca parameter**

- --ssl-ca: ✅ Encrypt network traffic via SSL
- ❌ KHÔNG cho phép sử dụng profile credentials (vẫn cần password)
- Chỉ giải quyết được 1/2 requirement

✅ **C. Set up an RDS database and enable the IAM DB Authentication**

- **IAM DB Authentication**:
  - ✅ Encrypt network traffic using **SSL**
  - ✅ Use **EC2 instance profile credentials** (KHÔNG cần password)
  - ✅ Authentication token (lifetime 15 minutes)
  - ✅ Centrally manage access via IAM
- ✅ Đáp ứng **cả 2 yêu cầu**

❌ **D. Launch new RDS database instance using Aurora with Backtrack enabled**

- Backtrack: ✅ Undo destructive actions (DELETE, UPDATE)
- ❌ KHÔNG liên quan đến SSL encryption
- ❌ KHÔNG liên quan đến profile credentials authentication

---

**📌 Note — IAM DB Authentication pipeline:**

```
EC2 instance
     ↓ EC2 instance profile (IAM role)
     ↓ request authentication token
RDS with IAM DB Auth enabled
     ├─ Generate token (valid 15 min)
     ├─ Token sent over SSL encrypted connection
     ├─ Verify token using IAM credentials
     └─ ✅ Connection established (NO password needed)
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Requirement | Solution | Method |
|---|---|---|
| **Network traffic encrypted** | ✅ IAM DB Auth | Uses SSL (mandatory) |
| **Use EC2 profile credentials** | ✅ IAM DB Auth | IAM role + token |
| **NO password** | ✅ IAM DB Auth | Token-based auth |

| Feature | At-Rest Encryption | IAM DB Auth | --ssl-ca | Backtrack |
|---|---|---|---|---|
| **Network SSL encryption** | ❌ No | ✅ Yes | ✅ Yes | ❌ No |
| **EC2 profile credentials** | ❌ No | ✅ Yes | ❌ No | ❌ No |
| **No password needed** | ❌ No | ✅ Yes | ❌ No | ❌ No |

> 💡 **Công thức ghi nhớ:**
> **"SSL encryption + EC2 profile credentials → IAM DB Authentication"**
> **"NO password required → IAM DB Auth"**

> 🔑 **Keyword bẫy:**
> ❌ **"RDS encryption"** → At-rest, KHÔNG network traffic
> ❌ **"--ssl-ca"** → SSL encryption only, vẫn cần password
> ❌ **"Backtrack"** → Undo changes, KHÔNG authentication
> ✅ **"IAM DB Authentication"** → SSL + profile credentials + no password
