## Câu hỏi 54

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A Solutions Architect needs to set up the required compute resources for the application which have **workloads that require high, sequential read and write access to very large data sets on local storage**.
>
> Which of the following instance type is the most suitable one to use in this scenario?

---

### 🔤 DỊCH CHI TIẾT

Một Solutions Architect cần thiết lập các **compute resources** (tài nguyên tính toán) cần thiết cho ứng dụng có **workloads** (khối lượng công việc) yêu cầu **high, sequential read and write access** (khả năng đọc và ghi tuần tự cao) đến **very large data sets on local storage** (các tập dữ liệu rất lớn trên bộ nhớ cục bộ). **Instance type** (loại instance) nào sau đây phù hợp nhất để sử dụng trong kịch bản này?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **High sequential read and write**: đọc/ghi **tuần tự** (sequential) với throughput cao
2. **Very large data sets**: dữ liệu rất lớn
3. **Local storage**: lưu trữ cục bộ (EBS hoặc instance store)
4. **High IOPS**: I/O operations per second cao

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

❌ **A. Compute Optimized Instances**

- Cho **CPU-intensive workloads** (batch processing, media transcoding, scientific simulation)
- ❌ KHÔNG tối ưu cho I/O và storage access

✅ **B. Storage Optimized Instances**

- ✅ Thiết kế cho **high sequential read/write** trên local storage
- ✅ **Tens of thousands IOPS** (10,000+ IOPS)
- ✅ **Low-latency** random I/O
- Ví dụ: i3, i4i instance families

❌ **C. General Purpose Instances**

- Balance giữa compute, memory, networking
- ❌ KHÔNG đủ mạnh cho **high read/write access** requirement

❌ **D. Memory Optimized Instances**

- Cho **in-memory workloads** (cache, database, analytics)
- ❌ KHÔNG tối ưu cho **local storage I/O**

---

**📌 Note — EC2 Instance Type Pipeline:**

```
Requirement: High sequential read/write on local storage
     ↓
Check instance family:
     ├─ Compute (CPU-bound) ❌
     ├─ Memory (In-memory) ❌
     ├─ General Purpose (balanced) ❌
     └─ Storage Optimized (I/O-intensive) ✅
     
Storage Optimized (i3, i4i):
     ├─ NVMe SSD storage
     ├─ High sequential throughput
     ├─ Tens of thousands IOPS
     └─ Low-latency random I/O
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Instance Type | Workload | Storage I/O | IOPS | Best for |
|---|---|---|---|---|
| **Storage Optimized** | I/O-intensive | ✅ **Very high** | ✅ **10K+** | **Large datasets, sequential access** |
| Memory Optimized | In-memory | ⚠️ Medium | ⚠️ 5K | Databases, caches |
| Compute Optimized | CPU-bound | ❌ Lower | ❌ < 5K | Batch, encoding |
| General Purpose | Balanced | ⚠️ Medium | ⚠️ 5K | Web servers, small workloads |

> 💡 **Công thức ghi nhớ:**
> **"High sequential read/write on local storage → Storage Optimized"**
> **"In-memory processing → Memory Optimized"**
> **"CPU-intensive → Compute Optimized"**

> 🔑 **Keyword bẫy:**
> ❌ **"High I/O"** → KHÔNG phải Compute hoặc Memory
> ✅ **"Sequential read/write on local storage"** → **Storage Optimized** (i3, i4i)
> ✅ **"Tens of thousands IOPS"** → Storage Optimized only
