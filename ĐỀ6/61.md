## Câu hỏi 62

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company plans to **launch an application that tracks the GPS coordinates** of **delivery trucks** in the country. The **coordinates are transmitted from each delivery truck every five seconds**. The **must be able to process coordinates from multiple consumers in real-time**. The **aggregated data will be analyzed in a separate reporting application**.
>
> Which AWS service should you use for this scenario?

---

### 🔤 DỊCH CHI TIẾT

Một công ty có kế hoạch **launch an application that tracks the GPS coordinates** (khởi chạy ứng dụng theo dõi tọa độ GPS) của **delivery trucks** (những chiếc xe giao hàng) ở **in the country** (trong đất nước). **Coordinates are transmitted from each delivery truck every five seconds** (tọa độ được truyền từ mỗi xe giao hàng cứ 5 giây một lần). **Must be able to process coordinates from multiple consumers in real-time** (phải có khả năng xử lý tọa độ từ nhiều người dùng/consumer trong thời gian thực). **Aggregated data will be analyzed in a separate reporting application** (dữ liệu tổng hợp sẽ được phân tích trong một ứng dụng báo cáo riêng). Bạn nên sử dụng dịch vụ AWS nào cho kịch bản này?

---

### 🎯 PHÂN TÍCH YÀU CẦU CHÍNH

1. **Real-time data ingestion**: nhập dữ liệu thời gian thực (GPS mỗi 5 giây)
2. **High-velocity stream**: dòng dữ liệu tốc độ cao (nhiều truck → nhiều coordinates)
3. **Process multiple consumers**: xử lý từ nhiều producer (truck)
4. **Separate reporting**: dữ liệu sau được phân tích riêng (batch analysis)

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **A. Amazon Kinesis**

- ✅ **Real-time streaming data**: ingest dữ liệu real-time
- ✅ **High-velocity**: xử lý dòng dữ liệu tốc độ cao (GPS 5s/lần)
- ✅ **Multiple producers**: nhiều truck gửi coordinates
- ✅ **Process on arrival**: xử lý khi dữ liệu tới (KHÔNG đợi batch)
- ✅ **Aggregated data**: tích lũy dữ liệu → gửi reporting app

❌ **B. Amazon Elastic MapReduce (EMR)**

- EMR: ✅ Big data processing (Hadoop, Spark)
- ❌ Batch mode (không real-time)
- ❌ Không tối ưu cho streaming data

❌ **C. Amazon Simple Queue Service (SQS)**

- SQS: ✅ Message queue, distributed messaging
- ❌ KHÔNG tối ưu cho **real-time, high-velocity** streams
- ❌ Có latency, KHÔNG tức thời

❌ **D. Amazon AppStream**

- AppStream: ✅ Stream desktop applications
- ❌ Dùng để stream **applications** (UI), KHÔNG phải data streams

---

**📌 Note — Amazon Kinesis Architecture:**

```
GPS Delivery Trucks (1000+ trucks)
     ├─ Truck 1: GPS coordinate every 5s
     ├─ Truck 2: GPS coordinate every 5s
     ├─ Truck 3: GPS coordinate every 5s
     └─ ... (1000+ trucks)
     
     ↓ gửi stream data
     
Amazon Kinesis Stream
     ├─ Shard 1 (receive data)
     ├─ Shard 2 (receive data)
     └─ Shard N (receive data)
     
     ↓ Real-time processing
     
Real-time Application
     ├─ Track truck locations live
     ├─ Send alerts/notifications
     └─ Dashboard updates
     
     ↓ Aggregate data
     
Reporting Application
     ├─ Batch analysis
     ├─ Generate reports
     └─ Historical data analysis
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Service | Real-time? | High-velocity? | Streaming? | Reporting? |
|---|---|---|---|---|
| **Amazon Kinesis** | ✅ Yes | ✅ Yes | ✅ Yes | ✅ Yes (via aggregation) |
| Amazon EMR | ❌ Batch | ❌ Batch | ❌ No | ✅ Yes |
| Amazon SQS | ⚠️ Latency | ❌ Lower | ⚠️ Limited | ❌ No |
| Amazon AppStream | ❌ No | ❌ No | ❌ No (UI) | ❌ No |

| Use Case | Best Service |
|---|---|
| **GPS streaming from trucks** | **Amazon Kinesis** ✅ |
| Big data batch analysis | Amazon EMR |
| Message queuing | Amazon SQS |
| Desktop app streaming | Amazon AppStream |

> 💡 **Công thức ghi nhớ:**
> **"Real-time streaming data (high-velocity) → Amazon Kinesis"**
> **"Batch processing big data → Amazon EMR"**
> **"Message queuing (can tolerate latency) → Amazon SQS"**
> **"Stream applications (UI) → Amazon AppStream"**

> 🔑 **Keyword bẫy:**
> ✅ **"Every five seconds"** → Real-time streaming = Kinesis
> ✅ **"Real-time processing"** → Kinesis, NOT SQS
> ❌ **"Aggregated data for reporting"** → Kinesis aggregates then sends to reporting app
> ❌ **"EMR"** → Batch processing, not real-time
