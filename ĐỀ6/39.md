## Câu hỏi 106

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> An HPC system uses **Amazon ECS** (EC2 launch type). **Hundreds of ECS tasks** run most of the time. These tasks require **high-frequency read and write** operations to a **shared storage system**. Output data per task is ~10 MB, total storage will not exceed **10 TB**.
>
> Which is the MOST suitable solution?

---

### 🔤 DỊCH CHI TIẾT

HPC system với ECS cần shared storage có:
- **High-frequency read/write** liên tục (không phải spiky)
- **Hàng trăm ECS tasks** đồng thời truy cập
- Tổng storage ≤ **10 TB** (nhỏ so với throughput demands)
- Mount được vào ECS task definition

**Từ khóa then chốt:** "high-frequency" + "hundreds of tasks" + "small storage but high throughput" → **EFS Provisioned Throughput + Max I/O**

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Cần phân biệt **2 lựa chọn EFS:**

**Throughput mode:**
- **Bursting** = throughput tỷ lệ theo storage size → nhỏ storage = ít throughput → không đủ
- **Provisioned** = throughput cố định, độc lập với storage size → phù hợp khi storage nhỏ nhưng cần throughput cao

**Performance mode:**
- **General Purpose** = latency thấp, phù hợp workload thông thường
- **Max I/O** = throughput cao hơn, phù hợp khi hàng trăm clients cùng truy cập

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án D:** EFS + **Provisioned Throughput** + **Max I/O** performance mode + mount vào ECS task definition.

- **Provisioned Throughput** = đảm bảo throughput cao dù storage chỉ 10 TB ✅
- **Max I/O** = tối ưu cho hàng trăm concurrent clients (ECS tasks) ✅
- EFS mount được vào ECS task definition ✅
- Shared storage cho tất cả tasks ✅

---

❌ **Đáp án A:** EFS + **Bursting Throughput** + **General Purpose**.

- **Bursting** = throughput phụ thuộc storage size ❌
- 10 TB storage → throughput burst có giới hạn → không đủ cho high-frequency workload liên tục ❌
- **General Purpose** = không tối ưu cho hàng trăm concurrent clients ❌

---

❌ **Đáp án B:** **FSx for Windows File Server** (SMB).

- Dành cho **Windows workloads** ❌
- Không cung cấp performance đủ cho high-frequency HPC workloads ❌
- SMB protocol không tối ưu cho Linux containers ❌

---

❌ **Đáp án C:** **DynamoDB + DAX + DynamoDB Streams** làm container mount point.

- DynamoDB là **database**, không phải file system ❌
- **Không thể mount** DynamoDB như file system vào container ❌
- Không có khái niệm "container mount point" với DynamoDB ❌

---

### Note / Pipeline đúng

**Yêu cầu:** Shared storage, high-frequency I/O, hundreds of ECS tasks.

```
Hundreds of ECS tasks (EC2 launch type)
    │ mount via task definition
    ▼
Amazon EFS
├── Throughput mode: PROVISIONED
│   → Throughput cố định, độc lập storage size
│   → 10 TB storage nhỏ nhưng throughput vẫn cao ✅
│
└── Performance mode: MAX I/O
    → Tối ưu cho hàng trăm concurrent connections
    → Higher throughput, slightly higher latency ✅

EFS tự động scale storage, multi-AZ redundancy ✅
```

---

### 📊 KẾT LUẬN & BÀI HỌC

**EFS Throughput Modes:**

| Mode | Throughput phụ thuộc | Dùng khi |
|---|---|---|
| **Bursting** | Storage size | Storage lớn, workload spiky |
| **Provisioned** | Bạn tự set | Storage nhỏ nhưng cần throughput cao liên tục |

**EFS Performance Modes:**

| Mode | Latency | Throughput | Dùng khi |
|---|---|---|---|
| **General Purpose** | Thấp nhất | Trung bình | < vài trăm clients |
| **Max I/O** | Cao hơn chút | Cao nhất | Hàng trăm/nghìn clients cùng lúc |

> 💡 **Công thức ghi nhớ:**
> "Storage nhỏ nhưng cần throughput cao" → **Provisioned Throughput**
> "Hàng trăm concurrent clients" → **Max I/O performance mode**
> "Workload spiky, storage lớn" → **Bursting Throughput + General Purpose**
> **DynamoDB không mount được** vào container — đây là database, không phải file system
