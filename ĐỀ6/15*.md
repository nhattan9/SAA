## Câu hỏi 99

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A company uses **AWS Certificate Manager (ACM)** provided certificates on ALBs to enable HTTPS. The security team wants to get **notified 30 days before the expiration** of the SSL certificates.
>
> Which of the following can the Solutions Architect implement? **(Select TWO.)**

---

### 🔤 DỊCH CHI TIẾT

Công ty dùng **ACM certificates** trên ALB cho HTTPS. Cần **cảnh báo trước 30 ngày** khi certificate sắp hết hạn.

**Từ khóa then chốt:** "30 days before expiration" + "ACM certificates" + "notify"

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

Có **2 cách** monitor ACM certificate expiry:
1. **EventBridge + AWS Health/ACM expiration events** → event-driven (tự động khi ACM phát event)
2. **EventBridge scheduled + CloudWatch DaysToExpiry metric** → batch check hàng ngày

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

✅ **Đáp án C:** EventBridge rule scheduled hàng ngày → check **DaysToExpiry metric** trong CloudWatch → SNS khi < 30 ngày.

- **DaysToExpiry** = CloudWatch metric của ACM, đo số ngày còn lại đến expiry ✅
- Schedule EventBridge chạy mỗi ngày → check metric → alert nếu ≤ 30 ngày ✅
- Một SNS notification cho tất cả certificates sắp hết hạn ✅

---

✅ **Đáp án D:** EventBridge rule check **AWS Health hoặc ACM expiration events** → SNS khi certificate sắp hết hạn 30 ngày.

- ACM tự động gửi **daily expiration events** từ 45 ngày trước expiry ✅
- EventBridge bắt events này → filter còn ≤ 30 ngày → SNS ✅
- Event-driven, không cần schedule ✅

---

❌ **Đáp án B:** **AWS Config** manually create rule + EventBridge → SNS.

- AWS Config đã có **managed rule sẵn** (`acm-certificate-expiration-check`) → tạo thủ công là thừa ❌
- ACM tự động generate AWS Health events → không cần Config ❌

---

❌ **Đáp án A:** Modify certificates → dùng **ACM Private CA** + EventBridge + Lambda → SNS.

- **Không cần chuyển sang Private CA** để monitor expiry ❌
- Private CA = tạo CA riêng, không liên quan đến notification ❌
- Thêm complexity không cần thiết ❌

---

❌ **Đáp án E:** **AWS Trusted Advisor** check ACM expiry → CloudWatch alarm → Systems Manager OpsItem.

- **Trusted Advisor không check ACM certificate expiry** ❌
- **OpsItem** (Systems Manager) không phải notification service phù hợp ❌

---

### Note / Pipeline đúng

**2 pipeline hợp lệ:**

```
Pipeline 1 (Event-driven):
ACM certificate sắp hết hạn
→ ACM tự động phát ACM Expiration Event / AWS Health Event
→ EventBridge bắt event (filter: DaysToExpiry ≤ 30)
→ SNS Topic
→ Email/SMS đến security team ✅

Pipeline 2 (Scheduled):
EventBridge rule (chạy mỗi ngày)
→ Kiểm tra CloudWatch metric: DaysToExpiry của tất cả ACM certs
→ Nếu DaysToExpiry ≤ 30 → trigger SNS
→ Email/SMS đến security team ✅
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Phương pháp | Cơ chế | Tự động? |
|---|---|---|
| **ACM Expiration Event + EventBridge** | Event-driven | ✅ Tự động |
| **CloudWatch DaysToExpiry + EventBridge Schedule** | Batch daily check | ✅ Scheduled |
| AWS Config custom rule | Manual, thừa | ❌ |
| Trusted Advisor | Không hỗ trợ ACM | ❌ |

> 💡 **Công thức ghi nhớ:**
> "ACM cert sắp hết hạn" → **EventBridge + ACM/AWS Health events** hoặc **CloudWatch DaysToExpiry metric**
> ACM tự phát expiration events từ **45 ngày trước** expiry
> **DaysToExpiry** = CloudWatch metric đo số ngày còn lại của ACM cert
> **Trusted Advisor không check ACM expiry** — hay bị nhầm!
