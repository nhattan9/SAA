## Câu hỏi 55

### 📋 NỘI DUNG CÂU HỎI (Tiếng Anh gốc)

> A Solutions Architect is managing a **three-tier web application** that processes **credit card payments** and **online transactions**. **Static web pages** are used on the **front-end tier** while the **application tier** contains a **single Amazon EC2 instance** that handles **long-running processes**. The data is stored in a **MySQL database**. The Solutions Architect is instructed to **decouple the tiers** to create a **highly available application**.
>
> Which of the following options can satisfy the given requirement?

---

### 🔤 DỊCH CHI TIẾT

Một Solutions Architect đang quản lý **three-tier web application** (ứng dụng web ba tầng) xử lý **credit card payments** (thanh toán thẻ tín dụng) và **online transactions** (giao dịch trực tuyến). **Static web pages** (các trang web tĩnh) được sử dụng ở **front-end tier** (tầng giao diện) trong khi **application tier** (tầng ứng dụng) chứa một **Amazon EC2 instance** (máy chủ ảo) duy nhất xử lý **long-running processes** (các quy trình chạy dài). Dữ liệu được lưu trữ trong **MySQL database** (cơ sở dữ liệu MySQL). Solutions Architect được hướng dẫn **decouple the tiers** (tách rời các tầng) để tạo **highly available application** (ứng dụng có độ sẵn sàng cao). Tùy chọn nào sau đây có thể đáp ứng yêu cầu đã cho?

---

### 🎯 PHÂN TÍCH YÊU CẦU CHÍNH

1. **Three-tier architecture**: Front-end (static) → App tier (EC2) → Database (MySQL)
2. **Decouple tiers**: Tách rời → từng tầng hoạt động độc lập
3. **Highly available**: Mỗi tầng phải có redundancy, failover, auto-scaling
4. **Long-running processes**: EC2 tier cần xử lý quy trình dài (Lambda max 15 min = không phù hợp)

---

### 🔍 PHÂN TÍCH TỪNG ĐÁP ÁN

❌ **A. CloudFront + Auto Scaling EC2 + RDS Multi-AZ**

- CloudFront: ❌ Là CDN (cache), KHÔNG phải nơi host static pages
- Cần **S3 để host static pages**, CloudFront chỉ là CDN ở phía trước

✅ **B. S3 + Amazon ECS + RDS Multi-AZ**

- **S3**: ✅ Host static web pages
- **ECS**: ✅ Container orchestration cho long-running processes
  - Có **Service Auto Scaling** → scale up/down theo demand
  - Docker containers → manage fleet of EC2 instances
- **RDS Multi-AZ**: ✅ Automated failover, high availability
- ✅ **Fully decoupled**: mỗi tầng độc lập, có redundancy

❌ **C. S3 + AWS Lambda + DynamoDB**

- S3: ✅ Static pages
- Lambda: ❌ Max timeout = **15 minutes**, KHÔNG thể xử lý **long-running processes**
- DynamoDB: ❌ NoSQL, KHÔNG hợp cho credit card payment system (cần ACID)

❌ **D. Larger EC2 + Auto Scaling EC2 + Aurora**

- Larger EC2: ❌ Gom hết (static + app) vào instance → KHÔNG decouple
- Static pages vào EC2: ❌ KHÔNG cost-effective, nên dùng S3

---

**📌 Note — Three-tier decoupled architecture:**

```
Tier 1: Front-end (Static)
     ├─ AWS S3 (host static pages)
     └─ CloudFront (CDN optional)

Tier 2: Application (Business logic)
     ├─ Amazon ECS (Docker containers)
     ├─ Service Auto Scaling (scale up/down)
     └─ Load Balancer (distribute requests)

Tier 3: Database
     ├─ Amazon RDS Multi-AZ (automated failover)
     └─ Highly available MySQL/Aurora
```

---

### 📊 KẾT LUẬN & BÀI HỌC

| Tầng | Best Service | Why | Decoupled? |
|---|---|---|---|
| **Static assets** | **S3 + CloudFront** | ✅ Cost-effective, CDN | ✅ Yes |
| **Long-running app** | **ECS + Auto Scaling** | ✅ Handles 15+ min processes | ✅ Yes |
| **Database** | **RDS Multi-AZ** | ✅ ACID, failover, HA | ✅ Yes |

| Service | Long-running? | Static hosting? | Use case |
|---|---|---|---|
| **Lambda** | ❌ Max 15 min | ❌ No | Event-driven, short jobs |
| **ECS** | ✅ Yes (unlimited) | ❌ No | Long-running applications |
| **EC2** | ✅ Yes | ✅ Yes (but expensive) | Custom, long processes |
| **S3** | ❌ No | ✅ Yes | Static content hosting |
| **CloudFront** | ❌ No | ❌ Cache only | CDN for S3/EC2 |

> 💡 **Công thức ghi nhớ:**
> **"Decouple 3-tier: S3 (static) + ECS (app) + RDS (database)"**
> **"Long-running process → ECS, KHÔNG Lambda"**
> **"Static pages → S3, KHÔNG CloudFront/EC2"**

> 🔑 **Keyword bẫy:**
> ❌ **"CloudFront để host pages"** → CloudFront là CDN, cần S3 phía sau
> ❌ **"Lambda cho long-running"** → Max 15 min, đề nói "long-running processes"
> ❌ **"Gom hết vào EC2 to hơn"** → Không phải decouple
> ✅ **"S3 + ECS + RDS Multi-AZ"** → **Fully decoupled, HA**
